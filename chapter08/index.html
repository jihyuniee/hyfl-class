    Chapter 8. AI에이전트 실전 활용 – 나만의 에이전트 워크플로 만들기¶

    Part 2. AI와 대화하는 기술 – 프롬프트 엔지니어링 | 8차시 (2시간)

    🎯 이 장에서 배우는 것¶

    • 복잡한 실생활 문제를 AI 에이전트 워크플로로 설계할 수 있다
    • AI에게 맡길 부분과 사람이 판단할 부분을 구분하여 협업 계획을 세울 수 있다
    • AI 에이전트를 활용해 실제 산출물(요약문, 비교표, 계획서 등)을 완성할 수 있다
    • 자신만의 관심 분야에서 AI 에이전트 활용 시나리오를 설계할 수 있다

    💡 왜 이걸 배우나요?¶

    생각해보기 💭 기말 보고서를 AI와 함께 쓴다면, 어떤 순서로 진행할까요? 자유롭게 머릿속에 떠오르는 순서를 적어보세요.

    여러분이 학교 축제를 기획한다고 상상해 보세요. 해야 할 일이 한두 가지가 아닙니다. 장소 조사, 아이디어 수집, 예산 계산, 발표자료 제작… 이 모든 것을 혼자 처음부터 끝까지 하면 며칠이 걸릴 수 있습니다.

    하지만 AI를 "똑똑한 조수"처럼 활용하면 어떨까요? 정보 조사는 AI가, 최종 판단은 내가, 초안 작성은 AI가, 수정과 보완은 내가 — 이렇게 역할을 나눠 협업하면 훨씬 빠르고 높은 품질의 결과물을 만들 수 있습니다.

    이번 장에서는 이런 협업 과정을 "워크플로"라는 체계적인 흐름으로 설계하고, 실제로 하나의 프로젝트를 AI와 함께 완성해 봅니다.

    📚 핵심 개념¶

    개념 1: 에이전트 워크플로란?¶

    🍳 비유로 시작하기

    요리를 떠올려 보세요. 맛있는 요리를 만들려면 레시피(순서)가 필요합니다.

    • 재료 준비 → 손질 → 조리 → 플레이팅(담기) → 완성

    AI 에이전트 워크플로도 마찬가지입니다. 복잡한 과제를 순서대로 나누고, 각 단계에서 AI와 사람이 할 일을 정하는 작업 흐름표입니다.

    📖 정확한 정의

    에이전트 워크플로(Agent Workflow): 하나의 큰 목표를 달성하기 위해, AI와 사람이 수행할 작업을 단계별로 배치한 협업 흐름도

    ✏️ 예시로 확인

    "여행 계획 세우기"를 워크플로로 나누면 이렇습니다.

    • 1단계 – 정보 수집: AI가 여행지 추천 목록을 생성
    • 2단계 – 선택·판단: 사람이 예산과 취향에 맞게 여행지 결정
    • 3단계 – 세부 계획: AI가 일자별 일정표 초안 작성
    • 4단계 – 최종 완성: 사람이 검토·수정하여 확정
    flowchart LR A["🔍 1단계\n정보 수집\n(AI)"] --> B["🧠 2단계\n선택·판단\n(사람)"] B --> C["📝 3단계\n세부 계획\n(AI)"] C --> D["✅ 4단계\n최종 완성\n(사람)"]

    개념 2: AI와 사람의 역할 분담¶

    모든 것을 AI에게 맡기면 안 됩니다. AI가 잘하는 일과 사람이 해야 하는 일은 다릅니다.

    🤖 AI가 잘하는 일 - 대량의 정보를 빠르게 요약·정리 - 다양한 아이디어를 한꺼번에 생성 - 정해진 형식(표, 목록, 초안)으로 변환 - 문법 교정, 번역 등 반복 작업

    🧑 사람이 해야 하는 일 - 목표와 방향 설정 (무엇을 할 것인지 결정) - 가치 판단과 윤리적 결정 - AI 결과의 정확성 검증 (팩트 체크) - 최종 승인과 책임

    알아보기 📌 AI가 생성한 정보에는 잘못된 내용(환각, hallucination)이 섞여 있을 수 있습니다. 그래서 "AI가 만들고, 사람이 확인한다"는 원칙이 반드시 필요합니다.

    flowchart TB subgraph AI["🤖 AI 담당"] A1["정보 수집·요약"] A2["아이디어 생성"] A3["초안·형식 작성"] end subgraph Human["🧑 사람 담당"] H1["목표·방향 설정"] H2["판단·선택"] H3["검증·최종 확정"] end H1 -->|"지시"| A1 A1 -->|"결과 전달"| H2 H2 -->|"피드백"| A2 A2 -->|"결과 전달"| H2 H2 -->|"확정 후 지시"| A3 A3 -->|"초안 전달"| H3

    개념 3: 워크플로 설계의 3단계 프레임워크¶

    복잡한 과제를 워크플로로 만들 때, 요리 레시피 비유를 활용한 3단계 프레임워크를 씁니다.

    🥕 1단계 – 재료 준비 (정보 수집) - 필요한 정보가 무엇인지 정의 - AI에게 검색·조사·요약을 요청 - 사람이 신뢰할 수 있는 자료인지 확인

    🍳 2단계 – 조리 (분석·정리) - 수집한 정보를 분석하고 구조화 - AI에게 비교, 분류, 아이디어 확장을 요청 - 사람이 핵심을 선별하고 방향 결정

    🍽️ 3단계 – 플레이팅 (최종 산출물) - 결과물의 형식을 결정 (보고서, 발표자료, 계획서 등) - AI에게 초안 작성을 요청 - 사람이 최종 검토·수정·완성

    flowchart LR subgraph S1["🥕 1단계: 재료 준비"] direction TB P1["정보 정의"] P2["AI 조사·요약"] P3["사람 검증"] end subgraph S2["🍳 2단계: 조리"] direction TB P4["AI 분석·분류"] P5["사람 선별·결정"] end subgraph S3["🍽️ 3단계: 플레이팅"] direction TB P6["AI 초안 작성"] P7["사람 검토·완성"] end S1 --> S2 --> S3

    개념 4: 반복과 피드백 루프¶

    워크플로는 한 번에 끝나지 않습니다. AI의 첫 결과가 완벽하지 않을 때, 피드백을 주고 다시 요청하는 과정이 핵심입니다.

    🔄 비유: 그림 그리기를 생각해 보세요. 스케치 → 피드백 → 수정 → 피드백 → 완성. AI와의 협업도 같습니다.

    정리하기 📋 - 1회차 요청: 대략적인 결과물 생성 - 피드백: "이 부분은 더 구체적으로", "예산을 추가해줘" - 2회차 요청: 개선된 결과물 - 필요시 반복: 만족할 때까지 피드백 루프를 돌림

    이 피드백 루프를 "반복 정제(Iterative Refinement)"라고 합니다.

    🔨 따라하기: "우리 학교 축제 기획안" AI 협업 프로젝트¶

    이제 실제로 AI와 함께 학교 축제 기획안을 만들어 봅시다! ChatGPT, Claude 등 어떤 AI 챗봇이든 사용할 수 있습니다.

    Step 1: 목표 정의와 워크플로 설계 (사람)¶

    먼저 사람이 전체 그림을 그립니다.

    우리의 목표: 학교 축제 기획안 완성 (부스 구성, 예산, 일정, 발표자료 초안)

    워크플로를 설계합니다.

    • Step 1: 목표 정의 → 사람
    • Step 2: 축제 아이디어 조사 → AI
    • Step 3: 아이디어 선별·결정 → 사람
    • Step 4: 예산표 초안 작성 → AI
    • Step 5: 발표자료 초안 작성 → AI
    • Step 6: 최종 검토·완성 → 사람

    Step 2: 정보 조사 – AI에게 아이디어 요청¶

    AI에게 다음과 같이 프롬프트를 입력합니다.

    너는 학교 축제 기획 전문가야.

고등학교 축제에서 운영할 수 있는 부스 아이디어 10개를 추천해줘.

조건:
- 학생들이 직접 운영 가능할 것
- 준비 비용이 부스당 5만 원 이내일 것
- 재미와 교육적 가치를 모두 갖출 것

각 아이디어마다 다음 정보를 포함해줘:
1. 부스 이름
2. 활동 내용 (2줄 설명)
3. 예상 준비 비용
4. 난이도 (상/중/하)

    알아보기 📌 이 프롬프트에는 7차시에서 배운 기법들이 녹아 있습니다. 역할 부여("축제 기획 전문가"), 구체적 조건, 출력 형식 지정 — 이것들이 좋은 워크플로의 기본입니다.

    Step 3: 사람이 선별하고 결정하기¶

    AI가 10개 아이디어를 제시하면, 사람이 다음 기준으로 3~4개를 선택합니다.

    • 우리 학교 시설에서 가능한가?
    • 친구들이 좋아할 만한가?
    • 준비 인원이 충분한가?

    생각해보기 💭 AI가 추천한 아이디어 중 현실적으로 불가능한 것은 없나요? 예를 들어 "VR 체험 부스"는 장비가 없으면 어렵겠죠. 이런 판단은 사람만 할 수 있습니다.

    Step 4: 예산표 초안 – AI에게 구조화 요청¶

    선택한 부스들로 예산표를 만들어 달라고 요청합니다.

    우리 축제에서 다음 4개 부스를 운영하기로 했어:

1. 미니 과학 실험실 (슬라임 만들기)
2. 포토존 & 즉석 사진관
3. 세계 간식 체험 부스
4. 방탈출 게임 부스

각 부스별로 다음 항목을 정리해줘:
- 필요한 재료/물품 목록
- 항목별 예상 비용
- 부스별 합계
- 전체 총 예산

전체 예산은 20만 원을 넘지 않도록 조정해줘.

    AI가 결과를 제시하면, 사람이 가격이 현실적인지 확인합니다. 인터넷으로 실제 가격을 검색해 보는 것이 좋습니다.

    Step 5: 발표자료 초안 – AI에게 구성 요청¶

    위 축제 기획안을 바탕으로 발표자료 초안을 만들어줘.

발표 대상: 학교 선생님과 학생회
발표 시간: 5분
슬라이드 수: 6장

각 슬라이드마다 다음을 포함해줘:
- 슬라이드 제목
- 핵심 내용 (3줄 이내)
- 발표자가 말할 스크립트 (50자 이내)

    Step 6: 최종 검토와 완성 (사람)¶

    AI가 만든 발표자료 초안을 검토합니다. 체크리스트를 활용하세요.

    • ✅ 정보가 정확한가? (가격, 인원 등)
    • ✅ 우리 학교 상황에 맞는가?
    • ✅ 빠진 내용은 없는가?
    • ✅ 발표 흐름이 자연스러운가?

    수정이 필요하면 AI에게 구체적으로 피드백합니다.

    2번째 슬라이드에서 부스 설명이 너무 길어.
각 부스를 한 줄로 요약하고, 핵심 키워드를 강조해줘.

    정리하기 📋 전체 과정을 돌아보면, AI가 작업한 단계(2, 4, 5)와 사람이 판단한 단계(1, 3, 6)가 번갈아 나타납니다. 이것이 바로 에이전트 워크플로의 핵심 패턴입니다.

    📝 전체 코드 (워크플로 프롬프트 모음)¶

    아래는 위 실습에서 사용한 프롬프트를 순서대로 정리한 것입니다. 복사하여 바로 사용할 수 있습니다.

    [프롬프트 1 – 아이디어 조사]

너는 학교 축제 기획 전문가야.

고등학교 축제에서 운영할 수 있는 부스 아이디어 10개를 추천해줘.

조건:
- 학생들이 직접 운영 가능할 것
- 준비 비용이 부스당 5만 원 이내일 것
- 재미와 교육적 가치를 모두 갖출 것

각 아이디어마다 다음 정보를 포함해줘:
1. 부스 이름
2. 활동 내용 (2줄 설명)
3. 예상 준비 비용
4. 난이도 (상/중/하)

    [프롬프트 2 – 예산표 작성]

우리 축제에서 다음 4개 부스를 운영하기로 했어:

1. 미니 과학 실험실 (슬라임 만들기)
2. 포토존 & 즉석 사진관
3. 세계 간식 체험 부스
4. 방탈출 게임 부스

각 부스별로 다음 항목을 정리해줘:
- 필요한 재료/물품 목록
- 항목별 예상 비용
- 부스별 합계
- 전체 총 예산

전체 예산은 20만 원을 넘지 않도록 조정해줘.

    [프롬프트 3 – 발표자료 초안]

위 축제 기획안을 바탕으로 발표자료 초안을 만들어줘.

발표 대상: 학교 선생님과 학생회
발표 시간: 5분
슬라이드 수: 6장

각 슬라이드마다 다음을 포함해줘:
- 슬라이드 제목
- 핵심 내용 (3줄 이내)
- 발표자가 말할 스크립트 (50자 이내)

    [프롬프트 4 – 피드백·수정 요청]

2번째 슬라이드에서 부스 설명이 너무 길어.
각 부스를 한 줄로 요약하고, 핵심 키워드를 강조해줘.

    ⚠️ 자주 하는 실수¶

    실수 1: 워크플로 없이 한 번에 모든 것을 요청한다 ❌

    "학교 축제 기획안 전체를 만들어줘"라고 한 번에 요청하면, AI는 두루뭉술하고 피상적인 결과를 내놓습니다. 단계를 나눠서 하나씩 요청해야 각 단계의 품질이 높아집니다.

    실수 2: AI 결과를 검증 없이 그대로 사용한다 ❌

    AI가 제시한 예산이 현실과 다를 수 있습니다. "슬라임 재료 1만 원"이라고 했는데 실제로는 2만 원일 수 있어요. 반드시 사람이 확인하세요.

    실수 3: 피드백 없이 첫 결과물에 만족한다 ❌

    AI의 첫 번째 응답이 완벽할 확률은 낮습니다. "여기를 이렇게 고쳐줘"라고 구체적 피드백을 2~3번 주면 결과물이 크게 개선됩니다.

    실수 4: AI에게 가치 판단을 맡긴다 ❌

    "어떤 부스가 가장 좋을까?"는 AI가 결정할 문제가 아닙니다. AI는 선택지를 제시하고, 최종 선택은 사람이 해야 합니다.

    ✅ 스스로 점검하기¶

    기본 문제¶

    문제 1. 다음 워크플로의 빈칸을 채우세요.

    목표: 독서 감상문 작성

    1단계: 책의 핵심 내용 ______ → AI 담당

    2단계: 감상문에 넣을 포인트 ______ → 사람 담당

    3단계: 감상문 ______ 작성 → AI 담당

    4단계: 최종 ______·완성 → 사람 담당

    정답 확인 1단계: **요약** / 2단계: **선택(결정)** / 3단계: **초안** / 4단계: **검토(수정)**

    문제 2. 다음 중 AI에게 맡기기에 적합하지 않은 작업은?

    • (가) 여행지 10곳의 특징 요약
    • (나) 가족 여행지로 어디를 갈지 최종 결정
    • (다) 여행 일정표 초안 작성
    • (라) 각 여행지의 날씨 정보 정리
    정답 확인 **(나)** — 최종 결정은 개인의 가치관과 상황에 따라 사람이 해야 합니다.

    응용 문제¶

    문제 3. "과학 탐구 보고서 작성"을 위한 에이전트 워크플로를 3단계 프레임워크(재료 준비 → 조리 → 플레이팅)로 설계해 보세요. 각 단계에서 AI가 할 일과 사람이 할 일을 구분하여 작성하세요.

    예시 답안 **🥕 재료 준비**: AI가 탐구 주제 관련 논문·자료 요약 → 사람이 신뢰할 수 있는 자료 선별 **🍳 조리**: AI가 실험 결과를 분석 형식으로 정리 → 사람이 실험의 의미를 해석하고 결론 도출 **🍽️ 플레이팅**: AI가 보고서 형식에 맞춰 초안 작성 → 사람이 내용 검증, 문장 다듬기, 최종 제출

    심화 문제 (도전!)¶

    문제 4. 여러분의 관심 분야(음악, 운동, 코딩, 미술 등)에서 AI 에이전트를 활용한 워크플로를 직접 설계해 보세요.

    다음 형식으로 작성합니다.

    • 주제: (예: "밴드 공연 준비")
    • 최종 산출물: (예: "공연 기획서 + 셋리스트")
    • 단계별 계획: 최소 4단계, 각 단계에 AI/사람 역할 표시
    • 핵심 프롬프트: 가장 중요한 단계의 프롬프트 1개 작성
