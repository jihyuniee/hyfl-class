    Chapter 8. 문제발견과 알고리즘 설계 — 해결할 문제를 찾아라¶

    Part 3 | [2권] 진짜 문제를 AI와 함께 해결하기 — 메인프로젝트

    🎯 이 장에서 배우는 것¶

    • 실생활의 불편함·비효율을 관찰하여 AI로 해결 가능한 문제를 구체적으로 정의할 수 있다
    • 문제 해결 과정을 입력-처리-출력 구조의 알고리즘으로 설계할 수 있다
    • 순서도(플로차트)를 활용하여 자신의 알고리즘을 시각적으로 표현하고 팀원에게 설명할 수 있다
    • 동일한 문제에 대해 두 가지 이상의 알고리즘을 비교하여 장단점을 분석할 수 있다

    💡 왜 이걸 배우나요?¶

    여러분은 지금까지 AI가 무엇인지, 어떻게 작동하는지를 배웠습니다. 이제 드디어 "진짜 문제"를 AI와 함께 해결하는 메인 프로젝트에 돌입합니다.

    그런데 잠깐 — 코딩부터 하면 될까요? 🤔

    절대 아닙니다. 좋은 해결책은 좋은 질문에서 시작됩니다.

    의사가 환자를 치료할 때를 떠올려 보세요. "어디가 아프세요?"라고 묻기도 전에 약을 처방하는 의사는 없습니다. 먼저 증상을 정확히 파악하고, 원인을 분석한 뒤, 치료 계획을 세우고, 그다음에 약을 처방하죠.

    AI 프로젝트도 마찬가지입니다.

    1. 문제를 발견하고 → 2. 문제를 정의하고 → 3. 해결 과정을 설계하고 → 4. 그다음에 코딩!

    이번 장에서는 1~3단계, 즉 코딩 전에 반드시 해야 할 사고 과정을 배웁니다. 연필과 종이만 있으면 충분합니다. ✏️

    📚 핵심 개념¶

    개념 1: 문제 발견 — "짜증"에서 "기회"로 🔍¶

    비유로 시작

    여러분이 매일 아침 등교할 때, 버스가 언제 오는지 몰라서 10분이나 허탕을 친 적 있나요? 그 "짜증나는 순간"이 바로 문제 발견의 씨앗입니다. 세상의 거의 모든 발명품은 누군가의 불편함에서 시작되었습니다.

    정확한 정의

    문제 발견(Problem Finding) 이란, 일상에서 겪는 불편함·비효율·반복적 어려움을 의식적으로 관찰하고, 그중 기술(특히 AI)로 개선할 수 있는 것을 골라내는 과정입니다.

    예시로 확인

    • 😤 "급식 메뉴를 매번 확인하기 귀찮아" → AI 챗봇이 오늘의 메뉴를 알려주면?
    • 😤 "영어 단어 외우는 게 지루해" → AI가 내 수준에 맞춰 퀴즈를 만들어 주면?
    • 😤 "분리수거를 어디에 버려야 할지 모르겠어" → 사진을 찍으면 AI가 분류해 주면?

    💡 생각해보기: 최근 1주일간 가장 짜증났던 순간 3가지를 떠올려 보세요. 노트에 적어 봅시다!

    1. __________________
    2. __________________
    3. __________________

    이 중에서 "AI가 도와줄 수 있을 것 같은 문제"에 ⭐ 표시를 해 보세요.

    개념 2: 문제 정의 캔버스 — 흐릿한 불만을 선명한 문제로 📋¶

    비유로 시작

    "배가 아파요"라고만 하면 의사가 진단하기 어렵습니다. "어제 저녁 회를 먹은 뒤, 오른쪽 아랫배가 찌르는 듯이 아파요"라고 해야 정확한 진단이 가능하죠. 문제도 마찬가지입니다. 막연한 불편함을 구체적인 문장으로 바꾸는 도구가 '문제 정의 캔버스'입니다.

    정확한 정의

    문제 정의 캔버스는 "누가(Who)·언제(When)·어디서(Where)·무엇이(What) 불편한가"라는 네 가지 질문으로 문제를 구체화하는 틀입니다.

    예시로 확인

    "분리수거가 어려워"라는 막연한 불만을 캔버스로 구체화하면:

    • 누가(Who): 우리 학교 학생들
    • 언제(When): 점심시간 후 교실에서 쓰레기를 버릴 때
    • 어디서(Where): 교실 뒤편 분리수거 구역
    • 무엇이(What) 불편한가: 플라스틱·비닐·일반쓰레기 구분이 헷갈려서 잘못 버리는 경우가 많다

    이렇게 구체화하면 해결 방향이 보입니다:

    "우리 학교 학생들이 교실에서 쓰레기를 버릴 때, 재질을 구분하지 못해 잘못 분류하는 문제를 AI 이미지 분류로 해결한다."

    flowchart LR A["😤 막연한 불편함<br>'분리수거 어려워'"] --> B["📋 문제 정의 캔버스<br>누가·언제·어디서·무엇이"] B --> C["✅ 구체적 문제 정의<br>'학생들이 교실에서<br>쓰레기 재질을 구분 못함'"] C --> D["💡 해결 아이디어<br>'AI 이미지 분류로<br>재질 자동 판별'"]

    개념 3: 입력-처리-출력 구조 — 알고리즘의 뼈대 🦴¶

    비유로 시작

    자판기를 떠올려 보세요. 동전을 넣고(입력) → 기계가 음료를 고르고(처리) → 음료가 나옵니다(출력). 세상의 거의 모든 문제 해결 과정은 이 세 단계로 나눌 수 있습니다.

    정확한 정의

    알고리즘(Algorithm) 이란, 문제를 해결하기 위한 단계별 절차입니다. 가장 기본적인 구조는 입력(Input) → 처리(Process) → 출력(Output) 입니다.

    예시로 확인

    분리수거 AI의 경우:

    • 입력: 쓰레기 사진 📸
    • 처리: AI가 이미지를 분석하여 재질(플라스틱/캔/유리/종이/일반쓰레기) 판별 🤖
    • 출력: "이것은 플라스틱입니다. 파란색 통에 버려주세요!" 💬

    영어 단어 퀴즈 AI의 경우:

    • 입력: 학생의 현재 영어 수준, 틀린 문제 기록 📊
    • 처리: AI가 취약한 단어를 분석하여 맞춤형 퀴즈 생성 🤖
    • 출력: 오늘의 퀴즈 10문제 + 힌트 📝
    flowchart LR subgraph 입력["📥 입력 Input"] I1["사진, 텍스트,<br>센서 데이터 등"] end subgraph 처리["⚙️ 처리 Process"] P1["AI 모델이<br>분석·판단·생성"] end subgraph 출력["📤 출력 Output"] O1["결과 안내,<br>추천, 경고 등"] end 입력 --> 처리 --> 출력

    개념 4: 순서도(플로차트) — 알고리즘을 눈에 보이게 🗺️¶

    비유로 시작

    친구에게 학교 가는 길을 말로 설명하는 것과, 지도를 그려서 보여주는 것 중 어느 쪽이 더 명확할까요? 당연히 지도죠! 순서도는 알고리즘의 지도입니다.

    정확한 정의

    순서도(Flowchart) 는 알고리즘의 각 단계를 도형과 화살표로 시각화한 다이어그램입니다. 주요 기호는 다음과 같습니다.

    • 타원(둥근 사각형): 시작 / 끝
    • 직사각형: 처리 단계 (어떤 작업을 수행)
    • 마름모: 조건 판단 (예/아니오로 갈림)
    • 평행사변형: 입력 / 출력
    • 화살표: 흐름의 방향

    예시로 확인 — 분리수거 AI 순서도

    flowchart TD A(["🟢 시작"]) --> B[/"📸 사용자가 쓰레기 사진 촬영"/] B --> C["🤖 AI가 이미지 분석"] C --> D{"재질 판별 성공?"} D -- 예 --> E["재질 종류 확인<br>(플라스틱/캔/유리/종이/일반)"] E --> F[/"💬 '○○입니다.<br>△△통에 버려주세요!' 출력"/] F --> G(["🔴 끝"]) D -- 아니오 --> H[/"⚠️ '인식 실패.<br>다시 촬영해 주세요' 출력"/] H --> B

    💡 알아보기: 순서도에서 마름모(조건 판단)는 프로그램이 "생각하는" 부분입니다. "만약 ~이면 A를 하고, 아니면 B를 해라"는 구조를 조건 분기라고 합니다. AI 프로젝트에서는 "AI가 판별에 성공했는가?"처럼 결과에 따라 다음 행동이 달라지는 경우가 많습니다.

    🔨 따라하기 — 나만의 AI 프로젝트 기획서 만들기¶

    ✏️ 이 활동은 코딩 없이 종이, 포스트잇, 필기도구만으로 진행합니다!

    Step 1: 불편함 수집하기 (10분) 🕵️¶

    노트에 최근 1주일간 겪은 불편한 순간을 최소 3가지 적습니다.

    적을 때 아래 형식을 따라 보세요:

    "나는 ____할 때 ____해서 불편했다."

    예시: - "나는 아침에 입을 옷을 고를 때 날씨를 몰라서 부적절한 옷을 입고 불편했다." - "나는 시험공부를 할 때 어떤 과목부터 해야 할지 몰라서 시간을 낭비했다." - "나는 도서관에서 책을 찾을 때 어디 있는지 몰라서 10분이나 헤맸다."

    📌 정리하기: 3가지 중 "AI가 도와줄 수 있을 것 같은 문제"에 ⭐ 표시를 하세요. AI는 패턴 인식, 분류, 추천, 예측, 텍스트 생성에 강하다는 점을 기억하세요!

    Step 2: 문제 정의 캔버스 채우기 (15분) 📋¶

    ⭐ 표시한 문제를 골라, 아래 네 가지 질문에 답합니다.

    노트에 크게 네 칸을 나누고, 각 칸에 질문과 답을 적으세요:

    • 누가(Who): 이 문제를 겪는 사람은? (예: 중학교 2학년 학생)
    • 언제(When): 어떤 상황에서? (예: 시험 2주 전 공부 계획을 세울 때)
    • 어디서(Where): 어떤 장소/환경에서? (예: 집 책상에서)
    • 무엇이(What) 불편한가: 구체적으로 어떤 점이? (예: 과목별 우선순위를 모르겠다)

    이 네 가지를 합쳐 한 문장으로 정리합니다:

    "중학교 2학년 학생이 시험 2주 전 집에서 공부 계획을 세울 때, 과목별 우선순위를 판단하지 못해 비효율적으로 공부하는 문제"

    Step 3: 입력-처리-출력 분해하기 (15분) ⚙️¶

    정의한 문제의 해결 과정을 세 덩어리로 나눕니다.

    포스트잇 3장(또는 노트를 3칸으로)을 준비하세요:

    • 📥 입력 (포스트잇 1): AI에게 무엇을 주어야 하나?
    • ⚙️ 처리 (포스트잇 2): AI가 어떤 작업을 하나?
    • 📤 출력 (포스트잇 3): 사용자가 무엇을 받나?

    공부 계획 AI 예시:

    • 📥 입력: 과목 목록, 각 과목별 시험 범위, 최근 성적, 남은 일수
    • ⚙️ 처리: 취약 과목 분석 → 우선순위 결정 → 일별 공부 시간 배분
    • 📤 출력: "Day 1: 수학 2시간, 영어 1시간..." 형태의 2주 학습 플랜

    Step 4: 순서도 그리기 (20분) 🗺️¶

    입력-처리-출력을 더 세밀하게 쪼개서 순서도로 그립니다.

    📌 팁: 처음부터 완벽하게 그리려 하지 마세요! 포스트잇에 각 단계를 하나씩 적고, 순서를 바꿔가며 배치한 뒤, 최종적으로 화살표를 그리면 됩니다.

    공부 계획 AI 순서도 예시:

    flowchart TD A(["🟢 시작"]) --> B[/"📥 과목 목록, 시험 범위,<br>최근 성적, 남은 일수 입력"/] B --> C["📊 과목별 취약도 점수 계산"] C --> D{"취약 과목이<br>3개 이상?"} D -- 예 --> E["⚠️ 상위 3개 취약 과목에<br>집중 배분"] D -- 아니오 --> F["📐 전체 과목에<br>균등 배분"] E --> G["📅 일별 공부 계획 생성"] F --> G G --> H[/"📤 '2주 맞춤 학습 플랜' 출력"/] H --> I(["🔴 끝"])

    노트에 그릴 때는 이렇게 하세요: - 둥근 도형 → 시작/끝 - 네모 → 처리 단계 - 마름모 → "예/아니오" 판단 - 비스듬한 네모(평행사변형) → 입력/출력 - 화살표 → 흐름 방향

    Step 5: 모둠 발표와 피드백 (20분) 🗣️¶

    모둠(3~4명)별로 돌아가며 자신의 기획서를 2분 이내로 발표합니다.

    발표 순서: 1. "내가 발견한 문제는 ____입니다." (문제 정의 캔버스 요약) 2. "AI 해결 과정은 ____를 입력받아 → ____를 처리하고 → ____를 출력합니다." 3. 순서도를 보여주며 흐름 설명

    듣는 사람은 피드백 카드를 작성합니다:

    • 👍 좋은 점: "문제 정의가 구체적이어서 해결 방향이 명확해요."
    • 🤔 궁금한 점: "입력 데이터를 어떻게 모을 건가요?"
    • 💡 제안: "조건 분기를 하나 더 추가하면 어떨까요?"

    Step 6: 보완 및 최종 정리 (10분) ✨¶

    피드백을 바탕으로 기획서를 수정합니다. 아래 체크리스트로 점검하세요:

    • 문제 정의가 한 문장으로 명확한가?
    • 입력-처리-출력이 각각 구체적인가?
    • 순서도에 조건 판단(마름모)이 최소 1개 있는가?
    • 실제로 만들 수 있을 것 같은가?

    📝 전체 코드¶

    이번 장은 코딩 없이 사고력을 키우는 활동입니다! 하지만 여러분이 설계한 알고리즘이 실제 코드로 어떻게 변환되는지 맛보기를 제공합니다.

    아래는 공부 계획 AI의 입력-처리-출력을 Python 의사코드(pseudocode)로 표현한 것입니다:

    # === 공부 계획 AI - 알고리즘 맛보기 ===

# 📥 입력
subjects = ["수학", "영어", "과학", "국어", "사회"]
recent_scores = [65, 80, 55, 90, 70]  # 최근 시험 점수
days_left = 14  # 시험까지 남은 일수

# ⚙️ 처리: 취약 과목 찾기
weakness_threshold = 70  # 70점 미만이면 취약 과목
weak_subjects = []

for i in range(len(subjects)):
    if recent_scores[i] < weakness_threshold:
        weak_subjects.append(subjects[i])
        print(f"⚠️ 취약 과목 발견: {subjects[i]} ({recent_scores[i]}점)")

# ⚙️ 처리: 공부 시간 배분
if len(weak_subjects) >= 3:
    print(f"\n취약 과목이 {len(weak_subjects)}개! 집중 배분 모드 🔥")
    strategy = "취약 과목에 하루 2시간, 나머지 1시간"
else:
    print(f"\n취약 과목이 {len(weak_subjects)}개. 균등 배분 모드 📐")
    strategy = "모든 과목에 균등하게 배분"

# 📤 출력
print(f"\n{'='*40}")
print(f"📅 {days_left}일 맞춤 학습 플랜")
print(f"전략: {strategy}")
print(f"취약 과목: {', '.join(weak_subjects) if weak_subjects else '없음'}")
print(f"{'='*40}")

    실행 결과: 

    ⚠️ 취약 과목 발견: 수학 (65점)
⚠️ 취약 과목 발견: 과학 (55점)

취약 과목이 2개. 균등 배분 모드 📐

========================================
📅 14일 맞춤 학습 플랜
전략: 모든 과목에 균등하게 배분
취약 과목: 수학, 과학
========================================

    💡 알아보기: 위 코드에서 if문(조건 분기)이 순서도의 마름모와 정확히 대응됩니다! len(weak_subjects) >= 3 이 "취약 과목이 3개 이상?"이라는 질문이고, 예/아니오에 따라 다른 전략을 선택합니다.

    🚫 실수 2: 입력을 구체적으로 정하지 않는다¶

    ❌ 입력: "데이터를 넣는다"

    ✅ 입력: "쓰레기 사진 1장 (스마트폰 카메라로 촬영, 정면에서 20cm 거리)"

    AI에게 어떤 형태의 데이터를 줄 것인지 구체적으로 정해야 합니다. 사진인지, 텍스트인지, 숫자인지, 어떤 조건에서 수집하는지를 명확히 하세요.

    🚫 실수 3: 순서도에 조건 분기가 없다¶

    시작 → 처리1 → 처리2 → 끝, 이렇게 일직선으로만 된 순서도는 현실 문제를 다루기 어렵습니다. "만약 ~이면?" 상황을 최소 1개 이상 포함하세요. 현실 세계에는 항상 예외 상황이 있기 때문입니다.
