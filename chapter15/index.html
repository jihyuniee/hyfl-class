    Chapter 15. 코딩을 몰라도 괜찮아 – AI 코딩 도구 시작하기¶

    Part 5. 웹앱 만들기 – AI와 함께 구현하기

    🎯 이 장에서 배우는 것¶

    • 노코드/AI 코딩 도구가 무엇이고, 어떤 원리로 작동하는지 설명할 수 있다
    • 웹의 3요소(HTML, CSS, JavaScript)가 각각 어떤 역할을 하는지 비유로 설명할 수 있다
    • AI에게 자연어로 지시하여 간단한 '자기소개 웹 페이지'를 생성할 수 있다
    • 생성된 코드를 직접 읽지 못해도 어느 부분이 무슨 역할인지 대략적으로 구분할 수 있다

    💡 왜 이걸 배우나요?¶

    생각해보기 💭

    "코딩은 영어도 아니고 수학도 아닌 외계어 같다…" 이런 생각, 한 번쯤 해본 적 있지 않나요?

    솔직하게 인정합시다. 코딩은 어렵게 느껴집니다. 화면 가득 펼쳐진 영어 단어, 괄호, 세미콜론을 보면 "이걸 언제 다 배우지?" 싶죠.

    하지만 지금은 2024년입니다. AI가 우리의 통역사가 되어줄 수 있는 시대예요.

    여러분이 한국어로 이렇게 말하면:

    "파란 배경에 내 이름이 크게 나오고, 아래에 취미 목록이 있는 웹 페이지 만들어줘"

    AI가 이 말을 코드로 번역해줍니다. 마치 외국 여행에서 통역 앱을 쓰는 것처럼요.

    이번 장에서는 그 첫 번째 성공 경험을 만들어봅니다. 코드를 한 줄도 외울 필요 없이, AI와 대화하며 나만의 웹 페이지를 완성할 거예요.

    📚 핵심 개념¶

    개념 1: AI 코딩 도구란?¶

    🏠 비유로 시작

    여러분이 멋진 집을 짓고 싶다고 상상해보세요. 두 가지 방법이 있습니다.

    • 방법 A: 건축학과를 졸업하고, 직접 설계도를 그리고, 벽돌을 하나씩 쌓는다
    • 방법 B: 건축가에게 "2층짜리, 하얀 벽, 넓은 창문이 있는 집을 지어주세요"라고 말한다

    AI 코딩 도구는 방법 B입니다. 여러분은 무엇을 만들고 싶은지만 말하면, AI가 코드를 작성해줍니다.

    📖 정확한 정의

    AI 코딩 도구란, 사용자가 자연어(일상 언어)로 원하는 결과물을 설명하면 AI가 자동으로 프로그래밍 코드를 생성해주는 소프트웨어입니다.

    🔍 예시로 확인

    대표적인 AI 코딩 도구들:

    • ChatGPT — 대화로 코드 생성 요청 가능, 코드 복사 후 실행
    • Google AI Studio — 구글 계정만 있으면 무료 접근 가능
    • Claude — 자연어 지시로 코드 생성 및 설명 제공
    • Replit AI — 코드 생성과 실행을 한 곳에서 가능한 온라인 환경

    알아보기 📌

    "노코드(No-Code)"와 "AI 코딩"은 비슷하지만 다릅니다. - 노코드: 드래그 앤 드롭으로 만드는 방식 (예: Wix, 네이버 모두) - AI 코딩: 자연어 지시로 실제 코드를 생성하는 방식

    이번 장에서는 AI 코딩 방식을 사용합니다. 실제 코드가 만들어지기 때문에, 나중에 코딩을 배울 때 연결 고리가 됩니다.

    개념 2: 웹의 3요소 – 집짓기 비유¶

    웹 페이지는 세 가지 언어로 이루어져 있습니다. 이걸 집짓기에 비유하면 쉽게 이해할 수 있어요.

    graph LR A["🏗️ HTML<br>뼈대·구조"] --> D["완성된<br>웹 페이지 🌐"] B["🎨 CSS<br>인테리어·디자인"] --> D C["⚡ JavaScript<br>전기·수도·기능"] --> D

    🏗️ HTML = 뼈대 (구조)

    집을 지을 때 가장 먼저 하는 일은 기둥을 세우고 벽을 올리는 것이죠. HTML은 웹 페이지의 뼈대입니다.

    • "여기에 제목이 있다"
    • "여기에 문단이 있다"
    • "여기에 이미지가 들어간다"

    이런 구조적 정보를 담당합니다.

    🎨 CSS = 인테리어 (디자인)

    뼈대만 있으면 콘크리트 건물이겠죠? CSS는 벽지를 바르고, 가구 색을 정하고, 조명을 설치하는 인테리어 담당입니다.

    • 배경색을 파란색으로
    • 글자 크기를 24px로
    • 제목을 가운데 정렬

    ⚡ JavaScript = 전기·수도 (기능)

    예쁜 집이 완성됐는데, 전기와 수도가 없으면? 스위치를 눌러도 불이 안 켜지고, 수도꼭지를 틀어도 물이 안 나옵니다. JavaScript는 웹 페이지의 동작과 기능을 담당합니다.

    • 버튼을 클릭하면 메시지가 나타남
    • 스크롤하면 메뉴가 따라옴
    • 입력값을 확인해서 결과를 보여줌

    정리하기 📋

    • HTML: 웹 페이지에 무엇이 있는지 (내용과 구조)
    • CSS: 그것이 어떻게 보이는지 (색상, 크기, 배치)
    • JavaScript: 그것이 어떻게 동작하는지 (클릭, 반응, 계산)

    개념 3: AI에게 지시하는 법 – 프롬프트 작성 원칙¶

    AI 코딩 도구에게 말할 때도 잘 말하는 법이 있습니다.

    🍕 비유로 시작

    피자를 주문할 때를 생각해보세요.

    • ❌ "피자 하나 주세요" → 어떤 피자? 크기는? 토핑은?
    • ✅ "라지 사이즈, 페퍼로니 토핑, 치즈크러스트 피자 하나 주세요" → 정확하게 나옴

    AI에게도 마찬가지입니다.

    좋은 프롬프트의 4가지 요소:

    • 무엇을 만들지 — "자기소개 웹 페이지"
    • 어떤 내용이 들어가는지 — "이름, 학교, 취미 3가지"
    • 어떤 디자인인지 — "파란 배경, 흰 글씨, 가운데 정렬"
    • 어떤 기능이 있는지 — "취미를 클릭하면 설명이 나타남"

    ❌ 나쁜 예시:

    "웹 페이지 만들어줘"

    ✅ 좋은 예시:

    "자기소개 웹 페이지를 HTML, CSS, JavaScript로 만들어줘. 이름은 '김하늘', 학교는 '한빛중학교', 취미는 '독서, 게임, 그림 그리기'야. 배경색은 연한 파란색(#E3F2FD), 글자는 진한 남색(#1A237E), 가운데 정렬이야. 취미 각각을 클릭하면 간단한 설명이 나타나게 해줘."

    개념 4: 코드 구조 읽기 – 읽지 못해도 보는 법¶

    AI가 코드를 만들어주면 화면에 코드가 쭉 나타납니다. 다 이해할 필요는 없지만, 어디가 어디인지 구분하는 눈은 길러야 합니다.

    graph TD A["전체 코드"] --> B["&lt;html&gt; ... &lt;/html&gt;<br>🏗️ HTML 영역"] A --> C["&lt;style&gt; ... &lt;/style&gt;<br>🎨 CSS 영역"] A --> D["&lt;script&gt; ... &lt;/script&gt;<br>⚡ JavaScript 영역"] B --> E["제목, 문단,<br>이미지, 목록 등"] C --> F["색상, 크기,<br>배치, 폰트 등"] D --> G["클릭 반응,<br>애니메이션 등"]

    알아보기 📌

    코드에서 태그(tag)란 < > 꺾쇠 괄호로 둘러싸인 것을 말합니다. - <h1> → 큰 제목 (heading 1) - <p> → 문단 (paragraph) - <img> → 이미지 (image)

    영어 단어의 줄임말이라고 생각하면 됩니다!

    🔨 따라하기: 나만의 자기소개 웹 페이지 만들기¶

    Step 1: AI 코딩 도구 접속하기¶

    이번 실습에서는 ChatGPT(chat.openai.com)를 사용합니다. 다른 AI 도구(Claude, Google AI Studio 등)를 사용해도 같은 결과를 얻을 수 있어요.

    1. 웹 브라우저에서 chat.openai.com에 접속합니다
    2. 구글 계정 또는 이메일로 로그인합니다
    3. 새 대화 화면이 나타나면 준비 완료!

    💡 학교에서 접속이 제한되면, 선생님께 허용된 AI 도구를 확인하세요.

    Step 2: 프롬프트 작성하기¶

    아래 프롬프트를 복사해서 그대로 붙여넣기하세요. 이름과 정보만 자신의 것으로 바꾸면 됩니다.

    자기소개 웹 페이지를 하나의 HTML 파일로 만들어줘.

[내용]
- 이름: 김하늘
- 학교: 한빛중학교 2학년
- 취미: 독서, 게임, 그림 그리기
- 한 줄 소개: "꿈을 코딩하는 학생입니다"

[디자인]
- 배경색: 연한 파란색(#E3F2FD)
- 글자색: 진한 남색(#1A237E)
- 전체 내용 가운데 정렬
- 이름은 큰 제목으로, 둥근 테두리가 있는 카드 형태

[기능]
- 취미 항목을 클릭하면 아래에 간단한 설명이 나타났다 사라짐
- 페이지 하단에 "방문해주셔서 감사합니다" 문구

모든 CSS와 JavaScript를 하나의 HTML 파일 안에 포함시켜줘.

    Step 3: AI 응답 확인하기¶

    AI가 코드를 생성하면, 코드 블록 우측 상단에 복사 버튼(📋)이 보입니다. 이것을 클릭하면 전체 코드가 복사됩니다.

    이 시점에서 코드를 처음부터 끝까지 이해할 필요는 없습니다. 다만 세 가지 영역이 보이는지만 확인하세요:

    • <body> ... </body> 안에 있는 내용 → 🏗️ HTML (뼈대)
    • <style> ... </style> 안에 있는 내용 → 🎨 CSS (디자인)
    • <script> ... </script> 안에 있는 내용 → ⚡ JavaScript (기능)

    Step 4: HTML 파일 만들고 실행하기¶

    1. 컴퓨터에서 메모장(Windows) 또는 텍스트 편집기(Mac)를 엽니다
    2. 복사한 코드를 붙여넣기 합니다
    3. 다른 이름으로 저장을 클릭합니다
    4. 파일 이름을 intro.html로 입력합니다
    5. 파일 형식(인코딩)은 UTF-8을 선택합니다
    6. 저장된 intro.html 파일을 더블 클릭합니다

    🎉 웹 브라우저가 열리면서 여러분의 자기소개 웹 페이지가 나타납니다!

    생각해보기 💭

    방금 여러분은 코드를 한 줄도 직접 작성하지 않았는데 웹 페이지를 만들었습니다. AI가 자연어를 코드로 번역해준 것이죠. 이것이 바로 AI 코딩 도구의 핵심입니다.

    Step 5: AI에게 수정 요청하기¶

    웹 페이지가 마음에 안 드는 부분이 있다면? 다시 AI에게 말하면 됩니다.

    수정 요청 예시:

    방금 만든 코드에서 다음 부분을 수정해줘:
1. 배경색을 연한 분홍색(#FCE4EC)으로 바꿔줘
2. 이름 옆에 ✨ 이모지를 추가해줘  
3. 취미 목록에 '음악 감상'을 하나 더 추가해줘

    AI가 수정된 코드를 보내주면, 다시 복사 → 메모장에 붙여넣기 → 저장 → 브라우저에서 열기 순서를 반복하면 됩니다.

    💡 팁: 브라우저에서 이미 파일이 열려있다면, Ctrl + R (새로고침)만 눌러도 변경사항이 반영됩니다.

    Step 6: 코드에서 3요소 찾아보기¶

    마지막으로, 생성된 코드를 메모장에서 열어 세 영역에 형광펜을 칠하듯 눈으로 구분해봅시다.

    graph TD A["intro.html 파일 전체"] --> B["&lt;!DOCTYPE html&gt;<br>&lt;html&gt;"] B --> C["&lt;head&gt;<br> &lt;style&gt;<br> 🎨 CSS 코드<br> &lt;/style&gt;<br>&lt;/head&gt;"] B --> D["&lt;body&gt;<br> 🏗️ HTML 내용<br> &lt;script&gt;<br> ⚡ JS 코드<br> &lt;/script&gt;<br>&lt;/body&gt;"]

    AI에게 이렇게 물어봐도 좋습니다:

    방금 만든 코드에서 HTML, CSS, JavaScript 부분을 각각 
구분해서 설명해줘. 각 부분이 어떤 역할을 하는지도 알려줘.

    📝 전체 코드¶

    아래는 AI가 생성해줄 코드의 예시입니다. AI의 응답이 이와 조금 다를 수 있지만 결과는 비슷합니다.

    <!DOCTYPE html>
<html lang="ko">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>김하늘의 자기소개</title>
  <style>
    /* 🎨 CSS 영역: 디자인을 담당합니다 */
    body {
      background-color: #E3F2FD;
      color: #1A237E;
      font-family: 'Segoe UI', sans-serif;
      display: flex;
      justify-content: center;
      align-items: center;
      min-height: 100vh;
      margin: 0;
    }
    .card {
      background: white;
      border-radius: 20px;
      padding: 40px;
      max-width: 500px;
      text-align: center;
      box-shadow: 0 4px 20px rgba(0,0,0,0.1);
    }
    h1 { font-size: 2em; margin-bottom: 5px; }
    .subtitle { color: #5C6BC0; margin-bottom: 20px; }
    .hobby {
      background: #E8EAF6;
      padding: 10px 15px;
      margin: 8px;
      border-radius: 10px;
      display: inline-block;
      cursor: pointer;
      transition: background 0.3s;
    }
    .hobby:hover { background: #C5CAE9; }
    .hobby-desc {
      display: none;
      color: #3949AB;
      font-size: 0.9em;
      margin-top: 5px;
    }
    .footer { margin-top: 30px; color: #9FA8DA; font-size: 0.85em; }
  </style>
</head>
<body>
  <!-- 🏗️ HTML 영역: 구조와 내용을 담당합니다 -->
  <div class="card">
    <h1>김하늘</h1>
    <p class="subtitle">한빛중학교 2학년</p>
    <p>"꿈을 코딩하는 학생입니다"</p>

    <h2>🎯 취미</h2>
    <div>
      <div class="hobby" onclick="toggle('desc1')">📚 독서
        <div class="hobby-desc" id="desc1">판타지 소설을 좋아해요!</div>
      </div>
      <div class="hobby" onclick="toggle('desc2')">🎮 게임
        <div class="hobby-desc" id="desc2">주말에 친구들과 함께 해요!</div>
      </div>
      <div class="hobby" onclick="toggle('desc3')">🎨 그림 그리기
        <div class="hobby-desc" id="desc3">태블릿으로 캐릭터를 그려요!</div>
      </div>
    </div>

    <p class="footer">방문해주셔서 감사합니다 💙</p>
  </div>

  <script>
    // ⚡ JavaScript 영역: 동작과 기능을 담당합니다
    function toggle(id) {
      var desc = document.getElementById(id);
      if (desc.style.display === 'block') {
        desc.style.display = 'none';
      } else {
        desc.style.display = 'block';
      }
    }
  </script>
</body>
</html>

    💡 위 코드를 메모장에 복사 → intro.html로 저장 → 더블 클릭하면 바로 웹 페이지가 열립니다!

    ⚠️ 자주 하는 실수¶

    실수 1: 파일 확장자가 .txt로 저장됨 😱

    메모장에서 저장할 때 파일 형식을 "모든 파일(.)"로 바꾸지 않으면, intro.html.txt로 저장됩니다. 이러면 브라우저가 코드로 인식하지 못합니다.

    👉 해결: 저장할 때 파일 형식을 "모든 파일"로 선택하고, 파일 이름 끝에 .html을 정확히 입력하세요.

    실수 2: 한글이 깨져서 보임 😵

    <meta charset="UTF-8"> 줄이 빠져있거나, 메모장에서 인코딩을 ANSI로 저장하면 한글이 깨집니다.

    👉 해결: 저장 시 인코딩을 반드시 UTF-8로 선택하세요.

    실수 3: AI에게 너무 모호하게 지시함 🤔

    "멋진 웹 페이지 만들어줘"라고만 하면, AI가 여러분이 원하는 것과 전혀 다른 결과를 내놓을 수 있습니다.

    👉 해결: Step 2의 프롬프트처럼 내용, 디자인, 기능을 구체적으로 나눠서 지시하세요.

    실수 4: 코드의 일부만 복사함 ✂️

    AI가 생성한 코드에서 맨 첫 줄 <!DOCTYPE html>이나 마지막 </html>을 빠뜨리면 페이지가 제대로 표시되지 않습니다.

    👉 해결: 코드 블록의 복사 버튼을 사용해서 전체를 한 번에 복사하세요.

    기본 문제¶

    문제 1. 다음 빈칸에 알맞은 단어를 넣으세요.

    • HTML은 웹 페이지의 ( ______ )를 담당한다.
    • CSS는 웹 페이지의 ( ______ )를 담당한다.
    • JavaScript는 웹 페이지의 ( ______ )를 담당한다.

    정답: 구조(뼈대) / 디자인(인테리어) / 기능(동작)

    문제 2. 다음 중 AI 코딩 도구에게 지시할 때 가장 좋은 프롬프트는?

    • (가) "웹 페이지 하나 만들어줘"
    • (나) "예쁜 웹 페이지를 빨리 만들어줘"
    • (다) "이름과 취미가 나오는 자기소개 웹 페이지를, 흰 배경에 검은 글씨로, 가운데 정렬해서 HTML 파일 하나로 만들어줘"

    정답: (다) — 무엇을, 어떤 내용으로, 어떤 디자인으로 만들지 구체적으로 설명했습니다.

    응용 문제¶

    문제 3. 코드에서 <style> 태그 안의 background-color: #E3F2FD;를 background-color: #FFF9C4;로 바꾸면 어떤 변화가 생길까요?

    정답: 배경색이 연한 파란색에서 연한 노란색으로 바뀝니다. CSS 영역의 값을 바꾸면 디자인이 변경됩니다.
