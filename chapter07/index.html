    미니프로젝트 4차시 — 발표, 시상, 성찰¶

    Part 2 | [1권] 미니프로젝트 — 우리만의 AI 분류기 만들기

    🎯 이 장에서 배우는 것¶

    • 프로젝트 과정과 결과를 '문제 → 데이터 → 모델 → 결과물 → 배운 점' 흐름으로 발표할 수 있다
    • 동료 평가 기준(창의성·완성도·협업도)에 따라 다른 팀의 프로젝트를 공정하게 평가할 수 있다
    • 프로젝트 전체를 돌아보며 성찰 일지를 작성하고, 배운 점을 한 문장으로 정리할 수 있다
    • 평가 결과를 집계하여 최우수 프로젝트를 선정하고 시상할 수 있다

    💡 왜 이걸 배우나요?¶

    여러분은 지난 3차시 동안 정말 대단한 일을 해냈습니다. 직접 문제를 정의하고, 데이터를 모으고, AI 모델을 학습시켜 분류기를 완성했죠. 그런데 여기서 끝이 아닙니다.

    세상의 모든 훌륭한 프로젝트에는 공통점이 하나 있습니다. 바로 다른 사람에게 자신의 작업을 설명하고, 피드백을 받고, 그 과정에서 더 성장한다는 것입니다.

    🤔 생각해보기: 과학자가 논문을 쓰는 이유, 개발자가 코드 리뷰를 하는 이유, 창업가가 투자자 앞에서 발표하는 이유… 모두 같습니다. 나의 생각을 정리하고, 타인의 시선으로 보완점을 발견하기 위해서입니다.

    이번 시간에는 세 가지를 합니다.

    • 🎤 발표 — 내가 만든 것을 남에게 설명하면, 스스로도 더 깊이 이해하게 됩니다
    • ⭐ 시상 — 서로의 노력을 인정하면, 다음 프로젝트의 동기가 됩니다
    • 📝 성찰 — 과정을 돌아보면, 같은 실수를 반복하지 않게 됩니다

    이 세 가지는 단순한 "마무리"가 아니라, 프로젝트에서 가장 많이 배우는 시간입니다.

    📚 핵심 개념¶

    개념 1: 프로젝트 발표 구조 — 5단계 스토리라인¶

    1. 비유로 시작

    영화를 생각해보세요. 좋은 영화에는 항상 흐름이 있습니다. 문제가 생기고(갈등), 해결하려 노력하고(과정), 결말에 도달하죠(결과). 프로젝트 발표도 똑같습니다. 여러분의 프로젝트는 하나의 이야기입니다. "이런 문제가 있었고, 이렇게 풀었고, 이런 결과가 나왔다"라는 이야기를요.

    2. 정확한 정의

    프로젝트 발표 구조란, 청중이 프로젝트의 맥락과 가치를 빠르게 이해할 수 있도록 정보를 논리적 순서로 배치하는 틀입니다. 우리는 아래 5단계를 사용합니다.

    flowchart LR A["🔍 문제\n무엇을 해결하려 했나"] --> B["📊 데이터\n어떤 데이터를 모았나"] B --> C["🤖 모델\n어떻게 학습시켰나"] C --> D["🎯 결과물\n무엇이 완성됐나"] D --> E["💡 배운 점\n무엇을 깨달았나"]

    3. 예시로 확인

    예를 들어 '급식 잔반 분류기'를 만든 팀이라면 이렇게 발표합니다.

    • 문제: "우리 학교 급식 잔반이 분리수거가 안 되고 있어서, AI로 잔반 종류를 자동 분류하면 좋겠다고 생각했습니다"
    • 데이터: "밥, 국, 반찬, 과일 네 가지 카테고리로 총 120장의 사진을 찍어 Teachable Machine에 학습시켰습니다"
    • 모델: "처음엔 60%대 정확도였지만, 배경을 통일하고 사진 각도를 다양하게 추가해서 88%까지 올렸습니다"
    • 결과물: "웹캠에 잔반을 비추면 실시간으로 종류를 알려주는 프로그램을 완성했습니다"
    • 배운 점: "데이터의 질이 모델 성능을 결정한다는 걸 몸소 느꼈습니다"

    📌 알아보기: 3분 발표에서 가장 중요한 건 시간 배분입니다. 5단계에 각각 약 30~40초씩 할당하면 딱 3분 안에 끝낼 수 있습니다.

    개념 2: 동료 평가 — 공정한 피드백의 기술¶

    1. 비유로 시작

    여러분이 요리 대회 심사위원이라고 상상해보세요. "맛있다/맛없다"로만 평가하면 공정할까요? 아닙니다. "맛(30점), 플레이팅(30점), 창의성(20점), 위생(20점)" 이렇게 기준을 나눠야 누구나 납득할 수 있는 평가가 됩니다. 프로젝트 평가도 마찬가지입니다.

    2. 정확한 정의

    동료 평가(Peer Evaluation)란, 사전에 합의된 평가 기준표(루브릭)에 따라 동료의 작업을 체계적으로 평가하는 활동입니다. 주관적 감상이 아닌, 기준에 근거한 평가를 하는 것이 핵심입니다.

    우리가 사용할 평가 기준은 세 가지입니다.

    • 창의성 ⭐ — 주제 선정이 독창적인가? 다른 팀과 차별화된 아이디어가 있는가?
    • 완성도 ⭐ — 분류기가 실제로 잘 작동하는가? 발표 자료가 깔끔한가?
    • 협업도 ⭐ — 팀원 모두가 역할을 수행했는가? 발표에서 팀워크가 느껴지는가?

    3. 예시로 확인

    flowchart TB subgraph 평가기준["📋 동료 평가 3가지 기준"] A["🎨 창의성\n5점 만점\n주제가 독창적인가?\n새로운 시도가 있는가?"] B["🔧 완성도\n5점 만점\n분류기가 작동하는가?\n발표가 명확한가?"] C["🤝 협업도\n5점 만점\n팀원 모두 참여했는가?\n역할 분담이 보이는가?"] end A --> D["합계: 15점 만점"] B --> D C --> D

    각 항목을 1~5점으로 채점합니다.

    • 5점: "와, 정말 인상적이다!"
    • 4점: "잘했다, 기대 이상이다"
    • 3점: "무난하게 해냈다"
    • 2점: "아쉬운 부분이 꽤 있다"
    • 1점: "많이 부족하다"

    ⚠️ 중요: 자기 팀에는 점수를 매기지 않습니다. 그리고 "친한 팀이니까 높게"가 아니라, 기준에 따라 점수를 매기는 연습이 이번 활동의 핵심입니다.

    개념 3: 시상과 투표 — 서로의 노력을 인정하는 문화¶

    1. 비유로 시작

    올림픽에서 금메달만 중요한 게 아닙니다. 은메달, 동메달, 그리고 "페어플레이상", "감동상" 같은 특별상도 있죠. 모든 참가자의 노력에는 각기 다른 가치가 있기 때문입니다.

    2. 정확한 정의

    시상은 프로젝트 과정의 다양한 가치를 인정하고 축하하는 의식입니다. 단순히 "1등"을 가리는 것이 아니라, 여러 관점에서 뛰어난 팀을 발견하는 과정입니다.

    3. 예시로 확인

    우리 수업에서 선정하는 상은 다음과 같습니다.

    • 🏆 최우수 프로젝트상 — 동료 평가 총점 1위 팀
    • 💡 최고 창의상 — 창의성 부문 최고 점수 팀
    • 🔧 기술 장인상 — 완성도 부문 최고 점수 팀
    • 🤝 최고 협업상 — 협업도 부문 최고 점수 팀
    • 🌟 인기 투표상 — 전체 학생 투표로 "가장 인상 깊었던 발표" 1위 팀

    💡 알아보기: 모든 팀에 칭찬할 포인트가 있습니다. 발표를 들으면서 각 팀의 "가장 잘한 점" 하나를 메모해두세요. 그 한 마디 칭찬이 그 팀에겐 가장 큰 보상이 됩니다.

    개념 4: 성찰 일지 — 경험을 지식으로 바꾸는 기술¶

    1. 비유로 시작

    운동선수는 경기가 끝나면 반드시 경기 영상을 돌려봅니다. "이 장면에서 왜 이런 판단을 했지?", "다음엔 이렇게 해야겠다"라고 분석하죠. 이걸 복기(復棋)라고 합니다. 프로젝트의 복기가 바로 성찰 일지입니다.

    2. 정확한 정의

    성찰 일지(Reflection Journal)란, 프로젝트 수행 과정을 되돌아보며 배운 점, 어려웠던 점, 개선할 점을 체계적으로 기록하는 글쓰기 활동입니다. 단순한 감상문이 아니라, 구체적인 경험에서 교훈을 뽑아내는 것이 핵심입니다.

    3. 예시로 확인

    나쁜 성찰 일지 vs 좋은 성찰 일지를 비교해봅시다.

    flowchart LR subgraph BAD["❌ 나쁜 예시"] A1["재미있었다"] A2["어려웠다"] A3["다음엔 잘하고 싶다"] end subgraph GOOD["✅ 좋은 예시"] B1["데이터 라벨링할 때\n기준이 모호해서\n팀원마다 다르게 분류한 게\n가장 어려웠다"] B2["민수가 모델 학습 중\n에러를 찾아줘서\n3시간을 아꼈다.\n정말 감사했다"] B3["다음엔 라벨링 가이드를\n먼저 만들고 시작하겠다"] end BAD -- "구체적으로!" --> GOOD

    좋은 성찰 일지의 특징은 구체적인 상황 + 그때의 감정이나 판단 + 앞으로의 행동 계획이 포함되어 있다는 것입니다.

    🔨 따라하기¶

    이제 실제로 발표, 평가, 시상, 성찰을 순서대로 진행합니다. 아래 단계를 하나씩 따라가세요.

    Step 1: 발표 자료 최종 점검 (30분)¶

    발표 전에 최종 점검 시간입니다. 아래 체크리스트를 팀원들과 함께 확인하세요.

    발표 내용 점검

    • 5단계(문제→데이터→모델→결과물→배운 점)가 모두 포함되어 있는가?
    • 3분 안에 끝나는가? (타이머로 리허설 해보기!)
    • 전문 용어를 쓸 때 쉬운 설명을 덧붙였는가?
    • 실제 분류기 시연(데모) 준비가 되어 있는가?

    발표 기술 점검

    • 발표자가 정해져 있는가? (1명 또는 릴레이 형식)
    • 목소리 크기와 속도가 적절한가?
    • 화면을 보지 않고 청중을 보며 말할 수 있는가?
    • 팀원 모두가 최소 한 마디 이상 참여하는가?

    💡 꿀팁: 발표가 3분을 넘기면 강제 종료됩니다! 리허설에서 2분 40초에 맞추면 실전에서 약간 늘어나도 안전합니다.

    리허설 연습 코드 — 타이머 프로그램

    발표 연습할 때 사용할 간단한 타이머를 만들어봅시다.

    import time

def presentation_timer(team_name, total_seconds=180):
    """발표 타이머: 기본 3분(180초)"""
    print(f"🎤 [{team_name}] 발표를 시작합니다!")
    print(f"⏱️ 제한 시간: {total_seconds}초")
    print("-" * 40)

    start = time.time()

    # 30초 단위로 알림
    checkpoints = [30, 60, 90, 120, 150, 160, 170, 180]
    passed = set()

    try:
        while True:
            elapsed = int(time.time() - start)
            remaining = total_seconds - elapsed

            for cp in checkpoints:
                if elapsed >= cp and cp not in passed:
                    passed.add(cp)
                    if cp == 160:
                        print(f"  ⚠️ {remaining}초 남았습니다! 마무리하세요!")
                    elif cp == 170:
                        print(f"  🚨 {remaining}초! 거의 끝!")
                    elif cp >= total_seconds:
                        print(f"  🔴 시간 종료! 총 {elapsed}초 사용")
                        return elapsed
                    else:
                        print(f"  ⏱️ {elapsed}초 경과 / 남은 시간 {remaining}초")

            time.sleep(1)

    except KeyboardInterrupt:
        elapsed = int(time.time() - start)
        print(f"\n⏹️ 발표 중단! 총 {elapsed}초 사용")
        if elapsed <= total_seconds:
            print(f"✅ 시간 내에 완료! ({total_seconds - elapsed}초 여유)")
        else:
            print(f"❌ {elapsed - total_seconds}초 초과!")
        return elapsed

    동료 평가 점수 집계 프로그램

    모든 학생의 평가를 입력받아 자동으로 집계하는 프로그램입니다.

    def collect_evaluations():
    """동료 평가 수집 및 집계"""

    teams = []
    team_count = int(input("총 몇 개 팀이 발표했나요? "))

    for i in range(team_count):
        name = input(f"\n{i+1}번째 팀 이름: ")
        teams.append(name)

    print(f"\n✅ 등록된 팀: {teams}")
    print("=" * 50)

    # 각 팀별 점수 저장
    scores = {}
    for team in teams:
        scores[team] = {
            "creativity": [],    # 창의성
            "completion": [],    # 완성도
            "teamwork": []       # 협업도
        }

    # 평가 입력
    evaluator_count = int(input("\n평가에 참여하는 학생 수: "))

    for e in range(evaluator_count):
        evaluator_name = input(f"\n👤 {e+1}번째 평가자 이름: ")
        my_team = input(f"  {evaluator_name}의 소속 팀: ")

        for team in teams:
            if team == my_team:
                continue  # 자기 팀은 평가하지 않음

            print(f"\n  📋 [{team}] 평가 (1~5점)")
            c1 = int(input(f"    🎨 창의성: "))
            c2 = int(input(f"    🔧 완성도: "))
            c3 = int(input(f"    🤝 협업도: "))

            # 1~5 범위 확인
            c1 = max(1, min(5, c1))
            c2 = max(1, min(5, c2))
            c3 = max(1, min(5, c3))

            scores[team]["creativity"].append(c1)
            scores[team]["completion"].append(c2)
            scores[team]["teamwork"].append(c3)

    return teams, scores


def show_results(teams, scores):
    """결과 출력 및 시상"""

    print("\n" + "=" * 50)
    print("📊 동료 평가 결과")
    print("=" * 50)

    results = []

    for team in teams:
        s = scores[team]
        avg_c = sum(s["creativity"]) / len(s["creativity"]) if s["creativity"] else 0
        avg_p = sum(s["completion"]) / len(s["completion"]) if s["completion"] else 0
        avg_t = sum(s["teamwork"]) / len(s["teamwork"]) if s["teamwork"] else 0
        total = avg_c + avg_p + avg_t

        results.append({
            "team": team,
            "creativity": round(avg_c, 1),
            "completion": round(avg_p, 1),
            "teamwork": round(avg_t, 1),
            "total": round(total, 1)
        })

        print(f"\n🏷️ [{team}]")
        print(f"  🎨 창의성 평균: {avg_c:.1f}점")
        print(f"  🔧 완성도 평균: {avg_p:.1f}점")
        print(f"  🤝 협업도 평균: {avg_t:.1f}점")
        print(f"  📊 총점: {total:.1f}점 / 15점")

    # 시상
    print("\n" + "=" * 50)
    print("🏆 시상 결과")
    print("=" * 50)

    # 총점 1위
    best_overall = max(results, key=lambda x: x["total"])
    print(f"\n🏆 최우수 프로젝트상: [{best_overall['team']}] (총 {best_overall['total']}점)")

    # 창의성 1위
    best_creative = max(results, key=lambda x: x["creativity"])
    print(f"💡 최고 창의상: [{best_creative['team']}] (창의성 {best_creative['creativity']}점)")

    # 완성도 1위
    best_complete = max(results, key=lambda x: x["completion"])
    print(f"🔧 기술 장인상: [{best_complete['team']}] (완성도 {best_complete['completion']}점)")

    # 협업도 1위
    best_teamwork = max(results, key=lambda x: x["teamwork"])
    print(f"🤝 최고 협업상: [{best_teamwork['team']}] (협업도 {best_teamwork['teamwork']}점)")

    return results
