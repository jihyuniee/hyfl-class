    Chapter 7. AI와 진짜 협업하기 – 에이전트 활용 입문¶

    Part 2. AI와 대화하는 기술 – 프롬프트 엔지니어링 | 7차시 (2시간)

    🎯 이 장에서 배우는 것¶

    • AI 에이전트의 개념을 '자율적으로 여러 단계를 수행하는 AI'로 설명할 수 있다
    • 단순 챗봇과 AI 에이전트의 차이를 3가지 이상 구분할 수 있다
    • 다단계 프롬프트를 설계하여 하나의 대화 흐름으로 복합 작업을 완성할 수 있다
    • 구글 Gemini 등 AI 도구를 활용해 실제 학교 과제에 에이전트 방식을 적용할 수 있다

    💡 왜 이걸 배우나요?¶

    생각해보기 🤔

    학교 축제 준비를 맡았다고 상상해 보세요. "포스터 디자인 → 홍보 문구 작성 → SNS 게시물 제작 → 일정표 정리"를 혼자 해야 합니다. 이때 누군가에게 "축제 홍보 전체를 알아서 준비해 줘"라고 한 마디만 하면, 스스로 계획을 세우고 하나씩 처리해 주는 비서가 있다면 어떨까요?

    지금까지 우리는 AI에게 한 번에 하나의 질문을 하고 답을 받는 방법을 배웠습니다. 그런데 현실의 과제는 하나의 질문으로 끝나지 않죠. 여러 단계를 연결해서 처리해야 합니다.

    AI 에이전트는 바로 이 문제를 해결합니다. 한 마디 지시로 AI가 스스로 계획을 세우고, 여러 단계를 순서대로 실행하며, 필요하면 도구까지 활용하는 방식입니다. 이것을 이해하면 AI 활용 능력이 "질문하는 수준"에서 "협업하는 수준"으로 올라갑니다.

    📚 핵심 개념¶

    개념 1: AI 에이전트란 무엇인가? 🤖¶

    ① 비유로 시작

    식당에 비유해 봅시다.

    • 자판기 = 버튼 하나 → 음료 하나 (단순 입출력)
    • 카운터 직원 = 주문을 받고 → 해당 메뉴 하나를 전달 (챗봇)
    • 전담 매니저 = "오늘 저녁 모임 준비해 줘"라고 하면 → 메뉴 추천, 예약, 좌석 배치, 음식 주문까지 알아서 순서대로 처리 (에이전트)

    ② 정확한 정의

    AI 에이전트(AI Agent): 사용자의 목표를 이해하고, 스스로 계획을 수립하여, 여러 단계의 작업을 자율적으로 수행하는 AI 시스템. 필요시 외부 도구(검색, 계산, 파일 생성 등)를 활용하기도 한다.

    핵심 키워드 세 가지를 기억하세요.

    • 자율성 — 사람이 매 단계마다 지시하지 않아도 스스로 다음 할 일을 판단
    • 계획 수립 — 목표를 달성하기 위한 단계를 스스로 나눔
    • 도구 활용 — 필요하면 검색, 코드 실행, 파일 생성 등 외부 기능을 사용

    ③ 예시로 확인

    "다음 주 영어 발표를 준비해 줘"라고 에이전트에게 말하면:

    1. 발표 주제 3개를 추천하고
    2. 선택한 주제로 개요를 작성하고
    3. 개요를 바탕으로 영어 대본 초안을 쓰고
    4. 어려운 단어의 발음 가이드까지 제공

    이 모든 것을 하나의 대화 흐름 안에서 처리합니다.

    개념 2: 챗봇 vs 에이전트 – 무엇이 다른가? ⚖️¶

    ① 비유로 시작

    • 챗봇 = 도서관 안내 데스크. "이 책 어디 있어요?" → "3층 A구역입니다." (질문 하나, 답변 하나)
    • 에이전트 = 개인 과외 선생님. "시험 준비 도와줘" → 범위 분석, 취약점 파악, 공부 계획 수립, 문제 출제까지 연속 수행

    ② 정확한 정의

    flowchart LR subgraph 챗봇["🗨️ 챗봇"] A1[사용자 질문] --> A2[AI 답변] A2 --> A3[끝] end subgraph 에이전트["🤖 에이전트"] B1[사용자 목표] --> B2[AI가 계획 수립] B2 --> B3[1단계 실행] B3 --> B4[2단계 실행] B4 --> B5[3단계 실행] B5 --> B6[결과 종합 전달] end

    둘의 차이를 핵심 기준으로 정리하면:

    • 대화 방식 — 챗봇은 "1문 1답", 에이전트는 "1목표 → 다단계 처리"
    • 주도권 — 챗봇은 사용자가 매번 지시, 에이전트는 AI가 스스로 다음 단계 진행
    • 도구 사용 — 챗봇은 텍스트 응답만, 에이전트는 검색·계산·파일 생성 등 외부 도구 활용 가능
    • 기억과 맥락 — 챗봇은 개별 질문 단위, 에이전트는 전체 목표를 기억하며 일관되게 작업

    ③ 예시로 확인

    같은 요청, 다른 결과:

    요청: "환경 오염에 대한 보고서를 써 줘"

    챗봇 방식: 환경 오염에 대한 일반적인 글 하나를 생성하고 끝.

    에이전트 방식: 1. "어떤 측면을 다룰까요? (대기오염/수질오염/토양오염)" 확인 2. 선택한 주제로 목차 구성 3. 각 섹션별 내용 작성 4. 참고자료 목록 제시 5. 최종 보고서 형태로 정리

    개념 3: 에이전트의 작동 원리 – 생각-행동-관찰 루프 🔄¶

    ① 비유로 시작

    요리사가 새로운 레시피에 도전하는 과정을 떠올려 보세요.

    1. 생각: "먼저 재료를 확인해야겠다"
    2. 행동: 냉장고를 열어서 재료를 확인한다
    3. 관찰: "토마토는 있는데 바질이 없네"
    4. 생각(다시): "바질 대신 깻잎을 써볼까?"
    5. 행동: 깻잎으로 대체해서 조리 시작
    6. 이 과정을 요리가 완성될 때까지 반복

    AI 에이전트도 정확히 이 패턴으로 작동합니다.

    ② 정확한 정의

    이것을 ReAct 패턴(Reasoning + Acting)이라고 부릅니다.

    flowchart TD A["🎯 사용자 목표 입력"] --> B["🧠 생각 Reasoning"] B --> C["⚡ 행동 Acting"] C --> D["👁️ 관찰 Observation"] D --> E{목표 달성?} E -->|아니오| B E -->|예| F["✅ 최종 결과 전달"]
    • 생각(Reasoning): 현재 상황을 분석하고 다음에 무엇을 해야 할지 판단
    • 행동(Acting): 실제로 텍스트 생성, 검색, 계산 등을 실행
    • 관찰(Observation): 행동의 결과를 확인하고 목표에 얼마나 가까워졌는지 평가

    ③ 예시로 확인

    "우리 반 체육대회 응원 구호를 만들어 줘"라고 요청하면:

    1. 🧠 생각: 반 이름, 특징, 분위기를 알아야겠다
    2. ⚡ 행동: 사용자에게 반 정보를 질문
    3. 👁️ 관찰: "2학년 3반, 파란색, 활발한 분위기"라는 정보 획득
    4. 🧠 생각: 파란색과 활발함을 살린 구호 5개를 만들자
    5. ⚡ 행동: 구호 5개 생성
    6. 👁️ 관찰: 사용자가 3번이 좋다고 함
    7. ⚡ 행동: 3번 구호를 다듬고 율동 제안까지 추가

    개념 4: 에이전트처럼 프롬프트 설계하기 📝¶

    ① 비유로 시작

    좋은 비서를 부리려면 좋은 지시를 내려야 합니다. "알아서 해"보다 "목표는 이것이고, 단계별로 이렇게 진행해 줘"라고 말하는 것이 훨씬 좋은 결과를 만듭니다.

    ② 정확한 정의

    다단계 프롬프트(Multi-step Prompt): 최종 목표와 함께 중간 단계를 명시하여, AI가 에이전트처럼 체계적으로 작업을 수행하도록 유도하는 프롬프트 설계 방식.

    ③ 핵심 구조

    좋은 에이전트형 프롬프트의 구성 요소:

    • 역할 부여: "너는 ~전문가야"
    • 최종 목표: "최종적으로 ~을 만들어 줘"
    • 단계 지정: "다음 순서로 진행해 줘: 1단계 → 2단계 → 3단계"
    • 출력 형식: "각 단계의 결과를 ~형태로 보여 줘"
    • 확인 요청: "각 단계가 끝나면 다음으로 넘어가기 전에 나에게 확인해 줘"

    알아보기 📌

    "확인 요청"이 왜 중요한가요? 에이전트가 엉뚱한 방향으로 가는 것을 중간에 잡아줄 수 있기 때문입니다. 사람과 AI의 협업이 바로 이 지점에서 일어납니다. 완전 자동이 아니라 반자동(사람이 중간 검수)이 현실적으로 가장 좋은 결과를 냅니다.

    🔨 따라하기: Gemini로 영어 발표 준비하기¶

    이 실습에서는 Google Gemini(gemini.google.com)를 사용합니다. ChatGPT, Claude 등 다른 AI 도구를 사용해도 됩니다.

    Step 1: 에이전트형 프롬프트 작성하기 ✍️¶

    아래 프롬프트를 복사하여 Gemini에 입력하세요.

    너는 영어 발표 준비를 도와주는 전문 코치야.
나는 고등학생이고, 다음 주에 3분짜리 영어 발표를 해야 해.

다음 순서대로 진행해 줘:

[1단계] 발표 주제 추천: 고등학생이 흥미를 가질 만한 주제 3개를 추천하고, 
각 주제의 장단점을 간단히 설명해 줘.

[2단계] 개요 작성: 내가 주제를 선택하면, 서론-본론-결론 구조로 개요를 작성해 줘.

[3단계] 영어 대본 초안: 개요를 바탕으로 3분 분량(약 400단어)의 영어 대본을 작성해 줘.

[4단계] 발음 가이드: 대본에서 발음하기 어려운 단어 5개를 골라 한글 발음과 함께 알려 줘.

각 단계가 끝나면 다음으로 넘어가기 전에 나에게 확인해 줘.
지금 1단계부터 시작해 줘.

    Step 2: 1단계 결과 확인하고 주제 선택하기 🎯¶

    Gemini가 3개의 주제를 추천하면, 그중 하나를 선택해서 답장하세요.

    2번 주제가 좋아. 2단계로 넘어가 줘.

    생각해보기 💭

    이 순간이 바로 사람과 AI의 협업 지점입니다. AI가 스스로 판단하지 않고, 여러분의 선택을 기다렸죠? 이것이 "완전 자동화"가 아닌 "반자동 협업"의 핵심입니다.

    Step 3: 개요를 검토하고 수정 요청하기 🔍¶

    개요를 받으면 내용을 검토합니다. 마음에 들지 않는 부분이 있으면 수정을 요청하세요.

    개요 좋아! 근데 본론에 개인 경험 사례를 하나 추가해 줘. 
그리고 3단계로 넘어가 줘.

    Step 4: 대본 초안 받기 📄¶

    3분 분량의 영어 대본이 생성됩니다. 이때 중요한 것은 그대로 사용하지 않고 자신의 스타일로 수정하는 것입니다.

    대본 고마워! 그런데 좀 더 캐주얼한 톤으로 바꿔 줘. 
친구들 앞에서 발표하는 느낌으로.

    Step 5: 발음 가이드 받고 연습하기 🗣️¶

    좋아, 4단계 발음 가이드도 부탁해!

    Gemini가 어려운 단어의 발음을 한글로 표기해 줍니다.

    Step 6: 전체 과정 되돌아보기 🔄¶

    마지막으로, 전체 작업을 정리해 달라고 요청합니다.

    지금까지 작업한 내용을 전체 정리해 줘.
주제, 개요, 대본, 발음 가이드를 한눈에 볼 수 있게.

    정리하기 📋

    방금 여러분은 6단계에 걸친 에이전트형 협업을 완료했습니다. - 1회의 대화가 아니라 연속된 대화 흐름으로 작업했고 - AI가 단계별로 자율 실행했으며 - 여러분이 중간에 방향을 조정했습니다

    이것이 바로 에이전트 활용의 핵심입니다!

    📝 전체 코드 (프롬프트 템플릿)¶

    실습에서 사용한 에이전트형 프롬프트의 범용 템플릿입니다. 다른 과제에도 활용할 수 있습니다.

    너는 [역할]이야.

[1단계] [작업 이름]: [구체적 지시]
[2단계] [작업 이름]: [구체적 지시]

조건:
- 각 단계가 끝나면 다음으로 넘어가기 전에 나에게 확인해 줘.
- [추가 조건이 있다면 여기에]

지금 1단계부터 시작해 줘.

    활용 예시 — 역사 보고서 작성:

    너는 역사 보고서 작성을 도와주는 전문 튜터야.
나는 고등학생이고, 한국 독립운동에 대한 보고서를 작성해야 해.

다음 순서대로 진행해 줘:

[1단계] 세부 주제 추천: 독립운동 관련 세부 주제 3개를 추천하고 각각의 특징을 설명해 줘.
[2단계] 자료 조사: 내가 선택한 주제에 대해 핵심 사건, 인물, 의의를 정리해 줘.
[3단계] 보고서 구조: 서론-본론-결론 형태의 보고서 초안을 작성해 줘.
[4단계] 검토: 보고서에서 보완할 점을 스스로 찾아서 알려 줘.

각 단계가 끝나면 다음으로 넘어가기 전에 나에게 확인해 줘.
지금 1단계부터 시작해 줘.

    ⚠️ 자주 하는 실수¶

    실수 1: "알아서 다 해 줘"라고만 말하기 🚫¶

    잘못된 예: "영어 발표 준비해 줘"

    문제점: AI가 어떤 수준의, 어떤 형태의 결과를 원하는지 모릅니다. 엉뚱한 방향으로 갈 확률이 높아요.

    해결법: 단계를 명시하고, 각 단계에서 원하는 결과물의 형태를 구체적으로 알려 주세요.

    실수 2: 중간 확인 없이 끝까지 달리기 🏃¶

    잘못된 예: 4단계를 모두 한꺼번에 실행하도록 지시

    문제점: 1단계에서 방향이 잘못되면 나머지 3단계가 모두 엉뚱한 결과를 냅니다. 전부 다시 해야 하죠.

    해결법: "각 단계가 끝나면 확인해 줘"를 반드시 포함하세요. 중간에 방향을 수정하는 것이 훨씬 효율적입니다.

    실수 3: AI가 만든 결과를 그대로 제출하기 📋¶

    잘못된 예: Gemini가 작성한 영어 대본을 수정 없이 그대로 발표 원고로 사용

    문제점: 여러분의 어휘 수준, 말투, 경험이 반영되지 않아 부자연스럽습니다. 무엇보다 학습 효과가 없습니다.

    해결법: AI의 결과물은 초안(draft)으로 활용하고, 반드시 자신의 언어로 수정·보완하세요. AI는 도구이지 대신 시험을 봐주는 존재가 아닙니다.

    실수 4: 에이전트와 챗봇을 구분하지 못하기 🤷¶

    "그냥 여러 번 질문하면 에이전트 아닌가요?"라고 생각할 수 있습니다.

    핵심 차이: 여러 번 질문하되 AI가 전체 목표를 기억하고, 이전 단계의 결과를 바탕으로 다음 단계를 자율 진행하면 에이전트 방식입니다. 반면 매번 새로운 질문을 독립적으로 하는 것은 챗봇 방식입니다.

    기본 문제 📗¶

    문제 1. 다음 중 AI 에이전트의 특징이 아닌 것을 고르세요.

    • ㄱ) 사용자가 매 단계마다 직접 지시해야만 작동한다
    • ㄴ) 최종 목표를 이해하고 스스로 계획을 수립한다
    • ㄷ) 필요시 검색, 계산 등 외부 도구를 활용한다
    • ㄹ) 여러 단계를 연속으로 수행한다
    정답 확인 **ㄱ)** 이 정답입니다. 에이전트는 매 단계마다 사용자가 지시하지 않아도 **자율적으로** 다음 단계를 진행합니다. 매번 지시가 필요한 것은 챗봇 방식에 가깝습니다.

    문제 2. 다음 시나리오가 챗봇 방식인지 에이전트 방식인지 분류하세요.

    • (a) "수학 문제 풀어 줘" → AI가 답을 알려줌 → 끝
    • (b) "기말고사 수학 공부 계획 세워 줘" → AI가 범위를 분석하고, 주차별 계획을 세우고, 연습문제까지 출제
    • (c) "이 영어 단어 뜻이 뭐야?" → AI가 뜻을 알려줌 → 끝
    • (d) "영어 에세이를 완성해 줘" → AI가 주제 선정, 아웃라인, 초안, 퇴고까지 순서대로 진행
    정답 확인 - (a) **챗봇** — 1문 1답 - (b) **에이전트** — 다단계 자율 수행 - (c) **챗봇** — 1문 1답 - (d) **에이전트** — 다단계 자율 수행
