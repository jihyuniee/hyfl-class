    제19장. 프로젝트 발표회 – 우리가 만든 솔루션¶

    🎯 이 장에서 배우는 것¶

    • 팀 프로젝트의 전 과정(문제 발견 → 설계 → 구현 → 테스트)을 발표를 통해 효과적으로 전달할 수 있다
    • 다른 팀의 발표를 경청하고 세 가지 기준에 따라 건설적인 피드백을 제공할 수 있다
    • 질의응답에서 자신의 프로젝트를 논리적으로 방어하고 설명할 수 있다
    • 동료 평가와 자기 평가를 통해 프로젝트를 객관적으로 돌아볼 수 있다

    💡 왜 이걸 배우나요?¶

    여러분은 지금까지 18차시에 걸쳐 정말 긴 여정을 걸어왔습니다. 데이터를 이해하고, AI의 원리를 배우고, 직접 문제를 정의하고, 솔루션을 설계하고, 코드로 구현했습니다.

    그런데 아무리 뛰어난 솔루션도 다른 사람에게 전달되지 못하면 세상을 바꿀 수 없습니다.

    🎬 생각해보기: 스티브 잡스가 아이폰을 처음 세상에 공개한 2007년 프레젠테이션을 떠올려 보세요. 아이폰이 아무리 혁신적이어도, 그 발표가 없었다면 사람들은 아이폰의 가치를 이해하지 못했을 것입니다.

    프로젝트 발표는 단순히 "우리가 뭘 했는지 보여주는 시간"이 아닙니다. 이것은 여러분이 발견한 문제와 해결책을 세상에 설득하는 과정입니다. 이 능력은 대학 입시 면접, 취업 면접, 사업 제안, 연구 발표 등 인생 전반에서 반복적으로 필요합니다.

    오늘은 여러분이 발표자이자 청중이자 평가자가 되는 특별한 시간입니다.

    📚 핵심 개념¶

    개념 1: 프로젝트 발표의 구조 – "스토리 아크"¶

    1. 비유로 시작

    영화를 떠올려 보세요. 좋은 영화에는 반드시 구조가 있습니다. 평화로운 일상 → 문제 발생 → 주인공의 도전 → 위기와 극복 → 해결. 프로젝트 발표도 하나의 이야기입니다.

    2. 정확한 정의

    프로젝트 발표의 스토리 아크(Story Arc)란, 청중이 자연스럽게 따라올 수 있도록 문제 → 탐색 → 해결 → 증명 순서로 내용을 배치하는 발표 구조입니다.

    3. 예시로 확인

    7~10분 발표를 다음과 같이 구성합니다.

    flowchart LR A["🔍 문제 발견\n(1~2분)"] --> B["🧠 설계\n(2분)"] B --> C["💻 구현\n(2~3분)"] C --> D["📊 결과와 의미\n(2분)"]
    • 문제 발견 (1~2분): "우리가 발견한 문제는 이것이고, 이것이 왜 중요한지"
    • 설계 (2분): "이 문제를 AI로 이렇게 접근했고, 이런 데이터를 활용했다"
    • 구현 (2~3분): "실제로 이렇게 만들었고, 핵심 기능은 이것이다" (데모 포함)
    • 결과와 의미 (2분): "결과는 이러하고, 한계점은 이것이며, 앞으로 이렇게 발전시킬 수 있다"

    📌 알아보기: 시간 배분이 핵심입니다. 초보자가 가장 많이 하는 실수는 문제 설명에 5분을 쓰고 정작 구현 결과를 30초 만에 끝내는 것입니다.

    개념 2: 동료 평가의 세 가지 기준¶

    1. 비유로 시작

    요리 대회의 심사위원을 떠올려 보세요. "맛있다/맛없다"로만 평가하면 요리사에게 아무런 도움이 되지 않습니다. 좋은 심사위원은 "재료 선택", "조리 기술", "플레이팅"처럼 구체적인 기준으로 피드백합니다.

    2. 정확한 정의

    동료 평가(Peer Evaluation)란, 같은 학습 과정에 참여한 동료가 사전에 합의된 명확한 기준에 따라 서로의 결과물을 평가하고 피드백을 제공하는 활동입니다.

    3. 예시로 확인

    오늘 발표에서 사용할 세 가지 평가 기준은 다음과 같습니다.

    flowchart TB subgraph 평가기준["📋 동료 평가 3대 기준"] A["🔍 문제 정의의 명확성\n문제가 구체적이고\n해결할 가치가 있는가?"] B["🤖 AI 활용의 적절성\nAI를 꼭 써야 하는 문제인가?\nAI를 효과적으로 활용했는가?"] C["💡 해결책의 창의성\n기존과 다른 접근인가?\n독창적 아이디어가 있는가?"] end

    각 기준에 대해 1~5점으로 점수를 매기되, 반드시 점수의 이유를 한 문장으로 적어야 합니다. 점수 없는 이유는 감정이고, 이유 없는 점수는 무의미합니다.

    💬 생각해보기: "AI 활용의 적절성"이라는 기준이 왜 중요할까요? 모든 문제에 AI가 필요한 것은 아닙니다. "AI를 쓰기 위해 억지로 AI를 붙인 프로젝트"와 "이 문제는 AI가 있어야 비로소 풀 수 있는 프로젝트"는 전혀 다릅니다.

    개념 3: 질의응답 – 논리적 방어의 기술¶

    1. 비유로 시작

    성(城)을 지키는 장군을 상상해 보세요. 적이 어디서 공격할지 모르지만, 좋은 장군은 성의 약점을 자기가 가장 잘 알고 있고, 그 약점에 대한 대비책을 미리 마련합니다. 질의응답도 마찬가지입니다.

    2. 정확한 정의

    질의응답에서의 논리적 방어란, 프로젝트의 강점과 약점을 모두 이해한 상태에서, 질문에 대해 솔직하고 근거 있는 답변을 제공하는 것입니다. 모르는 것을 아는 척하는 것이 아니라, 모르는 것은 인정하되 자신이 고민한 과정을 보여주는 것입니다.

    3. 예시로 확인

    질문에 답변하는 세 가지 유형별 전략입니다.

    • "왜 이 방법을 선택했나요?" → 대안을 비교한 과정을 설명합니다. "A, B, C 방법을 검토했는데, 우리 데이터 특성상 B가 가장 적합했습니다."
    • "이 부분의 한계는 무엇인가요?" → 솔직히 인정하고 개선 방향을 제시합니다. "맞습니다. 현재 정확도가 78%인데, 데이터를 더 수집하면 개선할 수 있다고 생각합니다."
    • "모르는 영역의 질문" → "좋은 질문입니다. 그 부분은 아직 검토하지 못했는데, 향후 연구 과제로 삼겠습니다."

    ⚠️ 절대 하면 안 되는 답변: "그냥 그렇게 했어요", "잘 모르겠어요" (고민 과정 설명 없이)

    개념 4: 건설적 피드백의 기술 – DESC 모델¶

    1. 비유로 시작

    친구가 그린 그림을 보고 "별로야"라고 말하는 것과 "색감은 정말 좋은데, 여기 원근감을 조금 조절하면 더 깊이감이 생길 것 같아"라고 말하는 것은 완전히 다릅니다. 둘 다 개선점을 지적하지만, 후자만이 성장에 도움이 됩니다.

    2. 정확한 정의

    DESC 모델은 건설적 피드백을 위한 구조입니다.

    • D(Describe, 묘사): 관찰한 사실을 구체적으로 말합니다
    • E(Express, 표현): 그것에 대한 자신의 생각이나 느낌을 표현합니다
    • S(Suggest, 제안): 구체적인 개선 방안을 제안합니다
    • C(Consequence, 결과): 개선했을 때 기대되는 효과를 말합니다

    3. 예시로 확인

    나쁜 피드백: "발표가 좀 지루했어요."

    DESC 피드백: "데이터 설명 부분에서 숫자가 많이 나열되었는데(D), 청중으로서 핵심이 무엇인지 파악하기 어려웠습니다(E). 가장 중요한 수치 2~3개만 강조하고 나머지는 부록으로 빼면(S), 메시지가 훨씬 명확해질 것 같습니다(C)."

    🔨 따라하기: 발표회 진행 가이드¶

    이 섹션은 발표회를 실제로 진행하는 절차입니다. 선생님 없이도 학생들이 자율적으로 운영할 수 있도록 구성했습니다.

    Step 1: 발표회 준비 (15분)¶

    발표 순서를 정합니다. 가장 공정한 방법은 추첨입니다. 파이썬으로 간단히 할 수 있습니다.

    import random

# 팀 이름을 입력하세요
teams = ["Team Alpha", "Team Beta", "Team Gamma", "Team Delta"]

# 랜덤 셔플로 발표 순서 결정
random.shuffle(teams)

print("🎤 발표 순서:")
for i, team in enumerate(teams, 1):
    print(f"  {i}번째: {team}")

    각 팀은 발표 전 마지막 점검을 합니다.

    📋 발표 전 체크리스트: - 슬라이드 또는 화면 공유 준비 완료? - 데모 실행 환경 테스트 완료? - 발표 시간 7~10분 내 리허설 완료? - 팀원 각자의 역할(누가 어떤 파트를 말할지) 확정?

    Step 2: 동료 평가지 세팅 (5분)¶

    각 학생은 다음 형식으로 평가를 준비합니다. 종이에 써도 되고, 아래 코드를 실행해서 디지털로 기록해도 됩니다.

    # 동료 평가 기록 도구
def create_evaluation_form():
    """발표를 듣고 평가를 기록합니다."""

    team_name = input("평가할 팀 이름: ")

    print(f"\n--- {team_name} 평가 ---")
    print("각 항목에 1~5점을 매기고, 이유를 적어주세요.\n")

    criteria = [
        "문제 정의의 명확성",
        "AI 활용의 적절성", 
        "해결책의 창의성"
    ]

    results = {}
    for criterion in criteria:
        score = int(input(f"  [{criterion}] 점수 (1~5): "))
        reason = input(f"  [{criterion}] 이유: ")
        results[criterion] = {"score": score, "reason": reason}

    # 추가 피드백 (DESC 모델)
    print("\n📝 DESC 피드백을 작성하세요:")
    strength = input("  가장 인상적이었던 점: ")
    suggestion = input("  개선 제안 (구체적으로): ")

    results["strength"] = strength
    results["suggestion"] = suggestion

    print(f"\n✅ {team_name} 평가 완료!")
    return team_name, results

# 실행
# team, evaluation = create_evaluation_form()

    Step 3: 발표 진행 (팀당 10~13분)¶

    각 팀의 발표는 다음 흐름으로 진행됩니다.

    flowchart LR A["발표\n7~10분"] --> B["질의응답\n3분"] B --> C["동료 평가\n작성 1분"]

    ⏱️ 타이머 역할이 중요합니다. 한 명이 타이머를 맡아, 7분에 1회, 9분에 1회, 10분에 최종 알림을 줍니다.

    import time

def presentation_timer(team_name):
    """발표 타이머: 7분, 9분, 10분에 알림"""

    print(f"🎤 {team_name} 발표 시작!")
    print("=" * 40)

    alerts = {
        420: "⏰ 7분 경과 - 마무리를 준비하세요",
        540: "⚠️ 9분 경과 - 1분 남았습니다!",
        600: "🛑 10분! 발표를 마무리해주세요"
    }

    start = time.time()
    alerted = set()

    while True:
        elapsed = int(time.time() - start)

        for seconds, message in alerts.items():
            if elapsed >= seconds and seconds not in alerted:
                print(f"\n{'='*40}")
                print(f"  {message}")
                print(f"{'='*40}\n")
                alerted.add(seconds)

        if elapsed >= 600:
            break
        time.sleep(1)

    print(f"\n✅ {team_name} 발표 종료. 질의응답 시간입니다!")

    Step 4: 질의응답 실전 연습 (팀당 3분)¶

    질문하는 법도 실력입니다. 좋은 질문의 세 가지 유형을 기억하세요.

    • 확인 질문: "~라고 이해했는데, 맞나요?"
    • 심화 질문: "만약 ~한 상황이라면 어떻게 되나요?"
    • 대안 질문: "~한 방법도 고려해 보셨나요?"

    💬 생각해보기: "왜 다른 방법을 안 쓰셨어요?"는 공격적으로 들릴 수 있습니다. 같은 내용을 "다른 방법도 검토하셨을 것 같은데, 이 방법을 최종 선택한 결정적 이유가 궁금합니다"로 바꾸면 훨씬 건설적입니다.

    Step 5: 전체 투표와 시상 (10분)¶

    모든 팀의 발표가 끝나면, 전체 투표를 진행합니다.

    def vote_and_award(teams):
    """발표회 투표 및 시상"""

    awards = {
        "most_innovative": "🏆 가장 혁신적인 솔루션",
        "best_ai_usage": "🤖 가장 효과적인 AI 활용",
        "best_presentation": "🎤 최고의 발표"
    }

    results = {award: {} for award in awards}

    print("📊 투표를 시작합니다!")
    print("각 부문별로 가장 뛰어났던 팀을 선택하세요.")
    print(f"참가 팀: {', '.join(teams)}\n")

    # 투표 수집 (실제로는 여러 학생이 투표)
    num_voters = int(input("투표 참여 인원 수: "))

    for i in range(num_voters):
        print(f"\n--- 투표자 {i+1} ---")
        for award_key, award_name in awards.items():
            vote = input(f"  {award_name}: ")
            if vote in teams:
                results[award_key][vote] = results[award_key].get(vote, 0) + 1

    # 결과 발표
    print("\n" + "🎉" * 20)
    print("       📢 투표 결과 발표!")
    print("🎉" * 20 + "\n")

    for award_key, award_name in awards.items():
        if results[award_key]:
            winner = max(results[award_key], key=results[award_key].get)
            votes = results[award_key][winner]
            print(f"  {award_name}")
            print(f"    👑 수상: {winner} ({votes}표)")
            print()

    print("모든 팀에게 박수를! 👏👏👏")

    Step 6: 자기 평가 작성 (10분)¶

    발표가 모두 끝난 후, 각자 자기 평가를 작성합니다. 이것은 다음 장(20차시)의 성찰 활동과 직접 연결됩니다.

    def self_evaluation():
    """프로젝트 자기 평가"""

    print("📝 자기 평가를 작성합니다.\n")

    questions = [
        "1. 프로젝트에서 내가 가장 잘한 점은?",
        "2. 프로젝트에서 가장 어려웠던 점은?",
        "3. 다시 한다면 바꾸고 싶은 점은?",
        "4. 이 프로젝트를 통해 새롭게 배운 것은?",
        "5. 우리 팀 발표에서 개선할 점은?"
    ]

    answers = {}
    for q in questions:
        print(q)
        answer = input("  → ")
        answers[q] = answer
        print()

    # 점수 자기 평가
    print("아래 항목에 1~5점을 매겨주세요:")
    scores = {
        "팀 기여도": int(input("  팀에 대한 나의 기여도 (1~5): ")),
        "학습 성장": int(input("  이 과정에서 나의 성장 (1~5): ")),
        "발표 만족도": int(input("  발표 결과에 대한 만족도 (1~5): "))
    }

    print("\n✅ 자기 평가 완료!")
    print(f"  평균 점수: {sum(scores.values()) / len(scores):.1f}")

    return answers, scores

    📝 전체 코드: 발표회 운영 통합 시스템¶

    아래 코드는 발표회 전체를 하나의 프로그램으로 운영할 수 있는 통합 코드입니다.

    import random
import time
from datetime import datetime

class PresentationDay:
    """프로젝트 발표회 운영 시스템"""

    def __init__(self, teams):
        self.teams = teams
        self.evaluations = {}  # {평가자: {팀: 평가내용}}
        self.votes = {"innovative": {}, "ai_usage": {}, "presentation": {}}

    def shuffle_order(self):
        """발표 순서 추첨"""
        order = self.teams.copy()
        random.shuffle(order)
        print("\n🎲 발표 순서 추첨 결과:")
        for i, team in enumerate(order, 1):
            print(f"  {i}. {team}")
        self.presentation_order = order
        return order

    def record_evaluation(self, evaluator, team, scores, feedback):
        """동료 평가 기록

        scores: {"clarity": 4, "ai_usage": 5, "creativity": 3}
        feedback: {"strength": "...", "suggestion": "..."}
        """
        if evaluator not in self.evaluations:
            self.evaluations[evaluator] = {}

        self.evaluations[evaluator][team] = {
            "scores": scores,
            "feedback": feedback,
            "timestamp": datetime.now().strftime("%H:%M")
        }
        print(f"✅ {evaluator}의 {team} 평가가 기록되었습니다.")

    def record_vote(self, voter, innovative, ai_usage, presentation):
        """투표 기록"""
        for category, team in [
            ("innovative", innovative),
            ("ai_usage", ai_usage),
            ("presentation", presentation)
        ]:
            self.votes[category][team] = self.votes[category].get(team, 0) + 1
        print(f"✅ {voter}의 투표가 기록되었습니다.")

    def announce_results(self):
        """결과 발표"""
        print("\n" + "=" * 50)
        print("        🏆 발표회 결과 발표 🏆")
        print("=" * 50)

        award_names = {
            "innovative": "🌟 가장 혁신적인 솔루션",
            "ai_usage": "🤖 가장 효과적인 AI 활용",
            "presentation": "🎤 최고의 발표"
        }

        for key, name in award_names.items():
            if self.votes[key]:
                winner = max(self.votes[key], key=self.votes[key].get)
                count = self.votes[key][winner]
                print(f"\n  {name}")
                print(f"  👑 {winner} ({count}표)")

        # 팀별 평균 점수 계산
        print(f"\n{'=' * 50}")
        print("  📊 팀별 동료 평가 평균")
        print(f"{'=' * 50}")

        for team in self.teams:
            scores_list = []
            for evaluator in self.evaluations:
                if team in self.evaluations[evaluator]:
                    s = self.evaluations[evaluator][team]["scores"]
                    avg = sum(s.values()) / len(s)
                    scores_list.append(avg)

            if scores_list:
                total_avg = sum(scores_list) / len(scores_list)
                print(f"  {team}: {total_avg:.1f} / 5.0")

        print(f"\n{'=' * 50}")
        print("  👏 모든 팀의 노력에 박수를 보냅니다!")
        print(f"{'=' * 50}")


# === 사용 예시 ===
teams = ["AI 건강지킴이", "스마트 분리수거", "감정 일기장", "소음 탐정단"]

# 발표회 시스템 생성
event = PresentationDay(teams)

# 1. 순서 추첨
event.shuffle_order()

# 2. 동료 평가 기록 예시
event.record_evaluation(
    evaluator="김학생",
    team="스마트 분리수거",
    scores={"clarity": 4, "ai_usage": 5, "creativity": 4},
    feedback={
        "strength": "이미지 분류 모델을 실제 분리수거에 적용한 점이 인상적",
        "suggestion": "다양한 쓰레기 종류에 대한 테스트 결과를 더 보여주면 좋겠음"
    }
)

# 3. 투표 예시
event.record_vote("김학생", "감정 일기장", "스마트 분리수거", "AI 건강지킴이")
event.record_vote("이학생", "스마트 분리수거", "스마트 분리수거", "감정 일기장")
event.record_vote("박학생", "감정 일기장", "AI 건강지킴이", "감정 일기장")

# 4. 결과 발표
event.announce_results()

    실수 1: "우리가 한 것 전부"를 보여주려 한다

    7분 안에 모든 것을 말할 수는 없습니다. 가장 핵심적인 메시지 1~2개에 집중하세요. 나머지는 질의응답에서 다루면 됩니다. 프레젠테이션은 "많이 보여주는 것"이 아니라 "기억에 남기는 것"이 목표입니다.
