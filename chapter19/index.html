    Chapter 19: 최종 프로젝트 — 웹앱 제작과 사용자 테스트¶

    Part 6 | [2권] 최종 프로젝트 — AI로 세상의 문제를 해결하라

    🎯 이 장에서 배우는 것¶

    • Google Sites를 활용하여 AI 모델이 연동된 웹앱을 제작할 수 있다
    • 프로젝트 소개 페이지, AI 기능 체험 페이지, 사용 설명서 페이지를 구성할 수 있다
    • 실제 사용자(베타 테스터)를 초대하여 테스트를 진행하고 피드백을 수집할 수 있다
    • 수집된 피드백을 분류·분석하여 웹앱을 개선할 수 있다

    💡 왜 이걸 배우나요?¶

    여러분이 지금까지 만들어 온 AI 모델은 "내 컴퓨터에서만 돌아가는 시제품" 상태입니다. 아무리 뛰어난 기능이라도 다른 사람이 쉽게 접근하지 못하면 세상의 문제를 해결할 수 없습니다.

    🍕 피자 가게를 떠올려 보세요. 맛있는 레시피를 개발했다면(= AI 모델 완성), 이제 가게 인테리어를 꾸미고(= 웹앱 제작), 시식 이벤트를 열어 손님 반응을 살피고(= 사용자 테스트), 의견을 반영해 메뉴를 개선하는(= 피드백 반영) 단계가 필요합니다.

    이번 장에서는 Google Sites라는 무료 웹사이트 도구를 이용해 여러분의 AI 프로젝트를 "누구나 방문할 수 있는 웹앱"으로 변환하고, 실제 사용자에게 테스트한 뒤 개선하는 전체 과정을 경험합니다. 이것이 바로 실제 IT 기업이 제품을 출시하는 방식입니다.

    📚 핵심 개념¶

    개념 1: 웹앱(Web App)¶

    🎒 비유로 시작

    스마트폰에 설치하는 앱은 "앱스토어에서 다운로드"해야 합니다. 반면 웹앱은 인터넷 브라우저에서 주소(URL)만 입력하면 바로 사용할 수 있는 앱입니다. 마치 친구에게 링크 하나만 보내면 바로 사용할 수 있는 셈이죠.

    📖 정확한 정의

    웹앱(Web Application)이란 별도의 설치 없이 웹 브라우저를 통해 접속하여 사용하는 응용 프로그램입니다. HTML, CSS, JavaScript 같은 웹 기술로 만들어지며, URL을 공유하는 것만으로 누구에게든 배포할 수 있습니다.

    ✏️ 예시로 확인

    • Google 문서: 브라우저에서 문서 작성 → 웹앱
    • Canva: 브라우저에서 디자인 작업 → 웹앱
    • 우리가 만들 것: 브라우저에서 AI 기능 체험 → 웹앱

    🔍 알아보기: 우리는 Google Sites로 웹앱의 "껍데기"(화면, 안내 페이지)를 만들고, 이전 챕터에서 완성한 AI 모델을 embed(삽입)하여 "속"을 채우는 방식으로 제작합니다.

    개념 2: 베타 테스트(Beta Test)¶

    🎒 비유로 시작

    영화가 개봉하기 전에 소수의 관객을 초대해서 시사회를 열죠? 감독은 관객의 반응을 보고 편집을 수정하기도 합니다. 베타 테스트는 소프트웨어의 시사회입니다.

    📖 정확한 정의

    베타 테스트(Beta Test)란 제품이 정식 출시되기 전, 실제 사용자를 대상으로 진행하는 테스트입니다. 개발자가 미처 발견하지 못한 문제점을 찾고, 사용자 경험(UX)을 개선하기 위한 목적으로 진행합니다.

    ✏️ 예시로 확인

    • 게임 회사의 "클로즈드 베타" → 일부 유저만 초대하여 플레이
    • 우리 프로젝트 → 다른 반 학생, 선생님을 초대하여 웹앱 사용
    flowchart LR A["🛠️ 개발 완료"] --> B["👥 베타 테스터 초대"] B --> C["📝 피드백 수집"] C --> D["🔧 개선 반영"] D --> E["🚀 정식 공개"]

    개념 3: 사용자 피드백(User Feedback)¶

    🎒 비유로 시작

    요리사가 손님에게 "맛이 어때요?"라고 묻는 것처럼, 개발자는 사용자에게 "사용하기 편한가요? 어떤 점이 불편한가요?"라고 물어봅니다. 이때 돌아오는 모든 의견, 평가, 제안이 피드백입니다.

    📖 정확한 정의

    사용자 피드백이란 제품을 실제로 사용한 사람이 제공하는 의견, 평가, 개선 제안을 말합니다. 정량적 피드백(점수, 별점)과 정성적 피드백(서술형 의견)으로 나뉩니다.

    ✏️ 예시로 확인

    • 정량적: "사용 편의성 4점 / 5점"
    • 정성적: "AI 결과가 나오는 데 시간이 너무 오래 걸려요"

    개념 4: 반복적 개선(Iteration)¶

    🎒 비유로 시작

    미술 시간에 그림을 그릴 때, 한 번에 완벽하게 완성하는 사람은 거의 없습니다. 밑그림 → 수정 → 채색 → 수정 → 마무리… 이렇게 여러 번 돌아가며 점점 완성도를 높이는 과정이 반복적 개선입니다.

    📖 정확한 정의

    반복적 개선(Iteration)이란 만들기 → 테스트 → 피드백 수집 → 개선의 사이클을 여러 번 반복하여 제품의 품질을 높이는 개발 방법론입니다.

    ✏️ 예시로 확인

    flowchart TB A["1차 웹앱 제작"] --> B["베타 테스트 1차"] B --> C["피드백 분석"] C --> D["개선 사항 반영"] D --> E["베타 테스트 2차"] E --> F["추가 피드백 분석"] F --> G["최종 버전 완성"]

    💡 생각해보기: 여러분이 자주 사용하는 앱(카카오톡, 인스타그램 등)도 업데이트를 반복하며 점점 나아지고 있습니다. 이것이 바로 반복적 개선의 실제 사례입니다.

    🔨 따라하기¶

    이제 실제로 웹앱을 만들고, 테스트하고, 개선하는 전체 과정을 6단계로 진행합니다.

    Step 1: Google Sites로 웹앱 뼈대 만들기¶

    목표: 3개 페이지(프로젝트 소개, AI 기능 체험, 사용 설명서)로 구성된 웹사이트를 생성합니다.

    1-1. Google Sites 접속

    1. 브라우저에서 sites.google.com에 접속합니다.
    2. 구글 계정으로 로그인합니다.
    3. "+" 빈 사이트를 클릭하여 새 사이트를 만듭니다.

    1-2. 사이트 기본 설정

    1. 왼쪽 상단의 사이트 이름을 프로젝트명으로 변경합니다.
    2. 예: "AI 급식 평가 도우미", "우리 동네 쓰레기 분류 AI"
    3. 상단 헤더에 프로젝트 제목과 한 줄 소개를 입력합니다.
    4. 헤더 이미지를 변경합니다. (이미지 → 업로드 또는 검색)

    1-3. 페이지 구성

    오른쪽 페이지 탭을 클릭하고, 아래 3개 페이지를 만듭니다.

    • 홈 (프로젝트 소개): 기본 생성된 첫 페이지를 활용
    • AI 체험하기: "+" 버튼 → 새 페이지 추가
    • 사용 설명서: "+" 버튼 → 새 페이지 추가
    flowchart TB Home["🏠 홈<br/>프로젝트 소개"] --> AI["🤖 AI 체험하기<br/>모델 연동 페이지"] Home --> Guide["📖 사용 설명서<br/>이용 방법 안내"]

    🔍 알아보기: Google Sites는 드래그 앤 드롭 방식이라 코딩 없이도 웹페이지를 만들 수 있습니다. 텍스트, 이미지, 동영상, 임베드(embed) 기능을 자유롭게 배치할 수 있습니다.

    Step 2: 각 페이지 콘텐츠 채우기¶

    🏠 홈 페이지 (프로젝트 소개) 구성 요소

    • 프로젝트 이름과 로고 (Canva 등으로 간단히 제작 가능)
    • 문제 정의: "우리가 해결하려는 문제는 무엇인가?"
    • 해결 방법: "AI를 어떻게 활용하는가?"
    • 팀 소개: 팀원 이름과 역할
    • AI 체험하기 버튼: AI 체험 페이지로 이동하는 링크 버튼

    아래는 홈 페이지에 넣을 프로젝트 소개 텍스트의 예시입니다.

    🌍 프로젝트: AI 급식 평가 도우미

❓ 문제
매일 급식을 먹지만, 학생들의 솔직한 평가가 영양사 선생님께 
전달되지 않아 메뉴 개선이 어렵습니다.

💡 해결
AI가 학생들의 급식 후기를 자동으로 분석하여 
긍정/부정/개선요청으로 분류하고, 핵심 키워드를 추출합니다.

👥 팀원
- 김하늘 (PM, 데이터 수집)
- 이바다 (AI 모델 학습)  
- 박산 (웹앱 제작, 디자인)

    🤖 AI 체험하기 페이지 구성 요소

    이 페이지가 웹앱의 핵심입니다. 이전 챕터에서 만든 AI 모델을 여기에 연동합니다.

    연동 방법은 사용한 AI 도구에 따라 다릅니다.

    • Teachable Machine 사용 시: 내보내기(Export)에서 얻은 공유 링크를 Google Sites의 "삽입(Embed)" → "URL로 삽입"에 붙여넣기
    • Colab/Gradio 사용 시: Gradio가 생성한 공개 URL을 "삽입(Embed)"으로 연동
    • 직접 만든 HTML 사용 시: "삽입(Embed)" → "코드 삽입"으로 HTML 코드 붙여넣기

    아래는 Teachable Machine 모델을 웹페이지에 삽입하는 HTML 코드 예시입니다.

    <!-- Teachable Machine 이미지 분류 모델 삽입 예시 -->
<!DOCTYPE html>
<html>
<head>
    <title>AI 체험하기</title>
    <script src="https://cdn.jsdelivr.net/npm/@tensorflow/tfjs@latest"></script>
    <script src="https://cdn.jsdelivr.net/npm/@teachablemachine/image@latest"></script>
</head>
<body>
    <h2>📷 이미지를 업로드하면 AI가 분류합니다</h2>
    <input type="file" id="imageUpload" accept="image/*">
    <div id="preview"></div>
    <div id="result" style="font-size: 20px; margin-top: 20px;"></div>

    <script>
        // 여러분의 Teachable Machine 모델 URL로 교체하세요
        const modelURL = 'https://teachablemachine.withgoogle.com/models/여러분의모델ID/';

        let model;

        async function loadModel() {
            const modelPath = modelURL + 'model.json';
            const metadataPath = modelURL + 'metadata.json';
            model = await tmImage.load(modelPath, metadataPath);
            document.getElementById('result').innerText = '✅ 모델 로드 완료! 이미지를 업로드하세요.';
        }

        document.getElementById('imageUpload').addEventListener('change', async (event) => {
            const file = event.target.files[0];
            if (!file) return;

            // 미리보기 표시
            const img = document.createElement('img');
            img.src = URL.createObjectURL(file);
            img.width = 300;
            document.getElementById('preview').innerHTML = '';
            document.getElementById('preview').appendChild(img);

            // 이미지 로드 후 예측
            img.onload = async () => {
                const predictions = await model.predict(img);
                let resultText = '🤖 AI 분석 결과:\n';
                predictions.forEach(p => {
                    const percent = (p.probability * 100).toFixed(1);
                    resultText += `${p.className}: ${percent}%\n`;
                });
                document.getElementById('result').innerText = resultText;
            };
        });

        // 페이지 로드 시 모델 불러오기
        loadModel();
    </script>
</body>
</html>

    ⚠️ 여러분의모델ID 부분을 실제 Teachable Machine에서 내보낸 모델의 ID로 교체해야 합니다.

    📖 사용 설명서 페이지 구성 요소

    • 시작하기: 웹앱에 처음 접속하면 무엇을 해야 하는지
    • AI 기능 사용법: 스크린샷과 함께 단계별 안내
    • 자주 묻는 질문(FAQ): 예상되는 질문과 답변 3~5개
    • 문의하기: 피드백 설문 링크
    📖 사용 설명서

1️⃣ "AI 체험하기" 페이지로 이동합니다
2️⃣ "파일 선택" 버튼을 클릭합니다
3️⃣ 분류하고 싶은 이미지를 선택합니다
4️⃣ AI가 자동으로 분석한 결과를 확인합니다

❓ 자주 묻는 질문
Q: 어떤 이미지를 올려야 하나요?
A: 급식 사진을 가까이서 찍은 사진이 가장 정확합니다.

Q: 결과가 이상하게 나와요.
A: 흐릿하거나 어두운 사진은 정확도가 떨어질 수 있습니다. 
   밝은 곳에서 찍은 선명한 사진을 사용해 주세요.

    Step 3: Google 설문지로 피드백 수집 양식 만들기¶

    목표: 베타 테스터가 웹앱을 사용한 후 작성할 피드백 설문지를 만듭니다.

    3-1. Google 설문지 접속

    1. forms.google.com에 접속합니다.
    2. "새 양식 시작"을 클릭합니다.
    3. 제목을 입력합니다: "[프로젝트명] 베타 테스트 피드백"

    3-2. 필수 질문 구성

    아래 질문들을 설문지에 추가합니다.

    정량적 질문 (점수로 답하는 질문)

    질문 1: 웹앱의 전체적인 만족도는?
→ 유형: "선형 배율" / 1(매우 불만족) ~ 5(매우 만족)

질문 2: AI 기능이 정확하다고 느꼈나요?

→ 유형: "선형 배율" / 1(매우 어려움) ~ 5(매우 쉬움)

    가능하다면 테스터가 사용하는 모습을 옆에서 관찰하세요. 다음을 기록합니다.

    • 어디서 멈칫하는가? (이해가 안 되는 부분)
    • 어디서 잘못 클릭하는가? (UI가 혼란스러운 부분)
    • 표정 변화가 있는가? (긍정적 반응 or 부정적 반응)
    • 테스터가 질문하는 내용은? (설명이 부족한 부분)

    💡 생각해보기: 옆에서 관찰하는 것이 설문지보다 더 정직한 피드백을 줄 때가 많습니다. 사람들은 글로는 "괜찮아요"라고 쓰지만, 실제로는 3분 동안 헤매고 있을 수 있습니다.

    Step 5: 피드백 분석하고 우선순위 정하기¶

    목표: 수집된 피드백을 체계적으로 분류하고, 무엇을 먼저 고칠지 우선순위를 정합니다.

    5-1. 정량적 데이터 분석

    Google 설문지의 "응답" 탭에서 자동으로 생성된 차트를 확인합니다. 각 질문별 평균 점수를 계산합니다.

    평균 점수 예시:
- 전체 만족도: 3.6 / 5.0
- AI 정확도: 3.2 / 5.0 ← 가장 낮음! 
- 사용 편의성: 4.1 / 5.0
- 디자인: 3.8 / 5.0

    5-2. 정성적 데이터 분석

    서술형 답변을 읽으면서 비슷한 내용끼리 묶습니다(그루핑). 아래는 Python으로 간단히 피드백을 분류하는 코드입니다.

    # 피드백 분류 도우미 코드
# 수집된 서술형 피드백을 카테고리별로 정리합니다

feedbacks = [
    "AI 결과가 나오는 데 너무 오래 걸려요",
    "디자인이 예뻐요!",
    "사용법을 모르겠어요. 설명이 더 필요해요",
    "사진을 올렸는데 분류가 틀렸어요",
    "로딩 시간이 길어요",
    "첫 페이지가 깔끔해서 좋았어요",
    "어떤 사진을 올려야 하는지 예시가 있으면 좋겠어요",
    "결과 정확도가 좀 아쉬워요"
]

# 카테고리 키워드 정의
categories = {
    "⚡ 속도 문제": ["오래", "느리", "로딩", "시간"],
    "🎯 정확도 문제": ["틀렸", "정확", "아쉬", "잘못"],
    "📖 설명 부족": ["모르겠", "설명", "예시", "이해"],
    "👍 긍정 피드백": ["좋았", "예뻐", "깔끔", "좋아", "최고"]
}

# 분류 실행
classified = {cat: [] for cat in categories}
unclassified = []

for fb in feedbacks:
    found = False
    for cat, keywords in categories.items():
        for keyword in keywords:
            if keyword in fb:
                classified[cat].append(fb)
                found = True
                break
        if found:
            break
    if not found:
        unclassified.append(fb)

# 결과 출력
print("=" * 50)
print("📊 피드백 분류 결과")
print("=" * 50)

for cat, items in classified.items():
    print(f"\n{cat} ({len(items)}건)")
    for item in items:
        print(f"  - {item}")

if unclassified:
    print(f"\n❓ 미분류 ({len(unclassified)}건)")
    for item in unclassified:
        print(f"  - {item}")

# 요약 통계
print("\n" + "=" * 50)
print("📈 카테고리별 건수 요약")
print("=" * 50)
for cat, items in classified.items():
    bar = "█" * len(items)
    print(f"{cat}: {bar} ({len(items)}건)")
