    Chapter 09. 메인프로젝트 — AI 협업으로 진짜 웹앱 만들기¶

    🎯 이 장에서 배우는 것¶

    • AI 에이전트에게 명확한 지시를 내려 웹앱의 HTML/CSS/JS 코드를 생성할 수 있다
    • AI가 만든 결과물을 확인하고, 대화를 통해 원하는 기능을 추가·수정할 수 있다
    • "프롬프트 → 확인 → 수정 요청"의 대화형 개발 사이클을 반복 수행할 수 있다
    • 완성된 웹앱을 실제로 배포하여 다른 사람이 사용할 수 있게 공유할 수 있다

    💡 왜 이걸 배우나요?¶

    생각해보기: 코딩을 몰라도 집을 지을 수 있을까? 🏠

    여러분이 꿈의 집을 짓고 싶다고 상상해보세요. 벽돌을 쌓는 기술을 몰라도 집을 지을 수 있을까요? 물론 가능합니다. 여러분은 설계자(건축주) 역할을 하고, 실제 시공은 시공사에게 맡기면 됩니다.

    단, 중요한 조건이 있습니다.

    • "예쁜 집 지어주세요"라고 하면? → 시공사가 알아서 짓겠지만, 원하는 집이 나올 확률은 매우 낮습니다
    • "2층짜리, 방 3개, 남향, 거실은 넓게"라고 하면? → 훨씬 원하는 결과에 가까워집니다

    AI와 함께 웹앱을 만드는 것도 똑같습니다. 여러분은 "무엇을 만들지" 설계하는 사람이고, AI는 실제 코드를 작성하는 시공사입니다. Chapter 08에서 열심히 설계한 알고리즘이 바로 여러분의 설계도입니다. 이제 그 설계도를 들고 AI라는 시공사에게 지시를 내릴 차례입니다!

    graph LR A["👤 여러분<br/>(설계자)"] -->|"명확한 지시<br/>(프롬프트)"| B["🤖 AI<br/>(시공사)"] B -->|"코드 결과물"| C["👀 확인"] C -->|"수정 요청"| B C -->|"만족!"| D["🎉 완성"]

    📚 핵심 개념¶

    개념 1: 대화형 개발 (Conversational Development) 💬¶

    비유로 시작

    음식점에서 주문하는 상황을 떠올려보세요. "맛있는 거 주세요"라고 하면 내 입맛에 맞는 음식이 나올까요? 아마 아닐 겁니다. "매운 떡볶이인데, 치즈 추가하고, 덜 맵게 해주세요"라고 구체적으로 말해야 원하는 결과를 얻습니다. 그리고 맛을 본 뒤 "조금 더 맵게 해주세요"라고 추가 요청을 할 수도 있죠.

    정확한 정의

    대화형 개발이란, AI에게 한 번에 완벽한 결과를 기대하는 것이 아니라 "요청 → 확인 → 수정 요청"을 반복하며 점진적으로 원하는 결과물을 만들어가는 개발 방식입니다.

    예시로 확인

    [1차 요청] "할 일 목록 웹앱을 만들어줘"
→ 기본적인 앱이 나옴. 하지만 디자인이 마음에 안 듦.

[2차 요청] "배경색을 하늘색으로 바꾸고, 글자를 더 크게 해줘"
→ 색상과 글자 크기가 변경됨. 하지만 삭제 기능이 없음.

[3차 요청] "각 할 일 옆에 삭제 버튼을 추가해줘"
→ 삭제 버튼이 생김!

    알아보기: 대화형 개발에서 가장 중요한 원칙은 "한 번에 하나씩" 요청하는 것입니다. 여러 가지를 동시에 요청하면 AI가 혼란스러워하거나, 문제가 생겼을 때 어디서 잘못됐는지 찾기 어렵습니다.

    개념 2: 효과적인 프롬프트의 구조 📋¶

    비유로 시작

    택배를 보낼 때 주소를 어떻게 쓰나요? "서울 어딘가"라고 쓰면 절대 도착하지 않습니다. "서울시 강남구 테헤란로 123, 4층 401호"처럼 구체적으로 써야 합니다. AI에게 보내는 프롬프트도 마찬가지입니다.

    정확한 정의

    효과적인 프롬프트란, AI가 정확한 결과를 만들어낼 수 있도록 역할, 목표, 세부 조건, 제약 사항을 명확하게 담은 지시문입니다.

    예시로 확인

    graph TD A["효과적인 프롬프트의 4요소"] --> B["🎭 역할<br/>너는 웹 개발자야"] A --> C["🎯 목표<br/>할 일 목록 앱을 만들어줘"] A --> D["📋 세부 조건<br/>할 일 추가, 완료 체크,<br/>삭제 기능 포함"] A --> E["🚫 제약 사항<br/>HTML 한 파일로,<br/>외부 라이브러리 없이"]
    • 나쁜 프롬프트: "웹앱 만들어줘"
    • 좋은 프롬프트: "너는 웹 개발자야. HTML, CSS, JavaScript를 하나의 파일에 담아서 할 일 목록 웹앱을 만들어줘. 기능은 (1) 할 일 입력 및 추가, (2) 완료 체크, (3) 삭제 버튼이야. 외부 라이브러리는 사용하지 마."

    개념 3: 코드를 몰라도 읽는 법 — 구조 파악하기 🔍¶

    비유로 시작

    외국어를 모르더라도 식당 메뉴판에서 🍕 피자 그림이 있으면 "아, 여기 피자를 파는구나"라고 알 수 있죠? 코드도 마찬가지입니다. 전체를 다 이해하지 못해도, 큰 구조만 파악하면 어디를 수정해야 할지 알 수 있습니다.

    정확한 정의

    웹앱의 코드는 크게 세 부분으로 나뉩니다.

    • HTML: 웹페이지의 뼈대 (어떤 요소가 있는지)
    • CSS: 웹페이지의 옷 (어떻게 보이는지)
    • JavaScript: 웹페이지의 두뇌 (어떻게 동작하는지)

    예시로 확인

    graph TB subgraph "웹앱 코드 구조" A["HTML 🦴<br/>뼈대<br/>버튼, 입력칸, 목록"] B["CSS 👗<br/>옷<br/>색상, 크기, 배치"] C["JavaScript 🧠<br/>두뇌<br/>클릭 시 동작, 계산"] end A --- B --- C

    생각해보기: AI가 만든 코드에서 글자 색상을 바꾸고 싶다면, HTML/CSS/JS 중 어느 부분을 수정해달라고 요청해야 할까요? 정답은 CSS입니다! 하지만 몰라도 괜찮습니다. AI에게 "글자 색상을 빨간색으로 바꿔줘"라고 하면 AI가 알아서 CSS를 수정해줍니다.

    개념 4: 배포 — 내 작품을 세상에 공개하기 🌍¶

    비유로 시작

    그림을 그렸다면, 서랍에 넣어두는 것보다 전시회에 걸어야 다른 사람도 볼 수 있겠죠? 웹앱도 마찬가지입니다. 내 컴퓨터에만 있으면 나만 볼 수 있지만, 배포(Deploy) 하면 전 세계 누구나 접속할 수 있습니다.

    정확한 정의

    배포란 내 컴퓨터에서 만든 웹앱 파일을 인터넷 서버에 올려서 누구나 웹 주소(URL)로 접속할 수 있게 만드는 과정입니다.

    예시로 확인

    이 수업에서는 GitHub Pages 또는 Netlify Drop을 사용합니다. 둘 다 무료이고, 파일을 드래그 앤 드롭하는 것만으로 배포가 완료됩니다.

    graph LR A["💻 내 컴퓨터<br/>index.html"] -->|"파일 업로드"| B["☁️ 서버<br/>Netlify / GitHub Pages"] B -->|"URL 생성"| C["🌐 누구나 접속 가능<br/>myapp.netlify.app"]

    🔨 따라하기¶

    이번 프로젝트에서는 Chapter 08에서 설계한 웹앱을 AI와 함께 실제로 만들어봅니다. 예시로 "오늘의 할 일 관리 앱" 을 함께 만들겠습니다. 여러분은 자신의 프로젝트 주제에 맞게 프롬프트를 변경하세요!

    🕐 1차시: 기본 화면 만들기 (2시간)¶

    Step 1: AI에게 첫 번째 지시 내리기¶

    ChatGPT, Claude, 또는 여러분이 사용하는 AI 도구를 열고, 다음과 같이 입력합니다.

    [프롬프트 예시]

너는 웹 개발 전문가야.
HTML, CSS, JavaScript를 하나의 HTML 파일 안에 담아서
"오늘의 할 일 관리 앱"을 만들어줘.

첫 번째 단계로 기본 화면만 먼저 만들어줘:
1. 상단에 앱 제목 "📋 오늘의 할 일"
2. 할 일을 입력하는 텍스트 입력칸
3. "추가" 버튼
4. 할 일이 표시될 빈 목록 영역

조건:
- 외부 라이브러리 사용하지 마
- 모바일에서도 보기 좋게 만들어줘
- 한국어로 주석을 달아줘

    알아보기: "한국어로 주석을 달아줘"라고 요청하면, AI가 코드 사이사이에 설명을 한국어로 적어줍니다. 나중에 어떤 부분이 무엇을 하는지 이해하는 데 도움이 됩니다.

    Step 2: 결과 확인하고 실행하기¶

    AI가 코드를 생성하면 다음 순서로 실행합니다.

    실행 방법: 1. AI가 준 코드를 전체 복사 (Ctrl+C 또는 Cmd+C) 2. 메모장(Windows) 또는 텍스트 편집기를 열기 3. 코드를 붙여넣기 (Ctrl+V 또는 Cmd+V) 4. 파일을 index.html 이라는 이름으로 저장 (확장자가 .html이어야 합니다!) 5. 저장한 파일을 더블클릭하면 웹 브라우저에서 열림

    ⚡ 팁: Visual Studio Code(VS Code)가 설치되어 있다면 더 편리합니다. VS Code에서 파일을 만들고, Live Server 확장을 사용하면 수정할 때마다 자동으로 새로고침됩니다.

    Step 3: 원하는 대로 안 나왔다면? — 수정 요청하기¶

    화면을 확인한 뒤, 마음에 들지 않는 부분이 있으면 구체적으로 수정을 요청합니다.

    [수정 요청 프롬프트 예시]

방금 만든 코드에서 다음을 수정해줘:
1. 배경색을 연한 하늘색(#E3F2FD)으로 변경
2. 제목 글자 크기를 더 크게 (32px)
3. 입력칸의 모서리를 둥글게 (border-radius: 10px)
나머지는 그대로 유지해줘.

    정리하기: 1차시의 핵심은 "첫 프롬프트를 보내고, 결과를 확인하고, 수정을 요청하는" 대화형 개발의 기본 사이클을 익히는 것입니다.

    🕐 2차시: 핵심 기능 구현하기 (2시간)¶

    Step 4: 기능 추가 요청하기¶

    화면이 만족스러우면, 이제 동작하는 기능을 추가합니다.

    [프롬프트 예시]

이제 기능을 추가해줘:

1. 추가 기능: 입력칸에 텍스트를 쓰고 "추가" 버튼을 누르면 
   목록에 할 일이 추가된다
2. 완료 기능: 할 일을 클릭하면 취소선이 그어지고 
   글자가 회색으로 변한다
3. 삭제 기능: 각 할 일 옆에 "❌" 삭제 버튼이 있고, 
   누르면 해당 항목이 사라진다
4. 빈 칸 방지: 아무것도 입력하지 않고 추가 버튼을 누르면 
   "할 일을 입력해주세요!" 알림이 뜬다

전체 코드를 다시 보여줘.

    알아보기: "전체 코드를 다시 보여줘"라고 말하는 것이 중요합니다. 부분만 보여주면 어디에 넣어야 할지 헷갈릴 수 있기 때문입니다.

    기능이 추가된 코드를 받으면, 다시 복사 → 저장 → 브라우저에서 확인합니다.

    Step 5: 문제 발견 시 디버깅 요청하기¶

    만약 기능이 제대로 작동하지 않는다면, 당황하지 마세요! 증상을 AI에게 그대로 설명하면 됩니다.

    [디버깅 요청 프롬프트 예시]

문제가 있어:
- "추가" 버튼을 눌러도 목록에 아무것도 나타나지 않아
- 브라우저 콘솔에 "Uncaught TypeError" 에러가 보여
- 입력칸에 "공부하기"라고 쓰고 추가 버튼을 눌렀어

원인을 찾아서 수정한 전체 코드를 보여줘.
    graph TD A["🔍 문제 발견!<br/>버튼이 안 눌려요"] --> B["📝 증상을 구체적으로 적기"] B --> C["🤖 AI에게 디버깅 요청"] C --> D["💻 수정된 코드 받기"] D --> E["✅ 다시 테스트"] E -->|"또 문제?"| A E -->|"해결!"| F["🎉 다음 단계로"]

    생각해보기: "안 돼요"와 "추가 버튼을 눌렀는데 목록에 항목이 추가되지 않아요"는 어떤 차이가 있을까요? AI도 사람처럼, 구체적인 증상을 알려줘야 정확한 해결책을 제시할 수 있습니다.

    🕐 3차시: 디자인 개선과 배포 (2시간)¶

    Step 6: 디자인 다듬기 + 배포하기¶

    기능이 완성되었으면, 마지막으로 디자인을 다듬습니다.

    [디자인 개선 프롬프트 예시]

앱의 디자인을 더 예쁘게 개선해줘:
1. 전체적으로 카드 형태의 깔끔한 디자인
2. 추가 버튼에 마우스를 올리면 색상이 변하는 효과
3. 할 일이 추가될 때 부드러운 애니메이션
4. 하단에 "완료한 일: 0개 / 전체: 0개" 카운터 표시
5. 페이지 하단에 내 이름과 "Made with AI ❤️" 표시

전체 코드를 다시 보여줘.

    배포하기 — Netlify Drop 사용:

    1. 웹 브라우저에서 https://app.netlify.com/drop 에 접속
    2. 완성된 index.html 파일이 들어있는 폴더 전체를 화면에 드래그 앤 드롭
    3. 잠시 기다리면 고유한 URL이 생성됨 (예: amazing-todo-app-12345.netlify.app)
    4. 이 URL을 친구에게 공유하면 끝! 🎉

    ⚡ 팁: index.html 파일 하나만 있는 경우에도, 반드시 폴더에 넣어서 폴더째로 업로드해야 합니다.

        // === JavaScript: 웹앱의 두뇌 (기능) ===

        // 할 일 추가 함수
        function addTodo() {
            const input = document.getElementById('todoInput');
            const text = input.value.trim(); // 입력값의 앞뒤 공백 제거

            // 빈 칸 방지
            if (text === '') {
                alert('할 일을 입력해주세요!');
                return;
            }

            // 새로운 할 일 항목 만들기
            const li = document.createElement('li');
            li.className = 'todo-item';
            li.innerHTML = `
                <span onclick="toggleDone(this)">${text}</span>
                <button class="delete-btn" onclick="deleteTodo(this)">❌</button>
            `;

            document.getElementById('todoList').appendChild(li);
            input.value = ''; // 입력칸 비우기
            input.focus();    // 입력칸에 다시 커서 놓기
            updateCounter();  // 카운터 업데이트
        }

        // 완료/미완료 토글 함수
        function toggleDone(span) {
            span.parentElement.classList.toggle('done');
            updateCounter();
        }

        // 삭제 함수
        function deleteTodo(btn) {
            btn.parentElement.remove();
            updateCounter();
        }

        // 카운터 업데이트 함수
        function updateCounter() {
            const total = document.querySelectorAll('.todo-item').length;
            const done = document.querySelectorAll('.todo-item.done').length;
            document.getElementById('counter').textContent = 
                `완료: ${done}개 / 전체: ${total}개`;
        }

        // Enter 키로도 추가할 수 있게
        document.getElementById('todoInput').addEventListener('keypress', function(e) {
            if (e.key === 'Enter') addTodo();
        });
    </script>
