    Chapter 9. 프롬프트 엔지니어링 기초 — AI에게 잘 부탁하는 법¶

    Part 3 | [1권] 생성형 AI와 대화하기 — LLM의 원리와 프롬프트 엔지니어링

    🎯 이 장에서 배우는 것¶

    • 좋은 프롬프트의 4가지 구성 요소(역할, 맥락, 형식, 제약)를 설명할 수 있다
    • 같은 목적의 프롬프트를 다양하게 변형하고 결과 차이를 비교할 수 있다
    • 프롬프트 템플릿을 활용하여 원하는 결과물을 효과적으로 얻어낼 수 있다
    • 나쁜 프롬프트를 분석하고 좋은 프롬프트로 개선할 수 있다

    💡 왜 이걸 배우나요?¶

    🍔 생각해보기: 식당에서 주문하기 vs AI에게 요청하기

    여러분이 식당에 갔다고 상상해 보세요.

    주문 A: "아무거나 주세요."

    주문 B: "매운 걸 잘 못 먹는데, 국물 있는 메뉴 중에서 1인분으로 만 원 이하인 걸로 추천해 주세요."

    어떤 주문이 원하는 음식을 받을 확률이 높을까요?

    당연히 주문 B입니다. AI에게 요청하는 것도 완전히 같습니다. "작문 도와줘"라고 말하면 AI는 무엇을 써야 할지, 누구를 위한 글인지, 얼마나 길어야 하는지 전혀 모릅니다. 반면 구체적으로 요청하면 놀라울 정도로 정확한 결과를 돌려줍니다.

    이 장에서 배우는 프롬프트 엔지니어링(Prompt Engineering) 은 AI에게 "잘 부탁하는 기술"입니다. 같은 AI라도 어떻게 물어보느냐에 따라 결과물의 품질이 완전히 달라집니다. 이 기술을 익히면 여러분은 AI를 10배 더 잘 활용할 수 있게 됩니다.

    📚 핵심 개념¶

    개념 1: 프롬프트란 무엇인가?¶

    🎯 비유로 시작하기

    프롬프트는 AI에게 보내는 주문서와 같습니다. 카페에서 음료를 주문할 때 "아이스 아메리카노, 샷 추가, 텀블러에 담아주세요"라고 구체적으로 말하듯이, AI에게도 정확한 주문서를 건네야 합니다.

    📖 정확한 정의

    프롬프트(Prompt) 란 AI 모델에게 입력하는 텍스트 지시문으로, AI가 어떤 작업을 수행하고 어떤 결과를 생성할지 안내하는 모든 내용을 말합니다.

    💬 예시로 확인

    • 약한 프롬프트: "지구 온난화에 대해 알려줘"
    • 강한 프롬프트: "중학생이 이해할 수 있는 수준으로 지구 온난화의 원인 3가지를 각각 한 문장으로 설명해 줘"

    두 프롬프트 모두 지구 온난화에 대해 묻고 있지만, 강한 프롬프트는 누구를 위한 것인지(중학생), 무엇을 원하는지(원인 3가지), 어떤 형식인지(한 문장씩) 모두 담고 있습니다.

    개념 2: 좋은 프롬프트의 4가지 구성 요소 — R.C.F.C¶

    좋은 프롬프트에는 네 가지 핵심 요소가 있습니다. 이를 R.C.F.C로 기억합시다.

    graph LR A["🎭 Role<br/>역할"] --> E["✨ 좋은 프롬프트"] B["📋 Context<br/>맥락"] --> E C["📐 Format<br/>형식"] --> E D["🚧 Constraint<br/>제약"] --> E

    🎭 ① Role (역할 부여)¶

    비유: 연극 무대에서 배우에게 "당신은 셰익스피어입니다"라고 말하면, 그 배우는 셰익스피어처럼 말하고 행동합니다. AI에게도 역할을 부여하면 그 전문가의 관점에서 답변합니다.

    정의: AI에게 특정 전문가나 인물의 역할을 지정하여, 그 관점과 전문성에 맞는 답변을 유도하는 것

    예시: - "너는 10년 경력의 과학 교사야." - "너는 초등학생에게 설명하는 것을 좋아하는 동화 작가야." - "너는 코드 리뷰를 담당하는 시니어 개발자야."

    📌 알아보기: 역할을 바꾸면 같은 질문에도 완전히 다른 답변이 나옵니다. "광합성을 설명해 줘"라는 질문에 '생물학 교수' 역할과 '요리사' 역할을 주면, 전자는 학술적으로, 후자는 "식물의 요리법"처럼 재미있게 설명합니다.

    📋 ② Context (맥락 제공)¶

    비유: 의사에게 진찰을 받을 때 "아파요"라고만 하면 의사도 난감합니다. "어제부터 목이 아프고 열이 38도예요"라고 말해야 정확한 진단이 가능하죠.

    정의: AI가 상황을 정확히 파악할 수 있도록 배경 정보, 목적, 대상 등을 제공하는 것

    예시: - "고등학교 1학년 정보 시간에 발표할 자료를 만들고 있어." - "영어를 잘 못하는 친구에게 보여줄 거야." - "학교 축제에서 사용할 포스터 문구가 필요해."

    📐 ③ Format (출력 형식 지정)¶

    비유: 택배를 보낼 때 "박스 포장", "에어캡 포장", "선물 포장" 중 원하는 것을 고르듯이, AI에게도 결과물의 형태를 지정해야 합니다.

    정의: AI의 답변이 어떤 구조와 형태로 나오길 원하는지 명시하는 것

    예시: - "번호 목록으로 정리해 줘" - "500자 이내로 요약해 줘" - "JSON 형식으로 출력해 줘" - "비유를 포함해서 설명해 줘"

    🚧 ④ Constraint (제약 조건)¶

    비유: 요리 대회에서 "30분 이내에, 주어진 재료 5가지만 사용해서" 요리하라는 제한이 있으면 더 창의적이고 집중된 요리가 나오듯이, AI에게도 제약을 주면 더 정확한 결과가 나옵니다.

    정의: AI가 하지 말아야 할 것, 지켜야 할 범위, 한계 등을 설정하는 것

    예시: - "전문 용어 사용하지 마" - "3가지 이내로만 답해 줘" - "한국어로만 답변해 줘" - "개인적인 의견은 빼고 사실만 알려줘"

    💡 정리하기: R.C.F.C 네 가지를 모두 갖춘 프롬프트를 만들면, AI는 마치 여러분의 마음을 읽은 것처럼 정확한 결과를 돌려줍니다. 물론 항상 네 가지를 전부 넣어야 하는 것은 아닙니다. 상황에 따라 필요한 요소를 조합하면 됩니다.

    개념 3: 핵심 프롬프트 기법 5가지¶

    R.C.F.C의 구성 요소를 이해했으니, 이제 실전에서 자주 쓰이는 구체적인 기법을 알아봅시다.

    graph TD A["프롬프트 핵심 기법 5가지"] --> B["1️⃣ 역할 부여법"] A --> C["2️⃣ 단계별 지시법"] A --> D["3️⃣ 예시 제공법"] A --> E["4️⃣ 출력 형식 지정법"] A --> F["5️⃣ 반복 개선법"]

    1️⃣ 역할 부여법¶

    앞서 배운 Role을 활용하는 기법입니다.

    너는 20년 경력의 한국사 교사야.
학생들이 흥미를 느낄 수 있도록 이야기하듯 설명하는 스타일이야.

    2️⃣ 단계별 지시법 (Step-by-Step)¶

    복잡한 작업을 여러 단계로 나누어 지시합니다.

    다음 순서대로 작업해 줘:
1단계: 아래 글의 핵심 주제를 한 문장으로 요약해
2단계: 핵심 키워드 5개를 뽑아
3단계: 중학생 수준으로 다시 쉽게 풀어서 설명해

    📌 알아보기: 왜 단계를 나누면 좋을까요? AI는 한꺼번에 여러 가지를 요구받으면 일부를 빠뜨리거나 품질이 떨어질 수 있습니다. 단계를 나누면 각 단계에 집중하여 더 정확한 결과를 만들어냅니다. 이것은 우리가 수학 문제를 풀 때 한 번에 답을 구하기보다 단계별로 풀이하는 것과 같은 원리입니다.

    3️⃣ 예시 제공법 (Few-Shot)¶

    원하는 결과의 예시를 보여주어 AI가 패턴을 파악하게 합니다.

    아래 예시처럼 영어 단어의 뜻과 예문을 만들어 줘.

예시:
- apple: 사과 → "I eat an apple every morning."
- book: 책 → "This book is very interesting."

이제 다음 단어들을 같은 형식으로 만들어 줘:
- courage
- adventure
- wisdom

    4️⃣ 출력 형식 지정법¶

    결과물의 구조를 구체적으로 지정합니다.

    다음 형식으로 답변해 줘:

[제목]
(한 줄 요약)

[핵심 내용]
- 포인트 1
- 포인트 2
- 포인트 3

[한 줄 결론]

    5️⃣ 반복 개선법 (Iterative Refinement)¶

    한 번에 완벽한 프롬프트를 만들기 어렵습니다. 결과를 보고 프롬프트를 수정하는 과정을 반복합니다.

    graph LR A["1차 프롬프트<br/>작성"] --> B["AI 결과<br/>확인"] B --> C["부족한 점<br/>분석"] C --> D["프롬프트<br/>수정"] D --> B

    💡 생각해보기: "좋은 프롬프트는 한 번에 완성되지 않는다." 마치 글쓰기에서 초안을 쓰고 퇴고(고쳐쓰기)하듯이, 프롬프트도 여러 번 다듬어야 최고의 결과를 얻을 수 있습니다.

    개념 4: 프롬프트 템플릿 — 나만의 만능 틀¶

    매번 처음부터 프롬프트를 만들면 시간이 많이 걸립니다. 자주 쓰는 상황에 맞는 템플릿(틀) 을 준비해 두면 빠르고 일관된 결과를 얻을 수 있습니다.

    🧩 만능 프롬프트 템플릿

    [역할]
너는 {역할}이야.

[맥락]
나는 {상황 설명}하고 있어.
대상은 {누구}이고, 목적은 {무엇}이야.

[작업 지시]
다음을 수행해 줘:
1. {첫 번째 할 일}
2. {두 번째 할 일}
3. {세 번째 할 일}

[출력 형식]
{원하는 형식 설명}

[제약 조건]
- {하지 말아야 할 것}
- {지켜야 할 것}

    이 템플릿의 {}(중괄호) 부분만 상황에 맞게 바꾸면 됩니다!

    📎 활용 예시 — 독후감 도우미

    [역할]
너는 고등학생의 글쓰기를 도와주는 국어 교사야.

[맥락]
나는 '데미안(헤르만 헤세)'을 읽고 독후감을 쓰려고 해.
학교 과제이고, 분량은 A4 한 장 정도야.

[작업 지시]
다음을 수행해 줘:
1. 책의 핵심 메시지 3가지를 정리해 줘
2. 각 메시지에 대한 내 생각을 쓸 수 있도록 질문 3개를 만들어 줘
3. 독후감의 개요(서론-본론-결론 구조)를 잡아 줘

[출력 형식]
각 항목을 번호와 소제목으로 구분해서 정리해 줘

[제약 조건]
- 독후감을 대신 써주지 마. 나의 생각을 이끌어내는 가이드만 제공해.
- 중학생도 이해할 수 있는 쉬운 말로 써 줘.

    📌 알아보기: 템플릿은 학습 도우미, 발표 자료 작성, 코드 작성 요청, 번역 등 자주 하는 작업별로 만들어두면 매우 유용합니다. 마치 편지를 쓸 때 "받는 사람 / 인사말 / 본문 / 맺음말" 형식을 따르듯이요.

    🔨 따라하기: 프롬프트 변형 실험¶

    이제 직접 프롬프트를 작성하고, 변형하며, 결과 차이를 비교해 봅시다.

    준비물: ChatGPT, Gemini, Claude 등 아무 AI 챗봇 하나

    Step 1: 기본 프롬프트 던져보기¶

    먼저 아주 간단한 프롬프트를 AI에게 보내봅시다.

    입력할 프롬프트:

    광합성에 대해 설명해 줘.

    AI의 답변을 읽어보세요. 아마 길고 일반적인 설명이 나올 것입니다. 결과를 메모장이나 노트에 복사해 두세요.

    🔍 관찰 포인트: 답변의 길이는? 어려운 용어가 있나요? 원하는 수준의 설명인가요?

    Step 2: 역할과 맥락 추가하기¶

    이번에는 역할과 맥락을 추가합니다.

    입력할 프롬프트:

    너는 초등학교 과학 교사야.
초등학교 5학년 학생에게 광합성을 설명하려고 해.
아이들이 재미있게 이해할 수 있도록 설명해 줘.

    🔍 관찰 포인트: Step 1의 결과와 비교해 보세요. 어떤 점이 달라졌나요? 설명의 수준, 사용된 단어, 비유의 사용 여부 등을 체크합니다.

    Step 3: 형식과 제약 추가하기¶

    이번에는 R.C.F.C 전체를 적용합니다.

    입력할 프롬프트:

    [역할] 너는 초등학교 과학 교사야.

[맥락] 초등학교 5학년 학생에게 광합성을 설명하려고 해.
아이들이 쉽고 재미있게 이해할 수 있어야 해.

[작업 지시]
1. 광합성을 음식 만들기에 비유해서 설명해 줘
2. 광합성에 필요한 재료 3가지를 알려줘
3. 광합성의 결과물이 무엇인지 설명해 줘

[출력 형식]
- 전체 분량은 300자 이내
- 각 항목을 번호로 구분해 줘
- 마지막에 한 줄 퀴즈를 하나 넣어 줘

[제약 조건]
- 화학식은 사용하지 마
- 초등학생이 모르는 어려운 단어는 쓰지 마

    🔍 관찰 포인트: Step 1, 2와 비교했을 때 결과의 정확도와 유용성이 어떻게 달라졌나요?

    Step 4: 세 가지 결과 비교 분석¶

    세 번의 결과를 나란히 놓고 비교해 봅시다.

    Step 5: 프롬프트 배틀 — 실전 도전! 🏆¶

    이제 배운 기법을 총동원하여 다음 과제에 도전합니다.

    🎯 미션: "AI에게 우리 학교를 소개하는 1분 영상 대본을 쓰게 하라"

    아래 템플릿을 활용하되, 여러분의 학교에 맞게 수정해 보세요.

    [역할]
너는 유튜브 교육 채널을 운영하는 영상 전문 작가야.
10대 시청자의 관심을 끄는 대본을 잘 써.

[맥락]
우리 학교(예: OO고등학교)를 소개하는 1분짜리 유튜브 영상 대본을 만들려고 해.
시청 대상은 중학교 3학년(예비 고등학생)이야.
학교의 장점을 매력적으로 보여주고 싶어.

[작업 지시]
1. 시선을 끄는 오프닝 문장 작성 (3초 이내 읽을 수 있는 분량)
2. 학교의 매력 포인트 3가지를 소개하는 본문
3. 기억에 남는 마무리 문장

[출력 형식]
- 영상 대본 형식 (화면 설명 + 내레이션)
- 전체 분량: 1분 영상 기준 (약 250~300자)

[제약 조건]
- 과장된 표현이나 거짓 정보는 넣지 마
- 10대가 공감할 수 있는 자연스러운 말투 사용
- 다른 학교를 비하하는 내용은 넣지 마

    📝 전체 코드¶

    이 장은 프로그래밍 코드보다는 프롬프트 자체가 "코드" 입니다. 앞서 배운 핵심 프롬프트들을 한 곳에 모아 정리합니다.

    🐍 Python으로 프롬프트 자동 생성하기 (보너스)

    프롬프트를 코드로 관리하고 싶다면, 간단한 Python 코드를 활용할 수 있습니다.

    def create_prompt(role, context, tasks, output_format, constraints):
    """R.C.F.C 기반 프롬프트를 자동 생성하는 함수"""

    # 작업 지시를 번호 목록으로 변환
    task_list = ""
    for i, task in enumerate(tasks, 1):
        task_list += f"{i}. {task}\n"

    # 제약 조건을 목록으로 변환
    constraint_list = ""
    for c in constraints:
        constraint_list += f"- {c}\n"

    # 최종 프롬프트 조합
    prompt = f"""[역할]
너는 {role}이야.

[맥락]
{context}

[작업 지시]
다음을 순서대로 수행해 줘:
{task_list}
[출력 형식]
{output_format}

[제약 조건]
{constraint_list}"""

    return prompt


# 사용 예시
my_prompt = create_prompt(
    role="10년 경력의 과학 교사",
    context="고등학교 1학년 학생에게 뉴턴의 운동 법칙을 설명하려고 해.",
    tasks=[
        "제1법칙을 일상 속 비유로 설명해",
        "제2법칙을 공식과 함께 쉽게 설명해",
        "제3법칙을 재미있는 예시로 설명해"
    ],
    output_format="각 법칙별로 소제목을 달고, 전체 500자 이내로 작성해 줘",
    constraints=[
        "수학 공식은 F=ma 하나만 사용해",
        "영어 용어에는 반드시 한글 뜻을 병기해"
    ]
)

print(my_prompt)
