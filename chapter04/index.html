    Chapter 4. 미니프로젝트 2~3차시 — 웹앱 제작, 발표, 시상¶

    Part 1 | [1권] AI 세계로의 초대 — 이해하고 체험하기

    🎯 이 장에서 배우는 것¶

    • 모델의 오분류 원인을 분석하고, 데이터 보강으로 정확도를 개선할 수 있다
    • 구글 Sites를 활용하여 AI 모델이 작동하는 노코드 웹앱을 제작할 수 있다
    • 프로젝트 과정과 결과를 청중 앞에서 논리적으로 발표할 수 있다
    • 동료 평가에 참여하고, 프로젝트 전 과정을 회고할 수 있다

    💡 왜 이걸 배우나요?¶

    여러분이 만든 AI 모델, 아직은 여러분의 컴퓨터 안에만 있습니다. 맛있는 요리를 만들고 혼자만 먹으면 아깝죠? 웹앱으로 공개하면 친구, 가족, 누구나 스마트폰으로 접속해 여러분의 AI를 직접 체험할 수 있습니다.

    하지만 그 전에 한 가지! 1차시에서 만든 모델을 다시 살펴보면, 분명 "틀리는 순간"이 있었을 겁니다. 왜 틀렸는지 원인을 찾고, 데이터를 보강해서 더 똑똑하게 만드는 과정이야말로 진짜 AI 개발자가 하는 일입니다.

    이번 장에서는 모델 개선 → 웹앱 제작 → 발표 → 시상까지, 미니프로젝트의 마지막 여정을 함께합니다. 🚀

    📚 핵심 개념¶

    개념 1: 오분류 분석 — AI의 실수에서 배우기¶

    1. 비유로 시작

    시험을 보고 채점만 하면 점수만 알 수 있습니다. 하지만 오답 노트를 쓰면 "왜 틀렸는지"를 알게 되고, 다음엔 같은 실수를 안 하게 되죠. AI 모델도 똑같습니다. 틀린 결과를 분석하는 것이 바로 오분류 분석입니다.

    2. 정확한 정의

    오분류(Misclassification) 란 AI 모델이 입력 데이터를 잘못된 클래스(범주)로 예측하는 것을 말합니다. 오분류 분석은 틀린 사례를 모아 공통 패턴(어두운 조명, 비슷한 색상, 특이한 각도 등)을 찾아내는 과정입니다.

    3. 예시로 확인

    생각해보기 💭

    "사과 vs 토마토" 분류 모델을 만들었는데, 빨간 사과 사진을 자꾸 토마토로 분류한다면? 원인은 "둘 다 빨갛고 둥글다"는 점입니다. 이럴 때 초록 사과, 줄무늬 사과 등 다양한 사과 사진을 추가하면 모델이 색상 외의 특징(꼭지 모양, 표면 질감)도 학습하게 됩니다.

    개념 2: 데이터 보강 — AI에게 더 다양한 경험을 주기¶

    1. 비유로 시작

    축구 연습을 할 때 항상 같은 위치에서만 슛을 차면, 실전에서 다른 위치의 슛은 못 찹니다. 다양한 위치에서 연습해야 실전에 강해지는 것처럼, AI도 다양한 조건의 데이터를 학습해야 합니다.

    2. 정확한 정의

    데이터 보강(Data Augmentation) 이란 기존 데이터에 변화를 주거나 새로운 데이터를 추가하여 학습 데이터의 양과 다양성을 늘리는 기법입니다.

    3. 예시로 확인

    보강할 수 있는 요소들을 살펴봅시다.

    • 조명: 밝은 곳, 어두운 곳, 그림자가 있는 곳
    • 각도: 정면, 위에서, 옆에서, 기울여서
    • 배경: 흰 배경, 책상 위, 손에 들고, 야외
    • 크기: 가까이서 크게, 멀리서 작게
    graph LR A["1차 데이터<br/>같은 조건 30장"] -->|오분류 분석| B["부족한 점 발견"] B -->|보강| C["2차 데이터<br/>다양한 조건 60장"] C --> D["정확도 향상! 🎯"]

    개념 3: 노코드 웹앱 — 코딩 없이 AI를 세상에 공개하기¶

    1. 비유로 시작

    예전에는 집을 지으려면 벽돌을 하나하나 쌓아야 했습니다. 하지만 지금은 조립식 주택처럼 부품을 끼워 맞추면 됩니다. 노코드 웹앱도 마찬가지로, 코딩 없이 블록을 조립하듯 웹페이지를 만들 수 있습니다.

    2. 정확한 정의

    노코드 웹앱이란 프로그래밍 코드를 직접 작성하지 않고, 드래그 앤 드롭 등 시각적 도구로 만드는 웹 애플리케이션입니다. 이번 프로젝트에서는 구글 Sites(무료 웹사이트 제작 도구)에 티처블머신 모델을 임베드(삽입)하여 만듭니다.

    3. 예시로 확인

    알아보기 🔍

    임베드(Embed) 란 다른 서비스의 콘텐츠를 내 웹페이지 안에 넣어 보여주는 것입니다. 유튜브 영상을 블로그에 삽입하는 것과 같은 원리입니다. 티처블머신은 학습된 모델을 임베드할 수 있는 링크를 제공합니다.

    개념 4: 프로젝트 발표와 동료 평가¶

    1. 비유로 시작

    영화를 만들었으면 시사회를 열어야 관객의 반응을 알 수 있습니다. 프로젝트도 발표를 통해 다른 사람에게 보여주고, 피드백을 받아야 완성됩니다.

    2. 정확한 정의

    동료 평가(Peer Review) 란 같은 과정을 수행한 동료들이 서로의 결과물을 정해진 기준에 따라 평가하는 방법입니다. 일방적인 점수 매기기가 아니라, 서로 배우고 성장하는 과정입니다.

    3. 예시로 확인

    발표 구성 예시 (3분)

    • 30초: 우리 모둠의 주제와 문제 정의
    • 1분: 데이터 수집, 모델 학습 과정
    • 1분: 오분류 분석과 개선 과정
    • 30초: 웹앱 시연 및 향후 개선 아이디어

    🔨 따라하기¶

    2차시: 모델 개선 + 웹앱 제작¶

    Step 1: 오분류 사례 수집하기¶

    1차시에서 만든 티처블머신 모델을 열어 테스트합니다.

    1. Teachable Machine에 접속하여 저장해둔 프로젝트를 엽니다
    2. 미리보기(Preview) 패널에서 다양한 물체를 카메라에 비춰봅니다
    3. 틀리는 경우를 기록합니다 — 아래 양식을 모둠 노트에 작성하세요

    📋 오분류 기록 양식

    • 시도한 물체: (예: 빨간 볼펜)
    • 모델의 예측: (예: 립스틱 85%)
    • 올바른 정답: (예: 볼펜)
    • 추정 원인: (예: 빨간색이 비슷하고, 길쭉한 형태가 유사)

    최소 5개 이상의 오분류 사례를 모아보세요!

    Step 2: 데이터 보강으로 모델 개선하기¶

    오분류 기록에서 발견한 패턴에 맞춰 데이터를 추가합니다.

    1. 티처블머신 프로젝트의 각 클래스로 돌아갑니다
    2. 부족했던 조건의 이미지를 추가 촬영합니다

    알아보기 🔍 — 효과적인 보강 전략

    • 틀린 클래스 쌍(예: 볼펜 ↔ 립스틱)의 차이점이 드러나는 사진을 집중 추가하세요
    • 클래스당 최소 50장 이상을 목표로 합니다
    • 촬영할 때 한 가지 조건만 바꿔가며 찍으면 체계적입니다
    1. "모델 학습(Train Model)" 버튼을 클릭하여 재학습합니다
    2. 다시 미리보기에서 테스트하고, 아까 틀렸던 사례가 맞는지 확인합니다
    graph TD A["1차 모델 테스트"] --> B["오분류 사례 수집"] B --> C["원인 분석<br/>(조명? 각도? 배경?)"] C --> D["해당 조건 데이터 추가"] D --> E["2차 모델 학습"] E --> F{"정확도 향상?"} F -->|"Yes ✅"| G["모델 내보내기"] F -->|"No ❌"| B

    Step 3: 티처블머신 모델 내보내기¶

    정확도가 만족스러우면 모델을 웹에 공개할 수 있도록 내보냅니다.

    1. 티처블머신에서 "모델 내보내기(Export Model)" 클릭
    2. 상단 탭에서 "Tensorflow.js" 선택
    3. "모델 업로드(Upload my model)" 클릭
    4. 업로드가 완료되면 공유 가능한 링크가 생성됩니다
    5. 이 링크를 복사하여 모둠 노트에 저장해 두세요

    ⚠️ 주의: 업로드 버튼을 누른 뒤 잠시 기다려야 링크가 나타납니다. 페이지를 닫지 마세요!

    Step 4: 구글 Sites로 웹앱 만들기¶

    이제 누구나 접속할 수 있는 웹페이지를 만듭니다.

    1. Google Sites에 접속하여 "+"(새 사이트 만들기) 클릭
    2. 사이트 제목 입력 (예: "우리 모둠 AI — 학용품 분류기")
    3. 페이지 구성 — 아래 내용을 순서대로 추가합니다

    첫 번째 섹션: 프로젝트 소개 - 텍스트 상자를 추가하고, 프로젝트 주제와 팀원 이름을 적습니다

    두 번째 섹션: AI 모델 체험 - 상단 메뉴에서 삽입(Insert) → 임베드(Embed) 클릭 - "URL로 임베드" 탭을 선택합니다 - Step 3에서 복사한 티처블머신 링크를 붙여넣습니다 - 삽입 버튼을 클릭하면 모델이 페이지 안에 나타납니다

    세 번째 섹션: 개선 과정 - 오분류 분석 결과와 데이터 보강 과정을 간단히 설명합니다

    1. 오른쪽 상단 "게시(Publish)" 버튼 클릭
    2. 웹 주소를 정하고 게시하면 완성! 🎉

    생각해보기 💭

    게시된 링크를 친구에게 보내보세요. 친구의 스마트폰 카메라로도 AI 모델이 잘 작동하나요? 다른 환경(교실 밖, 집 등)에서도 테스트해 보는 것이 진짜 실력 점검입니다.

    Step 5: 모둠 발표 (모둠당 3분)¶

    발표 전 아래 체크리스트를 확인하세요.

    📋 발표 전 체크리스트

    • 웹앱 링크가 정상 작동하는가?
    • 발표 역할이 분담되어 있는가? (발표자, 시연자, 타이머)
    • 3분 안에 끝나도록 리허설을 했는가?
    • 오분류 분석 → 개선 과정이 포함되어 있는가?

    발표 순서 가이드

    graph LR A["🎤 주제 소개<br/>30초"] --> B["📊 데이터 & 학습<br/>1분"] B --> C["🔍 오분류 분석<br/>& 개선<br/>1분"] C --> D["📱 웹앱 시연<br/>& 마무리<br/>30초"]

    Step 6: 동료 평가와 시상¶

    동료 평가 방법

    각 모둠의 발표가 끝나면, 다른 모둠 학생들이 아래 기준으로 평가합니다.

    • 실용성 ⭐⭐⭐⭐⭐ : 실생활에서 쓸모가 있는가?
    • 창의성 ⭐⭐⭐⭐⭐ : 주제 선정이 독창적인가?
    • 완성도 ⭐⭐⭐⭐⭐ : 모델 정확도와 웹앱이 잘 작동하는가?
    • 발표력 ⭐⭐⭐⭐⭐ : 설명이 명확하고 이해하기 쉬운가?

    알아보기 🔍 — 평가 팁

    "잘했다 / 못했다"보다 구체적 피드백이 더 도움됩니다. - ✅ "손에 들고 찍은 사진을 추가한 게 인상적이었어요" - ❌ "그냥 좋았어요"

    시상 부문

    모든 발표와 평가가 끝나면, 투표 결과를 집계하여 시상합니다.

    • 🏆 가장 실용적인 AI — 실용성 점수 1위
    • 🎨 가장 창의적인 AI — 창의성 점수 1위
    • 💎 가장 완성도 높은 AI — 완성도 점수 1위
    • 🎤 최고의 발표팀 — 발표력 점수 1위

    ⚠️ 자주 하는 실수¶

    실수 1: 모델을 업로드하지 않고 공유 링크를 찾는 경우

    티처블머신에서 "모델 내보내기" → "업로드"를 반드시 클릭해야 공유 링크가 생성됩니다. 로컬(내 컴퓨터)에만 다운로드하면 다른 사람이 접속할 수 없습니다.

    실수 2: 구글 Sites를 "게시"하지 않는 경우

    사이트를 아무리 예쁘게 꾸며도 "게시(Publish)" 버튼을 누르지 않으면 나만 볼 수 있습니다. 게시 후 나타나는 URL을 공유해야 다른 사람이 접속할 수 있습니다.

    실수 3: 데이터 보강 시 한쪽 클래스만 추가하는 경우

    "사과" 클래스에만 50장을 추가하고 "토마토" 클래스는 그대로 두면, 모델이 데이터가 많은 쪽으로 편향됩니다. 모든 클래스의 데이터 수를 비슷하게 유지하세요.

    실수 4: 발표에서 과정 없이 결과만 보여주는 경우

    "우리 모델은 정확도 95%입니다!"만 말하면 청중은 감동을 받지 못합니다. "처음엔 70%였는데, 이런 오류를 발견하고, 이렇게 고쳐서 95%가 되었습니다" 라는 개선 스토리가 훨씬 설득력 있습니다.

    응용 문제¶

    Q3. "운동화 vs 슬리퍼" 분류 모델에서, 검은 슬리퍼를 자꾸 운동화로 분류합니다. 어떤 데이터를 보강하면 좋을까요?

    예시 답안: 검은색 슬리퍼 사진을 다양한 각도에서 추가하고, 동시에 검은색이 아닌 다른 색상의 운동화 사진도 추가하여 모델이 "색상"이 아닌 "형태(발등이 막혀 있는지, 열려 있는지)"를 학습하도록 합니다.
