    미니프로젝트 1차시 — 문제 발견과 기획¶

    Part 2: [1권] 미니프로젝트 — 우리만의 AI 분류기 만들기¶

    🎯 이 장에서 배우는 것¶

    • 학교·일상 속에서 이미지 분류로 해결할 수 있는 문제를 스스로 발견할 수 있다
    • '좋은 문제'와 '나쁜 문제'를 구별하는 기준을 설명할 수 있다
    • 프로젝트 기획서(문제 정의, 분류 대상, 데이터 수집 계획)를 작성할 수 있다
    • 팀원과 역할을 분담하고 협업 계획을 세울 수 있다
    • 다른 팀의 기획서에 건설적인 피드백을 남길 수 있다

    💡 왜 이걸 배우나요?¶

    여러분은 앞선 3차시 동안 이미지 분류가 무엇인지, 데이터를 어떻게 모으고 라벨링하는지, 그리고 간단한 모델을 학습시키는 과정까지 배웠습니다. 이제 진짜 도전이 시작됩니다.

    🤔 생각해보기: 세상에서 가장 뛰어난 기술자가 있어도, "무엇을 만들지" 결정하지 못하면 아무것도 시작할 수 없습니다.

    실제 AI 프로젝트에서 가장 중요한 단계는 코딩이 아니라 "어떤 문제를 풀 것인가?"를 정하는 기획 단계입니다. 구글, 네이버 같은 기업에서도 엔지니어들이 코드를 작성하는 시간보다 문제를 정의하고 계획을 세우는 시간이 더 깁니다.

    이번 차시에서는 여러분이 직접 학교와 일상을 관찰하며 문제를 찾고, 그 문제를 AI 이미지 분류기로 해결하는 기획서를 작성합니다. 이 기획서가 앞으로 2~3차시에 걸친 미니프로젝트의 설계도가 됩니다. 설계도 없이 집을 짓는 사람은 없겠죠? 🏗️

    📚 핵심 개념¶

    개념 1: 문제 발견 — "불편함 사냥꾼" 되기¶

    1. 비유로 시작

    탐정이 사건 현장에서 단서를 찾듯이, AI 프로젝트의 첫걸음은 일상 속 불편함이라는 단서를 수집하는 것입니다. 셜록 홈즈는 누구나 지나치는 것에서 문제를 발견했습니다. 여러분도 매일 지나치던 학교 복도, 급식실, 교실에서 "이건 좀 불편한데…"라는 순간을 포착하는 불편함 사냥꾼이 되어야 합니다.

    2. 정확한 정의

    📖 문제 발견(Problem Finding): 주변 환경을 관찰하여, 개선이 필요한 상황이나 자동화하면 편리해질 작업을 찾아내는 과정

    문제 발견은 단순히 "뭐 만들까?" 하고 머릿속으로만 생각하는 게 아닙니다. 직접 관찰하고, 기록하고, 질문하는 체계적인 과정입니다.

    3. 예시로 확인

    학교에서 발견할 수 있는 문제들을 살펴볼까요?

    🔍 알아보기: 학교 속 문제 발견 사례들

    사례 A — 분리수거 혼란 🗑️ - 관찰: 점심시간 후 분리수거함 앞에서 학생들이 "이거 플라스틱이야? 비닐이야?" 하며 망설임 - 불편함: 잘못 분류된 쓰레기를 환경미화 당번이 다시 분류해야 함

    사례 B — 식물 관리 어려움 🌱 - 관찰: 교실 화분의 잎이 누렇게 변했는데, 물이 부족한 건지 병에 걸린 건지 모름 - 불편함: 검색해봐도 어떤 상태인지 판단하기 어려움

    사례 C — 급식 메뉴 확인 🍽️ - 관찰: 급식 배식대 앞에서 "이 반찬이 뭐지?" 하며 알레르기 때문에 걱정하는 친구 - 불편함: 음식 사진만 보고 재료를 파악하기 어려움

    이처럼 눈을 크게 뜨고 관찰하면, AI가 도울 수 있는 문제는 곳곳에 숨어 있습니다.

    개념 2: 문제 선별 — "이미지 분류로 풀 수 있는가?"¶

    1. 비유로 시작

    낚시를 하러 갔는데 모든 물고기를 다 잡을 수는 없습니다. 우리 그물(이미지 분류)에 잡히는 물고기만 골라야 합니다. 아무리 좋은 문제라도 "사진을 찍어서 분류"하는 방식으로 풀 수 없다면, 이번 프로젝트에는 적합하지 않습니다.

    2. 정확한 정의

    📖 문제 선별(Problem Screening): 발견한 여러 문제 중에서, 주어진 기술과 자원으로 실제로 해결 가능한 문제를 골라내는 과정

    좋은 프로젝트 주제는 세 가지 조건을 동시에 만족해야 합니다.

    graph TD A["🎯 좋은 프로젝트 주제"] --> B["📸 사진으로 구별 가능한가?"] A --> C["📦 데이터를 모을 수 있는가?"] A --> D["⏰ 주어진 시간 안에 가능한가?"] B --> E["눈으로 봤을 때 차이가 보여야 함"] C --> F["직접 촬영하거나 인터넷에서 모을 수 있어야 함"] D --> G["분류 클래스가 2~5개 정도면 적당"]

    3. 예시로 확인

    앞에서 발견한 문제들을 이 세 가지 기준으로 점검해봅시다.

    💡 생각해보기: 문제 선별 체크

    사례 A — 분리수거 분류기 ✅✅✅ - 📸 사진으로 구별 가능? → 페트병, 캔, 종이, 비닐은 생김새가 다르다 ✅ - 📦 데이터 모을 수 있나? → 학교 쓰레기통에서 직접 촬영 가능 ✅ - ⏰ 시간 안에 가능? → 4개 클래스, 충분히 가능 ✅

    사례 B — 식물 건강 진단 ✅✅⚠️ - 📸 사진으로 구별 가능? → 건강한 잎 vs 병든 잎은 색이 다르다 ✅ - 📦 데이터 모을 수 있나? → 교실 화분이 몇 개 없어서 데이터가 부족할 수 있음 ⚠️ - ⏰ 시간 안에 가능? → 클래스 2~3개면 가능 ✅

    "오늘 급식이 맛있을지 예측" ❌❌❌ - 📸 사진으로 구별 가능? → "맛"은 사진으로 판단할 수 없다 ❌ - 이건 이미지 분류 문제가 아닙니다!

    개념 3: 프로젝트 기획서 — 설계도 그리기¶

    1. 비유로 시작

    여행을 떠나기 전에 여행 계획표를 만들죠? 어디로 갈지, 무엇을 할지, 누가 무엇을 준비할지 정해놓으면 여행이 훨씬 순조롭습니다. 프로젝트 기획서는 AI 프로젝트의 여행 계획표입니다. 🗺️

    2. 정확한 정의

    📖 프로젝트 기획서(Project Proposal): 해결할 문제, 사용할 방법, 필요한 자원, 일정과 역할을 정리한 계획 문서

    기획서는 길고 복잡할 필요가 없습니다. 핵심만 담은 1페이지 기획서면 충분합니다.

    3. 예시로 확인 — 1페이지 기획서 구성 요소

    기획서에 반드시 들어가야 할 항목은 다음 다섯 가지입니다.

    graph LR A["📄 1페이지 기획서"] --> B["1️⃣ 프로젝트 이름"] A --> C["2️⃣ 문제 정의"] A --> D["3️⃣ 분류 클래스"] A --> E["4️⃣ 데이터 수집 계획"] A --> F["5️⃣ 역할 분담"]
    • 1️⃣ 프로젝트 이름: 한눈에 어떤 프로젝트인지 알 수 있는 이름 (예: "AI 분리수거 도우미")
    • 2️⃣ 문제 정의: "누가, 언제, 어떤 불편함을 겪는지"를 한두 문장으로 작성
    • 3️⃣ 분류 클래스: AI가 구별할 카테고리 목록 (예: 페트병, 캔, 종이, 비닐)
    • 4️⃣ 데이터 수집 계획: 어디서, 어떻게, 몇 장의 사진을 모을지
    • 5️⃣ 역할 분담: 팀원 각자가 맡을 역할

    🔍 알아보기: 기획서 예시 — "AI 분리수거 도우미"

    프로젝트 이름: AI 분리수거 도우미 🗑️🤖

    문제 정의: 점심시간 후 분리수거함 앞에서 학생들이 쓰레기를 잘못 분류하는 일이 자주 발생한다. 환경미화 당번의 부담을 줄이기 위해, 쓰레기 사진을 찍으면 올바른 분리수거 카테고리를 알려주는 AI 분류기를 만든다.

    분류 클래스 (4개): 페트병 / 캔 / 종이 / 비닐

    데이터 수집 계획: - 촬영 장소: 학교 분리수거장, 교실 쓰레기통 - 촬영 방법: 스마트폰으로 다양한 각도에서 촬영 - 목표 수량: 클래스당 50장, 총 200장 - 인터넷 보조 데이터: 부족한 경우 검색 이미지 추가

    역할 분담: - 김하나: 팀장 / 기획서 정리 - 이두리: 데이터 수집 총괄 / 촬영 - 박세찬: 라벨링 및 데이터 정리 - 정나래: 모델 학습 및 테스트

    개념 4: 팀 협업 — 함께 만드는 힘¶

    1. 비유로 시작

    축구 경기에서 11명이 모두 공격수인 팀은 이길 수 없습니다. 골키퍼, 수비수, 미드필더, 공격수가 각자의 포지션에서 최선을 다할 때 강한 팀이 됩니다. AI 프로젝트도 마찬가지입니다. ⚽

    2. 정확한 정의

    📖 역할 분담(Role Assignment): 프로젝트 목표를 달성하기 위해 팀원 각자의 강점에 맞는 업무를 배분하는 것

    3. 예시로 확인 — AI 프로젝트의 핵심 역할

    graph TD A["🏆 AI 프로젝트 팀"] --> B["👑 팀장<br/>전체 일정 관리, 의견 조율"] A --> C["📸 데이터 담당<br/>사진 촬영, 수집"] A --> D["🏷️ 라벨링 담당<br/>데이터 분류, 정리"] A --> E["💻 모델 담당<br/>학습, 테스트"] A --> F["📊 발표 담당<br/>결과 정리, 발표 준비"]

    💡 팁: 팀원이 3명이면 한 사람이 두 역할을 겸할 수 있습니다. 중요한 것은 "누가 무엇을 하는지"가 명확한 것입니다. "다 같이 하자"는 결국 아무도 안 하게 되는 지름길입니다! 😅

    Step 1: 브레인스토밍 — 불편함 수집하기 (30분) 🧠¶

    목표: "AI 이미지 분류기가 해결해주면 좋겠는 문제"를 최대한 많이 떠올린다.

    방법:

    1. 종이 한 장(또는 메모 앱)을 준비합니다
    2. 3분 타이머를 설정합니다
    3. 아래 질문에 대해 떠오르는 것을 판단 없이 모두 적습니다

    🤔 브레인스토밍 질문들

    • "학교에서 눈으로 직접 확인해야 해서 귀찮았던 일이 있나?"
    • "사진 한 장으로 상태를 알 수 있다면 편할 게 뭐가 있을까?"
    • "분류가 잘 안 되어서 문제가 생긴 적이 있나?"
    • "동생이나 부모님이 일상에서 겪는 불편함은?"
    • "우리 동네에서 사진으로 해결할 수 있는 문제는?"
    1. 팀원이 있다면, 각자 적은 아이디어를 돌아가며 발표합니다
    2. 모든 아이디어를 한곳에 모읍니다 (비판은 나중에!)

    ⚠️ 브레인스토밍의 황금 규칙: "그건 안 될 것 같아"는 절대 금지! 엉뚱한 아이디어에서 최고의 프로젝트가 탄생합니다.

    참고 — 학생들이 실제로 낸 아이디어 예시:

    • 교복 명찰 착용 여부 확인기
    • 급식 잔반 종류 분류기
    • 운동장 잡초 vs 잔디 구별기
    • 필통 속 문구류 자동 분류기
    • 도서관 책 훼손 상태 검사기
    • 교실 창문 열림/닫힘 감지기
    • 구름 모양으로 날씨 예측기

    Step 2: 학교 탐험 — 현장 관찰하기 (30분) 🏫¶

    목표: 교실 밖으로 나가 실제 문제를 눈으로 확인하고 사진을 찍는다.

    💡 이 단계가 왜 중요한가요? 머릿속으로만 생각한 문제와 실제로 관찰한 문제는 다릅니다. 직접 가서 보면 "생각보다 이건 별로네" 또는 "오, 이건 진짜 심하다!"를 판단할 수 있습니다.

    방법:

    1. 스마트폰(또는 태블릿)을 들고 학교 곳곳을 탐색합니다
    2. 아래 장소들을 체크하며 돌아다닙니다

    탐색 추천 장소:

    • ♻️ 분리수거장 / 쓰레기통 주변
    • 🌿 화단 / 텃밭
    • 📚 도서관 / 독서실
    • 🍽️ 급식실 / 매점
    • 🏃 운동장 / 체육관

    • 🧹 복도 / 계단

    • 문제가 될 만한 장면을 사진으로 찍습니다 (최소 5장)

    • 각 사진 옆에 메모를 남깁니다:
    • "여기서 무슨 불편함이 있지?"
    • "사진으로 분류할 수 있을까?"

    📸 촬영 팁: 먼 거리에서 전체 장면 1장, 가까이에서 세부 모습 1장, 이렇게 2장씩 찍어두면 나중에 유용합니다.

    현장 탐험이 어려운 경우 (온라인 수업 등): - 집 안에서 관찰할 수 있는 문제를 찾아봅니다 - 예: 냉장고 속 식재료 분류, 빨래 종류 분류, 재활용 쓰레기 분류 - 인터넷 검색으로 "이미지 분류 프로젝트 아이디어"를 참고합니다

    Step 3: 주제 선정 — 최고의 문제 고르기 (20분) 🎯¶

    목표: 브레인스토밍과 현장 관찰 결과를 합쳐서 하나의 주제를 선정한다.

    방법:

    1. Step 1과 Step 2에서 모은 아이디어를 모두 펼쳐놓습니다
    2. 각 아이디어를 3가지 기준으로 평가합니다
    graph TD subgraph 평가 기준 A["📸 사진 구별 가능성<br/>눈으로 봤을 때 차이가 확실한가?"] B["📦 데이터 수집 가능성<br/>2~3일 안에 충분한 사진을 모을 수 있는가?"] C["❤️ 흥미도<br/>우리 팀이 정말 만들고 싶은가?"] end
    1. 각 기준에 대해 ⭐~⭐⭐⭐ (별 1~3개)로 점수를 매깁니다
    2. 별이 가장 많은 아이디어를 주제로 선정합니다

    🔍 알아보기: 주제 선정 예시

    아이디어 1: 분리수거 분류기 - 📸 사진 구별: ⭐⭐⭐ (페트병, 캔, 종이 차이가 확실함) - 📦 데이터 수집: ⭐⭐⭐ (학교 쓰레기통에서 바로 촬영 가능) - ❤️ 흥미도: ⭐⭐ (좋긴 한데 다른 팀도 할 것 같음) - 총: 8점

    아이디어 2: 급식 잔반 분류기 - 📸 사진 구별: ⭐⭐ (밥, 국, 반찬은 구별되지만 반찬끼리는 비슷) - 📦 데이터 수집: ⭐⭐ (점심시간에만 가능) - ❤️ 흥미도: ⭐⭐⭐ (진짜 해보고 싶음!) - 총: 7점

    아이디어 3: 교복 명찰 착용 확인기 - 📸 사진 구별: ⭐⭐⭐ (명찰 있음/없음은 명확) - 📦 데이터 수집: ⭐ (친구들 동의 필요, 초상권 문제) - ❤️ 흥미도: ⭐ (선생님은 좋아하실 텐데…) - 총: 5점

    Step 4: 기획서 작성 — 1페이지 설계도 (40분) 📝¶

    목표: 아래 템플릿을 따라 프로젝트 기획서를 완성한다.

    이제 코드 에디터(또는 문서 편집기)를 열고, 기획서를 작성해봅시다. 아래 Python 코드는 기획서 내용을 정리하고 출력하는 도우미입니다.

    # ========================================
# 🤖 AI 분류기 프로젝트 기획서 작성 도우미
# ========================================

# ---- 1. 기본 정보 ----
project_name = "AI 분리수거 도우미"           # 프로젝트 이름을 적어보세요
team_name = "에코 AI"                         # 팀 이름
team_members = ["김하나", "이두리", "박세찬"]  # 팀원 이름

# ---- 2. 문제 정의 ----
who = "우리 학교 학생들"           # 누가 불편함을 겪는가?
when = "점심시간 후 분리수거 시간"  # 언제 발생하는가?
what = "쓰레기를 올바른 분류함에 넣지 못해서 환경미화 당번이 다시 분류해야 한다"  # 어떤 불편함?
solution = "쓰레기 사진을 찍으면 어떤 분류함에 넣어야 하는지 알려주는 AI 분류기"   # 해결 방법

# ---- 3. 분류 클래스 ----
classes = ["페트병", "캔", "종이", "비닐"]  # AI가 구별할 카테고리들

# ---- 4. 데이터 수집 계획 ----
data_source = "학교 분리수거장에서 직접 촬영"
photos_per_class = 50     # 클래스당 목표 사진 수
total_photos = photos_per_class * len(classes)
collection_days = 2       # 수집에 걸리는 예상 일수
backup_plan = "부족하면 인터넷 이미지로 보충"

# ---- 5. 역할 분담 ----
roles = {
    "김하나": "팀장 / 기획서 관리 / 발표",
    "이두리": "데이터 수집(촬영) 총괄",
    "박세찬": "라벨링 및 모델 학습",
}

# ========================================
# 기획서 출력
# ========================================
print("=" * 50)
print(f"📄 프로젝트 기획서: {project_name}")
print(f"👥 팀 이름: {team_name}")
print(f"👤 팀원: {', '.join(team_members)}")
print("=" * 50)

print("\n📌 [문제 정의]")
print(f"  - 누가: {who}")
print(f"  - 언제: {when}")
print(f"  - 불편함: {what}")
print(f"  - 해결책: {solution}")

print(f"\n🏷️ [분류 클래스] ({len(classes)}개)")
for i, cls in enumerate(classes, 1):
    print(f"  {i}. {cls}")

print(f"\n📸 [데이터 수집 계획]")
print(f"  - 수집 방법: {data_source}")
print(f"  - 클래스당: {photos_per_class}장")
print(f"  - 총 목표: {total_photos}장")
print(f"  - 수집 기간: {collection_days}일")
print(f"  - 백업 계획: {backup_plan}")

print(f"\n👤 [역할 분담]")
for member, role in roles.items():
    print(f"  - {member}: {role}")

print("\n" + "=" * 50)
print("✅ 기획서 작성 완료!")
print("=" * 50)
