    미니프로젝트 1차시 — 문제발견에서 AI 학습까지¶

    Part 1 · [1권] AI 세계로의 초대 — 이해하고 체험하기 | 3차시 / 전체 10차시

    🎯 이 장에서 배우는 것¶

    • 학교생활 속 불편함에서 AI로 해결할 수 있는 문제를 스스로 발견할 수 있다
    • 문제 해결에 필요한 이미지 데이터를 체계적으로 수집하고 분류할 수 있다
    • 티처블머신을 활용하여 자신이 설계한 분류 모델을 직접 학습시킬 수 있다
    • 모델의 1차 정확도를 기록하고 개선 방향을 말할 수 있다

    💡 왜 이걸 배우나요?¶

    지금까지 우리는 AI가 무엇인지 이해하고, 이미 만들어진 AI를 체험해 보았습니다. 하지만 진짜 실력은 "내 문제를 내가 AI로 풀어보는 경험"에서 나옵니다.

    여러분이 매일 마주하는 학교생활을 떠올려 보세요.

    • 급식 후 잔반통 앞에서 "이건 일반 쓰레기? 음식물 쓰레기?" 고민하는 친구
    • 재활용 분리수거함 앞에서 "페트병은 어디에?" 헷갈리는 순간
    • "오늘 교복 제대로 입었나?" 등교길에 드는 걱정

    이런 작은 불편함이 바로 AI 프로젝트의 출발점입니다. 오늘은 문제를 찾고, 데이터를 모으고, 모델을 학습시키는 전 과정을 한 번에 경험합니다. 완벽하지 않아도 괜찮아요 — 중요한 것은 처음부터 끝까지 직접 해보는 것입니다! 🚀

    📚 핵심 개념¶

    개념 1: 문제 정의 — AI 프로젝트의 첫 단추¶

    🎒 비유로 시작

    여행을 갈 때 "어디로 갈지"를 정하지 않으면 짐도 싸지 못합니다. AI 프로젝트도 마찬가지예요. "무엇을 분류할 것인가?"를 먼저 정해야 어떤 데이터가 필요한지, 몇 개의 클래스(분류 그룹)가 필요한지 결정할 수 있습니다.

    📖 정확한 정의

    문제 정의란 해결하고 싶은 상황을 구체적이고 명확한 문장으로 표현하는 것입니다. 좋은 문제 정의에는 세 가지가 들어갑니다.

    • 누가 불편한가 (대상)
    • 무엇이 불편한가 (상황)
    • AI가 어떻게 도와줄 수 있는가 (해결 방식)

    ✅ 예시로 확인

    • ❌ 나쁜 예: "급식이 불편하다" → 너무 막연함
    • ✅ 좋은 예: "급식 후 잔반을 버릴 때, 학생들이 일반 쓰레기와 음식물 쓰레기를 헷갈려 한다. → AI 이미지 분류로 쓰레기 종류를 알려주는 도우미를 만들자"

    생각해보기 💬 학교에서 "이걸 AI가 자동으로 분류해주면 좋겠다!"고 느낀 적이 있나요? 30초간 떠올려 보세요.

    개념 2: 클래스 설계 — 분류 기준 정하기¶

    🎒 비유로 시작

    서랍 정리를 할 때 칸을 먼저 나누듯, AI에게도 "몇 개의 칸(클래스)으로 나눌 것인지" 알려주어야 합니다.

    📖 정확한 정의

    클래스(Class)란 AI가 구분해야 하는 각각의 카테고리(범주)를 말합니다. 예를 들어 "재활용 분류기"라면 다음과 같은 클래스를 만들 수 있습니다.

    • 클래스 1: 페트병
    • 클래스 2: 캔
    • 클래스 3: 종이
    • 클래스 4: 일반 쓰레기

    ✅ 예시로 확인

    graph LR A["📷 촬영한 사진"] --> B{"AI 분류 모델"} B --> C["페트병 ♻️"] B --> D["캔 🥫"] B --> E["종이 📄"] B --> F["일반 쓰레기 🗑️"]

    알아보기 📌 클래스는 2개 이상, 5개 이하로 시작하는 것이 좋습니다. 너무 많으면 데이터도 많이 필요하고, 정확도가 떨어지기 쉽습니다.

    개념 3: 데이터 수집 — AI의 학습 재료 모으기¶

    🎒 비유로 시작

    요리사에게 좋은 재료가 필요하듯, AI에게는 좋은 데이터가 필요합니다. 아무리 훌륭한 AI라도 흐릿하고 편향된 데이터로는 제대로 배울 수 없어요.

    📖 정확한 정의

    데이터 수집이란 AI 학습에 사용할 이미지(또는 텍스트, 소리 등)를 체계적으로 모으는 과정입니다. 이미지 분류 모델의 경우, 각 클래스별로 비슷한 수량의 사진을 다양한 각도·배경에서 촬영해야 합니다.

    ✅ 좋은 데이터의 3가지 조건

    • 다양성 — 각도, 배경, 밝기를 바꿔가며 촬영
    • 균형 — 클래스마다 비슷한 수의 사진 (최소 30장씩 권장)
    • 명확성 — 분류 대상이 사진 중앙에 뚜렷하게 보일 것

    알아보기 📌 클래스 A에 100장, 클래스 B에 10장이면? AI는 "일단 A라고 하면 맞겠지"라고 편향된 학습을 합니다. 이것을 데이터 불균형이라 합니다.

    개념 4: 모델 학습 — 티처블머신으로 AI 가르치기¶

    🎒 비유로 시작

    선생님이 학생에게 문제와 정답을 반복해서 보여주면 학생이 패턴을 익히듯, AI도 이미지(문제)와 레이블(정답)을 반복적으로 보면서 스스로 규칙을 찾습니다.

    📖 정확한 정의

    모델 학습(Training)이란 수집한 데이터를 AI에게 반복적으로 보여주어, 입력(사진)과 출력(클래스 이름)의 관계를 스스로 알아내게 하는 과정입니다. 티처블머신은 이 복잡한 과정을 클릭 몇 번으로 가능하게 해 줍니다.

    ✅ 학습 흐름 한눈에 보기

    graph TD A["1️⃣ 문제 정의"] --> B["2️⃣ 클래스 설계"] B --> C["3️⃣ 데이터 수집 (촬영)"] C --> D["4️⃣ 데이터 업로드"] D --> E["5️⃣ 모델 학습 (Train)"] E --> F["6️⃣ 테스트 및 정확도 기록"]

    🔨 따라하기¶

    ⏱️ 전체 예상 시간: 약 80분 (나머지 시간은 개념 학습 + 정리)

    Step 1: 문제 발견 브레인스토밍 (15분)¶

    모둠(3~4명)을 구성하고, 아래 질문에 대해 자유롭게 아이디어를 내봅니다.

    "학교에서 AI가 자동으로 분류해주면 좋겠는 것은?"

    진행 방법

    1. 포스트잇이나 노트에 1인당 3개 이상 아이디어를 적습니다
    2. 모둠원끼리 돌아가며 발표합니다
    3. 아래 기준으로 하나의 주제를 선정합니다

    주제 선정 기준 체크리스트

    • ✅ 사진 촬영이 가능한가? (교실·급식실 등에서)
    • ✅ 클래스를 2~4개로 나눌 수 있는가?
    • ✅ 각 클래스별 30장 이상 촬영할 수 있는가?
    • ✅ 모둠원 모두 흥미가 있는가?

    인기 주제 예시

    • 🍱 급식 잔반 분류 (국물류 / 밥류 / 반찬류)
    • ♻️ 재활용 분리수거 (페트병 / 캔 / 종이 / 일반쓰레기)
    • 👔 교복 착장 검사 (정상 착용 / 비정상 착용)
    • 🌱 화단 식물 건강 상태 (건강 / 물 부족 / 병충해)

    Step 2: 기획서 작성 (10분)¶

    아래 형식을 노트나 문서에 옮겨 적고 빈칸을 채워 봅니다.

    📋 미니프로젝트 기획서
━━━━━━━━━━━━━━━━━━━━━

🔹 모둠 이름:
🔹 모둠원:

🔹 프로젝트 제목:
🔹 문제 정의: (누가 / 무엇이 불편 / AI가 어떻게 도와줄까)

🔹 클래스 목록:
   - 클래스 1:
   - 클래스 2:
   - 클래스 3:
   (필요 시 추가)

🔹 데이터 수집 계획:
   - 촬영 장소:
   - 클래스별 목표 사진 수:
   - 촬영 시 주의사항:

🔹 예상 결과:

    생각해보기 💬 기획서를 작성하면 "이 프로젝트가 실현 가능한가?"를 미리 점검할 수 있습니다. 기획 없이 바로 촬영을 시작하면 중간에 방향을 잃기 쉽습니다.

    Step 3: 데이터 수집 — 사진 촬영 (20분)¶

    스마트폰(또는 태블릿, 크롬북 카메라)을 이용해 사진을 촬영합니다.

    촬영 꿀팁 🍯

    • 배경을 바꿔가며 촬영하세요 (책상 위, 바닥, 손 위 등)
    • 각도를 바꿔가며 촬영하세요 (위에서, 비스듬히, 옆에서)
    • 밝기가 다른 곳에서도 찍으세요 (창가, 복도, 교실 안쪽)
    • 사진 속에 분류 대상만 크게 나오도록 하세요
    • 클래스별로 폴더를 나누어 저장하세요

    데이터 수집 진행표 (노트에 기록)

    클래스 이름     | 목표 수량 | 현재 수량 | 달성 여부
페트병            30장       ___장      □
캔                30장       ___장      □
종이              30장       ___장      □

    알아보기 📌 시간이 부족하면 티처블머신의 웹캠 촬영 기능을 활용하세요! 실물을 카메라 앞에 놓고 녹화 버튼을 누르면 1초에 여러 장을 빠르게 캡처할 수 있습니다.

    Step 4: 티처블머신에 데이터 업로드 (10분)¶

    1. 웹 브라우저에서 teachablemachine.withgoogle.com 에 접속합니다
    2. "시작하기" → "이미지 프로젝트" → "표준 이미지 모델" 을 선택합니다
    3. 왼쪽에 기본으로 보이는 Class 1, Class 2 이름을 여러분의 클래스 이름으로 바꿉니다
    4. 클래스가 더 필요하면 "클래스 추가" 버튼을 클릭합니다
    5. 각 클래스 칸에서 "업로드" 를 눌러 해당 폴더의 사진을 모두 선택합니다
    graph LR A["티처블머신 접속"] --> B["이미지 프로젝트 선택"] B --> C["클래스 이름 설정"] C --> D["사진 업로드"] D --> E["업로드 완료 ✅"]

    웹캠으로 직접 촬영하는 경우, "웹캠" 버튼을 누르고 실물을 카메라 앞에 가져다 대세요. 길게 누르기(Hold to Record)를 사용하면 연속 촬영이 됩니다.

    Step 5: 모델 학습시키기 (10분)¶

    1. 모든 클래스에 사진이 업로드되었는지 확인합니다
    2. 화면 중앙의 "모델 학습시키기(Train Model)" 버튼을 클릭합니다
    3. 학습이 진행되는 동안 브라우저 탭을 닫거나 이동하지 마세요! ⚠️
    4. 학습이 끝나면 오른쪽 "미리보기(Preview)" 영역이 활성화됩니다

    학습 중 화면에서 볼 수 있는 것

    • 에포크(Epoch): AI가 전체 데이터를 몇 번 반복 학습했는지
    • 손실(Loss): 오답률과 비슷한 개념. 낮을수록 좋음
    • 정확도(Accuracy): 학습 데이터에 대해 맞춘 비율. 높을수록 좋음

    알아보기 📌 기본 설정(에포크 50, 배치 크기 16, 학습률 0.001)으로 충분합니다. 나중에 정확도를 높이고 싶을 때 에포크를 늘려볼 수 있어요.

    Step 6: 테스트 및 정확도 기록 (15분)¶

    1. 미리보기 영역에서 웹캠을 켜거나, 학습에 사용하지 않은 새로운 사진을 업로드합니다
    2. AI가 어떤 클래스로 분류하는지, 확률(%)은 얼마인지 확인합니다
    3. 아래 형식으로 테스트 결과를 기록합니다
    📊 1차 테스트 결과
━━━━━━━━━━━━━━━━━━

테스트 사진 수: ___장

정답으로 분류한 수: ___장
오답으로 분류한 수: ___장

    1. 모둠원끼리 결과를 공유하고, 왜 틀렸을지 함께 이야기합니다

    정리하기 📋 1차 정확도가 낮아도 실망하지 마세요! 다음 차시에서 데이터를 보강하고 모델을 개선할 기회가 있습니다. 중요한 것은 "왜 틀렸는지"를 분석하는 것입니다.

    📝 전체 코드¶

    이번 차시는 티처블머신(웹 도구)을 사용하므로 별도의 프로그래밍 코드는 없습니다. 대신 학습이 끝난 모델을 내보내기(Export) 하여 다음 차시에서 활용할 수 있도록 저장해 둡니다.

    모델 저장 방법

    1. 티처블머신 화면 상단의 "모델 내보내기(Export Model)" 클릭
2. "Tensorflow.js" 탭 선택
3. "모델 업로드" 클릭 → 공유 가능한 링크가 생성됨
4. 생성된 링크를 복사하여 기획서에 기록

   예시 링크: https://teachablemachine.withgoogle.com/models/XXXXX/

    이 링크만 있으면 다른 컴퓨터에서도 여러분의 모델을 불러올 수 있습니다. 반드시 기록해 두세요! 📎

    ⚠️ 자주 하는 실수¶

    실수 1: 데이터를 한 가지 배경에서만 촬영한다 🖼️

    흰 책상 위에서만 페트병을 찍으면, AI는 "흰 배경 = 페트병"이라고 잘못 학습할 수 있습니다. 반드시 다양한 배경과 각도에서 촬영하세요.

    실수 2: 클래스별 사진 수가 크게 차이난다 ⚖️

    A 클래스에 80장, B 클래스에 10장이면 AI는 "일단 A라고 대답하자"는 편향이 생깁니다. 클래스 간 사진 수를 최대한 비슷하게 맞추세요.

    실수 3: 학습 도중에 브라우저 탭을 닫는다 🚫

    티처블머신은 브라우저 안에서 학습이 이루어집니다. 탭을 닫거나 다른 사이트로 이동하면 학습이 중단되고, 처음부터 다시 해야 합니다.

    실수 4: 테스트에 학습 데이터를 그대로 사용한다 🔄

    학습에 쓴 사진으로 테스트하면 당연히 정확도가 높게 나옵니다. 진짜 실력을 알려면 학습에 사용하지 않은 새로운 사진으로 테스트해야 합니다.

    도전 1: 데이터 증강 실험 🔬
