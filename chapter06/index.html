    Chapter 6. 프롬프트 실전 테크닉 – 단계별 지시와 예시 활용¶

    Part 2. AI와 대화하는 기술 – 프롬프트 엔지니어링 | 6차시 (2시간)

    🎯 이 장에서 배우는 것¶

    • Few-shot 프롬프트(예시 제공) 기법을 이해하고 직접 작성할 수 있다
    • Chain-of-Thought(단계별 사고 유도) 기법으로 AI의 추론을 이끌어낼 수 있다
    • 복잡한 과제를 여러 단계로 쪼개어 AI에게 지시할 수 있다
    • 초안 → 피드백 → 수정의 반복(Iterate) 과정을 실습할 수 있다

    💡 왜 이걸 배우나요?¶

    생각해보기 🤔

    친구에게 "맛있는 거 만들어줘"라고 부탁하면 어떻게 될까요? 아마 뭘 만들어야 할지 몰라 멍하니 서 있을 겁니다. 하지만 "어제 먹었던 계란볶음밥처럼, 밥에 계란이랑 파 넣고, 간장으로 간 맞춰서 볶아줘"라고 하면? 바로 행동에 옮길 수 있죠.

    AI도 마찬가지입니다. 5장에서 좋은 프롬프트의 기본 구조를 배웠다면, 이번 장에서는 실전에서 쓰는 고급 테크닉을 익힙니다.

    핵심은 세 가지입니다.

    • 예시를 보여주기 — "이런 식으로 해줘" (Few-shot)
    • 생각의 순서를 알려주기 — "먼저 ~하고, 그 다음 ~해줘" (Chain-of-Thought)
    • 한 번에 완벽을 기대하지 않기 — 초안을 받고, 고치고, 또 고치기 (Iterate)

    이 세 가지만 익히면, AI에게 어떤 복잡한 과제든 단계적으로 맡길 수 있습니다.

    📚 핵심 개념¶

    개념 1: Few-shot 프롬프트 🎯¶

    ① 비유로 시작

    새로운 아르바이트 첫날, 사장님이 "손님 응대 잘 해"라고만 하면 막막합니다. 하지만 옆에서 선배가 손님 한두 명을 응대하는 걸 직접 보여주면, 금방 감을 잡죠. Few-shot 프롬프트는 바로 이 "선배가 시범을 보여주는 것"입니다.

    ② 정확한 정의

    • Zero-shot: 예시 없이 지시만 하는 것 ("감정을 분석해줘")
    • One-shot: 예시를 1개 보여주고 지시하는 것
    • Few-shot: 예시를 2~5개 보여주고 지시하는 것

    정리하기 📌

    Few-shot 프롬프트란, AI에게 "이런 입력에는 이런 출력을 해줘"라는 예시를 먼저 보여준 뒤, 새로운 입력에 대해 같은 패턴으로 응답하게 하는 기법입니다.

    ③ 예시로 확인

    다음 예시를 참고해서 문장의 감정을 분석해줘.

[예시 1]
문장: "오늘 시험 망했어..."
감정: 😢 슬픔

[예시 2]
문장: "드디어 방학이다!"
감정: 😆 기쁨

[예시 3]
문장: "왜 나한테만 이러는 거야?"
감정: 😠 분노

이제 아래 문장을 분석해줘.
문장: "내일 소풍인데 비가 온대..."

    이렇게 하면 AI는 패턴을 파악해서 "😟 걱정/실망"처럼 일관된 형식으로 답합니다. 예시 없이 물어보면 형식이 들쭉날쭉해질 수 있어요.

    flowchart LR A["Zero-shot\n예시 0개"] --> B["One-shot\n예시 1개"] B --> C["Few-shot\n예시 2~5개"] A -.- D["형식이 불안정"] B -.- E["기본 패턴 학습"] C -.- F["안정적이고\n일관된 출력"]

    개념 2: Chain-of-Thought (CoT) 🔗¶

    ① 비유로 시작

    수학 선생님이 "답만 쓰세요"라고 하면 틀리기 쉽지만, "풀이 과정을 단계별로 쓰세요"라고 하면 실수가 줄어듭니다. Chain-of-Thought는 AI에게도 "생각하는 과정을 보여줘"라고 요청하는 것입니다.

    ② 정확한 정의

    정리하기 📌

    Chain-of-Thought(사고의 연쇄)란, AI에게 최종 답을 바로 말하지 말고, 중간 추론 과정을 단계별로 서술하도록 유도하는 기법입니다. "단계별로 생각해줘(Let's think step by step)"라는 한 마디만 추가해도 효과가 있습니다.

    ③ 예시로 확인

    ❌ CoT 없이 (바로 답 요구) 

    학교 축제에서 붕어빵을 팔려고 합니다. 
재료비 3만원, 봉투비 5천원입니다. 
붕어빵 1개에 500원이고 150개를 만들 수 있습니다.
이익은 얼마인가요?
    → AI가 바로 "40,000원"이라고 답할 수 있지만, 계산이 복잡해지면 틀릴 확률이 높아집니다.

    ✅ CoT 적용 (단계별 사고 유도) 

    학교 축제에서 붕어빵을 팔려고 합니다.
재료비 3만원, 봉투비 5천원입니다.
붕어빵 1개에 500원이고 150개를 만들 수 있습니다.

다음 단계에 따라 이익을 계산해줘:
1단계: 총 비용을 구해줘
2단계: 전부 팔았을 때 총 수입을 구해줘  
3단계: 수입에서 비용을 빼서 이익을 구해줘

    → AI가 각 단계를 보여주므로 어디서 틀렸는지 확인할 수 있고, 정확도도 올라갑니다.

    flowchart TD A["복잡한 문제"] --> B{"CoT 적용?"} B -->|No| C["바로 최종 답\n❌ 틀릴 가능성 높음"] B -->|Yes| D["1단계: 조건 정리"] D --> E["2단계: 중간 계산"] E --> F["3단계: 최종 답 도출\n✅ 정확도 높음"]

    개념 3: 과제 분할 (Task Decomposition) ✂️¶

    ① 비유로 시작

    "방 청소해!"라고 하면 막막하지만, "먼저 바닥의 옷을 모아 → 책상 위를 정리하고 → 마지막에 청소기를 돌려"라고 하면 바로 시작할 수 있습니다. 큰 과제를 작은 단계들로 쪼개는 것, 이것이 과제 분할입니다.

    ② 정확한 정의

    정리하기 📌

    과제 분할이란, 하나의 복잡한 요청을 여러 개의 작고 구체적인 하위 지시로 나누어 AI에게 순서대로 수행하게 하는 전략입니다. 각 단계의 출력이 다음 단계의 입력이 됩니다.

    ③ 예시로 확인

    ❌ 한 번에 모든 것을 요청 

    한국 학교생활을 소개하는 3분짜리 영상 스크립트를 써줘. 
재미있고 외국인이 이해하기 쉽게.

    ✅ 단계별로 나누어 요청 

    [1단계] 한국 학교생활에서 외국인이 신기해할 만한 주제 5개를 뽑아줘.
[2단계] 그 중 3개를 골라서 각각 한 줄 설명을 써줘.
[3단계] 그 내용으로 3분 영상 스크립트의 개요(인트로-본문-아웃트로)를 잡아줘.
[4단계] 개요를 바탕으로 전체 스크립트를 작성해줘.

    각 단계가 끝날 때마다 결과를 확인하고, 마음에 안 드는 부분을 수정한 뒤 다음 단계로 넘어갈 수 있습니다.

    개념 4: 반복 다듬기 (Iteration) 🔄¶

    ① 비유로 시작

    미술 시간에 그림을 그릴 때, 연필로 밑그림 → 수정 → 채색 → 세부 묘사 순서로 진행하죠? 처음부터 완성작이 나오지 않습니다. AI와의 대화도 마찬가지로 초안 → 피드백 → 수정을 반복해야 합니다.

    ② 정확한 정의

    정리하기 📌

    Iteration(반복 다듬기)이란, AI의 첫 응답을 완성품이 아닌 초안(draft)으로 보고, 구체적인 피드백을 주어 여러 차례 수정하는 과정입니다. 보통 3회 정도 반복하면 만족스러운 결과를 얻을 수 있습니다.

    ③ 예시로 확인

    [1회차 - 초안 요청]
"한국의 급식 문화를 소개하는 문단을 써줘."

[2회차 - 피드백]
"좋은데, 너무 딱딱해. 중학생 유튜버가 말하는 것처럼 
친근한 말투로 바꿔줘. 그리고 구체적인 메뉴 예시를 3개 넣어줘."

[3회차 - 추가 수정]
"훨씬 나아! 마지막에 외국인이 놀라는 리액션 대사를 
하나 추가하고, 전체를 150자 이내로 줄여줘."
    flowchart LR A["초안 요청\n(1회차)"] --> B["AI 응답\n확인"] B --> C["피드백 작성\n(2회차)"] C --> D["수정된 응답\n확인"] D --> E["추가 피드백\n(3회차)"] E --> F["최종 결과물 ✅"] style A fill:#e1f5fe style F fill:#c8e6c9

    알아보기 💡

    좋은 피드백의 세 가지 요소: - 구체적으로 — "더 좋게 해줘" ❌ → "말투를 반말로 바꿔줘" ✅ - 방향을 제시 — "이건 싫어" ❌ → "~처럼 바꿔줘" ✅ - 한 번에 1~2가지만 — 피드백이 너무 많으면 AI도 헷갈립니다

    🔨 따라하기¶

    실습 목표: "외국인 친구에게 한국 학교생활을 소개하는 3분 영상 스크립트"를 AI와 함께 단계적으로 완성합니다.

    사용 도구: ChatGPT, Gemini, Claude 등 어떤 AI 챗봇이든 가능합니다.

    Step 1: Few-shot으로 톤 설정하기¶

    AI에게 원하는 말투와 형식의 예시를 먼저 보여줍니다.

    프롬프트를 복사해서 AI에 붙여넣으세요:

    나는 외국인 친구에게 한국 학교생활을 소개하는 유튜브 영상 스크립트를 
쓰려고 해. 아래 예시처럼 친근하고 재미있는 말투로 써줘.

[예시 1]
주제: 한국의 편의점
스크립트: "여러분, 한국 편의점이 얼마나 대단한지 아세요? 
삼각김밥 하나에 천 원! 맛은 미슐랭급이에요. 
새벽 2시에 라면이 먹고 싶다? 편의점 가면 끝!"

[예시 2]  
주제: 한국의 지하철
스크립트: "서울 지하철은 진짜 다른 세계예요. 
와이파이 빵빵, 냉난방 완벽, 그리고 2분마다 한 대씩 와요.
이걸 경험하면 다른 나라 지하철이 슬퍼질 거예요."

이제 '한국 학교생활'을 주제로, 위 예시와 같은 톤으로 
소개할 만한 주제 5개를 뽑아줘. 
각 주제마다 한 줄 설명도 붙여줘.

    예상 결과: AI가 "급식", "야자(야간자율학습)", "체육대회", "교복", "쉬는 시간 간식" 등의 주제를 예시와 비슷한 톤으로 제안합니다.

    Step 2: Chain-of-Thought로 구조 잡기¶

    5개 주제 중 마음에 드는 3개를 골라, AI에게 단계적으로 구조를 만들게 합니다.

    위 주제 중에서 [급식], [체육대회], [야자]를 선택했어.

아래 단계를 순서대로 진행해줘:

1단계: 각 주제별로 외국인이 가장 놀랄 포인트 1가지를 정리해줘.
2단계: 3분 영상의 시간 배분을 정해줘 (인트로 / 주제1 / 주제2 / 주제3 / 아웃트로).
3단계: 각 파트에서 꼭 들어가야 할 핵심 한 문장을 써줘.

    알아보기 💡

    "[급식], [체육대회], [야자]"는 여러분이 실제로 고른 주제로 바꿔도 됩니다. Step 1에서 AI가 제안한 것 중 자유롭게 선택하세요!

    Step 3: 과제 분할로 스크립트 작성 요청¶

    구조가 잡혔으니, 이제 실제 스크립트 작성을 요청합니다.

    위에서 만든 구조를 바탕으로 전체 영상 스크립트를 작성해줘.

조건:
- 총 분량: 3분 읽기 분량 (약 500~600단어)
- 말투: 유튜버가 카메라 앞에서 직접 말하는 것처럼
- 각 주제 전환 시 재미있는 연결 문장 넣기
- 인트로에서 "안녕, 나는 한국 고등학생이야"로 시작
- 아웃트로에서 구독과 댓글 유도

    Step 4: 첫 번째 Iteration – 톤 조정¶

    AI의 초안을 읽어보고, 구체적인 피드백을 줍니다.

    고마워! 전체적으로 좋은데 몇 가지 수정해줘:

1. 인트로가 좀 길어. 3문장 이내로 줄여줘.
2. 급식 파트에서 실제 메뉴 이름을 3개 넣어줘 
   (예: 제육볶음, 떡볶이, 불고기).
3. 야자 설명이 너무 진지해. 유머를 하나 추가해줘.

    Step 5: 두 번째 Iteration – 디테일 보강¶

    수정본을 확인한 뒤, 마무리 피드백을 줍니다.

    거의 완벽해! 마지막으로:

1. 체육대회 파트에 "반 전체가 같은 색 티셔츠를 입는다"는 
   내용을 추가해줘.
2. 전체 스크립트에서 영어 단어를 괄호로 병기해줘.
   예: 급식(school lunch)
3. 아웃트로를 더 짧고 임팩트 있게 바꿔줘.

    Step 6: 최종 확인 및 프롬프트 템플릿 만들기¶

    완성된 스크립트를 읽어보고, 이 과정에서 사용한 프롬프트 패턴을 나만의 템플릿으로 정리합니다.

    방금 우리가 진행한 과정을 정리해줘:
1. 어떤 기법을 어떤 순서로 사용했는지
2. 각 기법이 결과물에 어떤 영향을 미쳤는지
3. 이 패턴을 다른 주제에도 쓸 수 있는 범용 템플릿으로 만들어줘

    생각해보기 🤔

    처음 한 번에 모든 것을 요청했을 때와, 단계별로 나누어 진행했을 때 결과물의 차이가 느껴지나요? 어떤 단계에서 결과가 가장 크게 좋아졌는지 생각해보세요.

    📝 전체 프롬프트 흐름 정리¶

    아래는 이번 실습에서 사용한 전체 프롬프트 흐름입니다. 다른 주제에도 이 순서를 적용할 수 있습니다.

    # === 나만의 프롬프트 템플릿 === #

# [1단계] Few-shot으로 톤/형식 설정
"""
나는 [목적]을 위한 [결과물 종류]를 만들려고 해.
아래 예시처럼 [원하는 톤/형식]으로 써줘.

[예시 1]
입력: ...
출력: ...

[예시 2]
입력: ...
출력: ...

이제 [새로운 입력]에 대해 같은 방식으로 해줘.
"""

# [2단계] Chain-of-Thought로 구조화
"""
아래 단계를 순서대로 진행해줘:
1단계: [첫 번째 작업]
2단계: [두 번째 작업]
3단계: [세 번째 작업]
"""

# [3단계] 과제 분할로 본문 작성
"""
위 구조를 바탕으로 [결과물]을 작성해줘.
조건:
- [분량]
- [말투]
- [포함할 요소]
"""

# [4~5단계] Iteration (2~3회 반복)
"""
[구체적 피드백 1]
[구체적 피드백 2]
[구체적 피드백 3]
"""

    ⚠️ 자주 하는 실수¶

    실수 1: 예시(Few-shot)의 형식이 제각각 😵

    ❌ 잘못된 예시 제공:
[예시 1] 문장: 기쁘다 → 긍정
[예시 2] "슬퍼..."라는 문장은 부정적 감정이에요.

→ 예시마다 형식이 다르면 AI가 패턴을 잡지 못합니다.

✅ 올바른 예시 제공:
[예시 1] 문장: "기쁘다" → 감정: 긍정
[예시 2] 문장: "슬퍼..." → 감정: 부정

→ 모든 예시의 형식을 통일하세요!

    실수 2: Chain-of-Thought 없이 복잡한 문제 던지기 🎯

    ❌ "이 데이터를 분석해서 결론 내려줘."
→ AI가 중간 과정을 건너뛰고 엉뚱한 결론을 낼 수 있습니다.

✅ "이 데이터를 분석해줘. 
    먼저 전체 경향을 파악하고, 
    그 다음 이상한 점을 찾고, 
    마지막으로 결론을 내려줘."
→ 단계를 나눠주면 정확도가 크게 올라갑니다.

    실수 3: 피드백이 모호함 🌫️

    ❌ "좀 더 좋게 해줘."
❌ "느낌이 별로야."

✅ "두 번째 문단의 말투를 반말에서 존댓말로 바꿔줘."
✅ "예시를 2개 더 추가하고, 전체 길이를 200자 줄여줘."

→ 피드백은 '무엇을', '어떻게'가 명확해야 합니다.

    실수 4: 한 번에 너무 많은 피드백 주기 📦

    ❌ 한 번에 수정사항 7~8개를 나열
→ AI가 일부를 빠뜨리거나 우선순위를 혼동합니다.

✅ 한 번에 2~3개씩, 여러 차례 나누어 피드백
→ 차근차근 반복하는 것이 결과가 훨씬 좋습니다.
