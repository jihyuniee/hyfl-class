    미니프로젝트 3차시 — 정확도 개선과 웹앱 제작¶

    Part 2 | [1권] 미니프로젝트 — 우리만의 AI 분류기 만들기

    🎯 이 장에서 배우는 것¶

    • 오류 분석 결과를 바탕으로 데이터를 보완하고 모델을 재학습시킬 수 있다
    • 데이터 증강이 무엇인지 설명하고 직접 적용할 수 있다
    • 학습된 모델을 웹앱으로 내보내고 Google Sites에 배포할 수 있다
    • 다른 사람의 웹앱을 사용해보고 사용성 피드백을 작성할 수 있다

    💡 왜 이걸 배우나요?¶

    지난 시간, 우리는 AI 분류기를 만들고 테스트하면서 "어? 이건 왜 틀리지?"라는 순간들을 만났습니다. 오류를 분석하고 원인을 찾았죠.

    하지만 원인을 아는 것과 문제를 해결하는 것은 다릅니다.

    병원에 비유해봅시다. 의사가 "감기입니다"라고 진단만 하고 약을 안 주면 어떨까요? 진단(오류 분석)은 2차시에서 했으니, 오늘은 처방과 치료(데이터 보완 + 재학습)를 합니다.

    그리고 한 걸음 더 나아갑니다. 완성된 AI 분류기를 누구나 사용할 수 있는 웹앱으로 만들어 공개합니다. 여러분이 만든 AI를 친구가, 가족이, 선생님이 직접 써보는 경험 — 이것이 바로 진짜 AI 프로젝트의 완성입니다.

    🔑 핵심 메시지: AI 개발은 "만들기 → 테스트 → 개선 → 배포" 순환의 연속입니다. 오늘 이 전체 과정을 완주합니다!

    📚 핵심 개념¶

    개념 1: 데이터 증강(Data Augmentation)¶

    1. 비유로 시작¶

    시험 공부를 한다고 상상해보세요. 교과서 예제 10문제만 풀면 실전 시험에서 당황할 수 있습니다. 하지만 같은 유형의 문제를 숫자만 바꾸거나, 조건을 살짝 비틀어서 50문제를 풀면 어떨까요? 훨씬 다양한 상황에 대비할 수 있겠죠.

    데이터 증강은 AI에게 이 "변형 문제집"을 만들어주는 것과 같습니다.

    2. 정확한 정의¶

    데이터 증강(Data Augmentation)이란, 기존 학습 데이터를 다양한 방법으로 변형하여 새로운 학습 데이터를 만들어내는 기법입니다. 원본 데이터의 핵심 의미는 유지하면서, AI가 더 다양한 상황을 경험할 수 있도록 돕습니다.

    이미지 분류에서 자주 사용하는 증강 방법은 다음과 같습니다.

    • 회전: 이미지를 살짝 기울이기 (예: 10도, 20도)
    • 반전: 좌우를 뒤집기
    • 밝기 조절: 밝게 또는 어둡게 만들기
    • 확대/축소: 일부를 크롭하거나 줄이기
    • 노이즈 추가: 약간의 흐림이나 점을 추가하기

    3. 예시로 확인¶

    "고양이"를 분류하는 모델을 만든다고 합시다. 원본 사진 1장에서 증강을 적용하면:

    graph LR A["🐱 원본 사진 1장"] --> B["↩️ 좌우 반전"] A --> C["🔄 15도 회전"] A --> D["🔆 밝기 높임"] A --> E["🔅 밝기 낮춤"] A --> F["🔍 일부 확대"] B --> G["총 6장의 학습 데이터 확보!"] C --> G D --> G E --> G F --> G A --> G

    Teachable Machine에서는 웹캠으로 촬영할 때 다양한 각도, 거리, 조명에서 찍는 것이 가장 쉬운 증강 방법입니다.

    📌 알아보기: Teachable Machine은 별도의 증강 코드 없이도, 촬영 시 다양한 조건을 만들면 자연스럽게 증강 효과를 얻을 수 있습니다. 그래서 "의도적으로 다양하게 찍기"가 핵심입니다!

    개념 2: 데이터 보완 전략¶

    1. 비유로 시작¶

    축구팀 감독이 경기 영상을 분석했더니, "왼쪽 수비가 약하다"는 문제를 발견했습니다. 해결책은? 왼쪽 수비 선수를 추가 훈련시키거나, 새로운 선수를 영입하겠죠. 약점을 정확히 파악했기 때문에 효과적인 보완이 가능합니다.

    2. 정확한 정의¶

    데이터 보완 전략이란, 오류 분석 결과를 기반으로 부족하거나 편향된 데이터를 체계적으로 추가·수정하는 과정입니다. 단순히 데이터를 "많이" 모으는 것이 아니라, "어디가 부족한지"를 알고 그 부분을 집중적으로 채우는 것이 핵심입니다.

    2차시에서 작성한 오류 분석 결과를 다시 꺼내봅시다. 오류의 원인별로 보완 방법이 다릅니다.

    • 데이터 수 부족 → 해당 클래스의 데이터를 추가 수집
    • 특정 조건에서만 촬영 → 다양한 조건(배경, 조명, 각도)으로 재촬영
    • 클래스 간 데이터 수 불균형 → 적은 쪽의 데이터를 늘려 균형 맞추기
    • 애매한 데이터 포함 → 잘못 분류된 데이터를 제거하거나 재분류

    3. 예시로 확인¶

    "재활용 분류기"를 만들고 있는데, 2차시 오류 분석에서 "투명 페트병"과 "유리병"을 자주 혼동한다고 발견했다면:

    graph TD A["🔍 오류 원인: 투명 페트병 ↔ 유리병 혼동"] --> B["원인 1: 둘 다 투명해서<br/>비슷하게 보임"] A --> C["원인 2: 유리병 데이터가<br/>15장으로 적음"] B --> D["✅ 보완: 라벨이 보이는 각도,<br/>손에 쥔 모습 등 차이점이<br/>드러나는 사진 추가"] C --> E["✅ 보완: 유리병 사진을<br/>40장 이상으로 늘리기"] D --> F["모델 재학습"] E --> F

    💡 생각해보기: 우리 팀의 AI 분류기에서 가장 많이 틀린 부분은 어디였나요? 그 원인은 무엇이었나요? 어떤 데이터를 추가하면 개선될 수 있을지 3가지 이상 적어봅시다.

    개념 3: 모델 내보내기와 웹앱 배포¶

    1. 비유로 시작¶

    맛있는 요리를 만들었다고 합시다. 주방(Teachable Machine)에서만 먹을 수 있다면 아쉽겠죠? 포장(내보내기)해서 배달(배포)하면 누구나 맛볼 수 있습니다. 웹앱 제작은 바로 이 "포장 + 배달" 과정입니다.

    2. 정확한 정의¶

    모델 내보내기(Export)란, 학습이 완료된 AI 모델을 다른 환경에서 사용할 수 있는 형태로 변환하는 것입니다.

    웹앱 배포(Deploy)란, 내보낸 모델을 웹 페이지에 삽입하여 인터넷 브라우저로 누구나 접근할 수 있게 만드는 것입니다.

    오늘 우리가 사용할 도구의 역할을 정리하면:

    graph LR A["🧠 Teachable Machine<br/>모델 학습 완료"] -->|"내보내기<br/>(업로드)"| B["☁️ 클라우드<br/>모델 저장소"] B -->|"공유 링크 생성"| C["📋 모델 URL 복사"] C -->|"HTML 코드에 삽입"| D["🌐 Google Sites<br/>웹앱 완성!"]

    3. 예시로 확인¶

    Teachable Machine에서 "모델 내보내기"를 누르면, 여러 옵션이 나타납니다.

    • Tensorflow.js → 웹 브라우저에서 바로 실행 (오늘 사용할 방식)
    • Tensorflow Lite → 모바일 앱에서 실행
    • Tensorflow → Python 프로그램에서 실행

    우리는 Tensorflow.js의 "업로드" 옵션을 선택하여, 생성된 코드 스니펫(짧은 코드 조각)을 Google Sites에 붙여넣을 것입니다.

    개념 4: 사용성 평가(Usability Evaluation)¶

    1. 비유로 시작¶

    새로 문을 연 식당에 처음 간 손님이 "메뉴판이 어디 있지?", "주문은 어떻게 하지?"라고 헤매고 있다면, 음식이 아무리 맛있어도 좋은 평가를 받기 어렵습니다. 웹앱도 마찬가지입니다. AI의 정확도가 높아도 사용하기 불편하면 의미가 줄어듭니다.

    2. 정확한 정의¶

    사용성 평가(Usability Evaluation)란, 실제 사용자가 제품이나 서비스를 얼마나 쉽고 효과적으로 사용할 수 있는지를 체계적으로 확인하는 과정입니다.

    좋은 웹앱의 사용성 기준을 정리하면:

    • 직관성: 설명 없이도 어떻게 사용하는지 바로 알 수 있는가?
    • 정확성: AI가 올바른 결과를 보여주는가?
    • 반응 속도: 결과가 빠르게 나오는가?
    • 안내 메시지: 무엇을 하면 되는지 화면에 안내가 있는가?
    • 오류 대응: 잘못된 입력에도 적절한 반응이 있는가?

    3. 예시로 확인¶

    graph TD A["👤 사용자가 웹앱에 접속"] --> B{"사용법을 바로<br/>이해할 수 있는가?"} B -->|"예"| C["✅ 직관성 통과"] B -->|"아니오"| D["❌ 안내 문구 추가 필요"] C --> E{"AI 분류 결과가<br/>정확한가?"} E -->|"대부분 맞음"| F["✅ 정확성 통과"] E -->|"자주 틀림"| G["❌ 데이터 보완 필요"] F --> H{"결과가 3초 안에<br/>나오는가?"} H -->|"예"| I["✅ 반응속도 통과"] H -->|"아니오"| J["❌ 최적화 필요"]

    📌 정리하기: 데이터 증강 → 데이터 보완 → 재학습 → 내보내기 → 웹앱 배포 → 사용성 평가. 이 흐름이 오늘 수업의 전체 과정입니다!

    🔨 따라하기¶

    이제 실제로 해봅시다! 크게 두 파트로 진행합니다.

    • 파트 A (Step 1~3): 데이터 보완 + 모델 재학습
    • 파트 B (Step 4~6): 웹앱 제작 + 사용성 평가

    🛠️ 활동: 보완 계획표 작성하기

    종이나 문서에 다음 내용을 적으세요.

    오류 1 - 증상: (예: "사과"를 "토마토"로 분류함) - 원인: (예: 빨간색만 보고 판단, 배경이 비슷함) - 보완 방법: (예: 사과의 꼭지 부분이 보이는 사진 20장 추가, 토마토는 단면 사진도 추가) - 목표 추가 데이터 수: 20장

    오류 2 - 증상: - 원인: - 보완 방법: - 목표 추가 데이터 수:

    (모든 주요 오류에 대해 작성)

    팁: 보완 계획은 구체적일수록 좋습니다. "사진을 더 찍겠다"보다 "어두운 조명에서 30cm 거리로 10장 촬영하겠다"가 훨씬 효과적입니다.

    Step 2: 데이터 보완 및 증강 실행하기¶

    Teachable Machine(https://teachablemachine.withgoogle.com/)에 접속하여 2차시에서 저장한 프로젝트를 엽니다.

    데이터 추가 촬영 체크리스트를 따라가며 데이터를 보완합니다.

    📋 촬영 체크리스트 (증강 효과를 자연스럽게 얻는 방법)

    ✅ 정면, 좌측, 우측, 위, 아래 — 최소 5가지 각도에서 촬영 ✅ 밝은 곳, 어두운 곳, 창가 — 최소 2가지 조명 조건에서 촬영 ✅ 가까이(20cm), 중간(50cm), 멀리(1m) — 최소 3가지 거리에서 촬영 ✅ 흰 배경, 나무 책상, 손에 들고 — 최소 2가지 배경에서 촬영 ✅ 혼동되는 클래스 간 차이점이 명확히 드러나는 사진을 의도적으로 추가

    각 클래스별 최종 데이터 수 점검:

    • 모든 클래스가 최소 50장 이상인지 확인
    • 클래스 간 데이터 수 차이가 2배를 넘지 않도록 균형 맞추기
    • 나쁜 예: A클래스 120장, B클래스 30장 (4배 차이 ❌)
    • 좋은 예: A클래스 80장, B클래스 60장 (1.3배 차이 ✅)

    💡 생각해보기: 왜 클래스 간 데이터 수 균형이 중요할까요? 힌트: 시험에서 "정답은 항상 ③번"이라고 학습한 학생을 떠올려 보세요.

    Step 3: 모델 재학습 및 성능 비교¶

    데이터 보완이 끝나면 "모델 학습시키기(Train Model)" 버튼을 눌러 재학습을 시작합니다.

    재학습 시 설정 확인:

    Teachable Machine의 학습 설정(고급 옵션)에서:

    • Epochs(에포크): 50 (기본값 유지)
    • 에포크란 전체 데이터를 몇 번 반복 학습할지를 의미합니다
    • Batch Size(배치 크기): 16 (기본값 유지)
    • 한 번에 몇 개의 데이터를 묶어서 학습할지를 의미합니다
    • Learning Rate(학습률): 0.001 (기본값 유지)
    • AI가 한 번에 얼마나 크게 조정할지를 의미합니다

    📌 알아보기: 기본값으로도 충분한 성능이 나옵니다. 고급 설정을 과도하게 바꾸면 오히려 성능이 나빠질 수 있으니, 처음에는 기본값으로 학습하고 결과를 확인한 후 조정하세요.

    성능 비교 기록:

    학습이 끝나면, 2차시에서 테스트했던 같은 테스트 데이터로 다시 테스트합니다.

    🛠️ 활동: 성능 비교표 작성하기

    종이나 문서에 다음을 기록하세요.

    개선 전 (2차시 결과) - 전체 정확도: _% - 가장 약한 클래스: _ (정확도 ____%)

    개선 후 (3차시 결과) - 전체 정확도: _% - 가장 약한 클래스: _ (정확도 ____%)

    변화 분석 - 정확도가 올랐다면: 어떤 보완이 가장 효과적이었나? - 정확도가 그대로이거나 떨어졌다면: 원인은 무엇일까?

    정확도가 만족스럽지 않다면 Step 2로 돌아가 데이터를 추가 보완할 수 있습니다. 하지만 시간 관계상 2회까지만 반복하고 다음 단계로 진행합니다.

    Step 4: Teachable Machine에서 모델 내보내기¶

    이제 완성된 모델을 웹에서 사용할 수 있도록 내보냅니다.

    따라하기:

    4-1. Teachable Machine 화면에서 "모델 내보내기(Export Model)" 버튼을 클릭합니다.

    4-2. 탭에서 "Tensorflow.js"를 선택합니다.

    4-3. "모델 업로드(Upload my model)" 버튼을 클릭합니다. - 업로드가 완료될 때까지 잠시 기다립니다 (약 30초~1분)

    4-4. 업로드가 완료되면 두 가지가 생성됩니다. - 공유 가능한 링크(Shareable Link): https://teachablemachine.withgoogle.com/models/xxxxx/ - 코드 스니펫(Code Snippet): HTML + JavaScript 코드

    4-5. 공유 가능한 링크를 복사하여 메모장에 저장해 둡니다.

    4-6. 코드 스니펫의 전체 내용을 복사합니다. (아래 "📝 전체 코드" 섹션에 형태가 나와 있습니다)

    ⚠️ 중요: 업로드 중에 브라우저를 닫거나 페이지를 이동하면 안 됩니다! 업로드가 완료되었다는 메시지를 반드시 확인하세요.

    Step 5: Google Sites로 웹앱 만들기¶

    Google Sites(https://sites.google.com/)를 사용하여 AI 분류기 웹앱 페이지를 만듭니다.

    따라하기:

    5-1. Google Sites에 접속하고 "새 사이트 만들기(빈 사이트)"를 클릭합니다.

    5-2. 사이트 이름을 입력합니다. - 예: "우리 팀의 AI 재활용 분류기", "꽃 종류 알려주는 AI"

    5-3. 페이지 제목을 작성합니다. - 예: "🌸 꽃 이름 알려주는 AI — 카메라에 꽃을 비춰보세요!"

    5-4. 소개 섹션을 추가합니다. "삽입 → 텍스트 상자"를 선택하고 다음 내용을 작성합니다. - 이 AI가 무엇을 하는지 (한 줄 설명) - 사용 방법 (1, 2, 3 단계) - 분류할 수 있는 카테고리 목록

    5-5. AI 모델을 삽입합니다. 이것이 핵심 단계입니다! - "삽입 → 소스 코드 삽입(Embed → Embed code)"을 선택합니다. - Step 4에서 복사한 코드 스니펫을 붙여넣습니다. - "다음(Next)" → "삽입(Insert)"을 클릭합니다.

    5-6. 게시(Publish)합니다. - 오른쪽 상단의 "게시" 버튼 클릭 - 웹 주소를 설정합니다 (예: our-ai-classifier) - "게시" 확인

    📌 알아보기: Google Sites의 "소스 코드 삽입" 기능은 HTML/JavaScript 코드를 직접 페이지에 넣을 수 있게 해줍니다. 이것을 통해 Teachable Machine의 AI 모델이 웹 페이지 안에서 직접 실행됩니다. 별도의 서버가 필요 없어요!

    웹앱에 반드시 포함할 내용:

    • 🏷️ 앱 이름: 무엇을 하는 AI인지 한눈에 알 수 있는 제목
    • 📖 사용 설명: "웹캠 접근을 허용하세요 → 분류할 물건을 카메라 앞에 놓으세요 → 결과를 확인하세요"
    • 🤖 AI 모델 영역: 코드 스니펫으로 삽입한 실행 영역
    • ⚠️ 주의사항: "이 AI는 학습 프로젝트용이며, 실제 분류 정확도와 다를 수 있습니다"
    • 👥 만든 사람: 팀 이름 또는 팀원 이름

    Step 6: 사용성 평가 — 사용자 피드백 카드 작성¶

    웹앱이 완성되면, 다른 팀의 웹앱을 직접 사용해보고 피드백을 주고받는 활동을 합니다.

    진행 방법:

    6-1. 각 팀은 완성된 웹앱의 URL을 칠판(또는 공유 문서)에 적습니다.

    6-2. 다른 팀의 웹앱에 접속하여 최소 5번 이상 분류를 시도합니다.

    6-3. 아래 형식의 사용자 피드백 카드를 작성하여 해당 팀에 전달합니다.

    🛠️ 활동: 사용자 피드백 카드

    평가 대상 팀: __ 평가자: ____

    1. 직관성 ⭐⭐⭐⭐⭐ (별 5개 중 ___개) - 사용법을 바로 이해할 수 있었나요? - 구체적 의견:

    2. 정확성 ⭐⭐⭐⭐⭐ (별 5개 중 개) - 5번 테스트 중 몇 번 맞았나요? (/5) - 어떤 것을 분류할 때 틀렸나요?

    3. 디자인/안내 ⭐⭐⭐⭐⭐ (별 5개 중 ___개) - 화면 구성이 보기 좋았나요? - 안내 문구가 충분했나요?
