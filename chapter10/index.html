    제10장: 프롬프트 엔지니어링 실전 — AI를 나의 전문 비서로¶

    Part 3 | [1권] 생성형 AI와 대화하기 — LLM의 원리와 프롬프트 엔지니어링

    🎯 이 장에서 배우는 것¶

    • 복잡한 과제를 여러 단계의 프롬프트로 분해하는 프롬프트 체이닝을 사용할 수 있다
    • AI의 출력을 비판적으로 평가하고, 구체적인 수정 요청을 작성할 수 있다
    • 자신의 진로·관심 분야에서 AI를 전문 비서처럼 활용하는 시나리오를 설계할 수 있다
    • AI가 잘못된 정보를 줄 때 팩트체크하고 교정을 요청할 수 있다

    💡 왜 이걸 배우나요?¶

    생각해보기: 기말고사가 2주 앞으로 다가왔습니다. 시험 과목은 6개, 남은 시간은 한정적입니다. AI에게 "기말고사 공부 계획 세워줘"라고 말하면 어떤 답이 올까요?

    아마 이런 답이 올 것입니다.

    "매일 아침 6시에 일어나서 2시간씩 공부하고, 쉬는 시간을 가진 뒤..."

    너무 일반적이죠? 내 과목이 뭔지, 어떤 과목이 약한지, 하루에 실제로 공부할 수 있는 시간이 몇 시간인지 — AI는 이 정보를 모릅니다. 한 번의 질문으로 완벽한 답을 얻기는 어렵습니다.

    이전 장에서 우리는 좋은 프롬프트를 작성하는 기본 원칙을 배웠습니다. 이번 장에서는 한 단계 더 나아갑니다. 여러 번의 대화를 전략적으로 이어가며 AI에게서 정말 쓸모 있는 결과를 끌어내는 기술, 그리고 AI의 답변을 무조건 믿지 않고 검증하는 습관을 배웁니다.

    이것을 익히면, AI는 단순한 질문-답변 도구가 아니라 여러분의 전문 비서가 됩니다.

    📚 핵심 개념¶

    개념 1: 프롬프트 체이닝 — 대화를 잇는 기술 🔗¶

    1. 비유로 시작

    요리사에게 "맛있는 음식 만들어주세요"라고 하면, 요리사는 당황합니다. 하지만 이렇게 대화하면 어떨까요?

    • 1단계: "오늘 저녁에 한식을 먹고 싶어요" → 메뉴 범위 좁히기
    • 2단계: "그중에서 국물 요리가 좋겠어요. 냉장고에 돼지고기와 두부가 있어요" → 구체적 조건 추가
    • 3단계: "김치찌개로 할게요. 2인분 레시피를 단계별로 알려주세요" → 최종 요청
    • 4단계: "아, 저 매운 거 못 먹어요. 덜 맵게 수정해주세요" → 결과 조정

    한 번에 완벽한 주문을 하는 것이 아니라, 대화를 이어가며 점점 정교하게 만드는 것입니다.

    2. 정확한 정의

    프롬프트 체이닝(Prompt Chaining): 하나의 복잡한 과제를 여러 개의 연결된 프롬프트로 분해하여, 이전 대화의 결과를 다음 프롬프트의 입력으로 활용하는 기법

    핵심은 세 가지입니다.

    • 분해: 큰 과제를 작은 단계로 나눈다
    • 연결: 이전 답변의 결과를 다음 질문에 활용한다
    • 정교화: 각 단계에서 결과를 검토하고 방향을 조정한다

    3. 예시로 확인

    "기말고사 공부 계획"을 프롬프트 체이닝으로 접근하는 모습을 봅시다.

    [프롬프트 1 - 상황 공유]
나는 고등학교 2학년이고, 2주 뒤에 기말고사야.
시험 과목: 국어, 수학, 영어, 한국사, 물리, 화학
각 과목 현재 성적: 국어 B, 수학 D, 영어 A, 한국사 C, 물리 C, 화학 B
하루 공부 가능 시간: 평일 4시간, 주말 8시간
이 정보를 바탕으로, 어떤 과목에 시간을 더 투자해야 할지 우선순위를 정해줘.

    [프롬프트 2 - 결과를 바탕으로 구체화]
좋아, 네가 제안한 우선순위에 동의해.
수학에 가장 많은 시간을 배분하자.
그런데 수학에서 특히 약한 단원은 '미적분'과 '확률과 통계'야.
이 두 단원을 2주 안에 공부할 수 있는 일별 세부 계획을 만들어줘.

    [프롬프트 3 - 조정 요청]
계획은 좋은데, 첫째 주 수요일에 학교 행사가 있어서 공부를 2시간밖에 못 해.
그날 못 하는 분량을 나머지 날에 분산시켜서 수정해줘.

    이처럼 한 번에 모든 것을 요구하지 않고, 단계별로 대화를 이어갑니다.

    flowchart LR A["🎯 큰 과제\n기말고사 계획"] --> B["1단계\n상황·조건 공유"] B --> C["2단계\n결과 기반 구체화"] C --> D["3단계\n조정·수정 요청"] D --> E["4단계\n최종 확인·보완"] style A fill:#FF6B6B,color:#fff style E fill:#51CF66,color:#fff

    개념 2: 비판적 대화 — AI를 의심하는 기술 🔍¶

    1. 비유로 시작

    친구가 "나폴레옹이 워털루 전투에서 이겼대"라고 말하면, 여러분은 "정말? 어디서 들었어?"라고 물을 것입니다. AI에게도 똑같이 해야 합니다. AI는 매우 자신 있는 말투로 틀린 정보를 말할 수 있습니다. 이것을 할루시네이션(Hallucination, 환각 현상)이라고 합니다. 9장에서 배운 내용이죠.

    차이점은, 친구는 "잘 모르겠는데..."라고 솔직히 말할 수 있지만, AI는 모르는 것도 아는 것처럼 유창하게 대답한다는 것입니다.

    2. 정확한 정의

    비판적 대화(Critical Dialogue): AI의 출력을 무조건 수용하지 않고, 사실 여부를 검증하고, 근거를 요청하며, 오류 발견 시 수정을 요구하는 상호작용 방식

    비판적 대화의 3단계를 기억하세요.

    • 의심하기: "이게 정말 맞나?" — 특히 숫자, 날짜, 고유명사가 나올 때
    • 검증하기: "출처가 뭐야?", "다른 관점은 없어?" — 근거를 요청
    • 교정하기: "이 부분이 틀렸어. ~로 수정해줘" — 정확한 정보를 제시하며 수정 요청

    3. 예시로 확인

    ❌ 나쁜 예 — 무비판적 수용

    [사용자] 한국의 노벨 문학상 수상자에 대해 알려줘.
[AI] 한국의 노벨 문학상 수상자는 2020년에 수상한 김영수 작가입니다...
[사용자] 고마워! (그대로 보고서에 인용)

    ⭕ 좋은 예 — 비판적 대화

    [사용자] 한국의 노벨 문학상 수상자에 대해 알려줘.
[AI] 한국의 노벨 문학상 수상자는 2020년에 수상한 김영수 작가입니다...
[사용자] 잠깐, 한국의 노벨 문학상 수상자가 정말 있어?
        내가 알기로 한강 작가가 2024년에 최초로 수상했는데.
        이 정보를 확인해주고, 틀렸다면 수정해줘.
[AI] 죄송합니다, 제가 잘못된 정보를 드렸습니다.
     맞습니다. 한강 작가가 2024년 한국인 최초로 노벨 문학상을 수상했습니다...

    📌 알아보기: AI가 특히 틀리기 쉬운 영역

    • 최신 뉴스나 실시간 정보 (학습 데이터에 없는 시점의 정보)
    • 구체적인 통계 숫자, 날짜
    • 실존 인물의 발언 인용
    • 학술 논문·법률 조항의 세부 내용
    • URL이나 참고문헌 (존재하지 않는 링크를 만들어내기도 함)

    개념 3: 역할 부여와 맥락 설정 — AI에게 전문가 옷 입히기 👔¶

    1. 비유로 시작

    같은 사람이라도 역할에 따라 말하는 방식이 달라집니다. 의사 가운을 입으면 의학 용어로 설명하고, 교사 역할을 하면 쉬운 비유를 사용합니다. AI도 마찬가지입니다. "너는 ○○ 전문가야"라고 역할을 부여하면, 해당 분야의 어휘, 관점, 깊이가 달라집니다.

    2. 정확한 정의

    역할 부여(Role Assignment): AI에게 특정 전문가·직업의 페르소나를 설정하여, 해당 관점과 전문성 수준으로 응답하도록 유도하는 프롬프트 기법

    맥락 설정(Context Setting): AI가 응답할 때 고려해야 할 배경 정보(대상, 목적, 제약 조건)를 미리 제공하는 것

    이 두 가지를 결합하면 AI의 응답 품질이 크게 달라집니다.

    3. 예시로 확인

    [역할 없이]
"지구온난화에 대해 설명해줘"
→ 일반적이고 백과사전적인 답변

[역할 + 맥락]
"너는 환경과학 분야의 고등학교 교사야.
 내가 기후변화에 관한 발표를 준비하고 있어.
 청중은 같은 반 친구들이고, 발표 시간은 5분이야.
 지구온난화의 원인과 영향을 쉽게 설명하되,
 친구들이 '와, 그렇구나' 할 만한 구체적 사례 2가지를 포함해줘."
→ 수준에 맞고, 목적에 맞고, 실용적인 답변
    flowchart TD A["🎭 역할 부여\n'너는 ~전문가야'"] --> D["🎯 정교한 응답"] B["📋 맥락 설정\n대상, 목적, 제약"] --> D C["📝 구체적 요청\n형식, 분량, 조건"] --> D style A fill:#4ECDC4,color:#fff style B fill:#45B7D1,color:#fff style C fill:#96CEB4,color:#fff style D fill:#FFEAA7,color:#333

    개념 4: AI 활용 시나리오 설계 — 나만의 비서 만들기 📋¶

    1. 비유로 시작

    비서를 새로 고용했다고 상상해보세요. 첫날부터 "알아서 해줘"라고 하면 비서는 무엇을 해야 할지 모릅니다. 좋은 상사는 비서에게 이렇게 합니다.

    • 담당 업무를 명확히 알려준다 (무엇을 할지)
    • 업무 절차를 공유한다 (어떤 순서로 할지)
    • 보고 형식을 정한다 (결과를 어떻게 줄지)
    • 확인 절차를 만든다 (실수가 없는지 어떻게 점검할지)

    AI 활용 시나리오 설계도 이와 같습니다.

    2. 정확한 정의

    AI 활용 시나리오 설계: 특정 과제에서 AI와의 대화 흐름(어떤 순서로, 어떤 프롬프트를, 어떤 검증 절차와 함께)을 미리 계획하는 것

    3. 예시로 확인 — 진로별 시나리오

    🌍 외교관 지망생의 시나리오

    • 과제: 국제 이슈 브리핑 문서 작성
    • 1단계: AI에게 최근 국제 이슈 목록 요청 → 관심 이슈 선택
    • 2단계: 선택한 이슈의 배경·이해관계국·쟁점 정리 요청
    • 3단계: 한국의 입장에서 분석 요청
    • 검증: 외교부 홈페이지, 뉴스 기사로 사실 확인
    • 4단계: 최종 브리핑 문서 형식으로 정리 요청

    🌐 통번역 지망생의 시나리오

    • 과제: 영한 번역 품질 검수
    • 1단계: 영어 원문을 AI에게 번역 요청
    • 2단계: 번역 결과에서 어색한 부분 지적 및 대안 요청
    • 3단계: 특정 용어의 번역 선택지를 비교 요청
    • 검증: 전문 용어는 해당 분야 사전/자료로 확인
    • 4단계: 최종 번역 완성 및 자연스러움 검토 요청

    📌 생각해보기: 여러분의 진로나 관심 분야는 무엇인가요? 그 분야에서 AI에게 맡길 수 있는 반복적이거나 복잡한 작업은 무엇일까요? 3가지만 떠올려보세요.

    🔨 따라하기: "기말고사 공부 계획"을 AI와 함께 세우기¶

    이 실습에서는 ChatGPT, Claude, Gemini 등 어떤 AI 챗봇이든 사용할 수 있습니다. 프롬프트 체이닝, 비판적 대화, 역할 부여를 모두 통합하여 실전 과제를 수행합니다.

    Step 1: 역할 부여와 상황 공유 🎭¶

    AI에게 역할을 부여하고, 여러분의 상황을 자세히 알려줍니다.

    아래 프롬프트를 여러분의 실제 상황에 맞게 수정해서 AI에 입력하세요.

    너는 10년 경력의 학습 코칭 전문가야.
고등학생의 시험 준비 계획을 세우는 데 특화되어 있어.

내 상황을 알려줄게:
- 고등학교 2학년
- 2주 뒤 기말고사
- 시험 과목과 현재 수준:
  * 국어: 중상 (문학은 괜찮은데 비문학 독해가 약함)
  * 수학: 하 (미적분 개념 자체가 부족)
  * 영어: 상 (어휘와 독해 모두 양호)
  * 한국사: 중 (근현대사 부분 암기 부족)
  * 물리: 중하 (공식 적용 문제에서 실수 많음)
  * 화학: 중 (개념은 아는데 계산 문제에서 막힘)
- 하루 가용 시간: 평일 4시간, 주말 8시간
- 특이사항: 다음 주 수요일 학교 행사로 공부 불가

이 정보를 바탕으로:
1. 과목별 시간 배분 비율을 추천해줘
2. 추천 이유를 각 과목별로 간단히 설명해줘

아직 세부 일정은 만들지 마. 우선 큰 방향만 잡자.

    📌 알아보기: 왜 "아직 세부 일정은 만들지 마"라고 했을까?

    프롬프트 체이닝의 핵심은 한 번에 하나씩 진행하는 것입니다. 첫 단계에서 큰 방향을 확인하고, 그 방향이 맞는지 검토한 뒤에 다음 단계로 넘어갑니다. 한 번에 모든 것을 요구하면 수정이 어려워집니다.

    Step 2: 결과 검토 및 조정 요청 🔧¶

    AI의 첫 번째 응답을 받으면, 무조건 수락하지 말고 검토합니다.

    고마워. 시간 배분 추천을 보니 몇 가지 조정하고 싶어:

1. 영어에 10% 배분했는데, 영어는 현재 수준이 높으니까
   5%로 줄이고 그 시간을 수학에 추가해줘.
2. 물리와 화학의 약점이 비슷한 유형(계산/공식 적용)인데,
   이 두 과목을 묶어서 공부하는 전략이 있을까?
3. 한국사는 암기 과목이니까, 잠자기 전 시간에 배치하면
   효과적일 것 같은데 어떻게 생각해?

이 피드백을 반영해서 시간 배분을 수정해줘.

    이 단계의 핵심: AI의 제안에 여러분의 실제 경험과 판단을 더합니다. AI는 일반적인 학습 이론을 알지만, 여러분 자신을 가장 잘 아는 사람은 여러분입니다.

    Step 3: 세부 일정 생성 📅¶

    큰 방향이 확정되면, 이제 구체적인 일정을 요청합니다.

    좋아, 수정된 시간 배분에 동의해.
이제 14일 세부 일정을 만들어줘.

조건:
- 매일 과목별 공부 시간과 구체적 학습 내용을 포함할 것
- 다음 주 수요일(8일차)은 학교 행사로 공부 불가
- 시험 3일 전부터는 전 과목 복습 시간을 넣어줘
- 각 날의 총 공부 시간이 평일 4시간, 주말 8시간을 넘지 않을 것

형식: 각 날짜별로 시간대-과목-학습내용 순서로 정리해줘

    Step 4: 비판적 검토 — 팩트체크 🔍¶

    AI가 만든 일정을 받으면, 다음을 확인합니다.

    일정 고마워. 몇 가지 확인할게:

1. 4일차에 수학 '미적분 기본정리' 학습이 배치되어 있는데,
   그 전에 '극한과 연속' 개념을 먼저 공부해야 하지 않아?
   수학 학습 순서가 교과서 진도와 맞는지 확인해줘.

2. 물리 파트에서 '뉴턴의 운동법칙 문제 풀이'가 1시간으로
   되어 있는데, 내가 공식 적용에 약하다고 했으니
   예제를 먼저 분석하는 시간을 30분 추가하는 게 나을까?

3. 전체 14일 동안 총 공부 시간을 합산해줘.
   평일 4시간 × 10일 + 주말 8시간 × 3일 (행사일 제외)
   = 64시간이 맞는지 확인해줘.

    📌 알아보기: 검증 포인트 체크리스트

    AI가 만든 학습 계획을 받으면 이 4가지를 꼭 확인하세요.

    • 순서의 논리성: 선수 학습이 먼저 배치되어 있나?
    • 시간의 현실성: 하루 배분 시간이 실제 가용 시간 이내인가?
    • 총량의 정확성: 전체 시간 합산이 계산과 일치하나?
    • 나의 약점 반영: 내가 알려준 약점이 일정에 실제로 반영되어 있나?

    Step 5: 진로 분야 AI 활용 실습 🎯¶

    이제 "기말고사 계획"이라는 예시를 벗어나, 여러분의 관심 분야에서 AI를 활용해봅니다.

    아래 3개의 시나리오 중 하나를 선택하거나, 자신의 분야에 맞게 변형하세요.

    시나리오 A: 국제 이슈 브리핑 작성 (사회·국제 관심 학생)

    [프롬프트 1]
너는 국제정치 분석가야.
최근 6개월 이내의 주요 국제 이슈 5가지를 간단히 나열해줘.
각 이슈를 한 줄로 요약하고, 한국과의 관련성을 표시해줘.
(단, 너의 학습 데이터 기준이라 최신 정보는 내가 별도 확인할게)

[프롬프트 2]
이 중에서 [선택한 이슈]에 대해 더 깊이 알려줘.
- 배경 (어떻게 시작됐나)
- 주요 이해관계국과 각국의 입장
- 한국에 미치는 영향
- 향후 전망
A4 1장 분량의 브리핑 문서 형식으로 작성해줘.

[프롬프트 3 — 비판적 검토]
이 브리핑에서 언급된 구체적 사실(날짜, 합의 내용 등)이
정확한지 내가 확인할 수 있도록, 각 사실의 검증 키워드를
알려줘. 내가 뉴스 검색으로 교차 확인할게.

    시나리오 B: 번역 품질 검수 (영어·언어 관심 학생)

    [프롬프트 1]
너는 한영 번역 전문가야.
다음 한국어 문장을 영어로 번역해줘:
"인공지능 기술의 발전은 교육 현장에 새로운 가능성과
함께 윤리적 과제를 던지고 있다."
번역을 3가지 버전으로 제시해줘:
- 격식체 (학술 보고서용)
- 일반체 (뉴스 기사용)  
- 구어체 (발표용)

[프롬프트 2]
세 버전의 차이를 설명해줘.
어떤 단어 선택이 격식도를 결정하는지 구체적으로 알려줘.

[프롬프트 3 — 비판적 검토]
격식체 버전에서 'ethical challenges'라고 했는데,
'ethical dilemmas'와 어떤 뉘앙스 차이가 있어?
이 맥락에서 어느 쪽이 더 적절한지 근거와 함께 설명해줘.

    시나리오 C: 과학 실험 보고서 검토 (자연과학 관심 학생)

    ``` [프롬프트 1] 너는 고등학교 화학 교사야. 내가 작성한 실험 보고서 초안을 검토해줘. [실험 보고서 초안 붙여넣기]
