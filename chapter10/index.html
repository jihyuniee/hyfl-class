    Chapter 10: 발표와 성찰 — 우리가 만든 AI 웹앱, 세상에 보여주기¶

    Part 3 · [2권] 진짜 문제를 AI와 함께 해결하기 — 메인프로젝트

    🎯 이 장에서 배우는 것¶

    • 프로젝트의 문제발견→설계→구현→결과를 논리적 스토리로 구성하여 발표할 수 있다
    • 동료의 프로젝트를 4가지 기준(창의성·실용성·완성도·발표력)에 따라 평가하고 건설적인 피드백을 제공할 수 있다
    • AI와 협업한 경험을 돌아보며 자신이 성장한 점과 AI 시대에 필요한 역량을 성찰할 수 있다
    • 성찰 에세이를 통해 앞으로의 방향을 자신의 언어로 정리할 수 있다

    💡 왜 이걸 배우나요?¶

    개발자가 코드를 잘 짜는 것만으로는 충분하지 않습니다. 세상의 어떤 훌륭한 앱도 사람들에게 전달되지 않으면 존재하지 않는 것과 같습니다.

    스티브 잡스가 아이폰을 처음 공개한 순간을 떠올려 보세요. 그는 기술 스펙을 나열하지 않았습니다. "왜 이것이 필요한지" 이야기를 들려주었고, 직접 시연했습니다. 여러분도 마찬가지입니다. Chapter 1부터 여기까지 달려온 여정을 하나의 이야기로 엮어 세상에 보여줄 차례입니다.

    그리고 발표 뒤에는 성찰이 옵니다. "내가 무엇을 배웠는가?"를 스스로 정리하는 사람만이 다음 도전에서 더 멀리 갑니다.

    생각해보기 🤔

    1분 동안 자유롭게 적어 보세요: "Chapter 1의 나와 지금의 나는 무엇이 달라졌는가?"

    코딩 실력? AI를 대하는 태도? 문제를 바라보는 시선? 무엇이든 좋습니다.

    📚 핵심 개념¶

    개념 1: 프로젝트 발표 스토리라인¶

    1) 비유로 시작

    영화 예고편을 생각해 보세요. 2분 안에 "어떤 문제가 있고 → 주인공이 어떻게 맞서며 → 결말이 어떤지"를 보여줍니다. 프로젝트 발표도 똑같습니다. 청중이 5분 안에 여러분의 프로젝트 전체를 이해하도록 압축하는 기술입니다.

    2) 정확한 정의

    프로젝트 발표 스토리라인이란, 프로젝트의 배경→문제→해결→결과→교훈을 시간순으로 연결하여 청중이 자연스럽게 따라올 수 있도록 구성한 논리적 이야기 흐름입니다.

    3) 구조 확인

    flowchart LR A["🔍 문제 발견<br/>왜 이 문제인가?"] --> B["🎨 설계<br/>어떻게 풀기로 했나?"] B --> C["🔨 구현<br/>AI와 어떻게 만들었나?"] C --> D["🖥️ 시연<br/>실제로 동작하는 모습"] D --> E["💡 배운 점<br/>성장과 한계"]

    알아보기 📖

    각 단계의 권장 시간 배분 (총 5분 발표 기준): - 문제 발견: 1분 — 청중의 공감을 이끌어내는 가장 중요한 순간 - 설계: 30초 — 핵심 아이디어만 간결하게 - 구현: 1분 — AI와의 협업 과정 중 인상적인 장면 - 시연: 1분 30초 — 실제 작동 화면, 라이브 데모 - 배운 점: 1분 — 솔직한 성찰과 다음 목표

    개념 2: 동료 평가 (Peer Review)¶

    1) 비유로 시작

    요리 대회에서 심사위원이 "맛·비주얼·창의성·난이도"로 나누어 점수를 매기는 것과 같습니다. 하나의 기준으로만 평가하면 편향되지만, 여러 기준으로 나누면 공정하고 구체적인 피드백이 됩니다.

    2) 정확한 정의

    동료 평가란, 같은 학습자가 미리 합의된 기준에 따라 서로의 결과물을 평가하고 개선점을 제안하는 활동입니다. 단순히 "잘했다/못했다"가 아니라, 어떤 점이 좋고 어떤 점을 바꾸면 더 좋을지 구체적으로 전달하는 것이 핵심입니다.

    3) 4가지 평가 기준

    mindmap root((동료 평가 기준)) 🎨 창의성 문제 선정이 독창적인가 해결 방식이 참신한가 🛠️ 실용성 실제 사용자에게 도움이 되는가 현실 문제를 해결하는가 ✅ 완성도 앱이 오류 없이 작동하는가 UI가 사용하기 편한가 🎤 발표력 스토리가 논리적인가 시연이 매끄러운가

    알아보기 📖

    건설적 피드백의 공식 — "칭찬 + 제안 + 격려" - ✅ 좋은 예: "로그인 없이 바로 쓸 수 있는 점이 편리했어요. 결과를 저장하는 기능이 추가되면 더 좋을 것 같아요. 전체적으로 완성도가 높아서 인상적이었습니다!" - ❌ 나쁜 예: "별로예요." / "그냥 괜찮아요."

    개념 3: 성찰 에세이 (Reflective Writing)¶

    1) 비유로 시작

    등산을 마치고 정상에서 올라온 길을 내려다보는 순간이 있습니다. "저 가파른 구간을 내가 올랐다고?" 하는 놀라움, "다음엔 저 길 대신 이 길로 가야지" 하는 깨달음. 성찰 에세이는 바로 그 정상에서의 되돌아봄입니다.

    2) 정확한 정의

    성찰 에세이란, 학습 경험을 돌아보며 무엇을 배웠고, 어떤 어려움이 있었으며, 앞으로 어떻게 성장할 것인지를 자신의 언어로 정리한 글입니다.

    3) 성찰 에세이의 3가지 질문

    • 과거: 이 프로젝트에서 가장 어려웠던 순간은 언제였나? 어떻게 극복했나?
    • 현재: AI와 협업하며 나는 어떤 역할을 했나? 나만의 강점은 무엇이었나?
    • 미래: AI 시대에 나는 어떤 사람이 되고 싶은가? 다음에 만들고 싶은 것은?

    개념 4: AI 시대의 핵심 역량¶

    1) 비유로 시작

    계산기가 등장했을 때 수학이 사라지지 않았습니다. 대신 무엇을 계산해야 하는지 판단하는 능력이 더 중요해졌습니다. AI도 마찬가지입니다. AI가 코드를 대신 써줄 수 있는 시대에, 진짜 중요한 것은 "무엇을 만들어야 하는가"를 결정하는 힘입니다.

    2) 정확한 정의

    AI 시대의 핵심 역량이란, AI 도구를 효과적으로 활용하면서도 인간만이 할 수 있는 문제 정의, 윤리적 판단, 창의적 기획, 소통과 협업 능력을 말합니다.

    3) 여러분이 이미 경험한 역량들

    flowchart TB subgraph 여러분이_한_것["🧑‍💻 여러분이 한 것 (인간의 역할)"] A["문제를 발견했다"] B["해결 방향을 설계했다"] C["AI에게 올바른 질문을 했다"] D["결과를 평가하고 수정했다"] E["사용자 관점에서 판단했다"] end subgraph AI가_한_것["🤖 AI가 한 것 (도구의 역할)"] F["코드를 생성했다"] G["데이터를 처리했다"] H["패턴을 분석했다"] end A --> C B --> C C --> F C --> G C --> H F --> D G --> D H --> D D --> E

    정리하기 📋

    AI는 도구이고, 여러분은 감독입니다. 감독 없는 도구는 방향을 모르고, 도구 없는 감독은 속도가 느립니다. 둘의 협업이 진짜 힘입니다.

    🔨 따라하기¶

    Step 1: 발표 슬라이드 구성하기¶

    발표를 위한 간단한 HTML 프레젠테이션 페이지를 만들어 봅시다. 별도 소프트웨어 없이 브라우저에서 바로 사용할 수 있습니다.

    아래 코드를 presentation.html로 저장하세요.

    <!DOCTYPE html>
<html lang="ko">
<head>
<meta charset="UTF-8">
<meta name="viewport" content="width=device-width, initial-scale=1.0">
<title>프로젝트 발표</title>
<style>
  * { margin: 0; padding: 0; box-sizing: border-box; }
  body { font-family: 'Segoe UI', sans-serif; background: #1a1a2e; color: #eee; }
  .slide {
    display: none; min-height: 100vh;
    padding: 60px 80px;
    flex-direction: column; justify-content: center;
  }
  .slide.active { display: flex; }
  .slide h1 { font-size: 2.8em; margin-bottom: 20px; color: #e94560; }
  .slide h2 { font-size: 2em; margin-bottom: 16px; color: #0f3460; background: #e94560; display: inline-block; padding: 8px 20px; border-radius: 8px; }
  .slide p, .slide li { font-size: 1.4em; line-height: 1.8; }
  .slide ul { margin-left: 40px; margin-top: 12px; }
  .nav { position: fixed; bottom: 30px; right: 40px; display: flex; gap: 12px; }
  .nav button {
    padding: 12px 24px; font-size: 1.1em; border: none;
    border-radius: 8px; cursor: pointer; background: #e94560; color: #fff;
  }
  .nav button:hover { background: #c73650; }
  .counter { position: fixed; bottom: 36px; left: 40px; font-size: 1.1em; color: #888; }
</style>
</head>
<body>

<!-- 슬라이드 1: 타이틀 -->
<div class="slide active">
  <h1>🚀 [프로젝트 이름]</h1>
  <p>팀명: [팀 이름] | 팀원: [이름들]</p>
  <p style="margin-top:20px; color:#aaa;">우리가 발견한 문제, AI와 함께 해결한 이야기</p>

<!-- 슬라이드 2: 문제 발견 -->

    <li>왜 이 문제를 골랐는가: ___</li>
    <li>누가 겪는 문제인가: ___</li>
  </ul>
</div>

<!-- 슬라이드 3: 설계 -->
<div class="slide">
  <h2>🎨 설계</h2>
  <ul>
    <li>핵심 아이디어: ___</li>
    <li>사용한 AI 기능: ___</li>
    <li>전체 구조 (간단히): ___</li>
  </ul>
</div>

<!-- 슬라이드 4: 시연 -->
<div class="slide">
  <h2>🖥️ 시연</h2>
  <p>여기서 실제 앱을 보여주세요!</p>
  <p style="margin-top:12px; color:#aaa;">(새 탭으로 앱을 열고 주요 기능 시연)</p>
</div>

<!-- 슬라이드 5: 배운 점 -->
<div class="slide">
  <h2>💡 배운 점</h2>
  <ul>
    <li>가장 어려웠던 점: ___</li>
    <li>AI와 협업하며 느낀 점: ___</li>
    <li>다음에 개선하고 싶은 점: ___</li>
  </ul>
</div>

<div class="counter" id="counter"></div>
<div class="nav">
  <button onclick="changeSlide(-1)">◀ 이전</button>
  <button onclick="changeSlide(1)">다음 ▶</button>
</div>

<script>
let current = 0;
const slides = document.querySelectorAll('.slide');
const counter = document.getElementById('counter');

function updateCounter() {
  counter.textContent = `${current + 1} / ${slides.length}`;
}

function changeSlide(dir) {
  slides[current].classList.remove('active');
  current = Math.max(0, Math.min(slides.length - 1, current + dir));
  slides[current].classList.add('active');
  updateCounter();
}

// 키보드 좌우 화살표로도 이동
document.addEventListener('keydown', (e) => {
  if (e.key === 'ArrowRight') changeSlide(1);
  if (e.key === 'ArrowLeft') changeSlide(-1);
});

updateCounter();
</script>
</body>
</html>

    [프로젝트 이름], [팀 이름] 등 ___ 부분을 여러분의 내용으로 채우세요. 슬라이드를 추가하고 싶으면 <div class="slide">...</div>를 복사하여 넣으면 됩니다.

const scores = {};

function rate(el, score) {
  const category = el.closest('.criteria').dataset.category;
  scores[category] = score;
  const stars = el.parentElement.querySelectorAll('span');
  stars.forEach((s, i) => {
    s.classList.toggle('selected', i < score);
  });
}

function submitReview() {
  const team = document.getElementById('teamName').value || '(미입력)';
  const feedback = document.getElementById('feedback').value || '(없음)';
  const categories = ['creativity', 'practicality', 'completeness', 'presentation'];
  const labels = ['창의성', '실용성', '완성도', '발표력'];

  let total = 0;
  let summaryText = `📋 ${team} 팀 평가 결과\n`;
  categories.forEach((cat, i) => {
    const s = scores[cat] || 0;
    total += s;
    summaryText += `${labels[i]}: ${'★'.repeat(s)}${'☆'.repeat(5 - s)}\n`;
  });
  summaryText += `합계: ${total}/20\n피드백: ${feedback}`;

  document.getElementById('summary').textContent = summaryText;
  document.getElementById('result').style.display = 'block';

  // 콘솔에도 출력 (추후 수집용)
  console.log(summaryText);
}

    Step 3: 발표 연습하기 (5분 리허설)¶

    코드 작성이 아닌 실제 발표 연습입니다. 아래 체크리스트를 따라가세요.

    발표 리허설 체크리스트 ✓

    • 타이머를 5분에 맞추고 시작한다
    • 첫 문장에서 문제를 한 줄로 말한다 (예: "급식 메뉴가 매일 뭔지 몰라서 불편했습니다")
    • 설계와 구현은 핵심만 말하고 넘어간다
    • 시연은 대표 기능 1~2개만 보여준다 (전체를 다 보여주려 하지 않기!)
    • 마지막에 "배운 점" 한 문장으로 마무리한다
    • 5분 내에 끝났는가?

    Step 4: 발표 & 동료 평가 진행¶

    실제 수업에서 진행하는 절차입니다.

    sequenceDiagram participant 발표팀 participant 청중 participant 진행자 진행자->>발표팀: 발표 시작 안내 (5분) 발표팀->>청중: 문제→설계→구현→시연→배운점 청중->>청중: peer-review.html로 평가 입력 청중->>발표팀: 질문 1~2개 (1분) 발표팀->>청중: 답변 진행자->>청중: 다음 팀 준비

    Step 5: 부문별 시상 투표¶

    모든 발표가 끝나면 전체 투표를 진행합니다. 평가 결과를 모아 부문별 수상 팀을 정합니다.

    • 🎨 최고 창의성 상 — 가장 독창적인 문제 & 해결
    • 🛠️ 최고 실용성 상 — 가장 실제로 쓸모 있는 앱
    • ✅ 최고 완성도 상 — 가장 안정적이고 완성된 앱
    • 🎤 최고 발표력 상 — 가장 설득력 있는 발표
