    📖 Chapter 14. 문제 분해와 패턴 인식 — 큰 문제를 작게 쪼개기¶

    Part 5 | [2권] 컴퓨팅 사고력 — 문제를 분해하고 알고리즘으로 설계하기

    🎯 이 장에서 배우는 것¶

    • 복잡한 문제를 하위 문제로 분해(Decomposition) 하는 사고 과정을 실행할 수 있다
    • 반복되는 패턴(Pattern) 을 발견하고 이를 일반화하여 표현할 수 있다
    • 추상화(Abstraction) 를 활용하여 문제의 핵심 요소만 추출할 수 있다
    • 분해 → 패턴 인식 → 추상화 → 알고리즘 설계의 4단계 컴퓨팅 사고력을 실생활 문제에 적용할 수 있다

    💡 왜 이걸 배우나요?¶

    선생님이 갑자기 이렇게 말씀하셨다고 상상해 보세요.

    "우리 반이 학교 축제 부스를 운영할 거야. 기획부터 정산까지 전부 너희가 해!"

    머릿속이 하얘지지 않나요? 😵 "어디서부터 시작하지?" 하는 막막함, 누구나 겪어 봤을 겁니다.

    하지만 컴퓨터 과학자들은 이런 거대한 문제를 매일 해결합니다. 비결은 간단합니다. 큰 문제를 작은 조각으로 쪼개고, 조각 안에서 반복되는 패턴을 찾고, 핵심만 뽑아내고, 그걸 순서대로 정리하는 것이죠.

    이 네 가지 기술을 묶어서 컴퓨팅 사고력(Computational Thinking) 이라고 부릅니다. 프로그래밍뿐 아니라 시험 공부 계획, 여행 준비, 팀 프로젝트 등 일상의 모든 문제 해결에 바로 쓸 수 있습니다.

    이번 장을 마치면, 어떤 복잡한 상황에서도 "일단 쪼개 보자!"라고 말할 수 있는 사고의 도구를 갖게 될 거예요. 🧩

    📚 핵심 개념¶

    개념 1: 문제 분해 (Decomposition) 🧩¶

    1. 비유로 시작

    레고로 거대한 성을 만든다고 생각해 보세요. 처음부터 성 전체를 한 번에 조립하는 사람은 없습니다. 성벽, 탑, 문, 깃발 — 부분별로 나눠서 각각 만든 뒤 합치죠. 문제 분해도 똑같습니다.

    2. 정확한 정의

    문제 분해(Decomposition) 란, 하나의 크고 복잡한 문제를 관리 가능한 작은 하위 문제(sub-problem) 들로 나누는 과정이다.

    분해의 핵심 원칙은 두 가지입니다.

    • 각 조각은 독립적으로 해결 가능해야 한다 — 다른 조각을 먼저 풀지 않아도 이해할 수 있어야 합니다.
    • 모든 조각을 합치면 원래 문제가 되어야 한다 — 빠진 조각이 있으면 안 됩니다.

    3. 예시로 확인

    🔍 생각해보기: "학교 축제 부스 운영"을 분해해 봅시다

    graph TD A["🎪 학교 축제 부스 운영"] --> B["📋 기획"] A --> C["💰 예산"] A --> D["📢 홍보"] A --> E["🏗️ 준비·설치"] A --> F["🎉 당일 운영"] A --> G["📊 정산·평가"] B --> B1["주제 선정"] B --> B2["역할 분담"] B --> B3["일정표 작성"] C --> C1["수입 예측"] C --> C2["지출 목록"] C --> C3["수지 계산"] D --> D1["포스터 제작"] D --> D2["SNS 홍보"] E --> E1["물품 구매"] E --> E2["부스 설치"] F --> F1["교대 근무"] F --> F2["판매·안내"] G --> G1["매출 정리"] G --> G2["후기 작성"]

    한 덩어리였던 과제가 6개의 중간 조각, 그리고 13개의 작은 할 일로 변했습니다. 이제 각각은 한 사람이나 한 팀이 독립적으로 처리할 수 있는 크기가 되었죠!

    개념 2: 패턴 인식 (Pattern Recognition) 🔄¶

    1. 비유로 시작

    음악을 들을 때 "아, 이 부분 아까 나왔던 멜로디네!" 하고 느낀 적 있죠? 노래에는 반복되는 후렴구(chorus) 가 있습니다. 문제 안에서도 똑같은 구조가 반복되는 것을 찾는 것이 패턴 인식입니다.

    2. 정확한 정의

    패턴 인식(Pattern Recognition) 이란, 분해된 하위 문제들 사이에서 공통점, 반복, 규칙을 발견하는 과정이다.

    패턴을 찾으면 같은 해결 방법을 여러 곳에 재사용할 수 있어서 시간과 노력을 크게 줄일 수 있습니다.

    3. 예시로 확인

    위에서 분해한 축제 부스 운영을 다시 살펴볼까요?

    🔍 알아보기: 반복되는 패턴 찾기

    • "포스터 제작"과 "SNS 홍보"는 둘 다 "정보를 전달한다" 는 공통 패턴을 가집니다.
    • "수입 예측"과 "매출 정리"는 둘 다 "숫자를 기록하고 계산한다" 는 패턴입니다.
    • "역할 분담"과 "교대 근무"는 둘 다 "사람을 배치한다" 는 패턴입니다.

    이 세 가지 패턴을 미리 정리해 두면, 비슷한 작업이 나올 때마다 처음부터 고민할 필요 없이 같은 틀을 재사용할 수 있습니다.

    프로그래밍에서도 마찬가지입니다. 아래 코드를 봅시다.

    # 패턴을 발견하기 전: 반복되는 코드
print("=== 1학년 평균 ===")
total_1 = 85 + 90 + 78
avg_1 = total_1 / 3
print(f"평균: {avg_1:.1f}")

print("=== 2학년 평균 ===")
total_2 = 92 + 88 + 95
avg_2 = total_2 / 3
print(f"평균: {avg_2:.1f}")

print("=== 3학년 평균 ===")
total_3 = 76 + 82 + 89
avg_3 = total_3 / 3
print(f"평균: {avg_3:.1f}")

    세 번이나 같은 구조가 반복되고 있죠? "점수 리스트를 받아서 평균을 출력한다" 라는 패턴입니다. 이 패턴을 함수로 만들면 훨씬 깔끔해집니다.

    # 패턴을 발견한 후: 함수로 일반화
def print_average(grade_name, scores):
    avg = sum(scores) / len(scores)
    print(f"=== {grade_name} 평균 ===")
    print(f"평균: {avg:.1f}")

print_average("1학년", [85, 90, 78])
print_average("2학년", [92, 88, 95])
print_average("3학년", [76, 82, 89])

    코드 양이 절반 이하로 줄었고, 학년이 아무리 늘어나도 한 줄만 추가하면 됩니다! 🎉

    개념 3: 추상화 (Abstraction) 🎯¶

    1. 비유로 시작

    지하철 노선도를 떠올려 보세요. 실제 지하철 터널은 구불구불하고 역 사이 거리도 제각각이지만, 노선도에서는 직선과 동그라미로 단순하게 표현합니다. 실제 거리나 각도는 과감하게 버리고, "어떤 역에서 어떤 역으로 갈 수 있는가" 라는 핵심 정보만 남긴 것이죠. 이것이 추상화입니다.

    2. 정확한 정의

    추상화(Abstraction) 란, 문제에서 핵심적인 요소만 남기고 불필요한 세부 사항을 제거하여 문제를 단순화하는 과정이다.

    추상화의 두 가지 핵심 질문:

    • "이 정보가 없으면 문제를 풀 수 없는가?" → 핵심! 남긴다. ✅
    • "이 정보가 없어도 문제를 풀 수 있는가?" → 세부 사항! 제거한다. ❌

    3. 예시로 확인

    🔍 생각해보기: "급식 만족도 조사"를 추상화해 봅시다

    급식 만족도를 조사한다고 할 때, 고려할 수 있는 정보는 많습니다.

    포함해야 할 핵심 요소 ✅ - 메뉴 이름 - 만족도 점수 (1~5점) - 응답자 수

    제거해도 되는 세부 사항 ❌ - 급식실 벽 색깔 - 그날의 날씨 - 응답자의 키와 몸무게 - 급식 아주머니의 이름

    이렇게 핵심만 추출하면 프로그래밍 모델이 단순해집니다.

    # 추상화 결과: 핵심 데이터만 담은 구조
menu_survey = {
    "메뉴": "김치찌개",
    "점수들": [4, 3, 5, 4, 2, 5, 3, 4, 4, 5],
    "응답자수": 10
}

avg_score = sum(menu_survey["점수들"]) / menu_survey["응답자수"]
print(f"{menu_survey['메뉴']} 만족도: {avg_score:.1f}점")

    김치찌개 만족도: 3.9점

    개념 4: 컴퓨팅 사고력 4단계 통합 🔗¶

    지금까지 배운 세 가지 개념에 알고리즘 설계를 더하면, 컴퓨팅 사고력의 4단계가 완성됩니다.

    graph LR A["1️⃣ 분해<br/>Decomposition"] --> B["2️⃣ 패턴 인식<br/>Pattern Recognition"] B --> C["3️⃣ 추상화<br/>Abstraction"] C --> D["4️⃣ 알고리즘 설계<br/>Algorithm Design"] style A fill:#FF6B6B,color:#fff style B fill:#4ECDC4,color:#fff style C fill:#45B7D1,color:#fff style D fill:#96CEB4,color:#fff

    정리하기:

    • 1단계 분해: 큰 문제를 작은 조각으로 나눈다 🧩
    • 2단계 패턴 인식: 조각들 사이 반복·공통점을 찾는다 🔄
    • 3단계 추상화: 핵심만 남기고 나머지는 제거한다 🎯
    • 4단계 알고리즘 설계: 해결 절차를 순서대로 정리한다 📝

    이 4단계는 순서가 고정된 것이 아닙니다. 실제로는 앞뒤로 오가며 반복적으로 적용합니다. 하지만 처음 배울 때는 이 순서대로 연습하면 가장 효과적입니다.

    🔨 따라하기: "우리 반 급식 만족도 조사 시스템" 만들기¶

    실제로 컴퓨팅 사고력 4단계를 적용하여, 급식 만족도 조사 프로그램을 처음부터 끝까지 만들어 봅시다!

    Step 1: 문제 정의하기 📋¶

    문제: 우리 반 학생들의 급식 만족도를 조사하고, 메뉴별 평균 점수와 가장 인기 있는 메뉴를 알아내자.

    이 문제를 그냥 "급식 만족도 프로그램 만들기"로 시작하면 막막합니다. 컴퓨팅 사고력 4단계를 적용해 봅시다!

    Step 2: 분해하기 🧩¶

    큰 문제를 하위 문제로 쪼갭니다.

    graph TD A["급식 만족도 조사 시스템"] --> B["① 데이터 입력<br/>메뉴와 점수를 받는다"] A --> C["② 데이터 저장<br/>입력받은 데이터를 보관한다"] A --> D["③ 통계 계산<br/>평균, 최고, 최저를 구한다"] A --> E["④ 결과 출력<br/>보기 좋게 화면에 보여준다"]

    이제 네 개의 독립적인 작은 문제가 생겼습니다. 하나씩 해결하면 됩니다!

    Step 3: 패턴 인식하기 🔄¶

    각 하위 문제를 살펴보면, "메뉴별로 같은 작업을 반복한다" 는 패턴이 보입니다.

    • 메뉴 A의 점수 평균 구하기
    • 메뉴 B의 점수 평균 구하기
    • 메뉴 C의 점수 평균 구하기
    • ...

    전부 "점수 리스트 → 평균 계산" 이라는 동일한 패턴입니다. 이 패턴을 함수 하나로 만들어 재사용하면 됩니다!

    Step 4: 추상화하기 🎯¶

    급식 만족도 조사에서 핵심만 남깁니다.

    남길 것 ✅: 메뉴 이름, 점수(1~5), 응답자 수

    버릴 것 ❌: 응답 시각, 학생 이름, 급식 가격, 칼로리

    이를 바탕으로 데이터 구조를 설계합니다.

survey_data = {

# 값: 점수들의 리스트 (1~5 사이 정수)

    Step 6: 코드로 구현하기 💻¶

    4단계를 모두 거쳤으니, 이제 코드를 작성합니다. 아래 코드를 복사해서 실행해 보세요!

    # ============================================
# 🍽️ 우리 반 급식 만족도 조사 시스템
# 컴퓨팅 사고력 4단계를 적용하여 만든 프로그램
# ============================================

# --- ① 데이터 입력 (분해: 하위문제 1) ---
survey_data = {
    "김치찌개": [4, 3, 5, 4, 2, 5, 3, 4, 4, 5],
    "돈까스":  [5, 5, 4, 5, 3, 4, 5, 5, 4, 5],
    "비빔밥":  [3, 2, 4, 3, 3, 4, 2, 3, 3, 4],
    "떡볶이":  [5, 4, 5, 5, 4, 3, 5, 4, 5, 5],
    "된장찌개": [3, 3, 4, 2, 3, 3, 4, 3, 2, 3]
}

# --- ② 패턴 인식: 반복되는 "평균 계산" 패턴을 함수로 ---
def calculate_average(scores):
    """점수 리스트를 받아 평균을 반환하는 함수"""
    return sum(scores) / len(scores)

def get_grade(avg):
    """평균 점수를 등급으로 변환하는 함수"""
    if avg >= 4.5:
        return "🌟 최고"
    elif avg >= 3.5:
        return "😊 좋음"
    elif avg >= 2.5:
        return "😐 보통"
    else:
        return "😟 개선 필요"

# --- ③ 통계 계산 (분해: 하위문제 3) ---
results = {}
for menu, scores in survey_data.items():
    avg = calculate_average(scores)
    grade = get_grade(avg)
    results[menu] = {"평균": avg, "등급": grade, "응답수": len(scores)}

# --- ④ 결과 출력 (분해: 하위문제 4) ---
print("=" * 40)
print("  🍽️  급식 만족도 조사 결과")
print("=" * 40)

for menu, info in results.items():
    print(f"\n📌 {menu}")
    print(f"   평균 점수: {info['평균']:.1f}점 / 5.0점")
    print(f"   등급: {info['등급']}")
    print(f"   응답자 수: {info['응답수']}명")

# 가장 인기 있는 메뉴 찾기
best_menu = max(results, key=lambda x: results[x]["평균"])
worst_menu = min(results, key=lambda x: results[x]["평균"])

print("\n" + "=" * 40)
print(f"  👑 가장 인기 메뉴: {best_menu} ({results[best_menu]['평균']:.1f}점)")
print(f"  📉 개선 필요 메뉴: {worst_menu} ({results[worst_menu]['평균']:.1f}점)")
print("=" * 40)

    📝 전체 코드¶

    아래는 위 따라하기의 완성 코드에 사용자 입력 기능을 추가한 확장 버전입니다.

    # ============================================
# 🍽️ 급식 만족도 조사 시스템 (입력 기능 포함)
# ============================================

def calculate_average(scores):
    """점수 리스트 → 평균 반환"""
    if len(scores) == 0:
        return 0
    return sum(scores) / len(scores)

def get_grade(avg):
    """평균 → 등급 반환"""
    if avg >= 4.5:
        return "🌟 최고"
    elif avg >= 3.5:
        return "😊 좋음"
    elif avg >= 2.5:
        return "😐 보통"
    else:
        return "😟 개선 필요"

def input_survey():
    """사용자에게 메뉴별 점수를 입력받는 함수"""
    survey = {}
    print("📋 급식 만족도 조사를 시작합니다!")
    print("   (메뉴 입력을 마치려면 '끝'을 입력하세요)\n")

    while True:
        menu = input("메뉴 이름: ").strip()
        if menu == "끝":
            break

        scores = []
        print(f"  '{menu}'의 점수를 입력하세요 (1~5, 'q'로 다음 메뉴)")
        while True:
            score_input = input("  점수: ").strip()
            if score_input.lower() == 'q':
                break
            try:
                score = int(score_input)
                if 1 <= score <= 5:
                    scores.append(score)
                else:
                    print("  ⚠️ 1~5 사이 숫자를 입력하세요!")
            except ValueError:
                print("  ⚠️ 숫자를 입력하세요!")

        if scores:
            survey[menu] = scores
            print(f"  ✅ {menu}: {len(scores)}개 응답 저장\n")

    return survey

def show_results(survey_data):
    """결과를 보기 좋게 출력하는 함수"""
    if not survey_data:
        print("📭 조사 데이터가 없습니다.")
        return

    print("\n" + "=" * 40)
    print("  🍽️  급식 만족도 조사 결과")
    print("=" * 40)

    results = {}
    for menu, scores in survey_data.items():
        avg = calculate_average(scores)
        grade = get_grade(avg)
        results[menu] = avg
        print(f"\n📌 {menu}")
        print(f"   평균: {avg:.1f}점  {grade}")
        print(f"   응답: {len(scores)}명")

    best = max(results, key=results.get)
    worst = min(results, key=results.get)
    print("\n" + "=" * 40)
    print(f"  👑 인기 메뉴: {best} ({results[best]:.1f}점)")
    print(f"  📉 개선 필요: {worst} ({results[worst]:.1f}점)")
    print("=" * 40)

# --- 실행 ---
if __name__ == "__main__":
    data = input_survey()
    show_results(data)
