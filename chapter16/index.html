    Chapter 16: AI와 함께하는 알고리즘 — 노코드로 자동화하기¶

    Part 5 | [2권] 컴퓨팅 사고력 — 문제를 분해하고 알고리즘으로 설계하기

    🎯 이 장에서 배우는 것¶

    이 장을 마치면 다음을 할 수 있습니다.

    • 설계한 알고리즘을 노코드 도구로 구현하는 방법을 설명할 수 있다
    • Google Sheets + AI 함수를 활용하여 간단한 자동화 시스템을 만들 수 있다
    • 알고리즘 설계(종이)와 실제 구현 사이의 차이를 발견하고 조정할 수 있다
    • 노코드 도구의 장점과 한계를 구분하여 정리할 수 있다

    💡 왜 이걸 배우나요?¶

    생각해보기: 지난 시간에 순서도와 의사코드로 알고리즘을 설계했습니다. 그런데 그 종이 위의 설계도를 컴퓨터가 대신 실행해준다면 어떨까요?

    여러분이 매일 아침 "오늘 입을 옷"을 고르는 과정을 떠올려 봅시다.

    1. 날씨를 확인한다
    2. 기온이 10도 이하면 → 패딩을 입는다
    3. 비가 오면 → 우산을 챙긴다
    4. 특별한 일정이 있으면 → 깔끔한 옷을 고른다

    이 과정을 매일 직접 판단하고 있죠? 만약 스마트폰이 날씨를 자동으로 확인하고, "오늘은 패딩 + 우산이 필요해요"라고 알려준다면 어떨까요? 이것이 바로 알고리즘의 자동화입니다.

    과거에는 자동화를 하려면 반드시 프로그래밍 언어를 배워야 했습니다. 하지만 지금은 노코드(No-Code) 도구 덕분에 코딩 없이도 알고리즘을 실행하는 시스템을 만들 수 있는 시대가 되었습니다. 그리고 여기에 AI가 결합되면, 더 똑똑한 자동화가 가능해집니다.

    이번 장에서는 여러분이 직접 설계한 알고리즘을 실제로 동작하는 시스템으로 만들어보면서, "설계"와 "구현" 사이의 간극을 체험합니다.

    📚 핵심 개념¶

    개념 1: 노코드(No-Code)란 무엇인가¶

    🎨 비유로 시작

    레고 블록을 떠올려 봅시다. 레고로 집을 만들 때, 벽돌을 직접 구워서 쌓지 않죠? 이미 만들어진 블록을 조립해서 원하는 모양을 만듭니다. 노코드 도구도 마찬가지입니다. 누군가가 미리 만들어 놓은 기능 블록을 끌어다 조합해서 프로그램을 만드는 것입니다.

    📖 정확한 정의

    노코드(No-Code) 란 프로그래밍 코드를 직접 작성하지 않고, 시각적인 인터페이스(드래그 앤 드롭, 메뉴 선택 등)를 통해 소프트웨어나 자동화 시스템을 만드는 방법입니다.

    ✅ 예시로 확인

    여러분이 이미 사용해 본 노코드 도구들이 있습니다.

    • Google Forms: 클릭만으로 설문조사를 만듦
    • Canva: 드래그 앤 드롭으로 디자인을 만듦
    • Scratch: 블록을 조립해서 게임을 만듦

    이들의 공통점은 코드를 한 줄도 쓰지 않고 결과물을 만들 수 있다는 것입니다.

    graph LR A["전통적 개발 🖥️"] --> B["코드 작성 필요"] B --> C["실행까지 오래 걸림"] D["노코드 개발 🧩"] --> E["블록 조립 방식"] E --> F["빠르게 결과물 완성"]

    개념 2: AI 함수 — 스프레드시트 안의 인공지능¶

    🎨 비유로 시작

    계산기를 생각해 봅시다. 계산기에 3 + 5를 입력하면 8이 나옵니다. 이것은 숫자 계산을 자동화한 것이죠. 이제 계산기가 한 단계 더 발전해서, "이 문장이 긍정적인지 부정적인지 판단해줘"라고 요청하면 답을 해준다면요? 이것이 바로 AI 함수입니다.

    📖 정확한 정의

    AI 함수란 Google Sheets 같은 스프레드시트에서 사용할 수 있는, AI 기반의 특수한 함수입니다. 기존의 SUM, IF 같은 함수가 숫자를 처리했다면, AI 함수는 텍스트를 이해하고, 분류하고, 생성하는 일을 합니다.

    ✅ 예시로 확인

    Google Sheets에서 사용할 수 있는 AI 활용 방식을 봅시다.

    • 기존 함수: =IF(A1>60, "합격", "불합격") → 숫자 비교
    • AI 활용: Gemini in Sheets 기능으로 "이 리뷰가 긍정인지 부정인지 분류해줘" → 텍스트 이해

    알아보기: Google Sheets의 AI 기능은 계속 업데이트되고 있습니다. 2024년부터 Google Workspace에서 Gemini AI를 스프레드시트 안에서 직접 활용할 수 있게 되었습니다. 우리는 이 장에서 기본 함수와 AI를 조합하는 방법을 배웁니다.

    개념 3: 자동화 워크플로 — 알고리즘이 실제로 동작하는 방식¶

    🎨 비유로 시작

    도미노를 세워 본 적이 있나요? 첫 번째 도미노를 넘기면 나머지가 연쇄적으로 쓰러집니다. 자동화 워크플로도 같습니다. 트리거(첫 번째 도미노)가 발생하면, 미리 정해 놓은 작업들이 순서대로 자동 실행됩니다.

    📖 정확한 정의

    자동화 워크플로(Automation Workflow) 란 특정 조건(트리거)이 충족되었을 때, 미리 설정한 일련의 작업이 자동으로 순서대로 실행되는 시스템입니다. 이것은 우리가 배운 알고리즘의 실제 구현과 같습니다.

    ✅ 예시로 확인

    학교 도서관 반납 알림 시스템을 자동화한다고 해봅시다.

    graph TD A["📅 트리거: 매일 오전 9시"] --> B{"반납일이 지난 책이 있는가?"} B -->|있다| C["📧 해당 학생에게 알림 메일 전송"] B -->|없다| D["⏹️ 아무 작업도 하지 않음"] C --> E["📊 알림 기록을 스프레드시트에 저장"]

    이 순서도, 어디서 많이 본 것 같지 않나요? 바로 이전 차시에서 설계한 알고리즘의 순서도와 동일한 구조입니다!

    개념 4: 설계와 구현의 간극¶

    🎨 비유로 시작

    건축 설계도를 그려 본 적이 있나요? 종이 위에서는 완벽한 꿈의 집이었는데, 실제로 지으려니 "이 벽은 너무 얇아서 세울 수 없어요", "이 위치에는 배관이 지나가서 창문을 못 넣어요" 같은 문제가 생깁니다. 알고리즘도 마찬가지입니다.

    📖 정확한 정의

    설계와 구현의 간극이란, 알고리즘을 종이 위에서 설계할 때는 고려하지 못했던 문제들이 실제 도구로 구현하는 과정에서 드러나는 현상을 말합니다. 이 간극을 발견하고 수정하는 과정이 바로 디버깅(Debugging) 의 시작입니다.

    ✅ 예시로 확인

    "학생 성적을 입력하면 등급을 자동으로 매겨주는 시스템"을 설계했다고 합시다.

    • 종이 위 설계: "90점 이상이면 A, 80점 이상이면 B..."
    • 실제 구현 시 발견하는 문제들:
    • 학생이 점수를 입력하지 않으면? (빈 칸 처리)
    • "구십"이라고 글자로 입력하면? (데이터 타입 문제)
    • 100점을 초과하는 점수를 입력하면? (범위 검증)

    생각해보기: 여러분이 이전 차시에서 만든 알고리즘에서도 이런 "예상하지 못한 상황"이 있을 수 있을까요? 어떤 것들이 있을지 잠깐 생각해 봅시다.

    🔨 따라하기: "학급 건의함 자동 분류 시스템" 만들기¶

    이제 실제로 노코드 도구를 사용하여 자동화 시스템을 만들어 봅시다!

    🎯 목표: Google Forms로 학급 건의 사항을 접수받고, Google Sheets에서 AI + 함수 조합으로 자동 분류하는 시스템을 만듭니다.

    📋 준비물: - Google 계정 - 웹 브라우저 (Chrome 권장) - 이전 차시에서 설계한 알고리즘 순서도 (없으면 이 장에서 새로 설계)

    Step 1: 알고리즘 설계하기 (종이 위의 설계도) 🗺️¶

    실습에 들어가기 전, 먼저 종이 위에서 알고리즘을 설계합니다. 노코드라도 "무엇을 만들 것인지"를 먼저 정리해야 합니다.

    우리가 만들 시스템의 알고리즘:

    1. 학생이 건의 사항을 제출한다
2. 건의 내용에 특정 키워드가 있는지 확인한다
3. 키워드에 따라 카테고리를 분류한다:
   - "급식", "메뉴", "음식" → 급식 관련
   - "시설", "고장", "수리" → 시설 관련
   - "수업", "시험", "과제" → 학습 관련
   - 위 키워드가 없으면 → 기타
4. 분류 결과와 함께 스프레드시트에 자동 정리한다
5. 긴급도를 판단한다:
   - "위험", "긴급", "당장" 키워드 포함 → 🔴 긴급
   - 그 외 → 🟢 일반

    이것을 순서도로 표현하면 다음과 같습니다.

    graph TD A["📝 학생이 건의 사항 제출"] --> B["키워드 확인"] B --> C{"급식 관련 키워드?"} C -->|있다| D["🍚 급식 관련으로 분류"] C -->|없다| E{"시설 관련 키워드?"} E -->|있다| F["🔧 시설 관련으로 분류"] E -->|없다| G{"학습 관련 키워드?"} G -->|있다| H["📖 학습 관련으로 분류"] G -->|없다| I["📌 기타로 분류"] D --> J{"긴급 키워드?"} F --> J H --> J I --> J J -->|있다| K["🔴 긴급 표시"] J -->|없다| L["🟢 일반 표시"] K --> M["📊 결과를 스프레드시트에 저장"] L --> M

    알아보기: 이 순서도가 바로 우리의 "설계도"입니다. 이제 이 설계도를 실제 도구로 구현해 봅시다. 구현 과정에서 설계도와 다른 점이 생기는지 주의 깊게 관찰하세요!

    Step 2: Google Forms로 입력 화면 만들기 📋¶

    ① Google Forms에 접속합니다

    • 브라우저에서 forms.google.com에 접속합니다
    • "새 양식 만들기(+)"를 클릭합니다

    ② 양식 제목과 설명을 입력합니다

    • 제목: 학급 건의함 📮
    • 설명: 우리 학급을 더 좋게 만들기 위한 건의 사항을 자유롭게 적어주세요.

    ③ 질문을 추가합니다

    첫 번째 질문: - 질문: 이름 - 유형: 단답형 - 필수 여부: ✅ 필수

    두 번째 질문 (+ 버튼으로 추가): - 질문: 건의 사항을 적어주세요 - 유형: 장문형 - 필수 여부: ✅ 필수

    세 번째 질문: - 질문: 긴급한 사항인가요? - 유형: 객관식 - 선택지: 예 / 아니오

    ④ 응답을 스프레드시트와 연결합니다

    • 상단의 "응답" 탭을 클릭합니다
    • 초록색 스프레드시트 아이콘(📊)을 클릭합니다
    • "새 스프레드시트 만들기" → 이름: 학급 건의함 데이터 → 만들기

    🎉 여기까지 완성되면, 학생들이 양식을 제출할 때마다 Google Sheets에 자동으로 데이터가 쌓입니다. 이것만으로도 벌써 입력 → 저장 과정이 자동화된 것입니다!

    Step 3: Google Sheets에서 자동 분류 함수 만들기 🔧¶

    이제 핵심 단계입니다. 스프레드시트에 함수를 넣어서 자동 분류가 되도록 만듭니다.

    ① 스프레드시트 구조 확인하기

    Google Forms와 연결된 스프레드시트를 열면 다음과 같은 열이 자동으로 만들어져 있습니다.

    • A열: 타임스탬프 (제출 시간)
    • B열: 이름
    • C열: 건의 사항 내용
    • D열: 긴급 여부

    ② 분류용 열 추가하기

    • E1 셀에 자동 카테고리라고 입력합니다
    • F1 셀에 긴급도라고 입력합니다

    ③ E2 셀에 카테고리 분류 함수 입력하기

    다음 함수를 E2 셀에 정확히 복사해서 붙여넣으세요.

    =IF(OR(ISNUMBER(SEARCH("급식",C2)),ISNUMBER(SEARCH("메뉴",C2)),ISNUMBER(SEARCH("음식",C2))),"🍚 급식 관련",IF(OR(ISNUMBER(SEARCH("시설",C2)),ISNUMBER(SEARCH("고장",C2)),ISNUMBER(SEARCH("수리",C2))),"🔧 시설 관련",IF(OR(ISNUMBER(SEARCH("수업",C2)),ISNUMBER(SEARCH("시험",C2)),ISNUMBER(SEARCH("과제",C2))),"📖 학습 관련","📌 기타")))

    이 함수가 하는 일을 한 줄씩 분석해 봅시다.

    • SEARCH("급식", C2): C2 셀에서 "급식"이라는 단어를 찾습니다
    • ISNUMBER(...): 찾았으면 숫자(위치)가 반환되므로 TRUE, 못 찾으면 FALSE
    • OR(...): 여러 키워드 중 하나라도 있으면 TRUE
    • IF(조건, 참일 때, 거짓일 때): 조건에 따라 분류

    알아보기: 이 함수 구조를 자세히 보면, Step 1에서 그린 순서도와 정확히 같은 구조입니다! IF 안에 IF가 들어가는 중첩(nested) 구조가 순서도의 분기(다이아몬드)에 해당합니다.

    graph LR A["순서도의 다이아몬드 ◇"] -->|"대응"| B["함수의 IF 조건"] C["순서도의 화살표 →"] -->|"대응"| D["함수의 중첩 구조"] E["순서도의 사각형 □"] -->|"대응"| F["함수의 결과값"]

    ④ F2 셀에 긴급도 판단 함수 입력하기

    =IF(OR(ISNUMBER(SEARCH("위험",C2)),ISNUMBER(SEARCH("긴급",C2)),ISNUMBER(SEARCH("당장",C2)),D2="예"),"🔴 긴급","🟢 일반")

    이 함수는 건의 내용에 "위험", "긴급", "당장" 키워드가 있거나, 학생이 직접 "긴급"이라고 답한 경우 🔴 긴급으로 표시합니다.

    ⑤ 함수를 아래로 확장하기

    • E2와 F2 셀을 선택합니다
    • 셀 오른쪽 아래 모서리의 작은 파란 점을 아래로 드래그합니다
    • 또는 E2:F2를 선택한 후 Ctrl+C → 아래 셀들을 선택 → Ctrl+V

    Step 4: 테스트 데이터로 검증하기 🧪¶

    알고리즘이 제대로 동작하는지 확인하려면 테스트가 필수입니다!

    ① Google Forms로 돌아가서 다음 건의 사항들을 제출합니다:

    테스트 건의 1: - 이름: 테스트학생1 - 건의: "급식 메뉴가 너무 단조로워요. 다양한 음식을 먹고 싶습니다." - 긴급: 아니오

    테스트 건의 2: - 이름: 테스트학생2 - 건의: "교실 에어컨이 고장 났어요. 당장 수리해주세요." - 긴급: 예

    테스트 건의 3: - 이름: 테스트학생3 - 건의: "체육 시간을 더 늘려주세요." - 긴급: 아니오

    ② 스프레드시트에서 결과를 확인합니다

    • 테스트 1 → 🍚 급식 관련 / 🟢 일반 ✅
    • 테스트 2 → 🔧 시설 관련 / 🔴 긴급 ✅
    • 테스트 3 → 📌 기타 / 🟢 일반 ✅

    ③ 여기서 "설계와 구현의 간극"을 발견합니다!

    테스트 3을 보세요. "체육 시간을 늘려달라"는 건의는 사실 학습 관련에 가깝지 않나요? 하지만 "수업", "시험", "과제" 키워드가 없어서 "기타"로 분류되었습니다.

    생각해보기: 이것이 바로 설계와 구현의 간극입니다! 종이 위에서는 "학습 관련 키워드가 있으면 학습으로 분류"라고 설계했지만, 실제로는 모든 가능한 키워드를 미리 예측하기 어렵습니다. 어떻게 해결할 수 있을까요?

    해결 방법: 키워드를 추가합니다. "체육", "시간표", "수행평가" 등 더 많은 키워드를 함수에 포함시키면 됩니다. 하지만 이렇게 계속 키워드를 추가하는 데는 한계가 있습니다. 이 한계를 극복하는 것이 바로 AI의 역할입니다.

    Step 5: AI를 활용한 더 똑똑한 분류 (Google Sheets + Gemini) 🤖¶

    키워드 기반 분류의 한계를 AI로 극복해 봅시다.

    방법 A: Google Sheets의 Gemini 사이드패널 활용

    최신 Google Sheets에서는 오른쪽에 Gemini 사이드패널이 있습니다.

    • 스프레드시트 오른쪽 상단의 Gemini 아이콘(✨)을 클릭합니다
    • 다음과 같이 요청합니다:
    C열의 건의 사항을 읽고, 각 건의를 "급식 관련", "시설 관련", 
"학습 관련", "기타" 중 하나로 분류해서 G열에 넣어줘.

    방법 B: AI 프롬프트를 활용한 수동 분류 (Gemini 기능이 없을 때)

    Google Sheets의 Gemini 기능을 사용할 수 없는 환경이라면, 별도의 AI 챗봇(ChatGPT, Gemini 웹 등)을 활용할 수 있습니다.

    • 스프레드시트의 건의 사항들을 복사합니다
    • AI 챗봇에 다음과 같이 입력합니다:
    아래 학생 건의 사항들을 읽고, 각각을 [급식 관련 / 시설 관련 / 
학습 관련 / 기타] 중 하나로 분류해주세요.

1. "급식 메뉴가 너무 단조로워요. 다양한 음식을 먹고 싶습니다."
2. "교실 에어컨이 고장 났어요. 당장 수리해주세요."
3. "체육 시간을 더 늘려주세요."

    AI의 응답 예시:

    1. 급식 관련 - "급식 메뉴", "음식" 키워드로 판단
2. 시설 관련 - "에어컨 고장", "수리" 키워드로 판단  
3. 학습 관련 - "체육 시간"은 수업 시간표와 관련되므로 학습 관련

    3번을 보세요! AI는 "체육 시간"이라는 표현이 키워드 목록에 없더라도, 문맥을 이해해서 "학습 관련"으로 올바르게 분류합니다. 이것이 키워드 기반 함수와 AI의 가장 큰 차이입니다.
