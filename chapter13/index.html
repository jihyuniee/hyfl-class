    Chapter 13. 해결책을 그려보자 – 알고리즘과 순서도¶

    Part 4. 문제 발견과 알고리즘 설계 – 프로젝트의 뼈대 세우기

    🎯 이 장에서 배우는 것¶

    • 알고리즘의 개념을 '문제 해결의 단계별 레시피'로 설명할 수 있다
    • 순서도(Flowchart)의 기본 기호 5가지를 구분하고 사용할 수 있다
    • 조건 분기와 반복을 포함한 순서도를 직접 설계할 수 있다
    • 자신의 프로젝트 해결 과정을 순서도로 표현하고 검증할 수 있다

    💡 왜 이걸 배우나요?¶

    여러분이 처음 가는 장소에 갈 때, 네비게이션 없이 "대충 저쪽으로 가면 되겠지"라고 출발하면 어떻게 될까요? 🗺️ 길을 잃거나, 돌아가거나, 시간을 낭비하게 됩니다.

    프로그래밍도 마찬가지입니다. 코드를 작성하기 전에 "어떤 순서로 문제를 풀 것인가"를 먼저 설계하지 않으면, 코드가 엉키고 버그가 쏟아집니다. 이 설계도가 바로 알고리즘이고, 그것을 눈에 보이게 그린 것이 순서도입니다.

    실제로 현업 개발자들도 복잡한 기능을 만들기 전에 순서도를 먼저 그립니다. 이번 장에서 이 습관을 익히면, 앞으로 어떤 프로젝트를 만들든 "먼저 그리고, 그다음 코딩한다"는 강력한 무기를 갖게 됩니다. 💪

    🤔 생각해보기: 라면 끓이기를 순서대로 적어보자

    지금 바로 노트에 "라면 끓이는 법"을 단계별로 적어보세요. 몇 단계가 나오나요? "물이 끓었는지 확인"처럼 판단이 필요한 순간이 있었나요? 이 과정 자체가 알고리즘을 만드는 것입니다!

    📚 핵심 개념¶

    개념 1: 알고리즘 — 문제 해결의 단계별 레시피 🍳¶

    비유로 시작해볼까요?

    요리 레시피를 떠올려보세요. 레시피에는 이런 것들이 있습니다.

    • 재료(입력): 물 550ml, 라면 1봉지, 계란 1개
    • 조리 순서(처리): 물 끓이기 → 면과 스프 넣기 → 4분 끓이기
    • 완성된 요리(출력): 맛있는 라면 한 그릇

    알고리즘도 정확히 같은 구조입니다!

    정확한 정의

    알고리즘(Algorithm): 어떤 문제를 해결하기 위해 정해진 입력을 받아서, 명확한 단계를 순서대로 수행하여, 원하는 출력을 만들어내는 절차

    핵심 특징 3가지

    • 명확성: 각 단계가 애매하지 않고 누구나 똑같이 실행할 수 있어야 합니다
    • 유한성: 반드시 끝나야 합니다 (영원히 반복되면 안 됩니다)
    • 효과성: 실제로 실행 가능한 동작이어야 합니다

    예시로 확인

    네비게이션의 경로 안내를 생각해봅시다.

    graph LR A["🏠 출발점<br/>(입력: 현재 위치)"] --> B["🛣️ 경로 계산<br/>(처리: 최단 경로 찾기)"] B --> C["📍 도착점<br/>(출력: 목적지 도착)"]

    네비게이션은 이 과정에서 조건 분기(막힌 도로가 있으면 우회)와 반복(목적지에 도착할 때까지 안내 반복)을 사용합니다. 알고리즘도 마찬가지입니다.

    개념 2: 순서도 — 알고리즘을 눈에 보이게 그린 지도 🗺️¶

    비유로 시작해볼까요?

    건축가가 집을 짓기 전에 설계도를 그리듯, 프로그래머는 코딩하기 전에 순서도를 그립니다. 머릿속 생각을 그림으로 꺼내면, 빠뜨린 단계나 논리적 오류를 훨씬 쉽게 발견할 수 있습니다.

    정확한 정의

    순서도(Flowchart): 알고리즘의 각 단계를 정해진 기호와 화살표로 연결하여 실행 순서를 시각적으로 표현한 다이어그램

    순서도 기본 기호 5가지

    graph TD A(["⬭ 단말기호<br/>시작 / 끝<br/>둥근 모서리"]) B["▭ 처리<br/>계산, 대입 등<br/>직사각형"] C{"◇ 판단<br/>조건 확인 (예/아니오)<br/>마름모"} D[/"▱ 입출력<br/>데이터 입력/출력<br/>평행사변형"/] E(["→ 흐름선<br/>실행 순서를 연결<br/>화살표"])

    각 기호를 정리하면 이렇습니다.

    • 단말 기호 (둥근 모서리 직사각형): 알고리즘의 시작과 끝을 나타냅니다
    • 처리 기호 (직사각형): 계산이나 동작 등 실행할 작업을 나타냅니다
    • 판단 기호 (마름모): "예/아니오"로 갈라지는 조건을 나타냅니다
    • 입출력 기호 (평행사변형): 데이터를 입력받거나 출력할 때 사용합니다
    • 흐름선 (화살표): 단계 간의 실행 순서를 연결합니다

    예시로 확인 — 라면 끓이기 순서도

    graph TD S(["시작"]) --> A["냄비에 물 550ml 넣기"] A --> B["가스레인지 켜기"] B --> C{"물이 끓는가?"} C -- 아니오 --> D["기다리기"] D --> C C -- 예 --> E["면과 스프 넣기"] E --> F["4분 타이머 시작"] F --> G{"4분 지났는가?"} G -- 아니오 --> H["계속 끓이기"] H --> G G -- 예 --> I["불 끄고 그릇에 담기"] I --> J(["끝"])

    위 순서도에서 마름모 기호가 2개 보이시나요? "물이 끓는가?"와 "4분 지났는가?"가 바로 조건 분기이고, 조건이 만족될 때까지 돌아가는 것이 반복입니다.

    개념 3: 조건 분기 — 갈림길에서의 선택 🔀¶

    비유로 시작해볼까요?

    등교길에 비가 오면 우산을 가져가고, 비가 안 오면 그냥 나가죠? 이처럼 상황에 따라 다른 길을 선택하는 것이 조건 분기입니다.

    정확한 정의

    조건 분기(Conditional Branch): 특정 조건의 참/거짓에 따라 실행 경로가 나뉘는 구조

    예시로 확인 — 로그인 과정

    graph TD S(["시작"]) --> A[/"아이디와 비밀번호 입력"/] A --> B{"아이디가 존재하는가?"} B -- 아니오 --> C[/"'존재하지 않는 아이디입니다' 출력"/] C --> A B -- 예 --> D{"비밀번호가 일치하는가?"} D -- 아니오 --> E[/"'비밀번호가 틀렸습니다' 출력"/] E --> A D -- 예 --> F[/"'로그인 성공!' 출력"/] F --> G(["끝"])

    조건 분기가 2개 연달아 나옵니다. 아이디를 먼저 확인하고, 그다음 비밀번호를 확인하는 단계적 판단이죠. 이런 설계를 미리 해두면 코딩할 때 if-else 구조를 자연스럽게 작성할 수 있습니다.

    개념 4: 반복 — 조건이 만족될 때까지 되풀이 🔄¶

    비유로 시작해볼까요?

    샴푸 뒷면에 "거품을 내서 헹구세요. 필요하면 반복하세요"라고 적혀 있죠? 머리가 깨끗해질 때까지 같은 동작을 반복하는 것, 이것이 알고리즘의 반복입니다.

    정확한 정의

    반복(Loop/Iteration): 특정 조건이 만족될 때까지 같은 처리 과정을 되풀이하는 구조

    예시로 확인 — 1부터 10까지 합 구하기

    graph TD S(["시작"]) --> A["합계 = 0, 숫자 = 1"] A --> B{"숫자 ≤ 10 인가?"} B -- 예 --> C["합계 = 합계 + 숫자"] C --> D["숫자 = 숫자 + 1"] D --> B B -- 아니오 --> E[/"합계 출력"/] E --> F(["끝"])

    마름모에서 "예"를 따라가면 처리 → 다시 마름모로 돌아오는 고리(loop) 모양이 보이시나요? 숫자가 11이 되는 순간 "아니오"로 빠져나와 결과를 출력합니다.

    📌 알아보기: 조건 분기 vs 반복 — 무엇이 다를까?

    • 조건 분기: 한 번 판단하고 한쪽 길로 갑니다 (갈림길)
    • 반복: 조건을 만족할 때까지 계속 되돌아옵니다 (회전 교차로)

    순서도에서 판단 기호(마름모) 뒤에 화살표가 되돌아가는 고리가 있으면 반복, 없으면 조건 분기입니다.

    🔨 따라하기: 프로젝트 핵심 기능을 순서도로 설계하기¶

    이제 직접 순서도를 그려봅시다! 여기서는 "할 일 관리(To-Do) 앱"의 핵심 기능을 순서도로 설계합니다.

    Step 1: 핵심 기능을 한 문장으로 정의하기 ✍️¶

    먼저 순서도로 그릴 기능을 명확하게 정합니다.

    "사용자가 할 일을 추가하고, 완료 표시를 하고, 삭제할 수 있는 기능"

    너무 큰 기능을 한 번에 그리지 마세요. 하나의 기능 단위로 나누는 것이 핵심입니다.

    • ✅ 좋은 예: "할 일 추가 기능"
    • ❌ 나쁜 예: "할 일 관리 앱 전체"

    Step 2: 입력 → 처리 → 출력 정리하기 📋¶

    노트에 아래 세 가지를 적어보세요.

    • 입력: 사용자가 입력하는 할 일 텍스트, 선택하는 메뉴 번호
    • 처리: 메뉴에 따라 추가/완료/삭제 수행
    • 출력: 업데이트된 할 일 목록 표시

    Step 3: 단계를 자연어로 나열하기 📝¶

    코드가 아니라 말로 순서를 적습니다.

    1. 프로그램 시작
2. 할 일 목록 보여주기
3. 메뉴 선택 받기 (1.추가 / 2.완료 / 3.삭제 / 4.종료)
4. 만약 "추가"를 선택했다면 → 할 일 텍스트 입력받기 → 목록에 추가
5. 만약 "완료"를 선택했다면 → 번호 입력받기 → 완료 표시
6. 만약 "삭제"를 선택했다면 → 번호 입력받기 → 목록에서 삭제
7. 만약 "종료"를 선택했다면 → 프로그램 끝
8. 종료가 아니라면 → 2번으로 돌아가기

    Step 4: 순서도로 변환하기 🎨¶

    자연어 단계를 기호로 바꿉니다.

    graph TD S(["시작"]) --> A[/"할 일 목록 출력"/] A --> B[/"메뉴 선택 입력<br/>1.추가 2.완료 3.삭제 4.종료"/] B --> C{"선택이 4(종료)인가?"} C -- 예 --> Z(["끝"]) C -- 아니오 --> D{"선택이 1(추가)인가?"} D -- 예 --> E[/"할 일 텍스트 입력"/] E --> F["목록에 할 일 추가"] F --> A D -- 아니오 --> G{"선택이 2(완료)인가?"} G -- 예 --> H[/"완료할 번호 입력"/] H --> I["해당 항목 완료 표시"] I --> A G -- 아니오 --> J[/"삭제할 번호 입력"/] J --> K["해당 항목 삭제"] K --> A

    Step 5: 파이썬 코드로 구현하기 💻¶

    순서도를 보면서 코드를 작성하면, 순서도의 각 기호가 코드의 어느 부분에 대응하는지 확인할 수 있습니다.

    # 할 일 관리 프로그램 - 순서도를 코드로 변환
todo_list = []  # 할 일을 저장할 리스트

while True:  # 순서도의 반복 고리에 해당
    # [입출력] 할 일 목록 출력
    print("\n📋 현재 할 일 목록:")
    if not todo_list:
        print("  (비어 있음)")
    for i, item in enumerate(todo_list, 1):
        status = "✅" if item["done"] else "⬜"
        print(f"  {i}. {status} {item['text']}")

    # [입출력] 메뉴 선택 입력
    print("\n1.추가  2.완료  3.삭제  4.종료")
    choice = input("메뉴를 선택하세요: ")

    # [판단] 선택이 4(종료)인가?
    if choice == "4":
        print("프로그램을 종료합니다. 👋")
        break  # 순서도의 '끝'으로 이동

    # [판단] 선택이 1(추가)인가?
    elif choice == "1":
        text = input("할 일을 입력하세요: ")  # [입출력]
        todo_list.append({"text": text, "done": False})  # [처리]
        print(f"'{text}' 추가 완료!")

    # [판단] 선택이 2(완료)인가?
    elif choice == "2":
        num = int(input("완료할 번호: ")) - 1  # [입출력]
        if 0 <= num < len(todo_list):
            todo_list[num]["done"] = True  # [처리]
            print("완료 표시했습니다! ✅")
        else:
            print("잘못된 번호입니다.")

    # [그 외 = 삭제]
    elif choice == "3":
        num = int(input("삭제할 번호: ")) - 1  # [입출력]
        if 0 <= num < len(todo_list):
            removed = todo_list.pop(num)  # [처리]
            print(f"'{removed['text']}' 삭제 완료! 🗑️")
        else:
            print("잘못된 번호입니다.")

    else:
        print("1~4 중에서 선택해주세요.")

    Step 6: AI로 순서도 검증하기 🤖¶

    설계한 순서도에 빠진 단계가 없는지, AI에게 물어봅시다. 아래처럼 프롬프트를 작성하면 됩니다.

    프롬프트 예시:
"아래는 할 일 관리 프로그램의 순서도입니다.
1. 시작 → 목록 출력 → 메뉴 선택
2. 종료 선택 시 → 끝
3. 추가 선택 시 → 텍스트 입력 → 목록에 추가 → 1로 돌아감
4. 완료 선택 시 → 번호 입력 → 완료 표시 → 1로 돌아감
5. 삭제 선택 시 → 번호 입력 → 삭제 → 1로 돌아감

이 순서도에서 빠진 단계나 예외 상황이 있을까요?"

    AI가 알려줄 수 있는 빠진 부분의 예시: - "잘못된 번호를 입력했을 때의 처리가 빠져 있습니다" - "목록이 비어 있을 때 완료/삭제를 선택하면 어떻게 되나요?" - "숫자가 아닌 값을 입력하면 오류가 발생할 수 있습니다"

    💡 정리하기

    순서도 설계 6단계를 기억하세요. 1. 기능을 한 문장으로 정의 2. 입력 → 처리 → 출력 정리 3. 자연어로 단계 나열 4. 순서도 기호로 변환 5. 코드로 구현 6. AI로 검증

    📝 전체 코드¶

    아래는 Step 5의 코드를 복사해서 바로 실행할 수 있는 전체 코드입니다.

    # ============================================
# 📋 할 일 관리 프로그램 (To-Do Manager)
# 챕터 13 - 알고리즘과 순서도 실습
# ============================================

todo_list = []

print("=" * 40)
print("  📋 할 일 관리 프로그램에 오신 걸 환영합니다!")
print("=" * 40)

while True:
    # 목록 출력
    print("\n📋 현재 할 일 목록:")
    if not todo_list:
        print("  (비어 있음)")
    for i, item in enumerate(todo_list, 1):
        status = "✅" if item["done"] else "⬜"
        print(f"  {i}. {status} {item['text']}")

    # 메뉴 선택
    print("\n1.추가  2.완료  3.삭제  4.종료")
    choice = input("메뉴를 선택하세요: ")

    if choice == "4":
        print("\n프로그램을 종료합니다. 👋")
        break

    elif choice == "1":
        text = input("할 일을 입력하세요: ")
        if text.strip():  # 빈 문자열 방지
            todo_list.append({"text": text, "done": False})
            print(f"✨ '{text}' 추가 완료!")
        else:
            print("⚠️ 빈 할 일은 추가할 수 없습니다.")

    elif choice == "2":
        if not todo_list:
            print("⚠️ 할 일이 없습니다. 먼저 추가해주세요.")
            continue
        try:
            num = int(input("완료할 번호: ")) - 1
            if 0 <= num < len(todo_list):
                todo_list[num]["done"] = True
                print("완료 표시했습니다! ✅")
            else:
                print("⚠️ 잘못된 번호입니다.")
        except ValueError:
            print("⚠️ 숫자를 입력해주세요.")

    elif choice == "3":
        if not todo_list:
            print("⚠️ 삭제할 할 일이 없습니다.")
            continue
        try:
            num = int(input("삭제할 번호: ")) - 1
            if 0 <= num < len(todo_list):
                removed = todo_list.pop(num)
                print(f"🗑️ '{removed['text']}' 삭제 완료!")
            else:
                print("⚠️ 잘못된 번호입니다.")
        except ValueError:
            print("⚠️ 숫자를 입력해주세요.")

    else:
        print("⚠️ 1~4 중에서 선택해주세요.")

    ⚠️ 자주 하는 실수¶

    실수 1: 시작/끝 기호를 빼먹는다 😅

    순서도에서 시작과 끝이 없으면, 프로그램이 어디서 시작하고 어디서 끝나는지 알 수 없습니다. 항상 둥근 모서리의 단말 기호로 시작과 끝을 명시하세요.

    실수 2: 판단 기호에서 "예/아니오" 표시를 안 한다 🔀

    마름모에서 나가는 화살표가 2개인데, 어느 쪽이 "예"이고 어느 쪽이 "아니오"인지 적지 않으면 순서도를 읽는 사람이 혼란스럽습니다. 반드시 화살표에 "예(Yes)/아니오(No)"를 표기하세요.

    실수 3: 반복의 탈출 조건을 설계하지 않는다 🔄

    반복 구조를 만들 때, "이 반복은 언제 끝나지?"를 꼭 확인하세요. 탈출 조건이 없는 반복은 무한 루프가 되어 프로그램이 멈추지 않습니다.

    # ❌ 나쁜 예: 탈출 조건이 없음
while True:
    print("영원히 실행됩니다...")

# ✅ 좋은 예: 탈출 조건이 있음
count = 0
while count < 5:
    print(f"{count}번째 실행")
    count += 1  # 이 줄이 없으면 무한 루프!

    실수 4: 너무 세세한 것까지 순서도에 넣는다 📏

    순서도는 전체적인 흐름을 파악하기 위한 도구입니다. 변수 이름이나 세부 코드 문법까지 넣으면 오히려 읽기 어려워집니다. "숫자 입력받기" 정도로 충분하지, "num = int(input())"까지 쓸 필요는 없습니다.

    • ① 직사각형 — 조건 판단
    • ② 마름모 — 처리/계산
