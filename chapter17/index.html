    Chapter 17: 최종 프로젝트 — 문제 발견과 아이디어 경진¶

    Part 6 | [2권] 최종 프로젝트 — AI로 세상의 문제를 해결하라

    🎯 이 장에서 배우는 것¶

    • 학교·지역사회의 실제 문제 중 AI로 해결 가능한 문제를 선정할 수 있다
    • 문제-사용자-해결방안-필요기술을 포함한 프로젝트 제안서를 작성할 수 있다
    • 아이디어를 1분 피칭으로 발표하고 동료 피드백을 줄 수 있다
    • 피드백을 반영하여 기획을 구체화하고 일정표를 작성할 수 있다

    💡 왜 이걸 배우나요?¶

    지금까지 여러분은 데이터 분석, 머신러닝, 딥러닝, 자연어 처리 등 다양한 AI 기술을 배웠습니다. 하지만 기술을 "아는 것"과 기술로 "문제를 해결하는 것"은 완전히 다른 일입니다.

    세계적인 AI 서비스들도 시작은 단순했습니다.

    • 배달 앱 → "메뉴판 없이 음식을 주문할 수 없을까?"라는 불편함에서 출발
    • 번역 AI → "외국어 간판을 바로 읽을 수 없을까?"라는 궁금증에서 출발
    • 자율주행 → "졸음운전 사고를 줄일 수 없을까?"라는 안전 문제에서 출발

    모두 "내 주변의 진짜 문제"를 발견한 사람들이 만든 결과물입니다.

    이번 장에서는 여러분이 직접 문제를 찾고, 팀과 함께 아이디어를 다듬어 프로젝트 제안서를 완성합니다. 이것이 앞으로 3개 차시에 걸쳐 진행할 최종 프로젝트의 출발점입니다.

    🔑 기억하세요: 좋은 AI 프로젝트는 좋은 기술이 아니라 좋은 문제에서 시작됩니다.

    📚 핵심 개념¶

    개념 1: 문제 발견 — "관찰의 눈"으로 세상 보기¶

    1. 비유로 시작

    의사가 환자를 치료하려면, 먼저 어디가 아픈지 정확히 진단해야 합니다. 아무리 뛰어난 수술 실력을 가져도, 아픈 곳을 모르면 소용없죠. AI 프로젝트도 마찬가지입니다. 기술(수술 도구)보다 문제 진단이 먼저입니다.

    2. 정확한 정의

    문제 발견(Problem Discovery)이란 일상에서 사람들이 겪는 불편함, 위험, 비효율을 체계적으로 관찰하고 기록하는 과정입니다. 단순히 "불편하다"를 넘어서, 누가, 언제, 어디서, 왜 불편한지를 구체적으로 파악하는 것이 핵심입니다.

    3. 예시로 확인

    문제를 발견하는 3가지 방법을 살펴봅시다.

    • 👀 관찰법: 통학길에 횡단보도에서 스마트폰을 보며 걷는 학생이 많다 → "보행 안전 문제"
    • 💬 인터뷰법: 학교 급식 담당 선생님에게 물어보니 잔반이 매일 20kg 나온다 → "음식물 쓰레기 문제"
    • 📊 데이터 탐색법: 우리 동네 미세먼지 데이터를 보니 특정 시간대에 급증한다 → "공기질 예측 문제"

    📌 알아보기: 좋은 문제의 4가지 조건

    • 구체적이다: "세상을 좋게 만들고 싶다" ❌ → "우리 학교 분리수거 오류를 줄이고 싶다" ✅
    • 측정 가능하다: 문제의 크기를 숫자로 표현할 수 있다
    • AI가 도울 수 있다: 이미지 인식, 텍스트 분석, 예측 등 AI 기술이 적용 가능하다
    • 데이터를 구할 수 있다: 학습에 필요한 데이터를 직접 모으거나 공개 데이터를 활용할 수 있다

    개념 2: 아이디어 구체화 — 문제에서 솔루션으로¶

    1. 비유로 시작

    건축가가 건물을 지을 때, 바로 벽돌을 쌓지 않습니다. 먼저 설계도(블루프린트)를 그립니다. "누가 사용할 건물인지, 몇 층인지, 어떤 재료를 쓸지"를 미리 정하는 거죠. AI 프로젝트에서 이 설계도가 바로 프로젝트 제안서입니다.

    2. 정확한 정의

    아이디어 구체화란 발견한 문제에 대해 "누구를 위해, 무엇을, 어떤 기술로 해결할 것인지"를 명확하게 정리하는 과정입니다. 이 과정에서 사용하는 프레임워크가 PUTS 프레임워크입니다.

    graph LR P["🔍 Problem<br/>문제 정의"] --> U["👤 User<br/>사용자 정의"] U --> T["🔧 Technology<br/>필요 기술"] T --> S["💡 Solution<br/>해결 방안"] S --> P style P fill:#FF6B6B,color:#fff style U fill:#4ECDC4,color:#fff style T fill:#45B7D1,color:#fff style S fill:#96CEB4,color:#fff

    3. 예시로 확인

    "학교 분리수거 오류 줄이기" 프로젝트를 PUTS 프레임워크로 정리하면:

    • 🔍 Problem(문제): 학생들이 플라스틱·종이·캔을 잘못 분류해서 재활용률이 60%에 그침
    • 👤 User(사용자): 우리 학교 학생 800명, 특히 1학년(분리수거 규칙에 익숙하지 않은)
    • 🔧 Technology(기술): 이미지 분류 모델 (카메라로 쓰레기를 촬영하면 분류 결과 안내)
    • 💡 Solution(해결방안): 분리수거 통 앞에 카메라를 설치, 올바른 통을 화면에 표시

    개념 3: 1분 피칭 — 아이디어를 설득력 있게 전달하기¶

    1. 비유로 시작

    엘리베이터를 타고 1층에서 10층까지 올라가는 약 1분. 이 짧은 시간 안에 상대방에게 내 아이디어를 전달해야 한다면? 이것이 실리콘밸리에서 유명한 엘리베이터 피치(Elevator Pitch)입니다.

    2. 정확한 정의

    1분 피칭이란 프로젝트의 핵심을 60초 이내로 발표하는 것입니다. 모든 내용을 담는 것이 아니라, 듣는 사람이 "오, 그거 좋은데!"라고 느낄 수 있도록 문제의 심각성과 해결의 매력을 집중적으로 전달합니다.

    3. 예시로 확인

    1분 피칭의 구조 (각 15초):

    • 🔴 15초 — 문제 공감: "여러분, 어제 점심 후 분리수거통을 보셨나요? 플라스틱 통에 종이컵이 가득했습니다."
    • 🟡 15초 — 원인과 규모: "학생 설문 결과, 73%가 '어디에 버려야 할지 모르겠다'고 답했습니다."
    • 🟢 15초 — 해결책: "저희 팀은 카메라로 쓰레기를 인식해 올바른 통을 알려주는 AI 도우미를 만들겠습니다."
    • 🔵 15초 — 기대 효과: "재활용률을 60%에서 90%로 올리고, 이 모델을 다른 학교에도 공유하겠습니다."

    📌 생각해보기: 좋은 피칭과 나쁜 피칭의 차이는 무엇일까요?

    • 나쁜 피칭: "저희는 이미지 분류 CNN 모델을 사용해서 ResNet 아키텍처로..." → 기술만 나열
    • 좋은 피칭: "매일 버려지는 잔반 20kg, 한 달이면 400kg입니다" → 문제에 공감하게 만듦

    개념 4: 피드백과 반복 — 아이디어를 "다이아몬드"로 만들기¶

    1. 비유로 시작

    다이아몬드 원석은 처음엔 투박한 돌덩이입니다. 여러 번 깎고(cutting), 다듬어야(polishing) 반짝이는 보석이 됩니다. 아이디어도 마찬가지입니다. 처음 생각은 원석이고, 동료 피드백이 바로 깎고 다듬는 과정입니다.

    2. 정확한 정의

    피드백 루프(Feedback Loop)란 아이디어를 발표 → 피드백 수집 → 개선 → 재발표하는 반복 과정입니다. 한 번에 완벽한 기획은 존재하지 않으며, 반복할수록 기획의 품질이 올라갑니다.

    3. 예시로 확인

    graph TD A["💡 초기 아이디어<br/>분리수거 AI 도우미"] --> B["🎤 1분 피칭<br/>팀별 발표"] B --> C["💬 동료 피드백 수집<br/>피드백 카드 작성"] C --> D["🔄 아이디어 수정<br/>부족한 점 보완"] D --> E["📋 최종 제안서 작성"] E --> F["✅ 주제 확정"] C -->|"좋은 점"| G["👍 유지"] C -->|"보완할 점"| H["🔧 개선"] C -->|"질문"| I["❓ 추가 조사"] style A fill:#FFE66D,color:#333 style F fill:#4ECDC4,color:#fff

    피드백을 줄 때는 SBI 방식을 사용합니다:

    • S (Situation, 상황): "분리수거 AI 도우미 아이디어에서"
    • B (Behavior, 구체적 내용): "카메라를 분리수거 통 앞에 설치한다고 했는데"
    • I (Impact, 영향/제안): "비가 오면 카메라가 망가질 수 있으니 방수 케이스를 고려하면 좋겠어요"

    🔨 따라하기: 프로젝트 아이디어 발굴부터 제안서 완성까지¶

    이번 실습은 코드 작성보다 사고 과정과 문서 작성에 집중합니다. 단, 아이디어를 정리하고 검증하는 데 Python을 활용합니다.

    Step 1: 내 주변의 문제 TOP 3 작성하기 🔍¶

    먼저, 혼자서 "AI가 해결해주면 좋겠는 문제"를 3가지 적어봅시다.

    방법: 아래 코드를 실행하고, 직접 입력하세요.

    # 나의 문제 발견 노트
print("=" * 50)
print("🔍 내 주변에서 AI가 해결해주면 좋겠는 문제 TOP 3")
print("=" * 50)

problems = []
for i in range(1, 4):
    print(f"\n--- 문제 {i} ---")
    what = input("무엇이 문제인가요? : ")
    where = input("어디서 발생하나요? : ")
    who = input("누가 불편을 겪나요? : ")
    why = input("왜 아직 해결되지 않았을까요? : ")

    problems.append({
        "번호": i,
        "문제": what,
        "장소": where,
        "대상": who,
        "미해결 이유": why
    })

print("\n" + "=" * 50)
print("📋 나의 문제 발견 노트 정리")
print("=" * 50)

for p in problems:
    print(f"\n🔹 문제 {p['번호']}: {p['문제']}")
    print(f"   📍 장소: {p['장소']}")
    print(f"   👤 대상: {p['대상']}")
    print(f"   ❓ 미해결 이유: {p['미해결 이유']}")

    Step 2: AI 적합성 평가하기 ⚖️¶

    찾은 문제 3가지 중에서 AI로 해결하기에 가장 적합한 것을 골라야 합니다. 다음 코드로 점수를 매겨봅시다.

    def evaluate_ai_fitness(problem_name):
    """AI 프로젝트 적합성을 평가하는 함수"""
    print(f"\n🔍 '{problem_name}' AI 적합성 평가")
    print("-" * 40)

    criteria = {
        "데이터를 구할 수 있는가? (공개 데이터, 직접 수집 등)": 0,
        "AI 기술이 도움이 되는가? (분류, 예측, 생성 등)": 0,
        "실제 사용자가 명확한가?": 0,
        "학생 수준에서 3주 안에 만들 수 있는가?": 0,
        "결과를 눈으로 확인할 수 있는가?": 0
    }

    for question in criteria:
        while True:
            try:
                score = int(input(f"  {question}\n  (1=전혀 아니다 ~ 5=매우 그렇다): "))
                if 1 <= score <= 5:
                    criteria[question] = score
                    break
                else:
                    print("  1~5 사이 숫자를 입력하세요.")
            except ValueError:
                print("  숫자를 입력하세요.")

    total = sum(criteria.values())
    max_score = len(criteria) * 5
    percentage = (total / max_score) * 100

    print(f"\n📊 총점: {total}/{max_score} ({percentage:.0f}%)")

    if percentage >= 80:
        print("🟢 훌륭합니다! AI 프로젝트로 매우 적합합니다.")
    elif percentage >= 60:
        print("🟡 괜찮습니다! 일부 보완하면 좋은 프로젝트가 됩니다.")
    else:
        print("🔴 다른 문제를 고려해보는 것이 좋겠습니다.")

    return total, criteria

    Step 3: PUTS 프레임워크로 제안서 작성하기 📋¶

    가장 높은 점수를 받은 문제를 선택해서, 프로젝트 제안서를 작성합니다.

    def create_proposal():
    """PUTS 프레임워크 기반 프로젝트 제안서 생성"""

    print("=" * 50)
    print("📋 AI 프로젝트 제안서 작성기")
    print("=" * 50)

    proposal = {}

    # 기본 정보
    proposal["프로젝트명"] = input("\n🏷️ 프로젝트 이름: ")
    proposal["팀명"] = input("👥 팀 이름: ")
    proposal["팀원"] = input("👤 팀원 (쉼표로 구분): ")

    # P - Problem
    print("\n--- 🔍 P: Problem (문제 정의) ---")
    proposal["문제 상황"] = input("어떤 문제를 해결하나요?\n: ")
    proposal["문제 규모"] = input("이 문제의 규모는? (숫자로 표현)\n: ")
    proposal["현재 해결 방식"] = input("현재는 이 문제를 어떻게 해결하고 있나요?\n: ")

    # U - User
    print("\n--- 👤 U: User (사용자 정의) ---")
    proposal["주요 사용자"] = input("누가 사용하나요?\n: ")
    proposal["사용자 수"] = input("예상 사용자 수는?\n: ")
    proposal["사용자 니즈"] = input("사용자가 가장 원하는 것은?\n: ")

    # T - Technology
    print("\n--- 🔧 T: Technology (필요 기술) ---")
    proposal["AI 기술"] = input("어떤 AI 기술을 사용하나요? (예: 이미지 분류, 텍스트 분석)\n: ")
    proposal["데이터 출처"] = input("데이터는 어디서 구하나요?\n: ")
    proposal["개발 도구"] = input("어떤 도구/라이브러리를 사용하나요?\n: ")

    # S - Solution
    print("\n--- 💡 S: Solution (해결 방안) ---")
    proposal["해결 방법"] = input("구체적으로 어떻게 해결하나요?\n: ")
    proposal["기대 효과"] = input("이 프로젝트가 성공하면 어떤 변화가 생기나요?\n: ")
    proposal["성공 기준"] = input("성공 여부를 어떻게 측정하나요?\n: ")

    # 제안서 출력
    print("\n" + "=" * 60)
    print(f"📋 프로젝트 제안서: {proposal['프로젝트명']}")
    print("=" * 60)

    sections = {
        "🔍 Problem (문제)": ["문제 상황", "문제 규모", "현재 해결 방식"],
        "👤 User (사용자)": ["주요 사용자", "사용자 수", "사용자 니즈"],
        "🔧 Technology (기술)": ["AI 기술", "데이터 출처", "개발 도구"],
        "💡 Solution (해결방안)": ["해결 방법", "기대 효과", "성공 기준"]
    }

    for section_name, keys in sections.items():
        print(f"\n{section_name}")
        print("-" * 40)
        for key in keys:
            print(f"  • {key}: {proposal[key]}")

    print(f"\n👥 팀: {proposal['팀명']} ({proposal['팀원']})")
    print("=" * 60)

    return proposal

# 제안서 작성 실행
proposal = create_proposal()

    Step 4: 1분 피칭 스크립트 만들기 🎤¶

    제안서를 바탕으로 발표 스크립트를 자동 생성합니다.

    ```python def generate_pitch_script(proposal): """1분 피칭 스크립트 자동 생성"""

    script = f"""

    {'=' * 50} 🎤 1분 피칭 스크립트 프로젝트: {proposal['프로젝트명']}

    ⏱️ [0:00 ~ 0:15] 🔴 문제 공감 만들기 "여러분, {proposal['문제 상황']} 현재 {proposal['문제 규모']}의 규모로 이 문제가 발생하고 있습니다."

    ⏱️ [0:15 ~ 0:30] 🟡 원인과 현재 상황 "지금까지는 {proposal['현재 해결 방식']} 하지만 이것만으로는 {proposal['주요 사용자']}의 문제를 해결하기 어렵습니다."

    ⏱️ [0:30 ~ 0:45] 🟢 AI 해결책 제시 "저희 {proposal['팀명']} 팀은 {proposal['AI 기술']}을(를) 활용해 {proposal['해결 방법']}"

    ⏱️ [0:45 ~ 1:00] 🔵 기대 효과 "이 프로젝트가 성공하면, {proposal['기대 효과']} 저희의 성공 기준은 {proposal['성공 기준']}입니다. 감사합니다!"

    {'=' * 50} 💡 발표 팁 • 첫 문장은 질문이나 충격적인 숫자로 시작하세요 • 청중의 눈을 보며 천천히 말하세요 • 1분 = 약 150~180단어입니다. 연습 필수! {'=' * 50} """ print(script) return script
