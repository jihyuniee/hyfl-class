    Chapter 2. AI는 어떻게 배울까? – 머신러닝의 원리¶

    Part 1. AI와의 첫 만남 – 인공지능, 넌 누구니? | 2차시 (2시간)

    🎯 이 장에서 배우는 것¶

    • 머신러닝의 핵심 원리를 '데이터 → 패턴 발견 → 예측' 3단계로 설명할 수 있다
    • 지도학습과 비지도학습의 차이를 실생활 비유로 구분할 수 있다
    • Teachable Machine을 사용해 직접 AI 모델을 학습시킬 수 있다
    • 학습 데이터의 양과 질이 AI 성능에 미치는 영향을 이해할 수 있다

    💡 왜 이걸 배우나요?¶

    생각해보기 🤔

    시험 기간을 떠올려 보세요. 여러분은 어떻게 공부하나요?

    기출문제를 반복해서 풀면서 "아, 이런 유형은 이렇게 풀면 되는구나!" 하고 패턴을 익히죠. 그리고 시험 당일, 처음 보는 문제도 비슷한 패턴이면 풀 수 있습니다.

    AI도 정확히 같은 방식으로 배웁니다.

    1장에서 우리는 AI가 무엇인지 알아봤습니다. 이번 장에서는 한 걸음 더 나아가, AI가 어떻게 배우는지 그 원리를 파헤칩니다.

    유튜브가 내 취향의 영상을 추천하고, 스마트폰이 내 얼굴을 알아보는 것 — 이 모든 것의 뒤에는 머신러닝(Machine Learning)이라는 기술이 있습니다. 이 원리를 이해하면, AI가 더 이상 마법이 아니라 논리적인 과정으로 보이기 시작할 거예요.

    📚 핵심 개념¶

    개념 1: 머신러닝이란?¶

    🎒 비유로 시작

    여러분이 수학 시험을 준비한다고 생각해 봅시다.

    • 1단계: 선생님이 준 기출문제 100개를 풉니다 → 데이터 수집
    • 2단계: 풀다 보니 "이차방정식 문제는 판별식부터 확인하면 빠르다"는 걸 깨닫습니다 → 패턴 발견
    • 3단계: 시험에서 처음 보는 이차방정식 문제가 나와도 풀 수 있습니다 → 예측

    📖 정확한 정의

    머신러닝(Machine Learning, 기계학습)이란, 컴퓨터가 명시적으로 프로그래밍되지 않아도 데이터로부터 스스로 패턴을 학습하여 새로운 상황에서 판단하거나 예측할 수 있게 되는 기술입니다.

    💬 쉽게 말하면: "사람이 일일이 규칙을 알려주지 않아도, 데이터를 많이 보면서 스스로 규칙을 찾아내는 컴퓨터"

    ✅ 예시로 확인

    • 스팸 메일 필터: "광고", "무료", "당첨" 같은 단어가 포함된 메일 수천 개를 학습 → 새로운 메일이 스팸인지 자동 판단
    • 음악 추천: 내가 들은 노래 목록(데이터)에서 선호 패턴 발견 → 비슷한 분위기의 새 노래 추천
    flowchart LR A["📊 1단계\n데이터 수집\n(기출문제를 모은다)"] --> B["🔍 2단계\n패턴 발견\n(풀이 규칙을 깨닫는다)"] B --> C["🎯 3단계\n예측\n(새 문제도 풀 수 있다)"] style A fill:#E3F2FD,stroke:#1565C0,color:#000 style B fill:#FFF3E0,stroke:#E65100,color:#000 style C fill:#E8F5E9,stroke:#2E7D32,color:#000

    개념 2: 지도학습 – 정답이 있는 공부¶

    🎒 비유로 시작

    영어 단어를 외울 때, 단어장에 영어 단어(문제)와 한글 뜻(정답)이 함께 적혀 있죠? 이처럼 정답(레이블)이 함께 주어진 상태에서 학습하는 방식이 지도학습입니다.

    📖 정확한 정의

    지도학습(Supervised Learning)은 입력 데이터와 그에 대응하는 정답(레이블, Label)을 함께 제공하여, 컴퓨터가 입력과 정답 사이의 관계를 학습하는 방법입니다.

    💬 "선생님이 채점까지 해주는 공부법"이라고 생각하면 됩니다.

    ✅ 예시로 확인

    • 고양이/강아지 분류: 사진마다 "이건 고양이", "이건 강아지"라고 정답을 붙여 학습
    • 시험 점수 예측: "공부 시간 3시간 → 80점, 5시간 → 92점" 같은 데이터로 학습 → 4시간 공부하면 몇 점일지 예측

    지도학습은 크게 두 가지로 나뉩니다.

    • 분류(Classification): 정해진 카테고리 중 하나를 고르기 (예: 스팸/정상, 고양이/강아지)
    • 회귀(Regression): 연속적인 숫자를 예측하기 (예: 내일 기온, 시험 점수)

    개념 3: 비지도학습 – 정답 없이 스스로 찾기¶

    🎒 비유로 시작

    처음 가본 학교 급식실에서 학생들이 자연스럽게 무리 지어 앉아 있는 걸 관찰한다고 생각해 보세요. 아무도 "1번 그룹, 2번 그룹"이라고 알려주지 않았지만, 가만히 보면 비슷한 학생들끼리 모여 있는 패턴이 보입니다. 이것이 비지도학습입니다.

    📖 정확한 정의

    비지도학습(Unsupervised Learning)은 정답(레이블)이 없는 데이터에서 컴퓨터가 스스로 숨겨진 구조나 패턴을 찾아내는 학습 방법입니다.

    💬 "정답지 없이 혼자 규칙을 발견하는 공부법"이라고 생각하면 됩니다.

    ✅ 예시로 확인

    • 고객 그룹 나누기: 쇼핑몰에서 구매 패턴이 비슷한 고객끼리 자동으로 묶기
    • 음악 장르 분류: 곡의 특성(빠르기, 악기 구성 등)으로 비슷한 노래끼리 그룹핑
    flowchart TB subgraph 지도학습["📗 지도학습 (Supervised)"] direction LR S1["데이터 + 정답표"] --> S2["패턴 학습"] --> S3["새 데이터 분류/예측"] end subgraph 비지도학습["📙 비지도학습 (Unsupervised)"] direction LR U1["데이터만\n(정답 없음)"] --> U2["스스로 그룹 발견"] --> U3["숨겨진 패턴 제시"] end style 지도학습 fill:#E8F5E9,stroke:#2E7D32,color:#000 style 비지도학습 fill:#FFF8E1,stroke:#F57F17,color:#000

    알아보기 📌

    지도학습 vs 비지도학습, 핵심 차이 한눈에!

    • 지도학습: 정답이 있다 / 선생님이 채점해 준다 / 분류·예측에 사용
    • 비지도학습: 정답이 없다 / 혼자 패턴을 찾는다 / 그룹핑·탐색에 사용

    🎮 게임으로 비유하면? - 지도학습 = 튜토리얼 모드 (가이드가 "여기를 누르세요"라고 알려줌) - 비지도학습 = 샌드박스 모드 (아무 설명 없이 직접 탐험하며 규칙 발견)

    개념 4: 학습 데이터의 중요성¶

    🎒 비유로 시작

    시험공부를 할 때, 수학 기출문제만 100개 풀고 영어 시험을 보면 어떻게 될까요? 당연히 망하겠죠! 마찬가지로, AI에게 어떤 데이터를 얼마나 주느냐가 AI의 성능을 결정합니다.

    📖 정확한 정의

    학습 데이터(Training Data)는 머신러닝 모델이 패턴을 학습하는 데 사용하는 데이터입니다. 데이터의 양, 질, 다양성이 모델의 정확도를 좌우합니다.

    ✅ 예시로 확인

    • 고양이 사진 10장으로 학습 → 정확도 낮음 😞
    • 고양이 사진 10,000장으로 학습 → 정확도 높음 😊
    • 밝은 곳의 고양이 사진만 학습 → 어두운 곳의 고양이는 못 알아봄 😱

    정리하기 📋

    좋은 학습 데이터의 3가지 조건 1. 양이 충분해야 한다 — 데이터가 많을수록 패턴을 잘 찾는다 2. 질이 좋아야 한다 — 잘못된 정답표가 붙으면 AI도 잘못 배운다 3. 다양해야 한다 — 편향된 데이터는 편향된 AI를 만든다

    🔨 따라하기: Teachable Machine으로 AI 학습시키기¶

    이제 직접 AI를 학습시켜 봅시다! 구글의 Teachable Machine은 코딩 없이 웹캠만으로 AI 모델을 만들 수 있는 무료 도구입니다.

    🎯 미션: 웹캠으로 "가위 ✌️ / 바위 ✊ / 보 🖐️" 손동작을 분류하는 AI 만들기

    Step 1: Teachable Machine 접속하기¶

    1. 웹 브라우저에서 teachablemachine.withgoogle.com 에 접속합니다
    2. 화면 중앙의 "시작하기(Get Started)" 버튼을 클릭합니다
    3. "이미지 프로젝트(Image Project)" → "표준 이미지 모델(Standard image model)" 을 선택합니다

    ⚡ 인터넷 연결만 되면 설치 없이 바로 사용할 수 있어요!

    Step 2: 클래스(분류 카테고리) 설정하기¶

    왼쪽에 기본으로 Class 1, Class 2 가 보입니다.

    1. Class 1 이름을 클릭하고 → "가위" 로 변경
    2. Class 2 이름을 클릭하고 → "바위" 로 변경
    3. 아래의 "Add a class" 버튼을 눌러 → "보" 클래스 추가

    💡 Class는 AI가 구분할 카테고리를 의미합니다. 지도학습에서 배운 레이블(정답)이 바로 이것입니다!

    Step 3: 학습 데이터 촬영하기¶

    각 클래스에 웹캠으로 이미지 데이터를 수집합니다.

    1. "가위" 클래스에서 "웹캠(Webcam)" 버튼 클릭
    2. 가위 손동작을 한 채로 "길게 눌러 촬영(Hold to Record)" 버튼을 3~5초 누르기
    3. 이때 손의 각도, 위치, 거리를 조금씩 바꿔가며 촬영 (약 50~100장)
    4. "바위", "보" 도 같은 방식으로 촬영

    생각해보기 🤔

    왜 손의 각도와 위치를 바꿔가며 촬영할까요?

    → 앞에서 배운 학습 데이터의 다양성 때문입니다! 한 각도에서만 촬영하면, 다른 각도의 손동작은 인식하지 못할 수 있어요.

    Step 4: 모델 학습시키기¶

    1. 화면 가운데의 "모델 학습시키기(Train Model)" 버튼을 클릭
    2. 약 10~30초간 학습이 진행됩니다 (화면에 진행 상태가 표시됨)
    3. 학습이 완료되면 오른쪽에 "미리보기(Preview)" 패널이 활성화됩니다

    이 짧은 시간 동안 AI는 여러분이 찍은 수백 장의 이미지에서 가위/바위/보의 패턴을 찾아낸 것입니다. 바로 머신러닝 3단계(데이터 → 패턴 발견 → 예측)가 일어난 거예요!

    Step 5: 테스트하기¶

    1. 오른쪽 미리보기 패널에서 웹캠이 자동으로 켜집니다
    2. 가위, 바위, 보 손동작을 차례로 해 보세요
    3. 각 클래스의 확률(%) 이 실시간으로 변하는 것을 관찰합니다
    🎉 결과 예시:
- 가위 손동작 → 가위: 95%, 바위: 3%, 보: 2%
- 바위 손동작 → 가위: 1%, 바위: 97%, 보: 2%

    생각해보기 🤔

    만약 정확도가 낮다면 어떻게 개선할 수 있을까요?

    1. 각 클래스의 데이터 수를 늘린다 (양)
    2. 배경이 깔끔한 곳에서 다시 촬영한다 (질)
    3. 다양한 각도로 촬영한다 (다양성)

    → 이것이 바로 학습 데이터의 중요성입니다!

    Step 6: 모델 내보내기 (선택)¶

    1. "모델 내보내기(Export Model)" 버튼 클릭
    2. 링크를 복사하면 친구에게 공유할 수도 있습니다
    3. 다운로드하면 나중에 웹사이트나 앱에서 활용할 수도 있습니다

    📝 전체 코드¶

    Teachable Machine은 코딩 없이 사용할 수 있지만, 학습시킨 모델을 Python으로 활용하는 코드도 살펴봅시다. 아래는 머신러닝의 원리를 가장 간단하게 보여주는 코드입니다.

    # 머신러닝의 3단계를 코드로 체험하기
# scikit-learn 라이브러리 사용 (설치: pip install scikit-learn)

from sklearn.tree import DecisionTreeClassifier

# ============================
# 1단계: 데이터 수집
# ============================
# 과일 데이터: [무게(g), 표면 매끄러움(0~10)]
데이터 = [
    [150, 8],   # 사과
    [160, 7],   # 사과
    [170, 9],   # 사과
    [120, 3],   # 귤
    [100, 2],   # 귤
    [110, 4],   # 귤
]

# 정답(레이블) - 지도학습이므로 정답이 있음!
정답 = ["사과", "사과", "사과", "귤", "귤", "귤"]

# ============================
# 2단계: 패턴 발견 (학습)
# ============================
모델 = DecisionTreeClassifier()     # AI 모델 생성
모델.fit(데이터, 정답)               # 데이터로 학습시키기

# ============================
# 3단계: 예측
# ============================
새로운_과일 = [[140, 7]]            # 무게 140g, 매끄러움 7
결과 = 모델.predict(새로운_과일)
print(f"이 과일은: {결과[0]}")       # 출력: 이 과일은: 사과

새로운_과일2 = [[105, 3]]           # 무게 105g, 매끄러움 3
결과2 = 모델.predict(새로운_과일2)
print(f"이 과일은: {결과2[0]}")      # 출력: 이 과일은: 귤

    💡 코드 해설

    • 데이터와 정답을 함께 제공했으므로 → 지도학습
    • 모델.fit()이 바로 패턴 발견 단계 (학습)
    • 모델.predict()가 바로 예측 단계 (새 데이터에 적용)
    • 단 6줄의 핵심 코드로 머신러닝의 전체 흐름이 담겨 있습니다!

    ⚠️ 자주 하는 실수¶

    실수 1: "데이터가 많으면 무조건 좋다"고 생각하기 ❌

    데이터의 양도 중요하지만 질이 더 중요합니다. 고양이 사진에 "강아지"라고 잘못된 레이블을 붙이면, AI도 잘못 학습합니다. 쓰레기 데이터를 넣으면 쓰레기 결과가 나옵니다(Garbage In, Garbage Out!).

    실수 2: 지도학습과 비지도학습을 "좋고 나쁨"으로 구분하기 ❌

    둘은 좋고 나쁜 것이 아니라, 목적이 다른 것입니다. 정답을 알고 있으면 지도학습, 데이터 속 숨겨진 패턴을 탐색하고 싶으면 비지도학습을 사용합니다.

    실수 3: Teachable Machine에서 한 자세로만 촬영하기 ❌

    웹캠 앞에서 완전히 같은 자세로만 50장을 찍으면, 조금만 각도가 달라져도 인식에 실패합니다. 반드시 위치, 각도, 거리를 조금씩 바꿔 촬영하세요. 이것이 데이터 다양성 확보입니다.

    실수 4: "AI가 스스로 생각한다"고 오해하기 ❌

    머신러닝은 데이터에서 통계적 패턴을 찾는 것이지, 인간처럼 "이해"하거나 "사고"하는 것이 아닙니다. AI가 고양이 사진을 맞추더라도, "귀엽다"고 느끼지는 않습니다!

    기본 문제 📗¶

    Q1. 머신러닝의 3단계를 순서대로 나열하세요.

    정답 보기

    데이터 수집 → 패턴 발견(학습) → 예측

    시험공부로 비유하면: 기출문제 풀기 → 풀이 규칙 깨닫기 → 새 문제 풀기

    Q2. 다음 중 지도학습에 해당하는 것을 모두 고르세요.

    • (가) 이메일에 "스팸/정상" 표시가 되어 있는 데이터로 학습
    • (나) 고객 구매 기록만 주고 비슷한 고객끼리 묶기
    • (다) 시험 점수와 공부 시간 데이터로 다음 시험 점수 예측
    • (라) 음악 데이터를 주고 자동으로 장르 그룹 만들기

    정답 보기

    (가), (다)

    (가)는 스팸/정상이라는 정답(레이블)이 있으므로 지도학습, (다)는 점수라는 정답이 있으므로 지도학습입니다. (나)와 (라)는 정답 없이 패턴을 찾으므로 비지도학습입니다.

    응용 문제 📙¶

    Q3. Teachable Machine에서 "교복 vs 체육복" 분류 모델을 만들었더니 정확도가 60%로 낮았습니다. 원인과 해결 방법을 각각 2가지 이상 제시하세요.

    정답 예시

    원인과 해결 방법: 1. 데이터 부족 → 각 클래스당 촬영 수를 100장 이상으로 늘린다 2. 다양성 부족 → 다양한 각도, 다양한 학생이 입은 모습을 촬영한다 3. 배경 혼란 → 배경이 일정한 장소에서 촬영한다 (AI가 옷이 아닌 배경으로 구분할 수 있음) 4. 레이블 오류 → 교복 사진이 체육복 클래스에 들어가지 않았는지 확인한다

    심화 문제 📕¶

    Q4. 어떤 회사가 고객 이탈을 예방하고 싶습니다. 다음 두 가지 접근법 중 각각 어떤 학습 유형에 해당하는지 설명하고, 어떤 상황에서 각 방법이 유리한지 서술하세요.

    • (A) 과거에 이탈한 고객과 남은 고객의 데이터(이탈 여부 표시됨)로 학습 → 새 고객의 이탈 가능성 예측
    • (B) 전체 고객 데이터에서 비슷한 행동 패턴을 보이는 그룹을 자동으로 찾기

    정답 예시

    • (A)는 지도학습: "이탈/잔류"라는 정답 레이블이 있으므로 지도학습(분류)에 해당합니다. 이미 충분한 이탈 데이터가 축적된 경우 유리합니다.
    • (B)는 비지도학습: 정답 없이 고객 패턴을 탐색하므로 비지도학습(군집화)에 해당합니다. 아직 이탈 데이터가 부족하거나, 어떤 유형의 고객이 있는지 파악이 안 된 초기 단계에서 유리합니다.
