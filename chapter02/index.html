    Chapter 02. 머신러닝 원리 탐구 — 가위바위보 분류 실습¶

    🎯 이 장에서 배우는 것¶

    • 머신러닝의 '학습 → 예측' 구조를 티처블머신 실습으로 직접 경험할 수 있다
    • 데이터의 양과 질이 AI 성능에 미치는 영향을 실험으로 확인하고 설명할 수 있다
    • 분류(Classification)의 개념을 실생활 비유와 실습 결과로 정의할 수 있다
    • 모델의 약점을 스스로 분석하고, 개선 방향을 제안할 수 있다

    💡 왜 이걸 배우나요?¶

    여러분은 가위바위보를 할 때, 상대방의 손 모양을 한눈에 구별합니다. 가위인지, 바위인지, 보인지 — 0.1초도 걸리지 않죠. 하지만 잠깐 생각해 볼까요?

    🤔 생각해보기: 컴퓨터에게 "이 사진이 가위야"라고 어떻게 알려줄 수 있을까요? 컴퓨터는 사진을 볼 때 숫자(픽셀 값)만 봅니다. 모양이나 의미를 저절로 이해하지 못합니다.

    바로 여기서 머신러닝이 등장합니다. 사람이 직접 규칙을 하나하나 프로그래밍하는 대신, 데이터를 보여주며 컴퓨터 스스로 패턴을 찾게 하는 방법입니다.

    이번 시간에는 티처블머신(Teachable Machine)이라는 도구로, 여러분이 직접 AI 모델을 만들어 봅니다. 코딩 한 줄 없이도 "학습 → 예측"의 전체 과정을 체험할 수 있습니다. 그리고 데이터를 적게 줄 때와 많이 줄 때, AI의 성능이 어떻게 달라지는지 눈으로 확인하게 됩니다.

    📚 핵심 개념¶

    개념 1: 머신러닝이란? 🧠¶

    1. 비유로 시작

    여러분이 처음 강아지와 고양이를 구별하게 된 과정을 떠올려 보세요. 부모님이 "이건 강아지야", "이건 고양이야"라고 여러 번 알려주셨고, 여러분은 점점 스스로 구별할 수 있게 되었습니다. 누군가가 "강아지는 코가 길고 꼬리를 흔들어"라고 규칙을 일일이 설명해준 게 아니라, 많은 예시를 보면서 스스로 패턴을 익힌 거죠.

    머신러닝도 똑같습니다. 컴퓨터에게 정답이 달린 데이터(예시)를 많이 보여주면, 컴퓨터가 스스로 패턴을 찾아냅니다.

    2. 정확한 정의

    머신러닝(Machine Learning): 사람이 명시적으로 규칙을 프로그래밍하지 않아도, 데이터를 통해 컴퓨터가 스스로 학습하여 판단하는 기술

    3. 예시로 확인

    • 📧 스팸 메일 필터: 수천 개의 스팸/정상 메일을 학습 → 새 메일이 스팸인지 판단
    • 🎵 음악 추천: 내가 들은 곡들을 학습 → 좋아할 만한 새 곡 예측
    • ✊ 가위바위보 분류: 손 사진 수백 장을 학습 → 새 손 사진이 무엇인지 판단

    개념 2: 분류(Classification) 📂¶

    1. 비유로 시작

    빨래를 할 때를 생각해 보세요. 세탁물을 흰옷 / 색깔옷 / 울 소재로 나누어 세탁기에 넣습니다. 이것이 바로 분류입니다 — 주어진 대상을 미리 정해진 카테고리(범주) 중 하나에 배정하는 것이죠.

    2. 정확한 정의

    분류(Classification): 입력 데이터를 미리 정의된 여러 범주(클래스) 중 하나로 구분하는 머신러닝 과제

    3. 예시로 확인

    • 손 사진 → 가위 / 바위 / 보 (3개 클래스)
    • 이메일 → 스팸 / 정상 (2개 클래스)
    • 동물 사진 → 개 / 고양이 / 토끼 (3개 클래스)

    우리가 오늘 만들 모델은 3개 클래스 분류 모델입니다.

    개념 3: 학습 → 검증 → 예측, 3단계 🍳¶

    1. 비유로 시작 — 요리 레시피

    새로운 요리를 배우는 과정에 비유하면 이해가 쉽습니다.

    flowchart LR A["🍳 <b>학습</b><br/>레시피 보고<br/>여러 번 연습"] --> B["👅 <b>검증</b><br/>맛을 보고<br/>간 조절"] B --> C["🍽️ <b>예측</b><br/>손님에게<br/>요리 제공"]
    • 학습(Training): 레시피(데이터)를 보면서 요리를 반복 연습하는 단계
    • 검증(Validation): 중간에 맛을 보며 "너무 짠가? 부족한가?" 확인하는 단계
    • 예측(Prediction): 실전에서 새로운 주문을 받고 요리를 만들어 내는 단계

    2. 정확한 정의

    • 학습: 정답이 달린 데이터를 모델에 넣어 패턴을 찾게 하는 과정
    • 검증: 학습에 사용하지 않은 데이터로 모델의 성능을 중간 점검하는 과정
    • 예측: 완전히 새로운 데이터를 넣어 결과를 얻는 과정

    3. 예시로 확인 — 오늘의 실습

    • 학습: 가위·바위·보 사진을 티처블머신에 업로드하여 모델 훈련
    • 검증: 훈련 후 나오는 정확도 수치 확인
    • 예측: 웹캠으로 새로운 손 모양을 보여주고 결과 확인

    개념 4: 데이터의 양과 질 📊¶

    1. 비유로 시작

    시험공부를 할 때, 기출문제를 3개만 풀어보는 것과 50개를 풀어보는 것, 어느 쪽이 실전에서 더 좋은 성적을 받을까요? 당연히 많이 풀어본 쪽이 유리합니다. 하지만 오답이 섞인 기출문제로 공부하면? 오히려 더 혼란스러워지겠죠.

    AI도 마찬가지입니다.

    2. 정확한 정의

    • 데이터의 양: 학습에 사용되는 데이터(사진, 텍스트 등)의 개수. 일반적으로 많을수록 성능 향상
    • 데이터의 질: 데이터에 잘못된 정답(라벨)이 없고, 다양한 상황을 포함하는 정도

    3. 예시로 확인

    오늘 실습에서 직접 비교합니다.

    • 📸 10장 학습 → 예측 정확도 측정
    • 📸 100장 학습 → 예측 정확도 측정
    • 이 차이를 통해 "데이터가 곧 AI의 실력"이라는 것을 체감하게 됩니다

    🔨 따라하기: 티처블머신으로 가위바위보 분류기 만들기¶

    📌 준비물: 웹캠이 달린 컴퓨터(또는 노트북), 인터넷 연결, 크롬 브라우저

    Step 1: 티처블머신 사이트 접속하기 🌐¶

    1. 크롬 브라우저를 열고 주소창에 입력합니다: teachablemachine.withgoogle.com
    2. 메인 화면에서 "시작하기(Get Started)" 버튼을 클릭합니다
    3. 프로젝트 유형 선택 화면에서 "이미지 프로젝트(Image Project)" 를 선택합니다
    4. "표준 이미지 모델(Standard image model)" 을 클릭합니다

    💡 알아보기: 티처블머신은 구글이 만든 무료 머신러닝 실습 도구입니다. 코딩 없이 웹브라우저만으로 이미지, 소리, 자세를 분류하는 AI 모델을 만들 수 있습니다.

    Step 2: 클래스 3개 설정하기 ✊✌️🖐️¶

    화면 왼쪽에 "Class 1", "Class 2"가 보입니다.

    1. Class 1의 이름을 클릭하여 "가위"로 변경합니다
    2. Class 2의 이름을 "바위"로 변경합니다
    3. 하단의 "Add a class(클래스 추가)" 버튼을 눌러 Class 3을 추가하고, "보"로 이름을 변경합니다
    flowchart LR subgraph 클래스 설정 A["✌️ Class 1: 가위"] B["✊ Class 2: 바위"] C["🖐️ Class 3: 보"] end

    Step 3: 데이터 수집 — 손 사진 촬영하기 📸¶

    여기서 두 번의 실험을 합니다.

    🔬 실험 A: 적은 데이터 (각 클래스 10장)

    1. "가위" 클래스에서 "웹캠(Webcam)" 버튼을 클릭합니다
    2. 가위 손 모양을 카메라 앞에 보여주세요
    3. "Hold to Record" 버튼을 길게 눌러 사진 10장을 촬영합니다
    4. 같은 방법으로 "바위" 10장, "보" 10장을 촬영합니다

    ⚡ 팁: 이 실험에서는 일부러 같은 각도, 같은 위치에서만 촬영하세요. 나중에 비교를 위해서입니다.

    🔬 실험 B: 많은 데이터 (각 클래스 100장) — Step 3 이후에 다시 진행

    실험 A의 결과를 먼저 확인한 뒤, 데이터를 추가 촬영합니다.

    Step 4: 모델 학습시키기 🏋️¶

    1. 화면 중앙의 "모델 훈련(Train Model)" 버튼을 클릭합니다
    2. 학습이 진행되는 동안 화면에 진행 상태가 표시됩니다 (보통 30초~1분 소요)
    3. 학습이 완료되면 오른쪽 "미리보기(Preview)" 패널이 자동으로 활성화됩니다
    flowchart LR D["📸 데이터 수집<br/>(사진 촬영)"] --> T["🏋️ 모델 훈련<br/>(Train 클릭)"] T --> P["🔍 미리보기<br/>(Preview)"] style T fill:#FFD700,color:#000

    🤔 생각해보기: "Train" 버튼을 누르는 순간, 컴퓨터 내부에서는 무슨 일이 벌어질까요? 여러분이 준 사진들의 공통 패턴을 수학적으로 찾아내는 과정이 진행됩니다. 마치 여러분이 시험 기출문제를 반복해서 풀며 "이런 유형은 이렇게 풀어야 하는구나"를 익히는 것과 같습니다.

    Step 5: 예측 테스트 & 정확도 비교하기 📊¶

    실험 A 결과 확인 (10장 학습)

    1. 미리보기 패널에서 웹캠 입력이 선택되어 있는지 확인합니다
    2. 카메라 앞에서 가위 손 모양을 보여줍니다 → 결과 확인
    3. 바위, 보도 차례로 보여주며 분류 결과를 기록합니다
    4. 이번에는 손의 각도를 살짝 바꾸거나, 손을 조금 멀리 두고 테스트합니다

    결과를 기록해 보세요 — 가위를 보여줬는데 "보"로 인식하는 경우가 있나요?

    실험 B 진행 (100장 학습)

    1. 다시 왼쪽 클래스로 돌아가서, 각 클래스에 사진을 90장씩 추가 촬영합니다
    2. 이번에는 다양한 각도, 다양한 거리, 손을 살짝 회전하며 촬영하세요
    3. 다시 "Train Model"을 클릭하여 재학습합니다
    4. 같은 방법으로 예측 테스트 → 결과를 기록합니다

    비교 기록

    📝 정리하기: 아래 항목을 노트에 기록하세요.

    • 실험 A (10장): 가위 정확도 %, 바위 정확도 %, 보 정확도 ___%
    • 실험 B (100장): 가위 정확도 %, 바위 정확도 %, 보 정확도 ___%
    • 두 실험의 가장 큰 차이점은 무엇인가요?
    • 왜 그런 차이가 발생했다고 생각하나요?

    Step 6: 모둠 활동 — "우리 모델의 약점 찾기" 🔍¶

    3~4명이 한 모둠을 이루어 활동합니다.

    1. 약점 탐색: 각자 만든 모델에 다양한 상황을 시도합니다
    2. 왼손으로 바꿔보기
    3. 장갑을 끼고 해보기
    4. 배경이 복잡한 곳에서 해보기

    5. 얼굴이 함께 나오게 해보기

    6. 약점 기록: 모델이 틀리는 상황을 모두 적습니다

    7. 원인 분석: "왜 이 상황에서 틀렸을까?"를 함께 토론합니다

    8. 발표: 모둠별로 발견한 약점과 원인을 30초 이내로 발표합니다

    💡 알아보기: 모델이 학습할 때 보지 못한 상황(다른 배경, 다른 손 크기 등)에서 성능이 떨어지는 현상은 AI 개발에서 매우 중요한 문제입니다. 이것을 일반화(Generalization) 문제라고 합니다. 다양한 데이터로 학습할수록 일반화 능력이 좋아집니다.

    📝 전체 코드¶

    티처블머신에서 만든 모델을 실제 코드로 활용하고 싶다면, 아래 단계를 따라하세요.

    모델 내보내기 (티처블머신에서)

    1. 학습 완료 후 "모델 내보내기(Export Model)" 클릭
    2. "Tensorflow.js" 탭 선택
    3. "모델 업로드(Upload my model)" 클릭
    4. 생성된 공유 링크를 복사합니다 (예: https://teachablemachine.withgoogle.com/models/xxxxx/)

    HTML로 실행하기

    아래 코드를 index.html 파일로 저장하고, 브라우저에서 열면 됩니다.

    <!DOCTYPE html>
<html>
<head>
    <title>가위바위보 분류기</title>
    <script src="https://cdn.jsdelivr.net/npm/@tensorflow/tfjs@latest"></script>
    <script src="https://cdn.jsdelivr.net/npm/@teachablemachine/image@latest"></script>
</head>
<body>
    <h1>✊✌️🖐️ 가위바위보 AI</h1>
    <button onclick="init()">시작하기</button>
    <div id="webcam-container"></div>
    <div id="label-container"></div>

    <script>
        // ⚠️ 아래 URL을 여러분의 모델 링크로 교체하세요!
        const URL = "https://teachablemachine.withgoogle.com/models/여러분의모델ID/";

        let model, webcam, labelContainer, maxPredictions;

        async function init() {
            const modelURL = URL + "model.json";
            const metadataURL = URL + "metadata.json";

            // 1단계: 모델 불러오기 (학습된 지식 로드)
            model = await tmImage.load(modelURL, metadataURL);
            maxPredictions = model.getTotalClasses();

            // 2단계: 웹캠 설정
            webcam = new tmImage.Webcam(300, 300, true);
            await webcam.setup();
            await webcam.play();
            document.getElementById("webcam-container").appendChild(webcam.canvas);

            // 3단계: 결과 표시 영역 준비
            labelContainer = document.getElementById("label-container");
            for (let i = 0; i < maxPredictions; i++) {
                labelContainer.appendChild(document.createElement("div"));
            }

            // 반복 예측 시작
            window.requestAnimationFrame(loop);
        }

        async function loop() {
            webcam.update();
            await predict();
            window.requestAnimationFrame(loop);
        }

        async function predict() {
            // 예측 수행: 웹캠 이미지 → 모델 → 결과
            const prediction = await model.predict(webcam.canvas);
            for (let i = 0; i < maxPredictions; i++) {
                const className = prediction[i].className;
                const probability = (prediction[i].probability * 100).toFixed(1);
                labelContainer.childNodes[i].innerHTML = 
                    `${className}: ${probability}%`;
            }
        }
    </script>
</body>
</html>

    코드 핵심 흐름: model.predict(webcam.canvas) — 이 한 줄이 바로 예측입니다. 웹캠에서 들어온 이미지를 모델에 넣으면, 각 클래스(가위/바위/보)에 대한 확률이 반환됩니다.

    ⚠️ 자주 하는 실수¶

    실수 1: 배경까지 학습시키기 🖼️

    "가위 사진을 찍었는데, 사실 모델은 손 모양이 아니라 뒤에 있는 파란 벽을 기준으로 판단하고 있었어요!"

    배경이 항상 같으면, AI는 손 모양 대신 배경을 패턴으로 인식할 수 있습니다. 촬영 시 배경을 다양하게 바꿔주세요.

    실수 2: 한 방향에서만 촬영하기 📐

    "정면에서는 잘 맞추는데, 손을 조금만 기울이면 완전히 다른 답을 내요."

    같은 각도의 사진만 100장 찍는 것보다, 여러 각도에서 50장 찍는 것이 훨씬 효과적입니다. 다양성이 핵심입니다!

    실수 3: 클래스 간 데이터 수 불균형 ⚖️

    "가위는 200장, 바위는 30장을 넣었더니 뭘 보여줘도 자꾸 '가위'라고 해요."

    모든 클래스의 데이터 수를 비슷하게 맞추세요. 한쪽이 압도적으로 많으면 AI가 그쪽으로 편향됩니다.

    실수 4: 모델 링크 복사 오류 🔗

    코드에서 모델을 불러올 때, URL 끝에 슬래시(/)가 빠지면 오류가 납니다. 복사한 링크를 그대로 붙여넣되, 끝에 /가 있는지 확인하세요.
