    제2장. 머신러닝의 비밀 — 기계는 어떻게 배울까?¶

    Part 1 | [1권] AI와 만나기 — 인공지능의 세계로 첫 발걸음 예상 소요 시간: 3시간

    🎯 이 장에서 배우는 것¶

    • 머신러닝의 학습 원리를 '분류'와 '패턴 인식'의 관점에서 설명할 수 있다
    • 규칙 기반 방식과 머신러닝 방식의 차이를 비유를 들어 설명할 수 있다
    • 학습 데이터의 양과 질이 AI 성능에 미치는 영향을 실험으로 확인할 수 있다
    • Teachable Machine을 활용하여 이미지 분류 모델을 직접 학습시킬 수 있다

    💡 왜 이걸 배우나요?¶

    여러분은 사과와 귤을 구분할 수 있나요? 당연하죠! 그런데 이걸 컴퓨터에게 설명하려면 어떻게 해야 할까요?

    "빨간색이면 사과, 주황색이면 귤!" — 정말 이렇게 간단할까요? 초록 사과는요? 아주 작은 귤은요? 규칙을 하나하나 만들다 보면 끝이 없다는 걸 느끼게 됩니다.

    머신러닝(Machine Learning)은 이 문제를 완전히 다른 방식으로 풀어냅니다. 규칙을 일일이 알려주는 대신, 수많은 사례를 보여주고 컴퓨터가 스스로 규칙을 찾게 하는 것이죠.

    이번 장에서는 이 놀라운 학습 원리를 직접 체험합니다. 글을 읽는 것만으로 끝나지 않고, 여러분이 직접 AI를 훈련시키는 시간이 될 거예요!

    📚 핵심 개념¶

    개념 1: 규칙 기반 vs. 머신러닝 — 두 가지 가르침의 방식¶

    1. 비유로 시작 🎣¶

    "물고기를 잡아주는 것 vs. 낚시하는 법을 깨닫게 하는 것"

    요리를 가르치는 두 명의 선생님이 있다고 상상해 봅시다.

    선생님 A (규칙 기반 방식) - "소금은 정확히 5g 넣어." - "불은 중불로 3분 30초 켜." - "색이 노릇해지면 뒤집어." - → 레시피에 없는 새로운 재료가 나오면? 🤷 아무것도 못 합니다.

    선생님 B (머신러닝 방식) - "여기 맛있는 요리 1,000가지가 있어. 직접 맛보면서 어떤 패턴이 있는지 찾아봐." - → 학생은 직접 경험하면서 "아, 단맛과 짠맛의 균형이 중요하구나!"를 스스로 깨닫습니다. - → 새로운 재료가 나와도 자기만의 감각으로 요리할 수 있습니다!

    2. 정확한 정의 📖¶

    • 규칙 기반 방식(Rule-Based): 사람이 모든 규칙을 하나하나 프로그래밍하는 방식입니다. if-else 문처럼 "이런 조건이면 이렇게 해라"를 직접 작성합니다.
    • 머신러닝(Machine Learning): 컴퓨터에게 대량의 데이터(사례)를 제공하고, 컴퓨터가 데이터 속에서 스스로 패턴과 규칙을 발견하게 하는 방식입니다.
    flowchart LR subgraph 규칙기반["🔧 규칙 기반 방식"] A["사람이 규칙 작성"] --> B["프로그램에 입력"] B --> C["결과 출력"] end subgraph 머신러닝["🤖 머신러닝 방식"] D["데이터 준비"] --> E["컴퓨터가 학습"] E --> F["스스로 규칙 발견"] F --> G["결과 출력"] end

    3. 예시로 확인 ✅¶

    🔍 생각해보기: 사과와 귤을 구분하는 설명서 만들기

    여러분이 외계인에게 "사과"와 "귤"을 구분하는 방법을 글로만 설명해야 한다고 상상해 보세요.

    ✏️ 종이에 직접 5가지 규칙을 적어보세요!

    (예시) - 규칙 1: 빨간색이면 사과다 - 규칙 2: 주황색이면 귤이다 - 규칙 3: …

    다 적었나요? 이제 반례를 생각해 봅시다. - 초록 사과는? 🍏 → 규칙 1에 걸리지 않습니다! - 아주 큰 귤(한라봉)은? → 크기 규칙이 맞지 않을 수 있습니다! - 사과 맛 사탕의 사진은? → 색과 모양이 비슷하면?

    👉 이것이 규칙 기반 방식의 한계입니다. 세상은 예외투성이니까요!

    개념 2: 머신러닝의 핵심 원리 — 분류와 패턴 인식¶

    1. 비유로 시작 🧩¶

    "우체부 아저씨의 편지 분류"

    상상해 보세요. 새로 온 우체부가 수천 통의 편지를 배달해야 합니다.

    • 처음에는 주소를 보고 어디로 가야 할지 모릅니다.
    • 하지만 매일 편지를 분류하다 보니, "101동은 왼쪽 길", "201동은 오른쪽 길"이라는 패턴을 발견합니다.
    • 나중에는 주소를 보자마자 자동으로 방향을 알 수 있게 됩니다!

    머신러닝도 마찬가지입니다. 수많은 데이터를 보면서 "이런 특징이 있으면 A 그룹, 저런 특징이 있으면 B 그룹"이라는 패턴을 스스로 찾아냅니다.

    2. 정확한 정의 📖¶

    • 분류(Classification): 입력된 데이터를 미리 정해진 카테고리(범주) 중 하나로 배정하는 작업입니다. 예를 들어, 사진을 보고 "고양이"인지 "강아지"인지 판단하는 것입니다.
    • 패턴 인식(Pattern Recognition): 데이터 속에 숨어 있는 규칙성이나 반복되는 특징을 찾아내는 과정입니다. 머신러닝이 학습하는 핵심 과정이 바로 이것입니다.
    • 특징(Feature): 데이터를 설명하는 개별 속성입니다. 과일이라면 색상, 크기, 모양, 무게 등이 특징입니다.

    3. 예시로 확인 ✅¶

    사과와 귤을 머신러닝으로 분류한다면, 과정은 이렇습니다.

    flowchart TD A["📸 사과 사진 500장 + 귤 사진 500장 준비"] --> B["🏷️ 각 사진에 라벨 붙이기\n사과=apple, 귤=orange"] B --> C["🤖 컴퓨터에게 학습시키기\n'이 사진들을 보고 차이를 찾아봐!'"] C --> D["💡 컴퓨터가 패턴 발견\n색상, 질감, 모양의 차이를 스스로 학습"] D --> E["🆕 새로운 과일 사진 입력"] E --> F["✅ '이건 사과입니다! 확신도 94%'"]

    📌 알아보기: 분류의 일상 속 예시

    여러분도 모르는 사이에 매일 AI 분류 기술을 사용하고 있습니다!

    • 이메일 스팸 필터: 정상 메일 vs. 스팸 메일 분류
    • 얼굴 인식 잠금해제: 주인 얼굴 vs. 다른 사람 얼굴 분류
    • 음악 추천: 좋아할 노래 vs. 좋아하지 않을 노래 분류
    • 번역기: 어떤 언어인지 분류한 후 번역

    개념 3: 학습 데이터 — AI의 교과서¶

    1. 비유로 시작 📚¶

    "시험공부와 기출문제"

    시험을 앞두고 기출문제로 공부하는 상황을 떠올려 보세요.

    • 기출문제를 5개만 풀면? → 시험 범위를 제대로 파악하기 어렵습니다.
    • 기출문제를 100개 풀면? → 출제 패턴을 꽤 잘 파악할 수 있습니다.
    • 그런데 기출문제 100개가 전부 1단원에서만 나왔다면? → 2단원 문제는 전혀 못 풉니다!

    AI도 똑같습니다. 학습 데이터는 AI의 기출문제집이에요. 양도 중요하고, 골고루 다양한 것도 중요합니다.

    2. 정확한 정의 📖¶

    • 학습 데이터(Training Data): 머신러닝 모델이 패턴을 찾기 위해 사용하는 데이터 묶음입니다. 보통 입력(사진, 텍스트 등)과 정답 라벨(이것은 고양이, 이것은 강아지)이 한 쌍으로 이루어져 있습니다.
    • 데이터의 양(Quantity): 학습 데이터가 많을수록 AI는 더 다양한 경우를 경험하여 일반적으로 성능이 올라갑니다.
    • 데이터의 질(Quality): 데이터가 정확하고, 다양하며, 편향되지 않아야 합니다. 아무리 많아도 잘못된 라벨이 붙어 있거나 한쪽으로 치우친 데이터는 AI를 오히려 망칩니다.

    3. 예시로 확인 ✅¶

    flowchart LR subgraph 적은데이터["📁 데이터 10장"] A1["사과 5장 + 귤 5장"] --> A2["정확도: 60%\n😕 자주 틀림"] end subgraph 많은데이터["📁 데이터 200장"] B1["사과 100장 + 귤 100장"] --> B2["정확도: 95%\n😊 거의 맞춤"] end subgraph 편향데이터["📁 편향된 데이터 200장"] C1["빨간 사과 190장 + 귤 10장"] --> C2["정확도: ???\n🤔 귤을 잘 모름"] end

    🔍 생각해보기: 편향된 데이터의 위험성

    만약 AI에게 "의사" 사진을 학습시킬 때, 남성 의사 사진만 1,000장 보여줬다면 어떻게 될까요?

    → 이 AI는 여성 의사 사진을 보면 "의사가 아니다"라고 판단할 수 있습니다!

    이것이 바로 데이터 편향(Data Bias)입니다. AI는 데이터에서 배우기 때문에, 데이터가 편향되면 AI의 판단도 편향됩니다. 이것은 기술 문제일 뿐 아니라 윤리적 문제이기도 합니다.

    개념 4: Teachable Machine — 누구나 만드는 AI¶

    1. 비유로 시작 🎮¶

    "레고 블록으로 집 짓기"

    전문 건축가가 아니어도 레고로 멋진 집을 만들 수 있죠? Teachable Machine은 AI 세계의 레고입니다. 복잡한 프로그래밍 없이도 웹브라우저에서 클릭 몇 번으로 AI 모델을 만들 수 있는 도구예요.

    2. 정확한 정의 📖¶

    • Teachable Machine: 구글(Google)이 만든 웹 기반 머신러닝 도구입니다. 코딩 없이 이미지, 소리, 자세(포즈)를 인식하는 AI 모델을 직접 학습시키고 테스트할 수 있습니다.
    • 접속 주소: teachablemachine.withgoogle.com

    3. 예시로 확인 ✅¶

    Teachable Machine으로 할 수 있는 것들은 다양합니다.

    • 🖐️ 가위바위보 판별기: 웹캠으로 손 모양을 찍어 가위/바위/보를 자동 분류
    • 🐶 반려동물 구분기: 우리 집 강아지와 고양이 사진을 학습시켜 분류
    • 🎵 소리 인식기: 박수 소리, 휘파람 소리를 구분하는 모델
    • 🧍 자세 판별기: 서 있는 자세와 앉아 있는 자세를 구분

    이번 실습에서는 가위바위보 이미지 분류 모델을 직접 만들어 볼 거예요!

    🔨 따라하기: 가위바위보 AI 만들기¶

    ⏰ 예상 소요 시간: 40~50분

    🛠️ 준비물: 인터넷 연결된 컴퓨터, 웹캠(노트북 내장 카메라 가능), 크롬(Chrome) 브라우저

    Step 1: Teachable Machine 접속하기¶

    1. 크롬 브라우저를 엽니다 (다른 브라우저도 가능하지만, 크롬을 권장합니다).
    2. 주소창에 teachablemachine.withgoogle.com 을 입력하고 접속합니다.
    3. 화면에서 "시작하기(Get Started)" 버튼을 클릭합니다.
    4. 세 가지 프로젝트 유형이 나타납니다.
    5. 이미지 프로젝트(Image Project) 👈 이것을 선택하세요!
    6. 오디오 프로젝트(Audio Project)
    7. 포즈 프로젝트(Pose Project)
    8. "표준 이미지 모델(Standard image model)"을 선택합니다.

    💡 팁: Teachable Machine은 완전 무료이고, 회원가입도 필요 없습니다!

    Step 2: 클래스 만들기 — 가위, 바위, 보¶

    프로젝트가 열리면 왼쪽에 "Class 1", "Class 2" 두 개의 클래스가 보입니다.

    1. Class 1의 이름을 클릭해서 "✌️가위"로 변경합니다.
    2. Class 2의 이름을 클릭해서 "✊바위"로 변경합니다.
    3. 아래쪽의 "클래스 추가(Add a class)" 버튼을 눌러 Class 3을 만들고, 이름을 "🖐️보"로 변경합니다.
    flowchart LR A["✌️ 가위\nClass 1"] ~~~ B["✊ 바위\nClass 2"] ~~~ C["🖐️ 보\nClass 3"]

    Step 3: 학습 데이터 촬영하기 — AI에게 보여줄 사진 찍기¶

    이제 각 클래스에 학습 데이터(사진)를 넣을 차례입니다!

    ✌️ 가위 클래스 1. "가위" 클래스에서 "웹캠(Webcam)" 버튼을 클릭합니다. 2. 카메라가 켜지면, 가위 모양 손 ✌️을 카메라 앞에 내밀어 주세요. 3. "길게 눌러 녹화(Hold to Record)" 버튼을 꾹 누른 채로 손 각도를 조금씩 바꿔가며 촬영합니다. 4. 최소 50장 이상 촬영하세요. (버튼을 2~3초 누르면 자동으로 여러 장 찍힙니다)

    🔑 중요! 손의 위치, 각도, 거리를 조금씩 바꾸면서 찍어야 AI가 다양한 상황을 배울 수 있습니다.

    ✊ 바위 클래스와 🖐️ 보 클래스도 같은 방법으로 각각 50장 이상 촬영합니다.

    ⚠️ 주의: 배경이 너무 복잡하면 AI가 손 모양이 아니라 배경을 학습할 수 있어요! 가능하면 단순한 배경 앞에서 촬영하세요.

    Step 4: 모델 학습시키기 — AI야, 배워봐!¶

    1. 세 클래스 모두에 사진이 들어갔는지 확인합니다.
    2. 화면 중앙의 "모델 학습시키기(Train Model)" 버튼을 클릭합니다.
    3. 잠시 기다립니다. (보통 30초~1분 소요)
    4. 학습이 완료되면 오른쪽 "미리보기(Preview)" 영역이 활성화됩니다!
    flowchart LR A["📸 데이터 수집\n가위 50장\n바위 50장\n보 50장"] --> B["🎯 모델 학습\nTrain Model 클릭\n약 30초~1분 대기"] B --> C["✅ 학습 완료!\n미리보기에서\n테스트 가능"]

    📌 알아보기: 학습 중에 무슨 일이 일어나는 걸까?

    "Train Model"을 누르는 순간, 컴퓨터 내부에서는 이런 일이 벌어집니다.

    1. 사진들의 픽셀(점) 정보를 숫자로 변환합니다
    2. 각 클래스의 사진들에서 공통된 특징을 찾습니다 (예: 가위는 두 손가락이 펴져 있다)
    3. 클래스 간의 차이점을 구별하는 기준을 만듭니다
    4. 이 기준을 반복적으로 조정하며 최적의 분류 규칙을 완성합니다

    이 모든 과정을 "학습(Training)"이라고 부릅니다!

    Step 5: 테스트하기 — 정말 잘 배웠을까?¶

    1. 오른쪽 "미리보기(Preview)" 영역에서 "웹캠"이 선택되어 있는지 확인합니다.
    2. 카메라 앞에서 가위 ✌️ 모양을 만들어 보세요.
    3. 오른쪽에 각 클래스의 확률(%) 이 실시간으로 표시됩니다!
    4. "가위" 옆에 90% 이상이 뜨면 성공!
    5. 바위 ✊, 보 🖐️도 차례대로 테스트해 봅니다.

    🎉 축하합니다! 여러분은 방금 인공지능 모델을 직접 만들고 학습시켰습니다!

    Step 6: 실험하기 — 데이터 양에 따른 정확도 변화 관찰¶

    이제 진짜 흥미로운 실험을 해봅시다! 데이터의 양이 AI 성능에 어떤 영향을 미치는지 직접 확인합니다.

    실험 방법

    1. 실험 1 — 데이터 5장만 사용하기
    2. 새 프로젝트를 시작합니다 (또는 기존 사진을 삭제합니다).
    3. 각 클래스에 사진을 딱 5장씩만 촬영합니다.
    4. "Train Model"을 눌러 학습시킵니다.

    5. 테스트하며 정확도를 관찰합니다. 얼마나 잘 맞추나요?

    6. 실험 2 — 데이터 50장 사용하기

    7. 각 클래스에 사진을 50장씩 촬영합니다.

    8. 학습시키고 테스트합니다.

    9. 실험 3 — 데이터 150장 사용하기 (시간이 되면)

    10. 각 클래스에 사진을 150장씩 촬영합니다.
    11. 학습시키고 테스트합니다.

    📋 결과 기록하기 (공책이나 메모장에 적어보세요!)

    • 실험 1 (5장): 가위 정확도 %, 바위 정확도 %, 보 정확도 ___%
    • 실험 2 (50장): 가위 정확도 %, 바위 정확도 %, 보 정확도 ___%
    • 실험 3 (150장): 가위 정확도 %, 바위 정확도 %, 보 정확도 ___%

    🔍 생각해보기

    • 데이터가 많아질수록 정확도는 어떻게 변했나요?
    • 5장일 때와 50장일 때, 가장 큰 차이를 느낀 순간은 언제였나요?
    • "데이터는 많을수록 좋다"라는 말이 와닿나요?

    Teachable Machine 모델 내보내기 (선택 사항)

    1. 학습이 완료된 후 "모델 내보내기(Export Model)" 버튼을 클릭합니다.
    2. "Tensorflow.js" 탭을 선택합니다.
    3. "모델 업로드(Upload my model)"를 클릭하면 고유한 URL이 생성됩니다.
    4. 이 URL을 사용해 웹페이지나 앱에서 모델을 불러올 수 있습니다.
