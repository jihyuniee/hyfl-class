    Chapter 18: 최종 프로젝트 — 데이터 수집과 AI 모델 구축¶

    Part 6 | [2권] 최종 프로젝트 — AI로 세상의 문제를 해결하라

    🎯 이 장에서 배우는 것¶

    • 프로젝트 목적에 맞는 데이터를 체계적으로 수집하고 스프레드시트로 정리할 수 있다
    • 수집한 데이터에 정확한 라벨을 붙여 AI가 학습할 수 있는 형태로 가공할 수 있다
    • 프로젝트 성격에 맞는 AI 도구(Teachable Machine 또는 LLM)를 선택하여 모델을 구축할 수 있다
    • 성능 테스트 프로토콜에 따라 모델을 테스트하고 반복적으로 개선할 수 있다

    💡 왜 이걸 배우나요?¶

    지난 시간에 여러분의 팀은 "AI로 해결할 문제"를 정하고, 데이터 수집 계획을 세웠습니다. 이제 진짜 손을 움직일 차례입니다.

    요리에 비유해 볼까요? 지난 시간이 레시피를 고르고 장보기 목록을 작성한 것이었다면, 오늘은 실제로 장을 보고(데이터 수집), 재료를 손질하고(데이터 정리), 요리를 시작하는(모델 구축) 시간입니다.

    아무리 훌륭한 레시피가 있어도 재료가 신선하지 않으면 맛있는 요리가 나오지 않듯, AI 프로젝트에서도 좋은 데이터가 좋은 모델의 시작입니다. 오늘 우리가 얼마나 꼼꼼하게 데이터를 모으고 정리하느냐가 최종 프로젝트의 완성도를 결정합니다.

    생각해보기 🤔

    여러분이 '교실 쓰레기 분류 AI'를 만든다고 상상해 보세요. 플라스틱 사진 10장과 종이 사진 100장만 모았다면, AI는 어떤 쪽을 더 잘 구분할까요? 왜 그럴까요?

    📚 핵심 개념¶

    개념 1: 체계적 데이터 수집¶

    🎣 비유로 시작

    데이터 수집은 낚시와 비슷합니다. 아무 곳에서나 낚싯줄을 던지면 원하는 물고기를 잡기 어렵습니다. 어떤 물고기를(어떤 데이터를), 어디서(어떤 환경에서), 얼마나(몇 개를) 잡을지 미리 정해야 효율적이죠.

    📖 정확한 정의

    체계적 데이터 수집이란, 프로젝트의 목적에 맞게 수집 대상, 수집 방법, 수집 수량, 수집 환경을 미리 정하고, 계획에 따라 일관된 방식으로 데이터를 모으는 과정입니다.

    ✅ 예시로 확인

    "학교 주변 식물 분류 AI"를 만든다면 다음과 같이 계획합니다.

    • 수집 대상: 은행나무, 단풍나무, 소나무 잎사귀 이미지
    • 수집 방법: 스마트폰 카메라로 촬영 (배경은 흰 종이)
    • 수집 수량: 각 종류당 최소 30장, 총 90장 이상
    • 수집 환경: 맑은 날 오전 / 흐린 날 오후 각각 촬영 (다양한 조건 확보)

    알아보기 📌

    데이터 수집의 3가지 원칙

    1. 균형(Balance): 각 분류 항목의 데이터 수가 비슷해야 합니다. 한쪽이 너무 많으면 AI가 편향됩니다.
    2. 다양성(Diversity): 같은 대상이라도 각도, 조명, 크기를 다르게 촬영해야 실제 환경에서 잘 작동합니다.
    3. 일관성(Consistency): 촬영 방법, 파일 이름 규칙 등을 통일해야 나중에 정리가 쉽습니다.

    개념 2: 데이터 라벨링과 정리¶

    🏷️ 비유로 시작

    도서관의 책을 상상해 보세요. 책이 아무리 많아도 분류 번호가 없으면 원하는 책을 찾을 수 없습니다. 라벨링은 데이터에 "이것은 ○○이다"라는 이름표를 붙이는 작업으로, AI에게 정답지를 만들어 주는 것과 같습니다.

    📖 정확한 정의

    데이터 라벨링(Data Labeling)이란, 수집한 데이터 각각에 해당 데이터가 무엇인지 설명하는 태그(분류명, 설명 등)를 부여하는 작업입니다. 라벨링된 데이터가 있어야 AI가 "이 입력에는 이 정답이 연결된다"는 것을 학습할 수 있습니다.

    ✅ 예시로 확인

    스프레드시트로 정리한 데이터 라벨링의 예시를 살펴보겠습니다.

    • 파일명: ginkgo_001.jpg → 라벨: 은행나무 → 촬영일: 2025-01-15 → 메모: 맑은 날, 정면 촬영
    • 파일명: maple_001.jpg → 라벨: 단풍나무 → 촬영일: 2025-01-15 → 메모: 흐린 날, 측면 촬영
    • 파일명: pine_001.jpg → 라벨: 소나무 → 촬영일: 2025-01-16 → 메모: 맑은 날, 클로즈업
    flowchart LR A["📷 원본 데이터\n(사진, 텍스트)"] --> B["🏷️ 라벨링\n(분류명 부여)"] B --> C["📊 스프레드시트 정리\n(파일명, 라벨, 메모)"] C --> D["✅ 검수\n(오류 확인)"] D --> E["🤖 AI 학습 준비 완료"]

    알아보기 📌

    파일 이름 규칙의 중요성

    사진1.jpg, IMG_20250115.jpg 같은 이름은 나중에 무엇인지 알 수 없습니다. 대신 [분류명]_[번호].jpg 형식(예: ginkgo_001.jpg)을 사용하면 파일명만 봐도 어떤 데이터인지 바로 알 수 있습니다.

    개념 3: 프로젝트에 맞는 AI 도구 선택¶

    🧰 비유로 시작

    못을 박을 때는 망치를, 나사를 조일 때는 드라이버를 사용합니다. AI 도구도 마찬가지입니다. 해결하려는 문제의 유형에 따라 적합한 도구가 다릅니다.

    📖 정확한 정의

    AI 도구 선택이란, 프로젝트에서 다루는 데이터의 종류(이미지, 텍스트, 소리 등)와 해결하려는 과제(분류, 생성, 분석 등)를 기준으로, 가장 효과적인 AI 플랫폼이나 모델을 결정하는 과정입니다.

    ✅ 예시로 확인

    flowchart TD Q["🤔 우리 프로젝트의 \n핵심 데이터는?"] Q --> IMG["📷 이미지 데이터"] Q --> TXT["📝 텍스트 데이터"] Q --> MIX["📷+📝 이미지+텍스트 혼합"] IMG --> TM["✅ Teachable Machine\n이미지 분류 모델 구축"] TXT --> LLM["✅ LLM 프롬프트 설계\nChatGPT, Gemini 등"] MIX --> BOTH["✅ 두 도구 조합\n각각의 강점 활용"]
    • 이미지 분류 프로젝트 (예: 쓰레기 분류, 식물 구별) → Teachable Machine : 웹에서 드래그 앤 드롭으로 이미지를 넣고, 클릭 몇 번이면 분류 모델이 완성됩니다.
    • 텍스트 처리 프로젝트 (예: 감정 분석, 문서 요약) → LLM 프롬프트 엔지니어링 : ChatGPT나 Gemini에게 역할과 규칙을 정해주는 프롬프트를 설계합니다.
    • 혼합 프로젝트 (예: 이미지를 찍으면 설명을 생성) → 두 도구 조합 : Teachable Machine으로 분류하고, 그 결과를 LLM에 전달하여 설명을 생성합니다.

    개념 4: 성능 테스트와 반복 개선¶

    🔬 비유로 시작

    새 자전거를 조립했다고 바로 장거리를 달리지는 않죠. 먼저 브레이크가 잘 작동하는지, 체인이 빠지지 않는지 테스트합니다. 문제가 발견되면 조정하고 다시 테스트하죠. AI 모델도 똑같습니다. 만들고 → 테스트하고 → 고치고 → 다시 테스트하는 반복 과정이 핵심입니다.

    📖 정확한 정의

    성능 테스트 프로토콜이란, AI 모델이 다양한 조건에서 얼마나 정확하게 작동하는지를 체계적으로 확인하는 절차입니다. 정확도(Accuracy)를 측정하고, 실패하는 케이스를 분석하여 모델을 개선합니다.

    ✅ 예시로 확인

    식물 분류 AI를 테스트한다면 이렇게 진행합니다.

    • 테스트 조건 1 — 학습에 사용하지 않은 새 사진 10장 → 정확도: 8/10 (80%)
    • 테스트 조건 2 — 어두운 환경에서 촬영한 사진 10장 → 정확도: 5/10 (50%)
    • 테스트 조건 3 — 잎이 부분만 보이는 사진 10장 → 정확도: 6/10 (60%)

    분석 결과: 어두운 환경 사진에서 정확도가 낮음 → 개선 방향: 어두운 환경에서 찍은 학습 데이터를 추가 수집

    flowchart LR A["🏗️ 모델 구축"] --> B["🧪 테스트"] B --> C["📊 결과 분석"] C --> D{"정확도 \n충분한가?"} D -- "아니오 ❌" --> E["🔧 개선\n(데이터 추가,\n프롬프트 수정 등)"] E --> A D -- "예 ✅" --> F["🎉 모델 완성"]

    알아보기 📌

    정확도 계산하기

    정확도(%) = (맞힌 개수 ÷ 전체 테스트 개수) × 100

    예) 20개 중 16개를 맞혔다면 → (16 ÷ 20) × 100 = 80%

    🔨 따라하기¶

    이제 실제로 데이터를 수집하고 AI 모델을 구축해 봅시다! 여기서는 "교실 쓰레기 분류 AI" 프로젝트를 예시로 진행합니다. 여러분의 팀 프로젝트에 맞게 응용하세요.

    Step 1: 데이터 수집하기 📷¶

    먼저 분류할 대상을 정하고 사진을 촬영합니다.

    우리 프로젝트의 분류 항목 (예시)

    • 플라스틱: 페트병, 플라스틱 컵, 비닐봉지
    • 종이: 노트 종이, 우유팩, 과자 상자
    • 일반 쓰레기: 과자 봉지, 휴지, 나무젓가락

    촬영 규칙

    1. 스마트폰 카메라를 사용한다
    2. 대상을 책상 위 흰 종이 배경에 놓고 촬영한다
    3. 한 대상당 정면, 측면, 위에서 3가지 각도로 찍는다
    4. 각 분류 항목당 최소 30장 (3가지 물건 × 3각도 × 약 3~4번 반복)
    5. 파일명은 plastic_001.jpg, paper_001.jpg, general_001.jpg 형식으로 저장한다

    생각해보기 🤔

    왜 3가지 각도로 찍어야 할까요? 만약 정면 사진으로만 학습시킨 AI에게 위에서 찍은 사진을 보여주면 어떤 일이 벌어질까요?

    Step 2: 스프레드시트로 데이터 정리하기 📊¶

    수집한 데이터를 구글 스프레드시트(또는 엑셀)에 기록합니다.

    스프레드시트 구성 항목

    • B열 — 라벨(분류): 플라스틱

    5. 학습 시작: "모델 학습시키기" 버튼을 클릭합니다 (약 1~3분 소요)
    6. 즉시 테스트: 학습이 끝나면 웹캠 또는 새 이미지를 넣어 테스트합니다

    Step 5: 성능 테스트 프로토콜 실행하기 🧪¶

    모델이 완성되었다면, 아래 프로토콜에 따라 체계적으로 테스트합니다.

    ```python

    성능 테스트 프로토콜 — 결과 기록 및 정확도 계산¶

    def run_performance_test(test_name, test_results): """ 성능 테스트 결과를 분석하는 함수

    test_results: [("입력 설명", "예상 라벨", "AI 예측", 맞음여부), ...]
"""
total = len(test_results)
correct = sum(1 for r in test_results if r[3])
accuracy = (correct / total) * 100

print(f"\n🧪 테스트: {test_name}")
print(f"   전체: {total}개 | 정답: {correct}개 | 오답: {total - correct}개")
print(f"   📈 정확도: {accuracy:.1f}%")

# 오답 분석
wrong_cases = [r for r in test_results if not r[3]]
if wrong_cases:
    print(f"\n   ❌ 틀린 케이스 분석:")
    for case in wrong_cases:
        print(f"      - '{case[0]}' → 예상: {case[1]}, AI 예측: {case[2]}")

return accuracy
