    Chapter 18. 설득력 있는 발표 만들기 – AI와 함께하는 프레젠테이션¶

    Part 6. 발표와 성찰 – 우리의 이야기를 세상에

    🎯 이 장에서 배우는 것¶

    • 프로젝트의 문제-해결-결과를 3막 구조로 논리적으로 구성할 수 있다
    • AI를 활용하여 발표 스크립트 초안과 슬라이드 구성안을 효율적으로 만들 수 있다
    • 청중을 고려한 핵심 메시지를 직접 설계할 수 있다
    • 리허설과 피드백을 통해 발표 전달력을 높일 수 있다

    💡 왜 이걸 배우나요?¶

    여러분은 지금까지 데이터를 분석하고, AI와 함께 문제를 해결하는 멋진 프로젝트를 완성했습니다. 하지만 아무리 훌륭한 결과물도 제대로 전달하지 못하면 아무도 모릅니다.

    영화를 떠올려 보세요. 같은 이야기라도 예고편이 지루하면 극장에 가고 싶지 않죠? 반대로, 2분짜리 예고편 하나가 심장을 두근거리게 만들기도 합니다. 발표는 여러분 프로젝트의 예고편입니다.

    🎬 생각해보기: '좋은 발표'와 '지루한 발표'의 차이를 3가지 키워드로 정리해 보세요.

    힌트: 구조, 시각자료, 전달력 같은 단어를 떠올려 보세요.

    이 장에서는 AI의 도움을 받아 발표 자료를 빠르게 만들면서도, 핵심 메시지와 감정은 직접 담는 방법을 배웁니다.

    📚 핵심 개념¶

    개념 1: 발표의 3막 구조 🎭¶

    1) 비유로 시작

    영화에는 언제나 3막이 있습니다.

    • 1막 — 갈등(문제 상황): 주인공이 위기에 처합니다 → "우리 학교 급식 잔반이 매일 50kg이나 나와요!"
    • 2막 — 여정(해결 과정): 주인공이 문제를 풀어가는 과정 → "데이터를 분석하고, AI로 원인을 찾았어요"
    • 3막 — 클라이맥스(결과와 제안): 결말과 교훈 → "이 방법으로 잔반을 30% 줄일 수 있습니다!"
    graph LR A["🔥 1막: 문제 상황<br/>청중의 관심 끌기"] --> B["🔍 2막: 해결 과정<br/>분석·실험·발견"] B --> C["🎉 3막: 결과와 제안<br/>핵심 메시지 전달"]

    2) 정확한 정의

    발표의 3막 구조란, 청중이 자연스럽게 이야기에 빠져들도록 "왜 이 문제가 중요한지(문제) → 어떻게 풀었는지(과정) → 무엇을 발견했는지(결과)" 순서로 내용을 배치하는 발표 설계 방법입니다.

    3) 예시로 확인

    📌 좋은 예: "매일 버려지는 급식 50kg, 1년이면 우리 교실 3개 분량입니다(문제) → 2주간 메뉴별 잔반량을 기록하고 AI로 패턴을 분석했습니다(과정) → 인기 없는 메뉴 3개를 대체하면 잔반이 30% 줄어듭니다(결과)"

    📌 나쁜 예: "저희 조는 급식에 대해 조사했습니다. 데이터를 수집했고, 분석했고, 결과가 나왔습니다." ← 무엇이 문제인지, 왜 중요한지 전혀 느껴지지 않습니다.

    개념 2: 핵심 메시지 — "한 문장의 힘" 💎¶

    1) 비유로 시작

    SNS 프로필을 생각해 보세요. 긴 자기소개를 아무도 읽지 않지만, 한 줄 소개는 누구나 읽습니다. 발표의 핵심 메시지도 마찬가지입니다.

    2) 정확한 정의

    핵심 메시지(Key Message)란, 발표가 끝난 뒤 청중이 딱 하나만 기억한다면 그것이어야 할 문장입니다. 모든 슬라이드와 스크립트는 이 한 문장을 뒷받침하기 위해 존재합니다.

    3) 예시로 확인

    • ✅ "메뉴 선택권을 주면 잔반이 30% 줄어듭니다"
    • ❌ "급식 잔반에 대해 여러 가지를 알아봤습니다" ← 기억에 남는 것이 없음

    🧠 알아보기: 핵심 메시지를 만드는 공식

    [대상]은 [행동]하면 [결과]를 얻을 수 있다.

    예: "우리 학교는(대상) 비인기 메뉴 3개를 교체하면(행동) 연간 잔반을 5톤 줄일 수 있다(결과)."

    개념 3: AI 활용 발표 자료 제작 🤖¶

    1) 비유로 시작

    AI는 발표를 대신 해주는 로봇이 아닙니다. 요리로 비유하면, AI는 재료를 손질해주는 보조 셰프입니다. 레시피(핵심 메시지)를 정하고 맛(감정)을 결정하는 것은 여러분의 몫입니다.

    2) 정확한 정의

    AI 활용 발표 자료 제작이란, 프롬프트(명확한 지시)를 통해 AI에게 스크립트 초안, 슬라이드 구성안, 발표 피드백을 요청하되, 핵심 메시지·감정·개인 경험은 발표자가 직접 채워 넣는 협업 방식입니다.

    3) 예시로 확인 — AI에게 맡기는 것 vs 직접 하는 것

    graph TB subgraph AI["🤖 AI에게 맡기기"] A1["스크립트 초안 작성"] A2["슬라이드 구성안 제안"] A3["문장 다듬기·요약"] A4["발표 연습 피드백"] end subgraph ME["👤 내가 직접 하기"] B1["핵심 메시지 결정"] B2["개인 경험·감정 추가"] B3["데이터 해석·의견"] B4["최종 발표 연습"] end

    개념 4: 리허설과 피드백 — 발표는 근육이다 💪¶

    1) 비유로 시작

    운동선수가 경기 전에 연습하듯, 발표도 리허설(rehearsal) 없이 본무대에 서면 안 됩니다. 리허설은 "발표 근육"을 키우는 훈련입니다.

    2) 정확한 정의

    리허설이란, 실제 발표와 동일한 조건(시간 제한, 슬라이드 사용)에서 미리 발표를 연습하고, 동료로부터 구체적인 피드백을 받아 개선하는 과정입니다.

    3) 예시로 확인 — 좋은 피드백 vs 나쁜 피드백

    • ✅ "2번 슬라이드에서 그래프 설명이 30초나 걸려서 흐름이 끊겼어. 핵심 수치만 말하면 어떨까?"
    • ❌ "잘한 것 같아." ← 구체적이지 않아서 개선할 수 없음

    📌 피드백의 3가지 기준 - 구조: 3막 흐름이 자연스러운가? - 시각자료: 슬라이드가 메시지를 돕고 있는가? - 전달력: 목소리·시선·속도가 적절한가?

    🔨 따라하기: AI와 함께 발표 자료 만들기¶

    ⏱️ 실습 예상 시간: 약 50분

    Step 1: 핵심 메시지 정하기 (10분)¶

    여러분의 프로젝트를 한 문장으로 요약하세요.

    [핵심 메시지 공식]
____은(는) ____하면 ____할 수 있다.

[예시]
우리 학교는 비인기 급식 메뉴 3개를 교체하면 연간 잔반을 5톤 줄일 수 있다.

    ✏️ 직접 써보세요:

    우리 프로젝트의 핵심 메시지: _________

    Step 2: 3막 구조 설계하기 (10분)¶

    핵심 메시지를 중심으로 3막을 채워봅니다.

    [1막 - 문제] 어떤 문제가 있는가? 왜 중요한가?
→ 

[2막 - 과정] 어떻게 해결했는가? 어떤 데이터/AI를 사용했는가?
→ 

[3막 - 결과] 무엇을 발견했는가? 어떤 제안을 하는가?
→

    Step 3: AI에게 스크립트 초안 요청하기 (10분)¶

    아래 프롬프트를 AI(ChatGPT 등)에 입력해보세요. 빈칸을 여러분의 내용으로 채우세요.

    prompt = """
당신은 중학생 발표 코치입니다.
아래 정보로 5분 발표 스크립트 초안을 작성해주세요.

[핵심 메시지]: {여러분의 핵심 메시지}
[1막 - 문제]: {문제 상황 요약}
[2막 - 과정]: {해결 과정 요약}
[3막 - 결과]: {결과와 제안 요약}

조건:
- 청중은 같은 반 친구들과 선생님입니다
- 쉬운 말로 작성해주세요
- 각 막의 시작 부분에 [1막], [2막], [3막]을 표시해주세요
- 총 5분 분량(약 800~1000자)으로 작성해주세요
"""

# ChatGPT, Claude 등 AI 도구에 이 프롬프트를 붙여넣기하세요
print(prompt)

    ⚠️ 중요: AI가 만들어준 초안을 그대로 사용하지 마세요! 다음 단계에서 여러분만의 경험과 감정을 넣어야 합니다.

    Step 4: 내 이야기로 다듬기 (10분)¶

    AI 초안을 받았으면, 아래 체크리스트로 수정하세요.

    # 발표 스크립트 점검 코드
checklist = {
    "핵심 메시지가 명확히 들어있는가?": False,
    "내가 직접 경험한 에피소드가 1개 이상 있는가?": False,
    "숫자/데이터가 구체적으로 제시되어 있는가?": False,
    "AI가 쓴 딱딱한 문장을 내 말투로 바꿨는가?": False,
    "마지막에 청중에게 행동을 요청하는가?": False
}

print("📋 스크립트 점검 체크리스트")
print("=" * 40)
for item, checked in checklist.items():
    status = "✅" if checked else "⬜"
    print(f"  {status} {item}")

print("\n모든 항목에 ✅가 되면 다음 단계로!")

    💡 팁: "데이터를 수집한 날, 비가 와서 설문조사를 온라인으로 바꿨던 경험"처럼 소소한 에피소드가 발표를 생생하게 만듭니다.

    Step 5: 슬라이드 구성안 만들기 (5분)¶

    # 리허설 타이머 & 피드백 기록 도구
import time

def rehearsal_timer(minutes=5):
    """발표 리허설 타이머"""
    print(f"🎤 {minutes}분 리허설을 시작합니다!")
    print(f"⏱️ 제한 시간: {minutes}분")
    print("-" * 30)
    print("준비되면 Enter를 누르세요...")
    input()

    start = time.time()
    print("🔴 발표 시작! (끝나면 Enter)")
    input()

    elapsed = time.time() - start
    minutes_used = int(elapsed // 60)
    seconds_used = int(elapsed % 60)

    print(f"\n⏱️ 소요 시간: {minutes_used}분 {seconds_used}초")

    if elapsed > minutes * 60:
        print("⚠️ 시간을 초과했어요! 내용을 줄여보세요.")
    elif elapsed < (minutes - 1) * 60:
        print("💡 시간이 많이 남았어요! 에피소드를 추가해보세요.")
    else:
        print("✅ 완벽한 시간 배분이에요!")

def collect_feedback():
    """동료 피드백 수집"""
    print("\n📝 피드백 카드")
    print("=" * 30)

    categories = {
        "구조": "3막 흐름이 자연스러웠나요? (1~5점)",
        "시각자료": "슬라이드가 이해에 도움이 됐나요? (1~5점)",
        "전달력": "목소리와 속도가 적절했나요? (1~5점)"
    }

    scores = {}
    for cat, question in categories.items():
        print(f"\n{cat}: {question}")
        score = input("점수 입력: ")
        scores[cat] = score

    print("\n💬 가장 좋았던 점:")
    good = input("> ")
    print("💬 하나만 개선한다면:")
    improve = input("> ")

    print("\n" + "=" * 30)
    print("📊 피드백 요약")
    for cat, score in scores.items():
        print(f"  {cat}: {'⭐' * int(score)}")
    print(f"  👍 좋았던 점: {good}")
    print(f"  🔧 개선할 점: {improve}")

    """
설득력 있는 발표 만들기 - AI 협업 발표 준비 도구
Chapter 18: 프레젠테이션 준비 워크시트

def create_presentation_worksheet():
    """발표 준비 워크시트 생성"""

    print("🎤 AI 협업 발표 준비 워크시트")
    print("=" * 50)

    # Step 1: 핵심 메시지
    print("\n📌 Step 1: 핵심 메시지 정하기")
    print("공식: [대상]은 [행동]하면 [결과]할 수 있다")
    subject = input("  대상: ")
    action = input("  행동: ")
    result = input("  결과: ")
    key_message = f"{subject}은(는) {action}하면 {result}할 수 있다."
    print(f"\n  ✅ 핵심 메시지: {key_message}")

    # Step 2: 3막 구조
    print("\n📌 Step 2: 3막 구조 설계")
    act1 = input("  1막(문제): ")
    act2 = input("  2막(과정): ")
    act3 = input("  3막(결과): ")

    # Step 3: AI 프롬프트 생성
    print("\n📌 Step 3: AI 프롬프트 자동 생성")
    prompt = f"""당신은 중학생 발표 코치입니다.
아래 정보로 5분 발표 스크립트 초안을 작성해주세요.

[핵심 메시지]: {key_message}
[1막 - 문제]: {act1}
[2막 - 과정]: {act2}
[3막 - 결과]: {act3}

조건:
- 청중은 같은 반 친구들과 선생님
- 쉬운 말로 작성
- 각 막 시작에 [1막], [2막], [3막] 표시
- 총 5분 분량(800~1000자)
"""

    print("\n  ⬇️ 아래 프롬프트를 AI에 붙여넣기하세요 ⬇️")
    print("-" * 50)
    print(prompt)
    print("-" * 50)

    # Step 4: 점검 체크리스트
    print("\n📌 Step 4: 스크립트 점검 체크리스트")
    checks = [
        "핵심 메시지가 명확히 들어있는가?",
        "내가 직접 경험한 에피소드가 있는가?",
        "숫자/데이터가 구체적인가?",
        "AI 문장을 내 말투로 바꿨는가?",
        "청중에게 행동을 요청하는가?"
    ]

    for i, check in enumerate(checks, 1):
        answer = input(f"  {i}. {check} (y/n): ")
        status = "✅" if answer.lower() == 'y' else "⬜"
        print(f"     {status}")

    print("\n🎉 발표 준비 워크시트 완성!")
    print(f"💎 기억하세요, 핵심 메시지: {key_message}")

# 실행
create_presentation_worksheet()

    ⚠️ 자주 하는 실수¶

    실수 1: AI 스크립트를 그대로 읽는다 🤖¶

    AI가 만든 문장은 깔끔하지만 여러분의 목소리가 아닙니다. "분석 결과에 따르면"을 "저희가 직접 조사해보니까"로 바꾸세요. 청중은 교과서가 아니라 사람의 이야기를 듣고 싶어합니다.

    실수 2: 슬라이드에 글자를 가득 채운다 📄¶

    슬라이드는 발표자를 도와주는 배경화면이지, 읽어야 할 문서가 아닙니다. 한 슬라이드에 문장이 5줄 이상이면 청중은 슬라이드를 읽느라 여러분의 말을 듣지 못합니다.

    graph LR A["❌ 나쁜 슬라이드<br/>글자 빼곡<br/>문장 7줄<br/>그래프 없음"] --> B["✅ 좋은 슬라이드<br/>키워드 3개<br/>큰 그래프 1개<br/>여백 충분"]

    실수 3: 리허설 없이 본무대에 선다 😰¶

    "머릿속으로 해봤어요"는 리허설이 아닙니다. 소리 내어, 시간을 재며, 슬라이드를 넘기면서 연습해야 합니다. 최소 2번은 리허설하세요. 첫 번째는 흐름 확인, 두 번째는 시간 맞추기입니다.

    실수 4: 피드백을 감정적으로 받아들인다 😢¶

    "네 발표 2번 슬라이드가 이해가 안 됐어"는 공격이 아니라 선물입니다. 피드백의 목적은 발표를 더 좋게 만드는 것이지, 사람을 평가하는 것이 아닙니다.
