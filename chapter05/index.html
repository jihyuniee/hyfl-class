    미니프로젝트 2차시 — 데이터 수집과 모델 학습¶

    Part 2 | [1권] 미니프로젝트 — 우리만의 AI 분류기 만들기

    🎯 이 장에서 배우는 것¶

    • 기획에 따라 목적에 맞는 이미지 데이터를 체계적으로 수집할 수 있다
    • 수집한 이미지를 클래스별로 분류·정리하여 학습 데이터셋을 구성할 수 있다
    • Teachable Machine으로 분류 모델을 학습시키고 정확도를 확인할 수 있다
    • 모델의 오류 원인을 분석하고 데이터 보완 전략을 수립할 수 있다

    💡 왜 이걸 배우나요?¶

    지난 시간에 우리는 "어떤 것을 분류할까?"를 기획했습니다. 멋진 설계도가 있어도 재료가 없으면 집을 지을 수 없듯이, AI 모델도 데이터라는 재료 없이는 아무것도 할 수 없습니다.

    오늘은 드디어 직접 카메라를 들고 교실, 복도, 운동장으로 나가 이미지를 촬영합니다. 그리고 그 이미지들로 AI 모델을 학습시킨 뒤, "과연 우리 모델이 잘 작동하는가?"를 팀끼리 서로 테스트해봅니다.

    🔍 생각해보기: 요리사가 좋은 레시피를 가지고 있어도, 재료의 품질이 나쁘면 맛있는 음식을 만들 수 없겠죠? AI도 마찬가지입니다. 좋은 데이터 = 좋은 모델입니다. 오늘은 "좋은 재료(데이터)를 모으고, 실제로 요리(학습)하는 날"입니다! 🍳

    📚 핵심 개념¶

    개념 1: 데이터 수집 전략 📷¶

    ① 비유로 시작

    시험 공부를 한다고 상상해보세요. 수학 시험을 준비하는데 한 유형의 문제만 100번 풀면 어떻게 될까요? 그 유형은 잘 풀겠지만, 조금만 다른 형태로 나오면 당황하게 됩니다. 다양한 유형의 문제를 골고루 풀어야 어떤 문제가 나와도 풀 수 있죠.

    AI의 데이터 수집도 똑같습니다. 다양한 조건에서 촬영한 이미지를 골고루 모아야 어떤 상황에서도 잘 분류하는 모델을 만들 수 있습니다.

    ② 정확한 정의

    데이터 수집 전략이란, 모델이 다양한 상황에서도 정확하게 분류할 수 있도록 촬영 조건(각도, 조명, 배경, 거리 등)을 의도적으로 변화시키며 체계적으로 이미지를 모으는 계획입니다.

    ③ 예시로 확인

    "교실 물건 분류기"를 만든다고 해봅시다. '가위'를 촬영할 때:

    • 각도 변화: 위에서, 옆에서, 비스듬히
    • 배경 변화: 책상 위, 손에 든 상태, 필통 옆
    • 조명 변화: 창가의 밝은 곳, 교실 안쪽의 어두운 곳
    • 거리 변화: 가까이서 크게, 멀리서 작게

    📌 알아보기: 한 클래스당 최소 30~50장 이상의 이미지를 수집하는 것이 좋습니다. 양이 너무 적으면 모델이 "외워버려서" 새로운 이미지에 대응하지 못합니다. 이것을 과적합(overfitting)이라고 합니다.

    flowchart LR A["📋 수집 계획 수립"] --> B["📷 다양한 조건으로 촬영"] B --> C["📁 클래스별 분류·정리"] C --> D["✅ 품질 검수"] D -->|부족하면| B D -->|충분하면| E["🤖 모델 학습 투입"]

    개념 2: 데이터셋 구성과 품질 관리 🗂️¶

    ① 비유로 시작

    도서관을 떠올려보세요. 수천 권의 책이 아무렇게나 쌓여 있으면 원하는 책을 찾을 수 있을까요? 도서관은 분류 기호에 따라 책을 정리하기 때문에 누구나 원하는 책을 빠르게 찾을 수 있습니다. AI에게 제공하는 데이터도 깔끔하게 분류·정리된 상태여야 합니다.

    ② 정확한 정의

    데이터셋(Dataset)이란, AI 모델을 학습시키기 위해 목적에 맞게 수집·분류·정리된 데이터의 모음입니다. 좋은 데이터셋은 다음 조건을 갖춥니다:

    • 균형(Balance): 각 클래스의 데이터 수가 비슷해야 합니다
    • 다양성(Diversity): 다양한 조건의 데이터가 포함되어야 합니다
    • 정확성(Accuracy): 잘못 분류된 데이터가 없어야 합니다

    ③ 예시로 확인

    "재활용 쓰레기 분류기"의 데이터셋을 구성한다면:

    • 균형 문제: 플라스틱 100장, 캔 20장, 종이 80장 → 캔 데이터가 너무 적어서 캔을 잘 인식하지 못할 수 있습니다
    • 다양성 문제: 플라스틱 병만 촬영 → 플라스틱 컵, 비닐은 인식 실패
    • 정확성 문제: 플라스틱 폴더에 캔 사진이 섞여 있으면 → 모델이 혼란

    ⚠️ 주의: 클래스 간 데이터 수 차이가 2배 이상 나지 않도록 하세요. 예를 들어 한 클래스가 50장이면 다른 클래스도 최소 25장 이상은 되어야 합니다.

    개념 3: 모델 학습과 정확도 📊¶

    ① 비유로 시작

    선생님이 학생에게 영어 단어를 가르치는 상황을 떠올려보세요. 단어장을 보여주면서 "apple은 사과, banana는 바나나..."라고 반복합니다. 처음에는 틀리지만, 반복할수록 점점 정확하게 답합니다. 그리고 단어 시험을 보면 몇 점을 맞았는지 확인하죠.

    AI 학습도 이와 같습니다. 데이터를 반복적으로 보여주면서 패턴을 익히게 하고, 마지막에 정확도(몇 %를 맞혔는지)를 확인합니다.

    ② 정확한 정의

    모델 학습(Training)이란 AI에게 분류된 데이터를 반복적으로 보여주어 각 클래스의 특징(패턴)을 스스로 찾아내게 하는 과정입니다. 학습이 끝나면 새로운 데이터에 대해 어떤 클래스에 속하는지 예측할 수 있게 됩니다.

    정확도(Accuracy)란 모델이 전체 테스트 중 올바르게 분류한 비율을 뜻합니다.

    예: 10장의 이미지를 테스트했을 때 8장을 맞혔다면 → 정확도 80%

    ③ 예시로 확인

    Teachable Machine에서 학습이 완료되면, 웹캠으로 실시간 테스트를 할 수 있습니다. 가위를 카메라에 비추면 화면에 각 클래스별 확률(confidence)이 막대 그래프로 표시됩니다:

    • 가위: 92% ← 가장 높으므로 "가위"로 분류
    • 풀: 5%
    • 테이프: 3%
    flowchart TD A["📷 데이터 입력<br/>(이미지들)"] --> B["🧠 학습 과정<br/>(패턴 발견)"] B --> C["📈 학습 완료<br/>(모델 생성)"] C --> D["🔍 테스트<br/>(새 이미지 분류)"] D --> E{"정확도 확인"} E -->|"90% 이상"| F["✅ 성공!"] E -->|"90% 미만"| G["🔄 데이터 보완 후<br/>재학습"] G --> A

    개념 4: 오류 분석과 데이터 보완 🔍¶

    ① 비유로 시작

    농구 경기를 마친 후 코치가 영상을 돌려보며 "이 장면에서 패스 실수가 있었고, 저 장면에서 수비가 늦었다"고 분석하는 것처럼, AI 모델도 틀린 부분을 하나하나 살펴보며 왜 틀렸는지 원인을 찾아야 합니다. 원인을 알아야 개선할 수 있으니까요.

    ② 정확한 정의

    오류 분석(Error Analysis)이란, 모델이 잘못 분류한 사례를 수집하여 오류의 패턴과 원인을 파악하고, 이를 바탕으로 데이터나 모델을 개선하는 과정입니다.

    흔한 오류 원인은 다음과 같습니다:

    • 데이터 부족: 특정 조건(어두운 배경 등)의 이미지가 부족
    • 클래스 간 유사성: 두 클래스의 외형이 너무 비슷함
    • 잘못된 라벨링: 데이터가 엉뚱한 클래스에 들어가 있음
    • 배경 편향: 물체가 아니라 배경의 특징을 학습해버림

    ③ 예시로 확인

    "과일 분류기"에서 사과와 토마토를 자주 혼동한다면:

    • 원인: 빨간색 + 둥근 모양이라는 공통 특징 때문
    • 해결: 꼭지 부분, 표면 질감이 잘 보이는 이미지를 추가 수집
    • 추가 전략: 초록 사과, 방울토마토 등 다양한 종류의 이미지 추가

    💡 정리하기: 오류 분석의 핵심 흐름은 "틀린 사례 수집 → 원인 분류 → 개선 전략 수립 → 데이터 보완 → 재학습" 입니다. 이 과정을 반복할수록 모델은 점점 더 정확해집니다.

    🔨 따라하기: 데이터 수집부터 모델 학습까지¶

    🎒 준비물: 스마트폰 또는 태블릿(카메라 기능), 컴퓨터(인터넷 연결), 1차시에서 만든 기획서

    Step 1: 촬영 계획 최종 점검 📋¶

    모델을 만들기 전, 지난 시간에 세운 기획서를 한 번 더 확인합니다.

    점검 사항:

    • 분류할 클래스: 3~5개가 적당 (예: 가위, 풀, 테이프)
    • 클래스당 목표 이미지 수: 최소 30장, 가능하면 50장 이상
    • 촬영 장소: 교실, 복도, 운동장 등 2곳 이상
    • 촬영 조건 변화: 각도 3가지 이상, 배경 2가지 이상

    📌 알아보기: 팀원별로 역할을 나누면 효율적입니다! - 촬영 담당: 이미지 촬영 - 기록 담당: 촬영 조건(장소, 시간, 각도) 메모 - 정리 담당: 촬영한 이미지를 클래스별 폴더에 분류 - 품질 담당: 흔들린 사진, 잘못된 분류 검수

    Step 2: 이미지 촬영하기 📷¶

    이제 실제로 촬영을 시작합니다! 다음 규칙을 기억하세요.

    촬영 핵심 규칙:

    1. 한 장에 대상 하나만: 여러 물건이 한 장에 찍히면 AI가 혼란스러워합니다
    2. 대상을 화면 중앙에: 너무 구석에 있거나 잘리지 않도록 합니다
    3. 조건을 바꿔가며 촬영: 같은 물건이라도 위치, 각도, 조명을 바꿉니다
    4. 흔들리지 않게: 초점이 맞지 않는 사진은 AI에게 나쁜 재료입니다
    flowchart LR subgraph 각도변화["📐 각도 변화"] A1["위에서 내려다보기"] A2["옆에서 수평으로"] A3["비스듬히 45도"] end subgraph 배경변화["🖼️ 배경 변화"] B1["밝은 책상 위"] B2["어두운 바닥"] B3["손에 들고"] end subgraph 거리변화["📏 거리 변화"] C1["가까이(물체가 크게)"] C2["보통 거리"] C3["멀리(물체가 작게)"] end

    💡 꿀팁: 스마트폰의 연속 촬영(버스트) 기능을 활용하면 빠르게 여러 장을 찍을 수 있습니다. 단, 너무 비슷한 사진이 많아지지 않도록 촬영 사이에 각도나 위치를 조금씩 변경하세요.

    ⏰ 촬영 시간 배분 (약 40분)

    • 교실 내 촬영: 20분
    • 교실 밖(복도, 운동장 등) 촬영: 15분
    • 촬영 결과 확인 및 재촬영: 5분

    Step 3: 이미지 정리 및 품질 검수 🗂️¶

    촬영이 끝나면 컴퓨터로 이미지를 옮기고 정리합니다.

    정리 방법:

    1. 프로젝트 이름으로 큰 폴더를 만듭니다 (예: 교실물건_분류기)
    2. 그 안에 클래스별 하위 폴더를 만듭니다 (예: 가위, 풀, 테이프)
    3. 촬영한 이미지를 해당 클래스 폴더에 넣습니다

    품질 검수 체크리스트:

    • ✅ 흔들리거나 초점이 맞지 않는 사진은 없는가?
    • ✅ 대상이 너무 작거나 잘린 사진은 없는가?
    • ✅ 잘못된 폴더에 들어간 사진은 없는가? (가위 사진이 풀 폴더에 있진 않은가?)
    • ✅ 각 클래스의 이미지 수가 비슷한가?

    ⚠️ 중요: 이 단계를 건너뛰고 싶은 마음이 들 수 있지만, 쓰레기 데이터로는 쓰레기 모델만 나옵니다(Garbage In, Garbage Out). 5분만 투자해서 꼼꼼히 검수하세요!

    Step 4: Teachable Machine에서 모델 학습시키기 🤖¶

    이제 가장 신나는 단계입니다. 수집한 데이터로 AI를 가르쳐봅시다!

    4-1. Teachable Machine 접속

    웹 브라우저에서 teachablemachine.withgoogle.com에 접속합니다.

    4-2. 프로젝트 생성

    1. "시작하기(Get Started)" 클릭
    2. "이미지 프로젝트(Image Project)" 선택
    3. "표준 이미지 모델(Standard image model)" 선택

    4-3. 클래스 설정 및 이미지 업로드

    1. 첫 번째 클래스의 이름을 기획서에 맞게 입력합니다 (예: "가위")
    2. "업로드(Upload)" 버튼을 클릭하고, 해당 폴더의 이미지를 모두 선택하여 업로드합니다
    3. "클래스 추가(Add a class)" 버튼으로 나머지 클래스도 동일하게 설정합니다

    📌 알아보기: Teachable Machine에서는 웹캠으로 직접 촬영하여 데이터를 추가할 수도 있습니다. 촬영한 이미지가 부족한 클래스는 웹캠 녹화로 빠르게 보충할 수 있어요. 웹캠 앞에서 물건을 천천히 돌려가며 녹화하면 다양한 각도의 이미지를 한꺼번에 수집할 수 있습니다.

    4-4. 모델 학습

    1. 모든 클래스에 이미지 업로드가 완료되었는지 확인합니다
    2. 가운데 "모델 학습시키기(Train Model)" 버튼을 클릭합니다
    3. 학습이 진행되는 동안 기다립니다 (보통 1~3분 소요)
    flowchart TD A["🏠 Teachable Machine 접속"] --> B["📁 이미지 프로젝트 선택"] B --> C["✏️ 클래스 이름 설정<br/>(가위, 풀, 테이프...)"] C --> D["📤 클래스별 이미지 업로드"] D --> E["🧠 '모델 학습시키기' 클릭"] E --> F["⏳ 학습 진행 (1~3분)"] F --> G["🎉 학습 완료!"] G --> H["🔍 미리보기에서 테스트"]

    4-5. 첫 번째 테스트

    학습이 완료되면 오른쪽에 미리보기(Preview) 패널이 나타납니다.

    1. 웹캠 입력을 선택합니다
    2. 분류할 물건을 카메라 앞에 가져다 대세요
    3. 각 클래스별 확률(%)이 실시간으로 표시됩니다

    잘 작동하나요? 🎉 아마 어떤 물건은 잘 맞히고, 어떤 물건은 틀릴 것입니다. 그게 정상입니다! 이제부터가 진짜 재미있는 단계입니다.

    Step 5: 크로스 테스트 — 팀끼리 서로 테스트하기 🔄¶

    혼자 테스트하면 자기도 모르게 "잘 되는 경우만" 시도하게 됩니다. 다른 팀이 테스트해주면 예상치 못한 상황에서의 약점을 발견할 수 있습니다.

    크로스 테스트 진행 방법:

    1. 짝 팀 지정: 옆 팀과 짝을 이룹니다
    2. 모델 교환: 각 팀의 컴퓨터 앞에서 상대 팀의 모델을 테스트합니다
    3. 도전적으로 테스트: 일부러 어려운 조건으로 시도해봅니다

    도전적 테스트 아이디어:

    • 💡 물건을 살짝 가린 상태에서 테스트
    • 💡 평소와 다른 배경(책 위, 가방 위 등)에서 테스트
    • 💡 물건을 거꾸로 들거나 세워서 테스트
    • 💡 비슷한 물건(가위 대신 집게 등)으로 테스트

    테스트 결과 기록:

    테스트하면서 다음 내용을 메모합니다:

    • 테스트한 물건: 무엇을 카메라에 보여줬는가
    • 모델의 예측: 어떤 클래스로 분류했는가
    • 정답 여부: 맞았는가, 틀렸는가
    • 틀렸다면 조건: 어떤 상황에서 틀렸는가 (어두운 곳, 특이한 각도 등)

    💡 생각해보기: 다른 팀의 모델을 테스트하면서 "아, 이런 경우에 틀리는구나!"라는 발견이 있었나요? 이런 발견은 우리 팀 모델을 개선하는 데에도 소중한 힌트가 됩니다.

    Step 6: 오류 분석 및 데이터 보완 🔧¶

    크로스 테스트에서 발견된 오류를 체계적으로 분석합니다.

    6-1. 오류 사례 모으기

    테스트에서 틀린 경우를 모두 모아봅니다. 그리고 비슷한 원인끼리 묶어봅니다.

    flowchart TD A["🔴 오류 사례 수집"] --> B{"원인 분류"} B --> C["📷 데이터 부족<br/>특정 조건의 이미지가 없었음"] B --> D["🔀 클래스 혼동<br/>두 클래스가 너무 비슷함"] B --> E["🏷️ 라벨링 오류<br/>잘못 분류된 데이터 존재"] B --> F["🖼️ 배경 편향<br/>물체 대신 배경을 학습"] C --> G["✅ 해당 조건으로<br/>추가 촬영"] D --> H["✅ 구별 특징이 드러나는<br/>이미지 추가"] E --> I["✅ 잘못된 이미지<br/>삭제 또는 이동"] F --> J["✅ 다양한 배경에서<br/>재촬영"]

    6-2. 개선 전략 수립

    오류 원인에 따라 구체적인 해결 방법을 정합니다:

    • "어두운 곳에서 자주 틀린다" → 어두운 조건에서 이미지 20장 추가 촬영
    • "가위와 집게를 혼동한다" → 가위의 고유한 특징(손잡이 구멍 등)이 잘 보이는 이미지 추가
    • "흰 배경에서만 잘 맞힌다" → 다양한 색상의 배경에서 재촬영
    • "특정 물건 한 개만 인식한다" → 같은 클래스의 다른 종류도 촬영 (빨간 가위, 파란 가위 등)

    6-3. 데이터 보완 후 재학습

    1. 개선 전략에 따라 추가 이미지를 수집합니다
    2. Teachable Machine에서 해당 클래스에 새 이미지를 추가합니다
    3. 다시 "모델 학습시키기"를 클릭합니다
    4. 동일한 테스트를 다시 수행하여 개선되었는지 확인합니다

    📌 정리하기: 이 "학습 → 테스트 → 분석 → 보완 → 재학습" 사이클이 바로 실제 AI 개발자들이 하는 일입니다. 한 번에 완벽한 모델을 만드는 것이 아니라, 반복적으로 개선해나가는 것이 핵심입니다!
